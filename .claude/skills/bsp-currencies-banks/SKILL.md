# BSP Currencies, Banks & Organizations (Валюты, Банки, Организации)

L3-скил по трём смежным подсистемам нормативно-справочной информации: получение и пересчёт курсов валют, работа с банковским классификатором (БИК), и сведения об организации (юр-лицо, ИП, обособленное подразделение). Подсистемы тесно интегрированы: сведения об организации включают банковский счёт; курсы валют и банки часто нужны в одних и тех же отчётах и печатных формах.

Скил **не покрывает** сопоставление адресов (`bsp-contact-info`), адресный классификатор (`bsp-contact-info`), работу с файлами классификаторов на уровне БСП-механизма «Поставляемые данные» (см. `РаботаСКурсамиВалютСлужебный.ЗагрузитьКурсы` — служебный API), и печатные формы с суммой прописью (см. `bsp-print-reports`).

## When to use

- Нужно получить **курс валюты на дату** для печатной формы, расчёта суммы или пересчёта по нескольким валютам.
- Нужно **пересчитать сумму** из одной валюты в другую (по курсу на дату или по заранее известным параметрам курса).
- Нужно **сформировать сумму прописью** на нужном языке (локализация через `КодЯзыка`).
- Нужно программно **добавить валюты в справочник** по цифровому коду (840 = USD, 978 = EUR и т. п.) — начальное заполнение или миграция.
- Нужно **найти банк по БИК** (или по БИК + корр. счёт) и вернуть таблицу с реквизитами.
- Нужно **открыть форму выбора БИК** из поля ввода на форме объекта (тонкий/веб-клиент).
- Нужно **объяснить пользователю**, почему банк помечен как недействительный (БИК сменился или деятельность прекращена).
- Нужно получить **сведения об организации** (ИНН, КПП, ОГРН, ОКВЭД, адреса, банковский счёт) для печатной формы, отчёта или обмена.
- Нужно проверить, **используется ли архив организаций** (режим, в котором старые организации остаются в ИБ, но помечаются как архивные).

## Не использовать, если

- Нужно сформировать **печатную форму** с числом прописью — используйте `bsp-print-reports` и `РаботаСКурсамиВалют.ПодключитьИсточникДанныхПечатиЧислоПрописью`; `СформироватьСуммуПрописью` напрямую используется только в коде, не из макета.
- Нужно программно **обновить курсы валют в ИБ** (регламентное задание, обработка поставляемых данных, копирование курсов в области данных) — это `РаботаСКурсамиВалютСлужебный` (⚠️ служебный API, обратная совместимость не гарантируется). В прикладном коде — вызов через обработку/команду `РаботаСКурсамиВалютКлиент.ПоказатьЗагрузкуКурсовВалют`, а не прямой вызов служебного модуля.
- Нужны **адреса в структурированном виде** (ФИАС, КЛАДР, ОКТМО) — это `bsp-contact-info` (`УправлениеКонтактнойИнформацией`), а не `ОрганизацииСервер.КонтактнаяИнформацияОрганизации` (последний отдаёт готовые строки/поля адреса, а не объект адреса).
- Нужно **создать/перезаписать элемент** справочника `Валюты` или `КлассификаторБанков` вручную — `ДобавитьВалютыПоКоду` создаёт элементы с дефолтными реквизитами, для интерактивной работы со справочником используйте стандартные формы.
- Нужно **сообщить пользователю** об ошибке — `ОбщегоНазначения.СообщитьПользователю` (`bsp-base-common`), а не `ПояснениеНедействительногоБанка` (последний возвращает форматированную строку для отображения в форме банка, а не сервис сообщений).

## Core concepts

### Карта «подсистема → общие модули»

В отличие от большинства подсистем, имена модулей **не совпадают** с именами подсистем напрямую. Это частая ловушка — не ищите модуль `Валюты` или `Банки`.

| Подсистема | Семейство общих модулей | Ключевое имя для стабильного API |
|---|---|---|
| `Валюты` | `РаботаСКурсамиВалют`, `РаботаСКурсамиВалютКлиент`, `РаботаСКурсамиВалютКлиентСервер`, `РаботаСКурсамиВалютСлужебный` (⚠️), `РаботаСКурсамиВалютВызовСервера` (⚠️), `РаботаСКурсамиВалютГлобальный` (⚠️), `РаботаСКурсамиВалютПереопределяемый`, `РаботаСКурсамиВалютЛокализация`, `РаботаСКурсамиВалютКлиентЛокализация` | `РаботаСКурсамиВалют` (сервер), `РаботаСКурсамиВалютКлиентСервер` (Клиент+Сервер) |
| `Банки` | `РаботаСБанками`, `РаботаСБанкамиКлиент`, `РаботаСБанкамиВызовСервера` (⚠️), `РаботаСБанкамиГлобальный` (⚠️), `РаботаСБанкамиПереопределяемый`, `РаботаСБанкамиСлужебный` (⚠️) | `РаботаСБанками` (сервер), `РаботаСБанкамиКлиент` (UI) |
| `Организации` | `ОрганизацииСервер`, `ОрганизацииПереопределяемый` | `ОрганизацииСервер` (стабильный API), `ОрганизацииПереопределяемый` (только переопределение) |

⚠️ Модули с суффиксами `Служебный`, `ВызовСервера`, `Глобальный` — служебные/внутренние, обратная совместимость между минорными версиями БСП **не гарантируется**. В Key methods отмечены явно.

### Справочники, на которые опираются подсистемы

- `Справочник.Валюты` — валюты, для которых хранятся курсы (`РегистрыСведений.КурсыВалют`).
- `Справочник.КлассификаторБанков` — банки с реквизитами БИК, корр. счёт, адрес, признак `ДеятельностьПрекращена`.
- `Справочник.Организации` (точное имя зависит от прикладной конфигурации через `ОпределяемыйТип.Организация`).

Все методы API возвращают **снимок данных** (структуру с полями), а не ссылки на актуальные объекты. Это норма для печатных форм и отчётов; для актуальных данных перечитайте объект через `ОбщегоНазначения.ЗначениеРеквизитаОбъекта` (`bsp-base-common`).

### Курсы валют: где живут и как кэшируются

`РегистрыСведений.КурсыВалют` — периодический РС с измерениями `Валюта` и `Дата`. `РаботаСКурсамиВалют.ПолучитьКурсВалюты` — стабильная обёртка над этим регистром с типизированным возвратом (`Структура` с полями `Курс`, `Кратность`, `Валюта`, `ДатаКурса`).

Внутри `РаботаСКурсамиВалют` есть вложенные области (`#Область ПрограммныйИнтерфейс` — стабильный API, `#Область СлужебныйПрограммныйИнтерфейс` — для других подсистем БСП, `#Область СлужебныеПроцедурыИФункции` — внутреннее, `#Область ОбновлениеКурсовВалют` — код регламентного обновления). В Key methods приведены **только** стабильные методы из `#Область ПрограммныйИнтерфейс`.

### Банки: устаревший API и актуальный

`ПолучитьДанныеКлассификатора` (модуль `РаботаСБанками`) помечен в исходнике как «Устарела. Следует использовать СведенияБИК». Возвращает данные через `ЗаписьОБанке` (выходной параметр — анти-паттерн 1С), а не через возврат значения. В новом коде — `СведенияБИК(БИК, КоррСчет, ТолькоАктуальные)`, возвращающий `ТаблицаЗначений`.

### Организации: разделение API и переопределения

API подсистемы разнесён по двум модулям:

- `ОрганизацииСервер` — стабильные функции **чтения** (`СведенияОбОрганизации`, `РегистрационныеДанныеОрганизации`, `БанковскийСчетОрганизации`, и т. д.). Принимают `Организация` (тип `ОпределяемыйТип.Организация`), `Поля` (через запятую), `Дата`, `КодЯзыка`. Возвращают `Структура` с запрошенными полями.
- `ОрганизацииПереопределяемый` — процедуры **замещения** (`ПриОпределенииРегистрационныхДанныхОрганизации` и т. д.). Вызывать их из прикладного кода — **анти-паттерн**: они предназначены для реализации в одноимённом переопределяемом модуле прикладной конфигурации, а не для вызова.

Параметр `Поля` — **не произвольные имена реквизитов справочника `Организации`**, а фиксированный набор полей, перечисленных в документации метода (например, `ИНН`, `КПП`, `ОГРН` для `РегистрационныеДанныеОрганизации`). Конечный состав полей в прикладной конфигурации определяется реализацией в `ОрганизацииПереопределяемый`. Пустая строка в `Поля` = вернуть все поля.

### Архив организаций

`ОрганизацииСервер.ИспользуетсяАрхив()` — без параметров, возвращает `Булево`. Если `Истина`, в данных могут быть организации с признаком `ВАрхиве` (см. поле в `РегистрационныеДанныеОрганизации`); их можно скрывать из интерфейса.

## Patterns

### 1. Получить курс и пересчитать сумму в другую валюту

```bsl
// Серверный обработчик
ДатаКурса = Объект.Дата;
КурсUSD = РаботаСКурсамиВалют.ПолучитьКурсВалюты(Объект.Валюта, ДатаКурса);

Если КурсUSD = Неопределено Тогда
    ОбщегоНазначения.СообщитьПользователю(
        НСтр("ru = 'Курс валюты на дату не задан.'"), , , "Объект");
    Отказ = Истина;
    Возврат;
КонецЕсли;

СуммаВРублях = РаботаСКурсамиВалют.ПересчитатьВВалюту(
    Объект.Сумма,
    Объект.Валюта,
    Справочники.Валюты.НайтиПоКоду("643"),  // рубль по ОКВ
    ДатаКурса);
```

`ПолучитьКурсВалюты` возвращает `Неопределено` при отсутствии записи курса — обязательно проверяйте. `ПересчитатьВВалюту` сам получит оба курса внутри, явный вызов `ПолучитьКурсВалюты` нужен только когда вы хотите показать курс пользователю или валидировать.

### 2. Сумма прописью на нужном языке

```bsl
// Сервер, формирование строки для печати
ТекстПрописью = РаботаСКурсамиВалют.СформироватьСуммуПрописью(
    1234.56,
    Валюта,
    ,           // БезДробнойЧасти = Ложь
    "ru");      // КодЯзыка

// "Одна тысяча двести тридцать четыре рубля 56 копеек"
```

Для многоязычных печатных форм передавайте `КодЯзыка` из параметров печати (`КодЯзыка = УправлениеПечатью.КодЯзыкаДляПечатныхФорм(КодЯзыка)` или явный `"en"`, `"en_US"`).

### 3. Выбор БИК из формы

```bsl
// Модуль формы документа
&НаКлиенте
Процедура БИКНачалоВыбора(Элемент, ДанныеВыбора, СтандартнаяОбработка)
    СтандартнаяОбработка = Ложь;
    РаботаСБанкамиКлиент.ВыбратьИзСправочникаБИК(
        Объект.БИК,
        ЭтотОбъект,
        Новый ОписаниеОповещения("ОбработкаВыбораБИК", ЭтотОбъект));
КонецПроцедуры

&НаКлиенте
Процедура ОбработкаВыбораБИК(Результат, ДопПараметры) Экспорт
    Если Результат <> Неопределено Тогда
        Объект.БИК = Результат.БИК;
        Объект.НаименованиеБанка = Результат.Наименование;
    КонецЕсли;
КонецПроцедуры
```

`СтандартнаяОбработка = Ложь` обязательно — иначе отработает штатный выбор из справочника. `ОбработчикОповещения` получает `Структура` с полями выбранного банка.

### 4. Сведения об организации для печатной формы

```bsl
// Сервер, модуль обработки печати
Сведения = ОрганизацииСервер.СведенияОбОрганизации(
    Организация,
    "ИНН,КПП,ОГРН,НаименованиеПолное,АдресОрганизации,БанковскийСчетНомерСчета",
    Дата,
    КодЯзыка);

Если Сведения.Свойство("ИНН") Тогда
    ОбластьИНН.Текст = Сведения.ИНН;
КонецЕсли;
```

`Поля` — **строка** имён через запятую, не массив. Пустая строка = все поля (дорого по производительности, не злоупотребляйте в горячих путях).

## Anti-patterns

### ❌ Вызывать `РаботаСКурсамиВалютСлужебный.ЗагрузитьКурсы()` напрямую

```bsl
// ❌ Служебный API, обратная совместимость не гарантируется
РаботаСКурсамиВалютСлужебный.ЗагрузитьКурсы();
```

```bsl
// ✅ Запускать через клиентскую форму (UI), которая сама вызовет служебный код
РаботаСКурсамиВалютКлиент.ПоказатьЗагрузкуКурсовВалют(Новый Структура("ОткрытиеИзСписка", Истина));
```

Прямой вызов служебного модуля работает в текущей версии, но в любой минорке может перестать. Этот модуль предназначен для обработчиков событий других подсистем БСП, а не для прикладного кода.

### ❌ Использовать `ПолучитьДанныеКлассификатора` (устарел)

```bsl
// ❌ Устаревший API с выходным параметром (анти-паттерн 1С)
ЗаписьОБанке = "";
РаботаСБанками.ПолучитьДанныеКлассификатора("044525225", , ЗаписьОБанке);
```

```bsl
// ✅ Стабильный API с возвратом значения
Таблица = РаботаСБанками.СведенияБИК("044525225", , Ложь);  // ТолькоАктуальные = Ложь
Если Таблица.Количество() > 0 Тогда
    БанкСсылка = Таблица[0].Ссылка;
КонецЕсли;
```

`ПолучитьДанныеКлассификатора` помечен в исходнике как «Устарела. Следует использовать СведенияБИК». Не вводите устаревший API в новый код.

### ❌ Присваивать `Валюта` через прямой запрос к `РегистрыСведений.КурсыВалют`

```bsl
// ❌ Обходит обёртку, ломает интеграцию с кэшированием и обработчиками обновления
Запрос = Новый Запрос("ВЫБРАТЬ * ИЗ РегистрСведений.КурсыВалют ГДЕ Валюта = &Валюта И Дата = &Дата");
Запрос.УстановитьПараметр("Валюта", Валюта);
Запрос.УстановитьПараметр("Дата", ДатаКурса);
Результат = Запрос.Выполнить().Выбрать();
```

```bsl
// ✅ Идиоматический БСП-стиль через обёртку
Курс = РаботаСКурсамиВалют.ПолучитьКурсВалюты(Валюта, ДатаКурса);
// Курс = Неопределено, если записи нет
```

`ПолучитьКурсВалюты` инкапсулирует работу с периодическим РС, кратностью и типизированным возвратом. Прямой запрос хрупок (смена структуры РС — и запрос сломан).

### ❌ Обращаться к модулям `Валюты`, `Банки`, `Организации` напрямую

```bsl
// ❌ Таких общих модулей НЕ существует в БСП 3.1.11
Валюты.ПолучитьКурс(Валюта, Дата);
Банки.НайтиПоБИК("044525225");
Организации.СведенияОбОрганизации(Организация);
```

```bsl
// ✅ Реальные имена модулей — `РаботаСКурсамиВалют`, `РаботаСБанками`, `ОрганизацииСервер`
РаботаСКурсамиВалют.ПолучитьКурсВалюты(Валюта, Дата);
РаботаСБанками.СведенияБИК("044525225");
ОрганизацииСервер.СведенияОбОрганизации(Организация);
```

Типовая ловушка — по аналогии с «должно быть `Валюты.ПолучитьКурс`» такие модули выдумывают. В БСП 3.1.11 есть `РаботаСКурсамиВалют`, `РаботаСБанками`, `ОрганизацииСервер`. Перед вызовом проверяйте `CommonModules/<Имя>/Ext/Module.bsl`.

## How to explore deeper

### Где искать модули в выгрузке конфигурации

- `CommonModules/РаботаСКурсамиВалют/Ext/Module.bsl` — серверный API валют (стабильный).
- `CommonModules/РаботаСКурсамиВалютКлиентСервер/Ext/Module.bsl` — клиент+сервер (`ПересчитатьПоКурсу`).
- `CommonModules/РаботаСКурсамиВалютСлужебный/Ext/Module.bsl` — ⚠️ служебный (загрузка курсов, обработка поставляемых данных).
- `CommonModules/РаботаСБанками/Ext/Module.bsl` — серверный API банков.
- `CommonModules/РаботаСБанкамиКлиент/Ext/Module.bsl` — клиентский API банков (выбор БИК из формы).
- `CommonModules/ОрганизацииСервер/Ext/Module.bsl` — стабильный API организаций.
- `CommonModules/ОрганизацииПереопределяемый/Ext/Module.bsl` — ⚠️ **только для переопределения**, не для вызова.
- `InformationRegisters/КурсыВалют/Ext/RecordSetModule.bsl` — модуль набора записей РС курсов.
- `Catalogs/КлассификаторБанков/` — справочник банков с реквизитами (`BIC`, `КоррСчет`, `ДеятельностьПрекращена`).
