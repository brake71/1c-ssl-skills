
# BSP Contact Info (КонтактнаяИнформация + АдресныйКлассификатор)

Скил по двум связанным подсистемам БСП 3.1.11 — **КонтактнаяИнформация** (хранение адресов, телефонов, e-mail в виде JSON / XML у владельцев) и **АдресныйКлассификатор** (загрузка КЛАДР / ФИАС, веб-сервис 1С, ОКТМО / ОКАТО / ИФНС). Покрывает **чтение / запись КИ**, **открытие формы**, **конвертацию JSON ↔ XML**, **валидацию адресов** по классификатору и **разбор адреса на регион / город / почтовый индекс / ОКТМО**.

Не покрывает загрузку файлов классификатора с диска и обновление — это разовая административная задача, выполняемая интерактивно через форму.

## When to use

- Нужно **прочитать контактную информацию** ссылки или объекта (адрес, телефон, e-mail) — `УправлениеКонтактнойИнформацией.КонтактнаяИнформацияОбъекта` / `КонтактнаяИнформацияОбъектов` с указанием вида или типа.
- Нужно **записать контактную информацию** в табличную часть объекта из JSON / XML — `УправлениеКонтактнойИнформацией.ЗаписатьКонтактнуюИнформацию` (Процедура, не Функция).
- Нужно получить **представление** адреса / телефона / e-mail строкой для UI, отчёта или печати — `УправлениеКонтактнойИнформацией.ПредставлениеКонтактнойИнформацииОбъекта` (рекомендуемый; `КонтактнаяИнформацияОбъекта` с `ТолькоПредставление = Истина` устарел).
- Нужно **открыть управляемую форму** ввода / редактирования КИ из обработчика команды формы — `УправлениеКонтактнойИнформациейКлиент.ОткрытьФормуКонтактнойИнформации` (универсальная) или `УправлениеКонтактнойИнформациейКлиент.ОткрытьФормуВводаАдреса` (только адрес).
- Нужно **проверить пакет адресов** на соответствие классификатору (через веб-сервис 1С — до 20 секунд на запрос, поэтому вызывать в фоне) — `АдресныйКлассификатор.ПроверитьАдреса`.
- Нужно **определить код региона** по наименованию «Московская», «Свердловская обл» и т. п. — `АдресныйКлассификатор.КодРегионаПоНаименованию`.
- Нужно **извлечь регион, город, ОКТМО, ОКАТО, ИФНС** из хранимой JSON-строки адреса (без записи / загрузки) — `РаботаСАдресами.РегионАдресаКонтактнойИнформации` / `ГородАдресаКонтактнойИнформации` / `СведенияОбАдресе` / `СведенияПоОКМТО`.
- Нужно **собрать полный адрес** (XML или JSON) по ФИАС-идентификатору адресного объекта — `РаботаСАдресами.АдресПоИдентификатору`.
- Нужно **определить формат** хранимой КИ (JSON / XML) перед разбором — `УправлениеКонтактнойИнформациейКлиентСервер.ЭтоКонтактнаяИнформацияВJSON` / `…ВXML`.

## Не использовать, если

- Нужны общие утилиты общего назначения (сериализация, проверка типа, безопасное хранилище) — `bsp-base-common`. Здесь только **специализированные** методы работы с КИ и адресами.
- Нужно напечатать адрес в печатной форме — `bsp-print-reports` (подсистема `Печать`). Скил `bsp-contact-info` даёт **данные** адреса, но не формирует табличный документ.
- Нужно **загрузить** файлы КЛАДР / ФИАС с диска или из веб-сервиса — это разовая интерактивная операция (`АдресныйКлассификаторКлиент.ПоказатьФормуЗагрузкиАдресногоКлассификатора`); скил описывает только **проверку** загруженных данных.
- Нужно **тяжёлую проверку адресов** (через веб-сервис) запустить прямо с клиента — оборачивать в `ДлительныеОперации.ВыполнитьФункцию` (`bsp-longs-and-jobs`); иначе форма «зависнет» на 20 секунд.
- Нужно определить, входит ли подсистема КИ в текущую конфигурацию — `ОбщегоНазначения.ПодсистемаСуществует("СтандартныеПодсистемы.КонтактнаяИнформация")` (`bsp-base-common`).
- Нужно **прикрепить файл** (скан, PDF) к адресу — `bsp-files-and-versions` (подсистема `РаботаСФайлами`); КИ хранит только текстовое представление.

## Core concepts

### Две подсистемы — один скил

`КонтактнаяИнформация` отвечает за **хранение и представление** (адреса, телефоны, e-mail — любых КИ) у произвольных владельцев (справочники, документы, регистры). `АдресныйКлассификатор` — за **валидацию** адресов по КЛАДР / ФИАС и **справочные данные** (коды регионов, ОКТМО, ФИАС-идентификаторы). Их нельзя путать: записать КИ в объект — это `УправлениеКонтактнойИнформацией.*`, а узнать код региона по имени — это `АдресныйКлассификатор.*`.

### Внутренний формат хранения КИ — JSON

Контактная информация хранится в табличных частях объектов-владельцев **в формате JSON** (начиная с БСП 3.1.x). XML-представление сохранено для обратной совместимости (XDTO-пакеты `КонтактнаяИнформация` и `Адрес`). Поле `ЗначенияПолей` (XML) заполняется параллельно, но **рабочий формат — JSON** (`Поле` `Значение` в таблице КИ).

Распознать формат строки перед обработкой: `УправлениеКонтактнойИнформациейКлиентСервер.ЭтоКонтактнаяИнформацияВJSON(Строка)` / `…ВXML(Строка)` (возвращают `Истина` / `Ложь`).

### Адресные форматы: КЛАДР vs ФИАР vs ФИАС vs ОКТМО

| Формат | Что это | Где встречается в БСП |
|---|---|---|
| **КЛАДР** (устар.) | Классификатор адресов России, 4 уровня: регион → район → город → улица. Код — до 11 цифр + код дома | `АдресныйКлассификатор.КодРегионаПоНаименованию`, `…СведенияПоОКМТО` (внутри используют КЛАДР-коды) |
| **ФИАС** (актуал.) | Федеральная информационная адресная система, иерархия объектов, у каждого `УникальныйИдентификатор` (GUID) | `РаботаСАдресами.АдресПоИдентификатору` |
| **ОКТМО** | Общероссийский классификатор территорий муниципальных образований (8 или 11 цифр) | `АдресныйКлассификатор.СведенияПоОКМТО` |
| **ОКАТО** | Общероссийский классификатор объектов АТД (устар., вытесняется ОКТМО) | возвращается в составе `СведенияПоОКМТО` (поле `OKATO`) |
| **ИФНС** | Код налоговой инспекции (ФЛ / ЮЛ + территориальный участок) | возвращается в `СведенияПоОКМТО` / `СведенияОбАдресе` |

### Суффиксная система модулей подсистем

Подсистема `КонтактнаяИнформация` обслуживается семейством модулей (без префикса, с суффиксами контекста — теми же, что описаны в `bsp-fundamentals`):

| Модуль | Контекст | Назначение |
|---|---|---|
| `УправлениеКонтактнойИнформацией` | Сервер, Толстый клиент, Внешнее соединение | Серверный API: чтение / запись КИ, виды, разбор |
| `УправлениеКонтактнойИнформациейКлиент` | Тонкий / Толстый клиент | UI: открытие форм, обработчики команд, показ на карте, Skype, e-mail |
| `УправлениеКонтактнойИнформациейКлиентСервер` | Сервер, Тонкий / Толстый клиент | Безопасный код: распознавание формата, формирование представления, детекторы типа |
| `УправлениеКонтактнойИнформациейЛокализация` | Сервер | Региональные особенности (РФ, ЕАЭС, иностранные адреса) |
| `УправлениеКонтактнойИнформациейКлиентЛокализация` | Клиент | Региональные особенности UI |
| `УправлениеКонтактнойИнформациейКлиентСерверЛокализация` | Клиент + Сервер | Региональный безопасный код |
| `УправлениеКонтактнойИнформациейПереопределяемый` | Сервер | «Крючки» для прикладной конфигурации |
| `УправлениеКонтактнойИнформациейСлужебный` | Сервер | ⚠️ **Служебный API** — для внутренних нужд БСП, обратная совместимость не гарантируется |
| `УправлениеКонтактнойИнформациейСлужебныйВызовСервера` | Сервер | ⚠️ Служебный, серверные вызовы из клиентского кода |
| `УправлениеКонтактнойИнформациейСлужебныйПовтИсп` | Сервер | ⚠️ Служебный, кэш повторного использования |

⚠️ `УправлениеКонтактнойИнформациейСлужебныйКлиент` / `…СлужебныйКлиентСервер` — **не существуют**. Все служебные варианты — только серверные.

Подсистема `АдресныйКлассификатор`:

| Модуль | Контекст | Назначение |
|---|---|---|
| `АдресныйКлассификатор` | Сервер, Толстый клиент, Внешнее соединение | Стабильный API: проверка адресов, коды регионов, ОКТМО, сведения о загрузке |
| `АдресныйКлассификаторКлиент` | Тонкий / Толстый клиент | UI: загрузка, очистка, выбор каталога файлов |
| `АдресныйКлассификаторПовтИсп` | Сервер | Кэш повторного использования (регионы, сокращения) |
| `АдресныйКлассификаторСлужебный` | Сервер | ⚠️ **Служебный API** — низкоуровневая работа с классификатором, идентификаторы, автоподбор |

Подсистема `РаботаСАдресами` (тесно интегрирована с обеими):

| Модуль | Контекст | Назначение |
|---|---|---|
| `РаботаСАдресами` | Сервер, Толстый клиент, Внешнее соединение | Разбор адреса на поля / регион / город; ФИАС; конвертация JSON ↔ XML; КЛАДР |
| `РаботаСАдресамиКлиент` | Тонкий / Толстый клиент | UI разбора |
| `РаботаСАдресамиКлиентСервер` | Клиент + Сервер | Безопасный код разбора (структура полей, типы адресов) |
| `РаботаСАдресамиПовтИсп` | Сервер | Кэш порядка уровней, типов объектов адресации |
| `РаботаСАдресамиСлужебныйКлиент` | Тонкий / Толстый клиент | ⚠️ Служебный клиентский (только этот суффикс существует) |

⚠️ `РаботаСАдресамиСлужебный` (без суффикса) — **не существует**.

### «Тесная интеграция» с другими подсистемами

- **`Длительные операции`** — `АдресныйКлассификатор.ПроверитьАдреса` через веб-сервис 1С занимает **до 20 секунд**; оборачивать в `ДлительныеОперации.ВыполнитьФункцию` (см. `bsp-longs-and-jobs`).
- **`Пользователи`** — для КИ пользователей-физлиц используется `ОпределяемыйТип.ВладелецКонтактнойИнформации`; владелец может быть любым справочником / документом.
- **`Печать`** — при печати адреса в печатной форме **сначала получить представление** через `УправлениеКонтактнойИнформацией.ПредставлениеКонтактнойИнформацииОбъекта`, **затем** передавать строку в макет (не подставлять JSON / XML напрямую).
- **`БСП_…` префикс** — **не используется**, как и в остальных подсистемах (`bsp-fundamentals`).

Возвращает массив результатов с полями `Ошибки`, `Варианты` (с `КодКЛАДР`, `OKATO`, `ОКТМО`, `КодИФНСФЛ`, `КодИФНСЮЛ`, `КодУчасткаИФНСФЛ`, `КодУчасткаИФНСЮЛ`). **До 20 секунд** на запрос — оборачивать в `ДлительныеОперации`. Перед вызовом имеет смысл проверить `АдресныйКлассификатор.АдресныйКлассификаторЗагружен()` (до 7 секунд HTTP) | `МассивАдресов = ОбщегоНазначенияКлиентСервер.ЗначениеВМассиве(Новый Структура("Адрес", АдресJSON)); Результат = АдресныйКлассификатор.ПроверитьАдреса(МассивАдресов);` | стабильный |
| `АдресныйКлассификатор.СведенияПоОКМТО` | `СведенияПоОКМТО(ОКТМО)` | Сервер, Толстый клиент, Внешнее соединение | Сведения об адресе по коду ОКТМО (8 или 11 цифр). Возвращает структуру с `КодРегиона`, `Регион` (+ типы полный / краткий), `ИдентификаторРегиона`, `МуниципальныйРайон`, `Поселение`, `НаселенныйПункт`, `ПочтовыйИндекс`, `OKATO`, `КодКЛАДР`, `КодИФНСФЛ` / `КодИФНСЮЛ` (+ участки) | `Данные = АдресныйКлассификатор.СведенияПоОКМТО("45380000");` | стабильный |
| `РаботаСАдресами.СведенияОбАдресе` | `СведенияОбАдресе(Адрес, ДополнительныеПараметры = Неопределено)` | Сервер, Толстый клиент, Внешнее соединение | Разбор строки адреса (JSON или XML) на отдельные части: `Регион`, `Район`, `Город`, `НаселенныйПункт`, `Улица`, `Дом`, `Корпус`, `Квартира`, + коды `КодРегиона`, `КодКЛАДР`, `КодИФНСФЛ` и т. д. В `ДополнительныеПараметры` — `КодыКЛАДР`, `ПроверитьАдрес`, `КодыАдреса` (ФИАС), `НаименованиеВключаетСокращение` | `Данные = РаботаСАдресами.СведенияОбАдресе(АдресJSON, Новый Структура("КодыКЛАДР", Истина));` | стабильный |
| `РаботаСАдресами.РегионАдресаКонтактнойИнформации` | `РегионАдресаКонтактнойИнформации(Знач Адрес)` | Сервер, Толстый клиент, Внешнее соединение | Извлечь **только регион** (наименование субъекта РФ) из строки адреса. `Адрес` — JSON или XML. Пустая строка, если субъект не определён. **Исключение**, если строка не является адресом | `Регион = РаботаСАдресами.РегионАдресаКонтактнойИнформации(АдресJSON);` | стабильный |
| `РаботаСАдресами.АдресПоИдентификатору` | `АдресПоИдентификатору(ИдентификаторАдреса, ДополнительнаяИнформацияАдреса = Неопределено)` | Сервер, Толстый клиент, Внешнее соединение | Собрать полный адрес (XML по умолчанию или JSON, если `ДополнительнаяИнформацияАдреса.АдресВJSON = Истина`) по ФИАС-идентификатору адресного объекта. В `ДополнительнаяИнформацияАдреса` можно передать `НомерДома`, `НомерОфиса`, `НомерСтроения`, `ПочтовыйИндекс`, `ДополнительнаяИнформация` (комментарий), `Муниципальный` | `АдресXML = РаботаСАдресами.АдресПоИдентификатору("d2dc5217-...-...", Новый Структура("НомерДома,НомерКвартиры", "5", "12"));` | стабильный |
| `АдресныйКлассификаторСлужебный.ПроверкаАдресов` | `ПроверкаАдресов(Знач НаборАдресов, ПроверятьУстареваниеАдресов = Ложь)` | Сервер | ⚠️ Служебный. Низкоуровневая проверка адресов по классификатору (использует `ПроверятьУстареваниеАдресов`). Стабильная обёртка — `АдресныйКлассификатор.ПроверитьАдреса`; эту — только если стабильного аналога не хватает | `Результат = АдресныйКлассификаторСлужебный.ПроверкаАдресов(МассивАдресов, Истина);` | ⚠️ служебный |

## Patterns

### 1. Прочитать КИ объекта и достать первое значение

```bsl
ВидАдреса = УправлениеКонтактнойИнформацией.ВидКонтактнойИнформацииПоИмени("ЮрАдресКонтрагента");

ТЗ = УправлениеКонтактнойИнформацией.КонтактнаяИнформацияОбъекта(
    Контрагент,
    ВидАдреса,
    ,       // Дата = Неопределено
    Ложь);  // ТолькоПредставление = Ложь — нужна таблица с JSON

Если ТЗ.Количество() > 0 Тогда
    JSONАдреса = ТЗ[0].Значение;
    Представление = ТЗ[0].Представление;
КонецЕсли;
```

`ТолькоПредставление = Ложь` — критично: при `Истина` метод **возвращает строку**, а не таблицу; обращение `ТЗ[0]` упадёт.

### 2. Прочитать представление КИ строкой (для UI / отчёта)

```bsl
ВидТелефона = УправлениеКонтактнойИнформацией.ВидКонтактнойИнформацииПоИмени("ТелефонКонтрагента");

СтрокаКИ = УправлениеКонтактнойИнформацией.ПредставлениеКонтактнойИнформацииОбъекта(
    Контрагент,
    ВидТелефона,
    ", ",   // Разделитель между несколькими телефонами
    ,       // Дата
    Новый Структура("ТолькоПервая, БезПробелов", Ложь, Ложь));
```

`ПредставлениеКонтактнойИнформацииОбъекта` — **рекомендуемый** метод для получения строки КИ. Старый путь `КонтактнаяИнформацияОбъекта(..., ТолькоПредставление = Истина)` помечен как устаревший.

### 3. Записать новый адрес контрагенту

```bsl
ВидАдреса = УправлениеКонтактнойИнформацией.ВидКонтактнойИнформацииПоИмени("ЮрАдресКонтрагента");
ОбъектСпр = Контрагент.ПолучитьОбъект();    // обязательно объект, не ссылка

// Предположим, что JSONАдреса — это строка в JSON-формате БСП,
// полученная из формы ввода или конвертации
УправлениеКонтактнойИнформацией.ЗаписатьКонтактнуюИнформацию(
    ОбъектСпр,
    JSONАдреса,
    ВидАдреса,
    Перечисления.ТипыКонтактнойИнформации.Адрес);

ОбъектСпр.Записать();
```

Это **Процедура** (без `Возврат`). `Объект` должен быть **модулем объекта** (`СправочникОбъект`), а не ссылкой — иначе `ЗаписатьКонтактнуюИнформацию` изменит копию в памяти, а `Записать()` не выполнится.

### 4. Открыть форму ввода КИ из обработчика команды формы

```bsl
&НаКлиенте
Процедура КомандаРедактироватьАдрес(Команда)
    ВидАдреса = УправлениеКонтактнойИнформацией.ВидКонтактнойИнформацииПоИмени("ЮрАдресКонтрагента");
    ЗначениеJSON  = ЭтаФорма[ВидАдреса.ИмяРеквизита];  // текущее значение из реквизита формы
    Представление = ЭтаФорма[ВидАдреса.ИмяРеквизита + "Представление"];

    ПараметрыФормы = Новый Структура;
    ПараметрыФормы.Вставить("Вид", ВидАдреса);
    ПараметрыФормы.Вставить("Значение", ЗначениеJSON);
    ПараметрыФормы.Вставить("Представление", Представление);

    УправлениеКонтактнойИнформациейКлиент.ОткрытьФормуКонтактнойИнформации(
        ПараметрыФормы,
        ЭтаФорма,
        Новый ОписаниеОповещения("ПослеРедактированияАдреса", ЭтаФорма));
КонецПроцедуры
```

Обработчик `ПослеРедактированияАдреса(Результат, ДополнительныеПараметры) Экспорт` получит `Результат = Неопределено` (отмена) или структуру с `Значение` (новый JSON) и `Представление`.

### 5. Пакетная проверка адресов в фоне (через длительные операции)

```bsl
// На сервере — обёртка для фонового вызова
&НаСервере
Функция ПроверитьАдресаВФоне(МассивJSON)
    ПараметрыПроцедуры = Новый Структура;
    ПараметрыПроцедуры.Вставить("МассивJSON", МассивJSON);

    ПараметрыВыполнения = ДлительныеОперации.ПараметрыВыполненияФункции(УникальныйИдентификатор);
    ПараметрыВыполнения.НаименованиеФоновогоЗадания = НСтр("ru = 'Проверка адресов'");

    Возврат ДлительныеОперации.ВыполнитьФункцию(
        ПараметрыВыполнения,
        "АдресныйКлассификатор.ПроверитьАдреса",
        МассивJSON);
КонецФункции

// Клиентский вызов с ожиданием
&НаКлиенте
Процедура КомандаПроверитьАдреса(Команда)
    ДлительнаяОперация = ПроверитьАдресаВФоне(МассивJSONАдресов);
    Оповещение = Новый ОписаниеОповещения("ПослеПроверкиАдресов", ЭтаФорма);
    ПараметрыОжидания = ДлительныеОперацииКлиент.ПараметрыОжидания(ЭтаФорма);
    ДлительныеОперацииКлиент.ОжидатьЗавершение(ДлительнаяОперация, Оповещение, ПараметрыОжидания);
КонецПроцедуры
```

`АдресныйКлассификатор.ПроверитьАдреса` через веб-сервис 1С занимает **до 20 секунд**; прямой вызов в синхронном коде заморозит управляемую форму.

### 6. Извлечь регион и ОКТМО из хранимой КИ

```bsl
JSONАдреса = ТЗ[0].Значение;   // строка из КонтактнаяИнформацияОбъекта

Если УправлениеКонтактнойИнформациейКлиентСервер.ЭтоКонтактнаяИнформацияВJSON(JSONАдреса) Тогда
    Регион = РаботаСАдресами.РегионАдресаКонтактнойИнформации(JSONАдреса);
    Данные = РаботаСАдресами.СведенияОбАдресе(JSONАдреса, Новый Структура("КодыКЛАДР", Истина));
    // Данные.Регион, Данные.Город, Данные.ПочтовыйИндекс, Данные.КодКЛАДР, Данные.КодИФНСФЛ ...
КонецЕсли;
```

Перед разбором **всегда** проверять формат строки через `ЭтоКонтактнаяИнформацияВJSON` / `…ВXML` — старые записи в базе могут быть в XML, и `РаботаСАдресами.СведенияОбАдресе` ожидает конкретный формат.

## Anti-patterns

### ❌ Подставлять JSON / XML КИ напрямую в макет печатной формы

```bsl
// ❌ В макете окажется JSON или XML — нечитаемо
ОбластьМакета.Параметры.Адрес = ТЗ[0].Значение;
```

```bsl
// ✅ Получить строковое представление
СтрокаАдреса = УправлениеКонтактнойИнформацией.ПредставлениеКонтактнойИнформацииОбъекта(
    Контрагент, ВидАдреса);
ОбластьМакета.Параметры.Адрес = СтрокаАдреса;
```

### ❌ Передавать ссылку в `ЗаписатьКонтактнуюИнформацию` вместо объекта

```bsl
// ❌ ОШИБКА: запись в копию в памяти, реальная запись в БД не произойдёт
УправлениеКонтактнойИнформацией.ЗаписатьКонтактнуюИнформацию(
    Контрагент,   // <-- ссылка
    JSONАдреса, ВидАдреса, ТипИнформации);
```

```bsl
// ✅ Объект (модуль объекта), затем .Записать()
ОбъектСпр = Контрагент.ПолучитьОбъект();
УправлениеКонтактнойИнформацией.ЗаписатьКонтактнуюИнформацию(
    ОбъектСпр, JSONАдреса, ВидАдреса, ТипИнформации);
ОбъектСпр.Записать();
```

### ❌ Вызывать `АдресныйКлассификатор.ПроверитьАдреса` синхронно с клиента

```bsl
// ❌ Управляемая форма «зависнет» на 20 секунд при недоступном веб-сервисе
Результат = АдресныйКлассификатор.ПроверитьАдреса(МассивАдресов);
```

```bsl
// ✅ Через ДлительныеОперации (см. Pattern 5)
Возврат ДлительныеОперации.ВыполнитьФункцию(
    ПараметрыВыполнения,
    "АдресныйКлассификатор.ПроверитьАдреса",
    МассивАдресов);
```

### ❌ Парсить JSON / XML КИ вручную через `СтрРазделить` / `ЧтениеJSON`

```bsl
// ❌ Ломается при изменении внутренней структуры КИ в новой версии БСП
Позиция = СтрНайти(JSONАдреса, "region");
Город = Сред(JSONАдреса, Позиция + 10, 30);
```

```bsl
// ✅ Только через методы БСП — они учитывают версионность формата
Город = РаботаСАдресами.ГородАдресаКонтактнойИнформации(JSONАдреса);
Регион = РаботаСАдресами.РегионАдресаКонтактнойИнформации(JSONАдреса);
```

### ❌ Хранить КИ в собственных реквизитах (строка / таблица значений) в обход БСП

```bsl
// ❌ Своё хранилище — обходит журнал регистрации, историю изменений, проверки
НовыйРеквизит = Справочники.Контрагенты.НайтиПоНаименованию("Адрес").Адрес;  // строка
```

```bsl
// ✅ Через табличную часть «КонтактнаяИнформация» и методы УправлениеКонтактнойИнформацией
//    (стандартный набор реквизитов: Значение, Представление, Вид, Тип, Дата)
```

### ❌ Использовать `АдресныйКлассификатор.КодРегионаПоНаименованию` для «Москва» / «г.Москва»

```bsl
// ❌ Возвращает Неопределено — функция работает только с именами СУБЪЕКТОВ (область, край)
Код = АдресныйКлассификатор.КодРегионаПоНаименованию("Москва");
```

```bsl
// ✅ Передавать наименование СУБЪЕКТА РФ, не населённого пункта
Код = АдресныйКлассификатор.КодРегионаПоНаименованию("Московская");
Код = АдресныйКлассификатор.КодРегионаПоНаименованию("г.Москва");  // допустимо
```

(Функция `КодРегионаПоНаименованию` ищет **субъект** РФ. Для городов и районов — `АдресныйКлассификатор.СведенияПоОКМТО` или `РаботаСАдресами.СведенияОбАдресе`.)

### ❌ Вызывать служебные модули `*Служебный` без явной отметки

```bsl
// ❌ Тихо вызывается служебный API — обратная совместимость не гарантируется,
//    и другие модули БСП могут перестать работать корректно при апгрейде
АдресныйКлассификаторСлужебный.УстановитьИдентификаторыАдреса(Адрес);
```

```bsl
// ✅ Если есть стабильный аналог — использовать его
//    Если нет — пометить комментарием ⚠️ и обосновать, почему
// ⚠️ Служебный API, аналога в стабильном слое нет: ...
```

Для поиска методов/модулей в выгрузке конфигурации используйте python scripts/bsp_data_search.py method <имя>.
