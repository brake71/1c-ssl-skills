
Скил по подсистеме «Защита персональных данных» (152-ФЗ): согласия субъектов, уничтожение/скрытие ПДн, регистрация событий доступа, встраивание подсистемы в формы списков субъектов и прикладные переопределяемые обработчики жизненного цикла ПДн.

## When to use

- Нужно программно узнать, есть ли у субъекта действующее согласие на обработку ПДн в указанной организации на дату (перед записью карточки, при печати, при формировании отчёта).
- Нужно определить, что объект содержит уничтоженные ПДн, чтобы скрыть его в списках/отчётах/печатных формах.
- Нужно инициировать уничтожение ПДн субъекта по истечении срока хранения или отзыве согласия (через `УничтожитьПерсональныеДанныеСубъектов`) или просто пометить данные как скрытые.
- Нужно добавить команду печати согласия на обработку ПДн в форму объекта-субъекта (из переопределения `ДобавитьКомандыПечати` прикладной конфигурации).
- Нужно встроить подсистему в форму списка субъектов: добавить колонку «ОтсутствуетСогласие» с картинкой, команду «ПоказыватьСоСкрытымиПДн», обработку оповещений об уничтожении ПДн из формы объекта/списка.
- Нужно в прикладной конфигурации переопределить логику уничтожения/скрытия ПДн (заполнить состав персональных данных, области данных, отказаться от уничтожения конкретного субъекта, выполнить дополнительные действия до/после уничтожения).
- Нужно узнать дату уничтожения ПДн субъекта или срок хранения ПДн (например, для информирования пользователя или для расчёта даты следующей очистки).
- Нужно изменить настройки уничтожения ПДн программно (включить/выключить функциональную опцию) из прикладного кода, в том числе из длительной операции.
- Нужно настроить регистрацию события «Доступ.Доступ» к ПДн (включить/выключить, опционально — по категориям данных) и узнать текущее состояние.

## Не использовать, если

- Задача про **управление доступом и правами** (роли, RLS, группы доступа, внешние пользователи) — это `bsp-users-access`.
- Нужно проверить, подключена ли подсистема БСП в конфигурации, или получить свой общий модуль `ОбщегоНазначения` (подсистемы/функциональные опции, безопасное хранилище, файлы, строки) — это `bsp-base-common`.
- Нужно **напечатать** что-то (формирование табличного документа, печатные формы объектов, команды печати в форме — кроме единственной команды «Согласие на обработку ПДн», которую добавляет эта подсистема) — это `bsp-print-reports`.
- Нужно отправить e-mail/SMS субъекту об уничтожении/истечении согласия — это `bsp-comms` или `bsp-users-access` (в части уведомлений).
- Нужно спроектировать процесс миграции данных при переходе на новую версию БСП (обработчики `ПриДобавленииОбработчиковОбновления`) — это `bsp-update`/L2 по обновлению ИБ, а не этот скил.
- Задача — очистка **помеченных на удаление** объектов, не связанных с ПДн — это `bsp-admin-tools`.

## Core concepts

### Закон и терминология

Подсистема «Защита персональных данных» обеспечивает соответствие требованиям **Федерального закона от 27.07.2006 № 152-ФЗ** «О персональных данных». Ключевые сценарии:

- **Согласие субъекта** — документ `СогласиеНаОбработкуПерсональныхДанных`; отзыв — `ОтзывСогласияНаОбработкуПерсональныхДанных`. Сведения о действующих согласиях хранятся в регистре сведений `СогласияНаОбработкуПерсональныхДанных`.
- **Категории ПДн** — ФИО, паспортные данные, ИНН, СНИЛС, адрес, контакты и т. п. Состав категорий для конкретной конфигурации **настраивается прикладным разработчиком** в переопределяемом модуле (`ЗащитаПерсональныхДанныхПереопределяемый.ЗаполнитьСведенияОПерсональныхДанных`).
- **Срок хранения** — заполняется для каждого субъекта (например, в карточке физического лица). Регламентное задание `РасчетСроковХраненияПерсональныхДанных` рассчитывает дату уничтожения.
- **Уничтожение** — безвозвратное удаление ПДн субъекта в случаях, перечисленных в ст. 21 закона (отзыв согласия, истечение срока, достижение цели обработки, неправомерная обработка). Фиксируется актом `АктОбУничтоженииПерсональныхДанных`. Запись о факте уничтожения попадает в регистр `УничтоженныеПерсональныеДанные`.
- **Скрытие** — программная пометка ПДн как скрытых (через `СкрытьПерсональныеДанныеСубъектов`) для отображения в форме списка фильтром «Показывать со скрытыми ПДн». Внутри `Скрыть…` сводится к `Уничтожить…` (см. реализацию).
- **События доступа** — 6 типов: вход, выход, аутентификация, отказ в аутентификации, доступ к ПДн, отказ в доступе к ПДн. Регистрация событий доступа к ПДн — опциональная, включается администратором через `УстановитьИспользованиеСобытияДоступ`.

### Семейство модулей

Подсистема `ЗащитаПерсональныхДанных` (см. `<Subsystems>/ЗащитаПерсональныхДанных.xml`) физически состоит из семи общих модулей с предсказуемой суффиксной системой:

| Модуль | Контекст | Назначение |
|---|---|---|
| `ЗащитаПерсональныхДанных` | Сервер | Основной API: уничтожение, скрытие, проверки, события доступа, обновление ИБ, формы |
| `ЗащитаПерсональныхДанныхКлиент` | Клиент | Обработчики оповещений форм, открытие формы согласия, клиентская часть настроек |
| `ЗащитаПерсональныхДанныхВызовСервера` | Клиент+Сервер (с вызовом сервера) | Запуск длительной операции смены настроек, серверные обёртки для клиента |
| `ЗащитаПерсональныхДанныхПереопределяемый` | Сервер | **Хуки** для прикладной конфигурации (заполнение состава ПДн, до/после уничтожения) — реализуются в прикладном коде, **не вызываются** напрямую |
| `ЗащитаПерсональныхДанныхПовтИсп` | Сервер+Кэш | Кэшированные справочники (типы субъектов, имена источников, категории) |
| `ЗащитаПерсональныхДанныхУничтожениеПовтИсп` | Сервер+Кэш | Кэшированные параметры для уничтожения |
| `ЗащитаПерсональныхДанныхСлужебный` | Сервер | ⚠️ **Служебный API** (обратная совместимость не гарантируется). Прямой вызов из прикладного кода — только если стабильного аналога нет |

Определяемые типы: `СубъектПерсональныхДанных` (ссылка), `СубъектПерсональныхДанныхОбъект` (объект).

### Ключевые регистры сведений

- `СогласияНаОбработкуПерсональныхДанных` — действующие и истёкшие согласия.
- `УничтоженныеПерсональныеДанные` — субъекты с уничтоженными ПДн + дата уничтожения.
- `ОбластиПерсональныхДанных` — защищаемые области (конфигуратор/администратор).
- `СрокиХраненияПерсональныхДанных`, `СубъектыДляРасчетаСроковХранения` — расчёт сроков.

### Документы и обработки

- `СогласиеНаОбработкуПерсональныхДанных`, `ОтзывСогласияНаОбработкуПерсональныхДанных`, `АктОбУничтоженииПерсональныхДанных` — журнал `СогласияНаОбработкуПерсональныхДанных`.
- `Обработка.УничтожениеПерсональныхДанных` — интерактивный список субъектов под уничтожение.
- `Обработка.СкрытиеКонфиденциальнойИнформации` — техническая обработка, используемая методом `УничтожитьПерсональныеДанныеСубъектов` внутри транзакции.

### Функциональная опция

`ИспользоватьСкрытиеПерсональныхДанныхСубъектов` (константа) — глобальный выключатель уничтожения/скрытия ПДн. По умолчанию выключен. Большинство методов подсистемы (`ЭтоОбъектСУничтоженнымиПерсональнымиДанными`, `СкрытьПерсональныеДанныеСубъектов`, добавление колонки в список) корректно отрабатывают и при выключенной опции (возвращают `Ложь` / не добавляют колонку) — но **настройки событий доступа и регламентное задание расчёта сроков** имеют смысл только при включённой опции.

### Устаревшие методы (deprecated)

`ЭтоОбъектСоСкрытымиПерсональнымиДанными` и `ИзменитьНастройкиСкрытияПерсональныхДанных` помечены в коде как «Устарела». Используйте `ЭтоОбъектСУничтоженнымиПерсональнымиДанными` и `ИзменитьНастройкиУничтоженияПерсональныхДанных` соответственно.

## Patterns

### 1. Проверка наличия действующего согласия перед записью

```bsl
// В модуле объекта-субъекта, в обработчике ПередЗаписью
Процедура ПередЗаписью(Отказ)
    Если ОбменДанными.Загрузка Тогда Возврат; КонецЕсли;
    Если Не ЭтоНовый() И Не ЭтоГруппа Тогда
        Согласие = ЗащитаПерсональныхДанных.ДействующееСогласиеНаОбработкуПерсональныхДанных(
            Ссылка, Организация, ТекущаяДатаСеанса());
        Если Согласие = Неопределено Тогда
            ОбщегоНазначения.СообщитьПользователю(
                НСтр("ru = 'Нет действующего согласия на обработку ПДн. Запись не рекомендуется.'"),
                , , , Отказ);
        КонецЕсли;
    КонецЕсли;
КонецПроцедуры
```

### 2. Интеграция подсистемы в форму списка субъектов

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    ЗащитаПерсональныхДанных.ПриСозданииНаСервереФормыСписка(ЭтаФорма, Элементы.Список);
КонецПроцедуры
&НаСервере
Процедура ПриПолученииДанныхНаСервере(ИмяСписка, Настройки, Строки)
    ЗащитаПерсональныхДанных.ПриПолученииДанныхНаСервере(Настройки, Строки);
КонецПроцедуры
&НаКлиенте
Процедура ОбработкаОповещения(ИмяСобытия, Параметр, Источник)
    ЗащитаПерсональныхДанныхКлиент.ОбработкаОповещенияФормыСписка(Элементы.Список, ИмяСобытия);
КонецПроцедуры
```

### 3. Программное уничтожение ПДн субъекта

```bsl
// В обработчике проведения акта об уничтожении ПДн
Процедура ОбработкаПроведения(Отказ, Режим)
    Если Не Отказ Тогда
        ЗащитаПерсональныхДанных.УничтожитьПерсональныеДанныеСубъектов(Субъекты);
    КонецЕсли;
КонецПроцедуры
```

### 4. Прикладной хук «отказаться от уничтожения конкретного субъекта»

```bsl
// В модуле ЗащитаПерсональныхДанныхПереопределяемый прикладной конфигурации
Процедура ПередСкрытиемПерсональныхДанныхСубъектов(Субъекты, ТаблицаИсключений, ОтказОтСкрытия) Экспорт
    Для Каждого Субъект Из Субъекты Цикл
        Если СубъектЯвляетсяДействующимСотрудником(Субъект) Тогда
            НоваяСтрока = ТаблицаИсключений.Добавить();
            НоваяСтрока.Субъект = Субъект;
            НоваяСтрока.ПричинаОтменыУничтожения = НСтр("ru = 'Субъект является действующим сотрудником.'");
        КонецЕсли;
    КонецЦикла;
КонецПроцедуры
```

Этот хук вызывается **самой подсистемой** перед уничтожением; прикладной код только **реализует** его в своём `ЗащитаПерсональныхДанныхПереопределяемый`. Прямой вызов из прикладного кода не нужен.

## Anti-patterns

### ❌ Прямой вызов методов из `ЗащитаПерсональныхДанныхСлужебный`

```bsl
// ❌ Служебный API — обратная совместимость не гарантируется
Сведения = ЗащитаПерсональныхДанныхСлужебный.СведенияОПерсональныхДанных();
```

### ❌ Универсальный «принудительный» DELETE через прямой запрос к регистрам

```bsl
// ❌ Обходит обработчики подсистемы, ломает регистрацию событий доступа
НаборЗаписей = РегистрыСведений.УничтоженныеПерсональныеДанные.СоздатьНаборЗаписей();
НаборЗаписей.Записать();
```

### ❌ Использование устаревших методов `СоСкрытымиПДн`

```bsl
// ❌ Устарел: используйте ...УничтоженнымиПерсональнымиДанными
Если ЗащитаПерсональныхДанных.ЭтоОбъектСоСкрытымиПерсональнымиДанными(Объект) Тогда
```

### ❌ Переопределение методов основного модуля

```bsl
// ❌ В прикладной конфигурации НЕ создавать свой «ЗащитаПерсональныхДанных»
```

```bsl
// ✅ Расширять только через ЗащитаПерсональныхДанныхПереопределяемый
// (реализовать хуки ПередУничтожением / ПослеУничтожения / ЗаполнитьСведенияОПерсональныхДанных)
```

### ❌ Игнорирование функциональной опции

