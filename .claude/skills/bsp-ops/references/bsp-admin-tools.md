
Скил по подсистемам **ЗавершениеРаботыПользователей** (управление сеансами: блокировка ИБ, завершение работы пользователей, информация о соединениях) и **УдалениеПомеченныхОбъектов** (программное и интерактивное удаление помеченных объектов с контролем ссылочной целостности, настройка отображения помеченных в формах списков, регламентное удаление по расписанию). Обе подсистемы относятся к административным задачам ИБ.

## When to use

- Нужно **заблокировать** соединения с ИБ перед обновлением — `СоединенияИБ.УстановитьБлокировкуСоединений`.
- Нужно **снять блокировку** ИБ — `СоединенияИБ.РазрешитьРаботуПользователей`.
- Нужно проверить, **установлена ли блокировка** — `СоединенияИБ.УстановленаБлокировкаСоединений`.
- Нужно получить **параметры блокировки** — `СоединенияИБ.ПараметрыБлокировкиСеансов`.
- Нужно получить **информацию о текущих соединениях** — `СоединенияИБ.ИнформацияОСоединениях`.
- Нужно **включить/выключить режим завершения** работы пользователей — `СоединенияИБКлиент.УстановитьРежимЗавершенияРаботыПользователей`.
- Нужно **получить список помеченных на удаление** — `УдалениеПомеченныхОбъектов.ПомеченныеНаУдаление`.
- Нужно **программно удалить** помеченные объекты — `УдалениеПомеченныхОбъектов.УдалитьПомеченныеОбъекты`.
- Нужно запустить **интерактивное удаление** — `УдалениеПомеченныхОбъектовКлиент.НачатьУдалениеПомеченных`.
- Нужно реализовать **переопределяемые крючки** перед/после удаления — `УдалениеПомеченныхОбъектовПереопределяемый.ПередУдалениемГруппыОбъектов`.
- Нужно переопределить **параметры блокировки сеансов** — `СоединенияИБПереопределяемый`.

## Не использовать, если

- Нужно **обычно записать/прочитать объект** — `bsp-base-common`.
- Нужно **запустить фоновое задание** — `bsp-longs-and-jobs`.
- Нужно **обновить конфигурацию или ИБ** — `bsp-update`.
- Нужно **создать резервную копию** ИБ — `bsp-backup`.

## Core concepts

### ЗавершениеРаботыПользователей — карта модулей

| Модуль | Контекст | Назначение |
|---|---|---|
| `СоединенияИБ` | Сервер | **Стабильный API**: блокировка, информация, параметры |
| `СоединенияИБКлиент` | Клиент | **Стабильный клиентский API**: режим завершения, параметры администрирования |
| `СоединенияИБПереопределяемый` | Сервер | **Переопределяемый**: модификация параметров блокировки |

> **Блокировка ИБ vs блокировка сеансов области данных.** `УстановитьБлокировкуСоединений` блокирует всю ИБ. `УстановитьБлокировкуСеансовОбластиДанных` — одну область данных в модели сервиса.

### УдалениеПомеченныхОбъектов — карта модулей

| Модуль | Контекст | Назначение |
|---|---|---|
| `УдалениеПомеченныхОбъектов` | Сервер | **Стабильный API**: получение помеченных, удаление, настройки |
| `УдалениеПомеченныхОбъектовКлиент` | Клиент | **Стабильный клиентский API**: интерактивное удаление, видимость в формах |
| `УдалениеПомеченныхОбъектовПереопределяемый` | Сервер | **Переопределяемый**: крючки перед/после удаления |
| `УдалениеПомеченныхОбъектовСлужебный` | Сервер | ⚠️ Служебный: ядро удаления |

### Режимы удаления

| Режим | Описание |
|---|---|
| `"Стандартный"` | Контроль ссылочной целостности + многопользовательская работа |
| `"Монопольный"` | Контроль ссылочной целостности с монопольным режимом |
| `"Упрощённый"` | Контроль ссылочной целостности только в непомеченных объектах |

### Событийная модель удаления

1. `ПередУдалениемГруппыОбъектов(Контекст, УдаляемыеОбъекты)` — вне транзакции
2. Удаление группы в транзакции
3. `ПослеУдаленияГруппыОбъектов(Контекст, Успешно)` — вне транзакции

## Patterns

### 1. Блокировка ИБ с гарантированным снятием

```bsl
Успех = СоединенияИБ.УстановитьБлокировкуСоединений("Техработы", "Код", 5, 60);
Если Не Успех Тогда ВызватьИсключение "Не удалось заблокировать"; КонецЕсли;
Попытка
    // ... администрирование ...
Исключение
    СоединенияИБ.РазрешитьРаботуПользователей();
    ВызватьИсключение;
КонецПопытки;
СоединенияИБ.РазрешитьРаботуПользователей();
```

### 2. Программное удаление помеченных

```bsl
Помеченные = УдалениеПомеченныхОбъектов.ПомеченныеНаУдаление();
Если Помеченные.Количество() = 0 Тогда Возврат; КонецЕсли;
Отбор = Новый СписокЗначений;
Отбор.Добавить("Справочник.Контрагенты");
ПомеченныеКонтр = УдалениеПомеченныхОбъектов.ПомеченныеНаУдаление(Отбор);
Результат = УдалениеПомеченныхОбъектов.УдалитьПомеченныеОбъекты(ПомеченныеКонтр, "Стандартный");
Если Не Результат.Успешно Тогда
    // обработка препятствующих ссылок
КонецЕсли;
```

### 3. Интеграция видимости помеченных в форму списка

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    УдалениеПомеченныхОбъектов.ПриСозданииНаСервере(ЭтотОбъект, Элементы.Список);
КонецПроцедуры
```

## Anti-patterns

### ❌ Не снимать блокировку при ошибке

```bsl
// ❌ Если код упадёт — ИБ останется заблокированной
СоединенияИБ.УстановитьБлокировкуСоединений("Обновление");
// ... долгая обработка ...
СоединенияИБ.РазрешитьРаботуПользователей();
```

### ❌ Вызывать `УдалитьПомеченныеОбъекты` внутри транзакции

```bsl
// ❌ Подсистема сама управляет транзакциями
НачатьТранзакцию();
Результат = УдалениеПомеченныхОбъектов.УдалитьПомеченныеОбъекты(Массив);
ЗафиксироватьТранзакцию();
```

### ❌ Игнорировать результат удаления

```bsl
// ❌ Объекты могут остаться неудалёнными
УдалениеПомеченныхОбъектов.УдалитьПомеченныеОбъекты(Массив);
```

### ❌ Вызывать `ПоказатьПараметрыАдминистрирования` без проверки

```bsl
// ✅ Проверить существование подсистемы
Если ОбщегоНазначенияКлиент.ПодсистемаСуществует("СтандартныеПодсистемы.ЗавершениеРаботыПользователей") Тогда
    СоединенияИБКлиент.ПоказатьПараметрыАдминистрирования(Оповещение);
КонецЕсли;
```
