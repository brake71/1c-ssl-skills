
Скил по четырём связанным коммуникационным подсистемам БСП 3.1.11 — **РаботаСПочтовымиСообщениями** (отправка / загрузка email), **ОтправкаSMS** (через внешнего провайдера), **ШаблоныСообщений** (подготовка писем и SMS из шаблонов с подстановкой параметров) и **Обсуждения** (чат через систему взаимодействия 1С:Диалог, интеграции с Telegram / ВКонтакте / WhatsApp / Webhook). Объединены, потому что все четыре решают задачу «отправить сообщение пользователю» — разными каналами.

**Гетерогенная группировка:** четыре подсистемы с разными API и областями ответственности. Если потребуется углубиться в один из каналов — см. `## How to explore deeper` в конце скила.

## When to use

- Нужно **отправить email** через настроенную учётную запись (SMTP) программно из серверного кода — `РаботаСПочтовымиСообщениями.ОтправитьПисьмо` / `ОтправитьПисьма` (пакетная отправка).
- Нужно **загрузить входящие письма** с IMAP/POP3 сервера в таблицу значений — `РаботаСПочтовымиСообщениями.ЗагрузитьПочтовыеСообщения` (возвращает `ТаблицаЗначений` с колонками `Тема / Отправитель / ДатаОтправления / Вложения / Тексты`).
- Нужно **проверить доступность канала**: есть ли настроенная учётная запись для отправки (`ДоступнаОтправкаПисем`), сконфигурирован ли провайдер SMS (`НастройкаОтправкиSMSВыполнена`, `ДоступнаОтправкаSMS`), подключены ли обсуждения (`ОбсужденияДоступны`).
- Нужно **получить список учётных записей** электронной почты с фильтром по назначению (только для отправки / только для получения) — `РаботаСПочтовымиСообщениями.ДоступныеУчетныеЗаписи`.
- Нужно **открыть форму нового письма** на клиенте с предзаполненными получателями / темой / вложениями — `РаботаСПочтовымиСообщениямиКлиент.СоздатьНовоеПисьмо` (при отсутствии учётной записи автоматически откроет мастер настройки).
- Нужно **отправить SMS** через настроенного провайдера (SMS4B / SMS.RU / SMS-ЦЕНТР / Билайн / МТС) программно — `ОтправкаSMS.ОтправитьSMS` (сервер) или `ОтправкаSMSКлиент.ОтправитьSMS` (клиент, с проверкой настроек).
- Нужно **сформировать сообщение по шаблону** (email или SMS) с подстановкой реквизитов предмета (документ, контрагент, организация) — `ШаблоныСообщений.СформироватьСообщение` (сервер) или `ШаблоныСообщенийКлиент.СформироватьСообщение` (открывает интерактивную форму выбора).
- Нужно **сформировать и сразу отправить** сообщение по шаблону — `ШаблоныСообщений.СформироватьСообщениеИОтправить`.
- Нужно **выбрать шаблон** сообщения в форме с отбором по виду (email / SMS) и предмету — `ШаблоныСообщенийКлиент.ВыбратьШаблон`.
- Нужно **отправить сообщение в обсуждение** (система взаимодействия) пользователю или группе — `Обсуждения.ОтправитьСообщение` (предварительно сформировав описание через `Обсуждения.ОписаниеСообщения`).
- Нужно **подключить / отключить обсуждения** из прикладной формы (включая интеграции с Telegram / ВКонтакте / WhatsApp / Webhook) — `ОбсужденияКлиент.ПоказатьПодключение` / `ПоказатьОтключение`.

## Не использовать, если

- Нужны общие утилиты (сериализация, безопасное хранилище, форматирование строк, безопасное хранилище паролей) — `bsp-base-common`. Здесь только каналы коммуникации.
- Нужно **обернуть долгий вызов** отправки в фоновое задание (для админ-операций или вебхуков) — `bsp-longs-and-jobs` (`ДлительныеОперации.ВыполнитьПроцедуру` / `ВыполнитьФункцию`).
- Нужно **распечатать документ и приложить файлом** к письму — `bsp-print-reports` (подсистема `Печать`); скил `bsp-comms` отвечает только за **отправку**, не за формирование печатных форм.
- Нужны **адреса / телефоны / ОКТМО / ФИАС** — `bsp-contact-info` (подсистема `КонтактнаяИнформация`); здесь нет хранения адресов, только **отправка сообщений**.
- Нужно **настроить учётную запись электронной почты** через UI — это интерактивная операция в форме списка «Учётные записи электронной почты», не программный API; программный путь — только создание элемента справочника `УчетныеЗаписиЭлектроннойПочты` (если действительно нужно программно).
- Нужно **управлять правами** на отправку / чтение почты, SMS, шаблонов — роли БСП (`ОтправкаSMS`, `ДобавлениеИзменениеШаблоновСообщений`, `ЧтениеЭлектроннойПочты`); роли устанавливаются в Конфигураторе / из кода через `УправлениеДоступом`.
- Нужно **загружать классификаторы провайдеров SMS** — это разовая интерактивная операция в форме настроек.
- Нужно **работать с подсистемой `Взаимодействия`** (хранение входящих/исходящих писем в базе, CRM-сценарии, привязка к документам) — это **отдельная** подсистема (`Взаимодействия`), не покрыта этим скилом.

## Core concepts

### Четыре подсистемы — разные каналы, общая задача

Все четыре подсистемы решают «отправить сообщение пользователю», но различаются по:

| Подсистема | Канал | Внешняя зависимость | Требует подключения |
|---|---|---|---|
| `РаботаСПочтовымиСообщениями` | Email (SMTP/IMAP/POP3) | Учётная запись электронной почты в справочнике `УчетныеЗаписиЭлектроннойПочты` | Нет, работает сразу |
| `ОтправкаSMS` | SMS через провайдера | Настройки провайдера (SMS4B, SMS.RU, SMS-ЦЕНТР, Билайн, МТС) | Да, настройка провайдера |
| `ШаблоныСообщений` | Универсальный (Email + SMS + произвольный) | Справочник `ШаблоныСообщений` с заполненными реквизитами | Нет, но нужен хотя бы один шаблон |
| `Обсуждения` | Чат (1С:Диалог, Telegram, ВК, WhatsApp) | Сервер 1С:Диалог или локальный сервер взаимодействия | Да, требуется `Подключить обсуждения` в админ-разделе |

### Разделение API на стабильный / служебный

В каждом модуле БСП есть `#Область ПрограммныйИнтерфейс` (стабильный, можно вызывать из прикладного кода) и `#Область СлужебныйПрограммныйИнтерфейс` / `#Область СлужебныеПроцедурыИФункции` (служебный, ⚠️ обратная совместимость не гарантируется). В таблице `## Key methods` методы из `*Служебный` модулей помечены ⚠️.

Большинство «публичных» методов подсистем — это **тонкие обёртки** над `*Служебный` модулями. Например, `РаботаСПочтовымиСообщениями.ОтправитьПисьмо` (стабильный) сразу вызывает `РаботаСПочтовымиСообщениямиСлужебный.ОтправитьПисьмо` (служебный). **Правило:** из прикладного кода вызывать **только** стабильный слой. `*Служебный` — только если стабильного аналога нет, с явной отметкой в коде.

### Суффиксная система модулей (без префикса)

Общие модули БСП именуются по подсистеме без префикса, с суффиксами контекста (`bsp-fundamentals`):

| Суффикс | Контекст |
|---|---|
| (без суффикса) | Сервер, Толстый клиент, Внешнее соединение |
| `Клиент` | Тонкий / Толстый клиент (UI-обработчики, диалоги) |
| `КлиентСервер` | Клиент + Сервер (безопасный код) |
| `Служебный` | ⚠️ Служебный серверный |
| `СлужебныйКлиент` | ⚠️ Служебный клиентский |
| `Переопределяемый` | «Крючки» для прикладной конфигурации |
| `ПовтИсп` | Кэш повторного использования |

### Карта «подсистема → модули»

**`РаботаСПочтовымиСообщениями`** — общие модули:
- `РаботаСПочтовымиСообщениями` — стабильный серверный API (отправка, загрузка, проверки, учётные записи).
- `РаботаСПочтовымиСообщениямиКлиент` — стабильный клиентский (открытие форм, проверка учётной записи).
- `РаботаСПочтовымиСообщениямиВызовСервера` — клиентский с серверным вызовом без контекста формы.
- `РаботаСПочтовымиСообщениямиПереопределяемый` — «крючки».
- `РаботаСПочтовымиСообщениямиКлиентПереопределяемый` — клиентские «крючки».
- `РаботаСПочтовымиСообщениямиЛокализация` — региональные особенности.
- `РаботаСПочтовымиСообщениямиСлужебный` — ⚠️ служебный серверный (низкоуровневая работа с SMTP/IMAP, профиль, токены).
- `РаботаСПочтовымиСообщениямиСлужебныйКлиент` — ⚠️ служебный клиентский.
- `РаботаСПочтовымиСообщениямиСлужебныйКлиентСервер` — ⚠️ служебный общий.

**`ОтправкаSMS`** — общие модули:
- `ОтправкаSMS` — стабильный серверный API (отправка, статусы, настройки).
- `ОтправкаSMSКлиент` — стабильный клиентский (UI отправки с проверкой настроек).
- `ОтправкаSMSПереопределяемый` — «крючки» для нестандартного провайдера.
- `ОтправкаSMSКлиентПереопределяемый` — клиентские «крючки».
- `ОтправкаSMSПовтИсп` — кэш настроек провайдера.
- `ОтправкаSMSЧерезSMS4B` / `ОтправкаSMSЧерезSMSRU` / `ОтправкаSMSЧерезSMSЦЕНТР` — ⚠️ служебные модули провайдеров (для добавления своего — реализовать интерфейс в `ОтправкаSMSПереопределяемый`).

**`ШаблоныСообщений`** — общие модули:
- `ШаблоныСообщений` — стабильный серверный API (формирование сообщений, параметры, инициализация структур).
- `ШаблоныСообщенийКлиент` — стабильный клиентский (открытие форм, выбор шаблона, формирование сообщения из формы).
- `ШаблоныСообщенийКлиентСервер` — безопасный код (описание параметров шаблона).
- `ШаблоныСообщенийПереопределяемый` — «крючки» для формирования сообщений.
- `ШаблоныСообщенийСлужебный` — ⚠️ служебный серверный (низкоуровневая работа с шаблонами, СКД-параметры, конструктор сообщения).

**`Обсуждения`** — общие модули:
- `Обсуждения` — стабильный серверный API (отправка сообщений, описание, пользователи, проверки подключения).
- `ОбсужденияКлиент` — стабильный клиентский (подключение, отключение, проверка доступности).
- `ОбсужденияЛокализация` — региональные особенности.
- `ОбсужденияСлужебный` — ⚠️ служебный серверный (блокировки, интеграции, обновление пользователей).
- `ОбсужденияСлужебныйКлиент` — ⚠️ служебный клиентский (команды, предупреждения, открытие вложений).
- `ОбсужденияСлужебныйКлиентСервер` — ⚠️ служебный общий (типы внешних систем).
- `ОбсужденияСлужебныйВызовСервера` — ⚠️ служебный серверный вызов.

⚠️ **Не существует:** `РаботаСПочтовымиСообщениямиСервер`, `ОтправкаSMSСервер`, `ШаблоныСообщенийСервер` (без суффикса — есть только базовые модули с именем подсистемы).

### «Тесная интеграция» с другими подсистемами

- **`Длительные операции`** — массовая отправка писем / SMS (рассылки) или проверка подключения (`РежимТестирования = Истина` в `ЗагрузитьПочтовыеСообщения`) занимает секунды — оборачивать в `ДлительныеОперации.ВыполнитьФункцию` (`bsp-longs-and-jobs`).
- **`Безопасное хранилище`** — пароли учётных записей электронной почты хранятся через `ОбщегоНазначения.ЗаписатьДанныеВБезопасноеХранилище` / `ПрочитатьДанныеИзБезопасногоХранилища` (`bsp-base-common`); пароль = `Владелец = УчетнаяЗапись.Ссылка`, ключ = `"Пароль"`.
- **`Пользователи`** — в `Обсуждения.ОтправитьСообщение` автор и получатели передаются как `СправочникСсылка.Пользователи`; внутри модуль сам маппит их в `ПользовательСистемыВзаимодействия`.
- **`Внешние компоненты`** — провайдеры SMS (особенно Билайн / МТС) могут требовать внешнюю компоненту для подписи запросов; при ошибках подключения проверить `ОтправкаSMS.ДополнительныеРазрешения` и права на внешние ресурсы.
- **`Регламентные задания`** — `Получение и отправка электронных писем` (в `bsp-longs-and-jobs`) периодически забирает письма; ручной вызов `ЗагрузитьПочтовыеСообщения` — для разовых загрузок.

Возвращает `Структура`: `ОтправленныеСообщения` (массив `НомерПолучателя / ИдентификаторСообщения`) + `ОписаниеОшибки` (пустая строка — успех). Перед вызовом проверить `ОтправкаSMS.НастройкаОтправкиSMSВыполнена()` | `Рез = ОтправкаSMS.ОтправитьSMS(МассивНомеров, "Текст SMS"); Если Рез.ОписаниеОшибки <> "" Тогда ... КонецЕсли;` | стабильный |
| `ОтправкаSMS.ДоступнаОтправкаSMS` | `ДоступнаОтправкаSMS()` | Сервер, Толстый клиент, Внешнее соединение | Проверить, настроена ли отправка и есть ли у текущего пользователя права. Использовать перед UI-командой «Отправить SMS» | `Если ОтправкаSMS.ДоступнаОтправкаSMS() Тогда // открыть форму отправки` | стабильный |
| `ОтправкаSMSКлиент.ОтправитьSMS` | `ОтправитьSMS(НомераПолучателей, Текст, ДополнительныеПараметры)` | Клиент | Клиентский аналог `ОтправкаSMS.ОтправитьSMS`. Перед отправкой проверяет настройки провайдера — при отсутствии открывает мастер настройки. `ДополнительныеПараметры` — `Структура` с `ИмяОтправителя` / `ПеревестиВТранслит` | `ОтправкаSMSКлиент.ОтправитьSMS(МассивНомеров, "Текст", Новый Структура("ПеревестиВТранслит", Истина));` | стабильный |
| `ШаблоныСообщений.СформироватьСообщение` | `СформироватьСообщение(Шаблон, Предмет, УникальныйИдентификатор, ДополнительныеПараметры = Неопределено)` | Сервер, Толстый клиент, Внешнее соединение | Сформировать сообщение (email / SMS / произвольное) по шаблону с подстановкой реквизитов `Предмет` (документ, контрагент, организация и т. п. — типы из `ОпределяемыйТип.ПредметШаблонаСообщения`). `ДополнительныеПараметры.ПреобразовыватьHTMLДляФорматированногоДокумента` — нужен для вывода в форматированный документ. Возвращает структуру с `Тема / Текст / Получатель / Вложения` | `Сообщение = ШаблоныСообщений.СформироватьСообщение(ШаблонСсылка, ЗаказСсылка, УникальныйИдентификатор);` | стабильный |
| `ШаблоныСообщений.СформироватьСообщениеИОтправить` | `СформироватьСообщениеИОтправить(Шаблон, Предмет, УникальныйИдентификатор, ДополнительныеПараметры = Неопределено)` | Сервер, Толстый клиент, Внешнее соединение | Сформировать **и сразу отправить** сообщение по шаблону. `ДополнительныеПараметры.УчетнаяЗапись` — какую учётку использовать; `ОтправитьСразу = Истина` — отправить немедленно, иначе письмо попадёт в папку исходящих (отправится при общей отправке) | `Рез = ШаблоныСообщений.СформироватьСообщениеИОтправить(ШаблонСсылка, ЗаказСсылка, УИД, ШаблоныСообщений.ПараметрыОтправкиПисьмаПоШаблону());` | стабильный |
| `ШаблоныСообщенийКлиент.СформироватьСообщение` | `СформироватьСообщение(ПредметСообщения, ВидСообщения, ОписаниеОповещенияОЗакрытии = Неопределено, ВладелецШаблона = Неопределено, ПараметрыСообщения = Неопределено)` | Клиент | Открыть **интерактивную** форму формирования сообщения по шаблону. `ВидСообщения` — `"Письмо"` (email) или `"СообщениеSMS"`. После закрытия формы — `ОписаниеОповещенияОЗакрытии` с результатом | `ШаблоныСообщенийКлиент.СформироватьСообщение(ЗаказСсылка, "Письмо", Новый ОписаниеОповещения("ПослеФормирования", ЭтаФорма));` | стабильный |
| `Обсуждения.ОтправитьСообщение` | `ОтправитьСообщение(Знач Автор, Знач Получатели, Сообщение, ОбсуждениеКонтекст = Неопределено)` | Сервер, Толстый клиент, Внешнее соединение | Отправить сообщение в обсуждение (1С:Диалог). `Автор` — `СправочникСсылка.Пользователи` или `ПользовательСистемыВзаимодействия`. `Получатели` — массив пользователей / `ПользовательСистемыВзаимодействия`. `Сообщение` — структура из `Обсуждения.ОписаниеСообщения`. `ОбсуждениеКонтекст` — `Неопределено` (создаётся негрупповое обсуждение 1:1), `ИдентификаторОбсужденияСистемыВзаимодействия` (в существующее) или `ЛюбаяСсылка` (создаётся контекстное по навигационной ссылке объекта) | `Обсуждения.ОтправитьСообщение(ТекущийПользователь, МассивПолучателей, Обсуждения.ОписаниеСообщения("Привет!"));` | стабильный |
| `Обсуждения.ОписаниеСообщения` | `ОписаниеСообщения(Знач Текст)` | Сервер, Толстый клиент, Внешнее соединение | Создать **описание** сообщения для `Обсуждения.ОтправитьСообщение`. `Текст` — `Строка` или `ФорматированнаяСтрока`. Возвращает структуру с `Текст / Вложения / Данные / Действия`. Для вложений использовать `Обсуждения.ОписаниеВложения(Поток, Наименование)` | `Описание = Обсуждения.ОписаниеСообщения("Текст сообщения"); Описание.Вложения.Добавить(Обсуждения.ОписаниеВложения(ПотокФайла, "report.pdf"));` | стабильный |
| `Обсуждения.ОбсужденияДоступны` | `ОбсужденияДоступны()` | Сервер, Толстый клиент, Внешнее соединение | Проверить, подключены ли обсуждения в текущей конфигурации. Использовать перед `Обсуждения.ОтправитьСообщение` или перед открытием UI-формы | `Если Обсуждения.ОбсужденияДоступны() Тогда // показать кнопку чата` | стабильный |

## Patterns

### 1. Отправить одно письмо с обработкой ошибки

```bsl
Письмо = Новый ИнтернетПочтовоеСообщение;
Письмо.Тема = НСтр("ru = 'Счёт на оплату'");
Письмо.Тексты.Добавить(НСтр("ru = 'Здравствуйте, высылаем счёт.'"));
Письмо.Получатели.Добавить(Новый ИнтернетПочтовыйАдрес("client@example.com"));

Попытка
    РаботаСПочтовымиСообщениями.ОтправитьПисьмо(УчетнаяЗапись, Письмо);
Исключение
    ОбщегоНазначения.СообщитьПользователю(
        НСтр("ru = 'Не удалось отправить письмо.'"),
        , , , Отказ);
КонецПопытки;
```

`ОтправитьПисьмо` **может вызвать исключение** при сетевых ошибках / неверных настройках / превышении лимита провайдера — обязательно `Попытка / Исключение`. Пароль учётной записи автоматически берётся из безопасного хранилища (`bsp-base-common`).

### 2. Проверить доступность канала перед UI-командой

```bsl
&НаКлиенте
Процедура КомандаОтправитьПисьмо(Команда)
    Если РаботаСПочтовымиСообщениями.ДоступнаОтправкаПисем() Тогда
        РаботаСПочтовымиСообщениямиКлиент.СоздатьНовоеПисьмо(
            РаботаСПочтовымиСообщениямиКлиент.ПараметрыОтправкиПисьма(),
            Новый ОписаниеОповещения("ПослеОтправкиПисьма", ЭтаФорма));
    КонецЕсли;
КонецПроцедуры
```

`ДоступнаОтправкаПисем` сам проверит и наличие настроенной учётки, и права. При отсутствии учётки `СоздатьНовоеПисьмо` сам откроет мастер настройки.

### 3. Отправить SMS с проверкой настроек

```bsl
МассивНомеров = Новый Массив;
МассивНомеров.Добавить("+79991234567");
МассивНомеров.Добавить("+79997654321");

Результат = ОтправкаSMS.ОтправитьSMS(МассивНомеров, "Ваш заказ подтверждён.");

Если НЕ ПустаяСтрока(Результат.ОписаниеОшибки) Тогда
    ОбщегоНазначения.СообщитьПользователю(
        СтроковыеФункцииКлиентСервер.ПодставитьПараметрыВСтроку(
            НСтр("ru = 'Ошибка отправки SMS: %1'"),
            Результат.ОписаниеОшибки));
Иначе
    // Опционально: сохранить ИдентификаторыСообщений для отслеживания доставки
    Для Каждого Отправленное Из Результат.ОтправленныеСообщения Цикл
        // Отправленное.НомерПолучателя, Отправленное.ИдентификаторСообщения
    КонецЦикла;
КонецЕсли;
```

Перед вызовом рекомендуется проверить `ОтправкаSMS.НастройкаОтправкиSMSВыполнена()` — если провайдер не сконфигурирован, метод вернёт `ОписаниеОшибки` (но **не** выбросит исключение).

### 4. Сформировать сообщение по шаблону (сервер)

```bsl
Шаблон = Справочники.ШаблоныСообщений.НайтиПоНаименованию("Уведомление об отгрузке");

ДопПараметры = ШаблоныСообщений.ПараметрыОтправкиПисьмаПоШаблону();
ДопПараметры.УчетнаяЗапись   = УчетнаяЗапись;
ДопПараметры.ОтправитьСразу  = Истина;

Результат = ШаблоныСообщений.СформироватьСообщениеИОтправить(
    Шаблон,                      // ссылка на шаблон
    ЗаказКлиента,                // предмет — реквизиты подставляются из него
    УникальныйИдентификатор,     // для вложений во временном хранилище
    ДопПараметры);

// Результат содержит Тема / Текст / Получатель / Вложения / СообщенияПользователю
```

`ПараметрыОтправкиПисьмаПоШаблону()` — фабричный метод, возвращает структуру с правильными ключами. `УникальныйИдентификатор` — обычно `ЭтаФорма.УникальныйИдентификатор` или `Новый УникальныйИдентификатор()` для серверных вызовов без формы.

### 5. Отправить сообщение в обсуждение (1С:Диалог)

```bsl
ТекстСообщения = НСтр("ru = 'Документ согласован, можно приступать к отгрузке.'");

Описание = Обсуждения.ОписаниеСообщения(ТекстСообщения);
// Можно добавить вложение:
// Описание.Вложения.Добавить(Обсуждения.ОписаниеВложения(ПотокФайла, "akt.pdf"));

МассивПолучателей = Новый Массив;
МассивПолучателей.Добавить(Ответственный);
МассивПолучателей.Добавить(Кладовщик);

Обсуждения.ОтправитьСообщение(
    Пользователи.ТекущийПользователь(),  // автор
    МассивПолучателей,                    // получатели
    Описание,                             // сообщение
    ЗаказКлиента);                       // контекст — обсуждение будет привязано к документу
```

Когда `ОбсуждениеКонтекст` — `ЛюбаяСсылка` (например, документ), создаётся **контекстное обсуждение**, привязанное к этому объекту (видно в его форме). При `Неопределено` и одном получателе — негрупповое обсуждение 1:1. При `Неопределено` и нескольких — групповое обсуждение.

## Anti-patterns

### ❌ Вызывать `ОтправитьПисьмо` без `Попытка / Исключение`

```bsl
// ❌ При сетевой ошибке / неверном пароле — неперехваченное исключение
РаботаСПочтовымиСообщениями.ОтправитьПисьмо(Учетка, Письмо);
```

```bsl
// ✅ Метод документированно «может вызвать исключение»
Попытка
    РаботаСПочтовымиСообщениями.ОтправитьПисьмо(Учетка, Письмо);
Исключение
    ОбщегоНазначения.СообщитьПользователю(НСтр("ru='Не удалось отправить.'"), , , , Отказ);
КонецПопытки;
```

### ❌ Прятать пароль учётной записи электронной почты в коде

```bsl
// ❌ Своя обёртка для пароля — обходит безопасное хранилище
Учетка.Пароль = "secret123";
Учетка.Записать();
```

```bsl
// ✅ Пароль — в безопасном хранилище (bsp-base-common)
ОбщегоНазначения.ЗаписатьДанныеВБезопасноеХранилище(Учетка, "secret123", "Пароль");
// При отправке ОтправитьПисьмо сам прочитает пароль из хранилища
```

### ❌ Вызывать `ОтправкаSMS.ОтправитьSMS` без проверки настроек

```bsl
// ❌ Если провайдер не сконфигурирован — метод вернёт ОписаниеОшибки,
//    но при отладке это легко пропустить
ОтправкаSMS.ОтправитьSMS(Номера, "Текст");
```

```bsl
// ✅ Сначала проверить готовность провайдера
Если ОтправкаSMS.НастройкаОтправкиSMSВыполнена() Тогда
    Рез = ОтправкаSMS.ОтправитьSMS(Номера, "Текст");
    Если НЕ ПустаяСтрока(Рез.ОписаниеОшибки) Тогда
        // ... обработка ошибки
    КонецЕсли;
Иначе
    // Показать форму настроек или сообщить о необходимости настроить
    ОтправкаSMSКлиент.ОткрытьФормуНастроек(Новый ОписаниеОповещения("ПослеНастройкиSMS", ЭтаФорма));
КонецЕсли;
```

### ❌ Парсить строки получателей вручную для `ОтправитьSMS`

```bsl
// ❌ Своя логика разделения — не учитывает формат `+7XXX...` и пробелы
СтрокаНомеров = "+7 999 123-45-67, +7 999 765-43-21";
МассивНомеров = СтрРазделить(СтрокаНомеров, ",");
```

```bsl
// ✅ Использовать методы из подсистемы контактной информации
Номера = Новый Массив;
Номера.Добавить(УправлениеКонтактнойИнформацией.ПредставлениеКонтактнойИнформацииОбъекта(
    Контрагент, ВидТелефона));
// или просто нормализовать строку в +7XXXXXXXXXX до вызова
```

### ❌ Вызывать `ОтправитьСообщение` обсуждений без `ОписаниеСообщения`

```bsl
// ❌ Передавать голую строку — параметр `Сообщение` имеет тип `Структура`
Обсуждения.ОтправитьСообщение(Автор, Получатели, "Привет!");
```

```bsl
// ✅ Обязательно оборачивать в ОписаниеСообщения (для вложений, действий, данных)
Описание = Обсуждения.ОписаниеСообщения("Привет!");
Обсуждения.ОтправитьСообщение(Автор, Получатели, Описание);
```

### ❌ Дублировать логику шаблонов собственными обработками

```bsl
// ❌ Своя подстановка реквизитов в текст шаблона
Текст = СтрЗаменить(Шаблон.Текст, "[Номер]", Документ.Номер);
Текст = СтрЗаменить(Текст, "[Дата]", Формат(Документ.Дата, "ДЛФ=D"));
// ...50 параметров...
```

```bsl
// ✅ Использовать ШаблоныСообщений.СформироватьСообщение — поддерживает СКД, реквизиты-колонки, вложенные объекты
Сообщение = ШаблоныСообщений.СформироватьСообщение(ШаблонСсылка, Документ, УникальныйИдентификатор);
```

Шаблоны БСП поддерживают параметры СКД (динамический набор реквизитов), реквизиты табличных частей, условное форматирование — ручная подстановка этих возможностей не даст.

### ❌ Вызывать `*Служебный` модули без явной отметки

```bsl
// ❌ Тихо вызывается служебный API — обратная совместимость не гарантируется
ШаблоныСообщенийСлужебный.СформироватьСообщение(ПараметрыОтправки);
```

```bsl
// ✅ Если стабильного аналога не хватает — пометить комментарием ⚠️
// ⚠️ Служебный API БСП, стабильного аналога нет:
// ШаблоныСообщенийСлужебный.СформироватьСообщение(ПараметрыОтправки);
```


Для поиска методов/модулей в выгрузке конфигурации используйте python scripts/bsp_ops_search.py method <имя>.
