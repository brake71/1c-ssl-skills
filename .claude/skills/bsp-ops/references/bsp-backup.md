
Скил по подсистеме `РезервноеКопированиеИБ` библиотеки стандартных подсистем 3.1.11. Покрывает программный интерфейс файлового резервного копирования «из коробки» (UI-форма, сохранение/чтение настроек, вычисление даты следующего копирования, навигационная ссылка на обработку), а также единственный переопределяемый крючок подсистемы.

Главная задача скила — научить агента отличать «ручной запуск UI-формы» от «программной работы с настройками», знать про разделение «файловый режим / клиент-сервер», не выдумывать несуществующие служебные модули, и корректно переопределять показ предупреждений о необходимости настройки копирования.

## When to use

- Открыть **форму создания/восстановления резервной копии** из прикладной команды, из обработки или из формы объекта — в том числе с предзаполненным каталогом или режимом работы.
- **Сохранить настройки резервного копирования** (расписание, каталог, период хранения, пароль администратора) программно — например, из обработки первоначального заполнения или из сценария миграции.
- **Прочитать текущие настройки** резервного копирования, чтобы показать их в форме, в отчёте или в уведомлении.
- **Получить текущую настройку строкой** — готовую формулировку для UI («копирование по расписанию», «выполняется при завершении работы», «не настроено» и т. п.).
- **Сформировать навигационную ссылку** на обработку резервного копирования (для подстановки в форматированную строку, в оповещение или в email).
- **Вычислить дату следующего автоматического копирования** — например, чтобы показать пользователю в карточке настроек «следующее копирование: …».
- **Переопределить поведение подсистемы** в прикладной конфигурации: отключить показ предупреждений о необходимости настройки копирования (если в компании бэкапы делаются сторонними средствами и предупреждения мешают).

## Не использовать, если

- Нужна **обычная запись/чтение объектов, работа с хранилищем общих настроек, безопасное хранилище, журнал регистрации** — это `bsp-base-common` (`ОбщегоНазначения.ХранилищеОбщихНастроекСохранить`, `ОбщегоНазначения.СообщитьПользователю` и т. п.). Метод `УстановитьНастройкиРезервногоКопирования` — узкоспециализированная обёртка над хранилищем общих настроек с фиксированным ключом `ПараметрыРезервногоКопирования`; для произвольных настроек используйте прямые API из `bsp-base-common`.
- Нужно **создать или изменить регламентное задание** (например, для регулярного архивирования файлов в произвольный каталог через сторонний скрипт) — `bsp-longs-and-jobs` (`РегламентныеЗаданияСлужебный.ДобавитьЗадание` / `ИзменитьЗадание`). Расписание резервного копирования хранится **внутри** настроек подсистемы и редактируется через `ПараметрыРезервногоКопирования`, а не через регламентные задания.
- Нужна **миграция данных при обновлении** или работа с версиями ИБ — `bsp-update` (`ОбновлениеИнформационнойБазы.ЗаписатьДанные`, `ВерсияИБ` и т. п.). Задачи обновления и резервного копирования формально пересекаются в момент «бэкап перед обновлением», но **выполняются разными подсистемами**: бэкап — `РезервноеКопированиеИБ`, обработка — `ОбновлениеИнформационнойБазы`.
- Нужно **выгрузить/загрузить ИБ через `.dt`** вручную, работать с SQL-бэкапами СУБД, монтировать сетевые тома — это вне БСП. Подсистема `РезервноеКопированиеИБ` поддерживает **только** файловый вариант ИБ и использует собственный формат (не `.dt`). В клиент-серверном варианте — настройте резервное копирование средствами СУБД, а в подсистеме оставьте признак «копирование организовано сторонними приложениями» (флаг `РезервноеКопированиеНастроено = Истина` без `ВыполнятьАвтоматическоеРезервноеКопирование`).
- Нужна **настройка подсистемы в дереве метаданных** (внедрение БСП, подключение к прикладной конфигурации) — задача архитектора конфигурации, не покрывается этим скилом.

## Core concepts

### Архитектура подсистемы «Резервное копирование ИБ»

Подсистема обслуживается **семейством** общих модулей `РезервноеКопированиеИБ*` (без префикса, с суффиксами) и двумя обработками. Все методы, которые вызывает прикладной код, находятся в **стабильной** области `Программный интерфейс` модулей `РезервноеКопированиеИБКлиент`, `РезервноеКопированиеИБСервер` и `РезервноеКопированиеИБВызовСервера`. Служебных модулей `*Служебный` в этом семействе **нет** — вся служебная логика встроена в основные модули (см. Anti-patterns).

Полная карта модулей:

| Модуль | Контекст | Назначение |
|---|---|---|
| `РезервноеКопированиеИБКлиент` | Тонкий клиент, Толстый клиент | **Стабильный API**: `ОткрытьФормуРезервногоКопирования`. Служебный интерфейс: обработчики событий подсистем (`ПриНачалеРаботыСистемы`, `ПередЗавершениемРаботыСистемы`, `ПриПроверкеВозможностиРезервногоКопированияВПользовательскомРежиме`, `ПриПредложенииПользователюСоздатьРезервнуюКопию`) и таймер ожидания — ⚠️ вызываются БСП, не из прикладного кода |
| `РезервноеКопированиеИБСервер` | Сервер, Внешнее соединение | **Стабильный API**: `ПараметрыРезервногоКопирования`, `УстановитьНастройкиРезервногоКопирования`, `ТекущаяНастройкаРезервногоКопирования`, `НавигационнаяСсылкаОбработкиРезервногоКопирования`, `СброситьПризнакРезервногоКопирования`. Служебный интерфейс: `ПриЗаполненииСпискаТекущихДел`, `ПриДобавленииПараметровРаботыКлиента`, `ПриВключенииИспользованияПрофилейБезопасности` — ⚠️ для БСП, не для прикладного кода |
| `РезервноеКопированиеИБГлобальный` | Глобальный | Один экспортный метод: `ОбработчикДействийРезервногоКопирования` — глобальный обработчик события. Используется самой БСП, из прикладного кода не вызывается |
| `РезервноеКопированиеИБВызовСервера` | Серверный вызов с клиента | **Стабильный API**: `УстановитьЗначениеНастройки`, `ДатаСледующегоАвтоматическогоКопирования`, `УстановитьДатуПоследнегоНапоминания` — безопасный серверный вызов из клиентского кода формы настроек |
| `РезервноеКопированиеИБКлиентПереопределяемый` | Тонкий клиент, Толстый клиент | **Переопределяемый** крючок: `ПриОпределенииНеобходимостиПоказаПредупрежденийОРезервномКопировании`. **Не вызывать** — реализовать в одноимённом модуле прикладной конфигурации |
| `РезервноеКопированиеОбластейДанных` | Сервер | Заглушки для модели сервиса (SaaS): `РезервноеКопированиеИспользуется`, `УстановитьФлагАктивностиПользователяВОбласти`, `МенеджерСервисаПоддерживаетРезервноеКопирование`, `ПроксиКонтроляРезервногоКопирования`, `ИмяПодсистемыДляСобытийЖурналаРегистрации`, `НаименованиеФоновогоРезервногоКопирования`. Методы помечены `@skip-warning ПустойМетод` — реализация подставляется в модели сервиса, в локальной ИБ — пустые |
| `РезервноеКопированиеОбластейДанныхКлиент` | Тонкий клиент, Толстый клиент | `ПредложитьПользователюСоздатьРезервнуюКопию` — клиентский сценарий для SaaS, в локальной ИБ не используется |

⚠️ **Не существует** модуля `РезервноеКопированиеИБСервер` (с суффиксом `Сервер`) — он **есть**, но это **сам** стабильный серверный модуль. ⚠️ **Не существует** модулей `РезервноеКопированиеИБСлужебный` и `РезервноеКопированиеИБПереопределяемый`: в этом семействе только `*КлиентПереопределяемый` (для переопределения), а служебная логика встроена в основные модули. ⚠️ **Не существует** модуля `РезервноеКопированиеИБКлиентСервер` (нет общего клиент-серверного кода в этой подсистеме). Перед вызовом сверяйтесь с реальным каталогом общих модулей в выгрузке конфигурации.

### Обработки подсистемы

Подсистема содержит **две** обработки:

- `Обработка.РезервноеКопированиеИБ` — основная форма `РезервноеКопированиеДанных`. Открывается через `ОткрытьФормуРезервногоКопирования`. Поддерживает режимы `ВыполнитьПриЗавершенииРаботы`, ручной запуск, восстановление.
- `Обработка.НастройкаРезервногоКопированияИБ` — форма настроек. Не имеет экспортных методов в общих модулях для прямого открытия — открывается через UI-команду подсистемы или через навигационную ссылку `НавигационнаяСсылкаОбработкиРезервногоКопирования` (для подсистемы это та же обработка, форма `РезервноеКопированиеДанных` с параметром `РежимРаботы`).

### Хранилище настроек

Настройки резервного копирования хранятся в **хранилище общих настроек** подсистемы (ключ `ПараметрыРезервногоКопирования`), плюс дублируются в **константе** `ПараметрыРезервногоКопирования` (тип `ХранилищеЗначения`, сжатие 9) для передачи пользователя при завершении работы. Метод `ПараметрыРезервногоКопирования` возвращает **структуру** (не `ФиксированнаяСтруктура`) с предопределённым набором полей, описанным в `НовыеНастройкиРезервногоКопирования`. Не пытайтесь добавлять собственные поля в эту структуру — метод `УстановитьНастройкиРезервногоКопирования` запишет только фиксированный набор.

### Файловый и клиент-серверный вариант

Подсистема поддерживает **только файловый** вариант ИБ. В клиент-серверном варианте методы `НеобходимостьАвтоматическогоРезервногоКопирования` сразу возвращают `Ложь`, а `ТекущаяНастройкаРезервногоКопирования` выдаёт «Резервное копирование не выполняется (организовано средствами СУБД)». Перед вызовом — проверять `ОбщегоНазначения.ИнформационнаяБазаФайловая()` (см. `bsp-base-common`), иначе сценарий может оказаться бессмысленным.

### Варианты выполнения

`ВариантВыполнения` в настройках — `ПоРасписанию` или `ПриЗавершенииРаботы`. В обоих случаях фактически выполняется **одна и та же** серверная процедура; различие — в том, **когда** инициируется запуск: по таймеру в форме либо при `ПередЗавершениемРаботыСистемы`. Агент не должен программно «выполнять резервное копирование» в обход этих сценариев — метод `ЗавершитьРезервноеКопирование` существует, но находится в `#Область СлужебныйПрограммныйИнтерфейс` (⚠️) и предназначен для обработки результата после выполнения, а не для запуска.

## Patterns

### 1. Программное открытие формы копирования из прикладной команды

```bsl
// В обработчике команды формы (клиент)
&НаКлиенте
Процедура СоздатьРезервнуюКопию(Команда)
    Если ОбщегоНазначенияКлиент.ИнформационнаяБазаФайловая() Тогда
        РезервноеКопированиеИБКлиент.ОткрытьФормуРезервногоКопирования();
    Иначе
        ПоказатьПредупреждение(,
            НСтр("ru = 'В клиент-серверном варианте резервное копирование выполняется средствами СУБД.'"));
    КонецЕсли;
КонецПроцедуры
```

`ОткрытьФормуРезервногоКопирования` — единственная стабильная клиентская точка входа подсистемы. Без предварительной проверки `ИнформационнаяБазаФайловая()` форма откроется «пустой».

### 2. Сохранение настроек копирования из прикладной обработки

```bsl
// На сервере
Настройки = РезервноеКопированиеИБСервер.ПараметрыРезервногоКопирования();
Настройки.ВыполнятьАвтоматическоеРезервноеКопирование = Истина;
Настройки.ВариантВыполнения = "ПоРасписанию";
Настройки.РасписаниеКопирования = ОбщегоНазначенияКлиентСервер.РасписаниеВСтруктуру(
    НовоеРасписание);
Настройки.КаталогХраненияРезервныхКопий = "D:\Backups";
Настройки.РезервноеКопированиеНастроено = Истина;
РезервноеКопированиеИБСервер.УстановитьНастройкиРезервногоКопирования(Настройки);
```

Всегда читайте настройки через `ПараметрыРезервногоКопирования` перед изменением — структура содержит обязательные поля, и `УстановитьНастройкиРезервногоКопирования` ожидает полный набор. **Не собирайте структуру с нуля** — потеряете значения по умолчанию.

## Anti-patterns

### ❌ Вызывать несуществующий `РезервноеКопированиеИБСлужебный`

```bsl
// ❌ ОШИБКА: модуль не существует
Параметры = РезервноеКопированиеИБСлужебный.ПолучитьВнутренниеНастройки();
```

```bsl
// ✅ Стабильный API — в основном модуле РезервноеКопированиеИБСервер
Настройки = РезервноеКопированиеИБСервер.ПараметрыРезервногоКопирования();
```

### ❌ Вызывать переопределяемый модуль как обычный

```bsl
// ❌ ОШИБКА: модуль *Переопределяемый не предназначен для вызова
Результат = РезервноеКопированиеИБКлиентПереопределяемый.ПриОпределенииНеобходимостиПоказаПредупрежденийОРезервномКопировании(Ложь);
```

```bsl
// ✅ Реализуйте одноимённую процедуру в прикладном модуле — БСП вызовет её сама
Процедура ПриОпределенииНеобходимостиПоказаПредупрежденийОРезервномКопировании(ПоказыватьПредупреждение) Экспорт
    ПоказыватьПредупреждение = Ложь;
КонецПроцедуры
```

### ❌ Собирать структуру настроек «с нуля»

```bsl
// ❌ Потеряются значения по умолчанию и обязательные поля
Настройки = Новый Структура;
Настройки.Вставить("ВыполнятьАвтоматическоеРезервноеКопирование", Истина);
Настройки.Вставить("РасписаниеКопирования", НовоеРасписание);
РезервноеКопированиеИБСервер.УстановитьНастройкиРезервногоКопирования(Настройки);
```

```bsl
// ✅ Всегда читайте текущие настройки, затем изменяйте нужные поля
Настройки = РезервноеКопированиеИБСервер.ПараметрыРезервногоКопирования();
Настройки.ВыполнятьАвтоматическоеРезервноеКопирование = Истина;

```

### ❌ Использовать форму копирования в клиент-серверном варианте

```bsl
// ❌ В клиент-серверном варианте БСП UI-форма копирования бесполезна
РезервноеКопированиеИБКлиент.ОткрытьФормуРезервногоКопирования();
```

