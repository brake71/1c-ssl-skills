# Подключаемые команды и дополнительные отчёты/обработки

Подсистемы **ПодключаемыеКоманды** и **ДополнительныеОтчетыИОбработки** — инфраструктура
размещения в формах «подключаемых» команд (печать, отчёты, заполнение, «создать на
основании», шаблоны сообщений) и подключения внешних отчётов/обработок (epf/erf) через
справочник `ДополнительныеОтчетыИОбработки`. Когда нужно добавить в форму объекта
команду, источник которой — внешняя обработка или другой подключаемый объект, а не
собственный код формы.

## Модули

Семейство **ПодключаемыеКоманды** (суффиксная логика):

- `ПодключаемыеКоманды` — сервер: размещение команд в форме, выполнение серверных
  команд (`ПриСозданииНаСервере`, `ВыполнитьКоманду`, `ДобавитьУсловиеВидимостиКоманды`).
- `ПодключаемыеКомандыКлиент` — клиент: обработчики клика/записи
  (`НачатьВыполнениеКоманды`, `ВыполнитьКоманду`, `ПослеЗаписи`, `НачатьОбновлениеКоманд`).
- `ПодключаемыеКомандыКлиентСервер` — общий: `ОбновитьКоманды` (стабильный);
  `УсловияВыполняются`, `ПараметрыВыполненияКоманды`, `ВладелецКомандыПоИмениКоманды` —
  ⚠️ служебные.
- `ПодключаемыеКомандыВызовСервера` — серверный вызов с клиента без контекста формы.
- `ПодключаемыеКомандыПереопределяемый` — **хуки**: БСП вызывает, прикладной код
  реализует (`ПриОпределенииВидовПодключаемыхКоманд`,
  `ПриОпределенииКомандПодключенныхКОбъекту`). НЕ вызывается из прикладного кода.

Семейство **ДополнительныеОтчетыИОбработки**:

- `ДополнительныеОтчетыИОбработки` — сервер: `ПодключитьВнешнююОбработку`,
  `ОбъектВнешнейОбработки`, `ВыполнитьКоманду`, `СведенияОВнешнейОбработке`,
  `СохранитьНастройки`, `ЗагрузитьНастройки` (стабильные).
- `ДополнительныеОтчетыИОбработкиКлиент` — клиент: `ОткрытьФормуКоманд…`,
  `ОткрытьВариантДополнительногоОтчета`, `ВыполнитьКомандуВФоне`,
  `ПараметрыВыполненияКомандыВФоне` (стабильные).
- `ДополнительныеОтчетыИОбработкиКлиентСервер` — общий: константы видов и типов
  команд (`ВидОбработкиЗаполнениеОбъекта()`, `ТипКомандыВызовСерверногоМетода()` …).
- ⚠️ `ДополнительныеОтчетыИОбработкиКлиент.ОткрытьСписокДополнительныхОтчетовИОбработок`
  и `ДополнительныеОтчетыИОбработки.ИспользуютсяДополнительныеОтчетыИОбработки` —
  регион `СлужебныйПрограммныйИнтерфейс`; обратная совместимость не гарантируется.

Семейство **СозданиеНаОсновании**:

- `СозданиеНаОсновании` — сервер: `ДобавитьКомандуСозданияНаОсновании` (стабильный).
  `ПриОпределенииКомандПодключенныхКОбъекту` здесь — ⚠️ служебный (БСП-внутренний);
  прикладное переопределение состава — через `ПодключаемыеКомандыПереопределяемый`.
- `СозданиеНаОснованииПереопределяемый` — хук регистрации объектов с командами
  создания на основании.

> ⚠️ Не существует модулей `ПодключаемыеКомандыСервер`, `ДополнительныеОтчетыИОбработкиСервер`
> — частая ошибка по аналогии. Серверный модуль всегда без суффикса `Сервер`.

## Сценарии

### 1. Разместить подключаемые команды в форме объекта

**Задача:** в форме документа/справочника автоматически построить подменю «Печать»,
«Отчёты», «Заполнить», «Создать на основании» из всех зарегистрированных источников.

**Функции:**
`ПодключаемыеКоманды.ПриСозданииНаСервере(Форма, Знач ПараметрыРазмещения = Неопределено) Экспорт`
`ПодключаемыеКомандыКлиент.НачатьВыполнениеКоманды(Форма, Команда, Знач Источник = Неопределено) Экспорт`
`ПодключаемыеКомандыКлиент.ПослеЗаписи(Форма, Объект, ПараметрыЗаписи) Экспорт`
`ПодключаемыеКомандыКлиент.НачатьОбновлениеКоманд(Форма) Экспорт`
— все Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Первая — Сервер;
остальные — Клиент.

**Параметры:**
- `Форма` (УправляемаяФорма) — форма объекта; в `ПриСозданииНаСервере` это `ЭтотОбъект`.
- `ПараметрыРазмещения` (Структура / Неопределено) — тонкая настройка подменю
  (источники, командная панель, префикс групп). По умолчанию `Неопределено` —
  стандартное размещение. Структуру возвращает `ПодключаемыеКоманды.ПараметрыРазмещения()`.
- `Команда` (КомандаФормы) — команда подключаемого подменю.
- `Источник` (Произвольный) — объект/список, к которому привязана команда
  (`Элементы.Список`, `Форма.Объект`).

**Пример:**
```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    // ...свой код инициализации...
    ПодключаемыеКоманды.ПриСозданииНаСервере(ЭтотОбъект);
КонецПроцедуры

&НаКлиенте
Процедура ПодключаемаяКоманда(Команда)
    // НачатьВыполнениеКоманды сам проверит ТребуетсяЗапись/ТребуетсяПроведение,
    // спросит пользователя и только потом вызовет сервер.
    ПодключаемыеКомандыКлиент.НачатьВыполнениеКоманды(ЭтотОбъект, Команда, Элементы.Список);
КонецПроцедуры

&НаКлиенте
Процедура ПослеЗаписи(ПараметрыЗаписи)
    ПодключаемыеКомандыКлиент.ПослеЗаписи(ЭтотОбъект, Объект, ПараметрыЗаписи);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Забыть `ПриСозданииНаСервере` — подменю не появятся, вся инфраструктура БСП для
  формы отключена. Всегда вызывайте после своего кода инициализации.
- ❌ Вызывать `ПодключаемыеКоманды.ВыполнитьКоманду` (серверный) прямо из клиентского
  обработчика `&НаКлиенте` — метод недоступен на клиенте. С клиента —
  `ПодключаемыеКомандыКлиент.НачатьВыполнениеКоманды` (он сам уходит на сервер).
- ❌ Дублировать подключаемую команду обычной кнопкой формы «в обход» БСП — команда
  не попадёт в общий список, не учтётся в «Текущих делах», ролях, видимости.
- `НачатьВыполнениеКоманды` выстраивает диалог «Сохранить? Провести?» и только после
  подтверждения уходит на сервер — не вызывайте серверный `ВыполнитьКоманду` «в лоб»
  из клиента без проверки, что объект записан.

### 2. Задать условие видимости команды по значению реквизита

**Задача:** показывать команду печати/заполнения только для проведённых документов
или только когда реквизит заполнен.

**Функция:**
`ПодключаемыеКоманды.ДобавитьУсловиеВидимостиКоманды(Команда, Реквизит, Значение = Неопределено, Знач ВидСравнения = Неопределено) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Команда` (СтрокаТаблицыКоманд) — строка коллекции команд (из `КомандыПечати.Добавить()`
  или из `Команды` в `ПриОпределенииКомандПодключенныхКОбъекту`).
- `Реквизит` (Строка) — имя реквизита объекта, напр. `"Проведен"`, `"Контрагент"`.
- `Значение` (Произвольный) — сравниваемое значение; `Неопределено` для `Заполнено`/
  `НеЗаполнено`.
- `ВидСравнения` (ВидСравненияКомпоновкиДанных) — `Равно`, `НеРавно`, `Заполнено`,
  `НеЗаполнено`, `ВСписке`, `НеВСписке`, `Больше`, `Меньше`, `БольшеИлиРавно`,
  `МеньшеИлиРавно`. По умолчанию `Неопределено` → `Равно`.

**Пример:**
```bsl
// В модуле менеджера объекта, в ДобавитьКомандыПечати (или в
// ПодключаемыеКомандыПереопределяемый.ПриОпределенииКомандПодключенныхКОбъекту):
КомандаПечати = КомандыПечати.Добавить();
КомандаПечати.Идентификатор = "Акт";
КомандаПечати.Представление = НСтр("ru = 'Акт выполненных работ'");

// Показывать только для проведённых документов
ПодключаемыеКоманды.ДобавитьУсловиеВидимостиКоманды(
    КомандаПечати, "Проведен", Истина, ВидСравненияКомпоновкиДанных.Равно);
```

**Нюансы / антипаттерны:**
- ❌ Реализовывать видимость «руками» в обработчике `ПриИзменении` реквизита,
  переключая `Элементы.КомандаПечать.Видимость` — рассинхрон с БСП-инфраструктурой,
  ломает обновление команд после записи. Только `ДобавитьУсловиеВидимостиКоманды`.
- Условия вычисляются при `ПодключаемыеКомандыКлиентСервер.ОбновитьКоманды` (вызывается
  БСП после записи и при изменении контекста).
- Для команд печати есть параллельный метод
  `УправлениеПечатью.ДобавитьУсловиеВидимостиКоманды(КомандаПечати, Реквизит, Значение, Знач МетодСравнения = Неопределено)`
  — 4-й параметр называется `МетодСравнения`, не `ВидСравнения`. См. `print-reports.md`.

### 3. Объявить команды «Создать на основании» для объекта

**Задача:** в подменю «Создать на основании» формы документа показать команды
создания других документов на основании текущего.

**Функция:**
`СозданиеНаОсновании.ДобавитьКомандуСозданияНаОсновании(КомандыСозданияНаОсновании, ОбъектМетаданных) Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `КомандыСозданияНаОсновании` (Массив) — коллекция команд создания, заполняемая в
  обработчике `ДобавитьКомандыСозданияНаОсновании(Команды, Объект, Параметры)` модуля
  менеджера объекта.
- `ОбъектМетаданных` (ОбъектМетаданных) — метаданные объекта-результента, напр.
  `Метаданные.Документы.СчетФактураВыданный`.

**Пример:**
```bsl
// Модуль менеджера документа-источника
Процедура ДобавитьКомандыСозданияНаОсновании(Команды, Объект, Параметры) Экспорт
    СозданиеНаОсновании.ДобавитьКомандуСозданияНаОсновании(Команды, Метаданные.Документы.СчетФактураВыданный);
    СозданиеНаОсновании.ДобавитьКомандуСозданияНаОсновании(Команды, Метаданные.Документы.ПлатежноеПоручение);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `СозданиеНаОсновании.ПриОпределенииКомандПодключенныхКОбъекту` — это
  ⚠️ служебный (`СлужебныйПрограммныйИнтерфейс`) внутренний метод БСП. Прикладное
  переопределение состава — в `ПодключаемыеКомандыПереопределяемый.ПриОпределенииКомандПодключенныхКОбъекту`.
- Регистрация самого объекта как участника подсистемы — в
  `СозданиеНаОснованииПереопределяемый.ПриОпределенииОбъектовСКомандамиСозданияНаОсновании`
  (хук переопределения, реализуется в прикладной конфигурации).
- Создание объекта **программно** (`Документы.Х.СоздатьДокумент()`) — это платформенный
  API, БСП-инфраструктура подключаемых команд здесь ни при чём.

### 4. Зарегистрировать внешнюю обработку (описать параметры регистрации)

**Задача:** во внешней обработке (epf/erf) описать вид, назначение, команды и
безопасный режим, чтобы БСП корректно загрузила её в справочник
`ДополнительныеОтчетыИОбработки`.

**Функция:**
`ДополнительныеОтчетыИОбработки.СведенияОВнешнейОбработке(ВерсияБСП = "") Экспорт`
— Функция → Структура `ПараметрыРегистрации`, регион `#Область ПрограммныйИнтерфейс`
(stable). Сервер.

**Параметры:**
- `ВерсияБСП` (Строка) — версия БСП, для которой формируется шаблон (влияет на набор
  полей). По умолчанию `""` — текущая.

Возвращаемая структура содержит поля: `Вид`, `Назначение` (массив полных имён
метаданных-владельцев), `Наименование`, `Информация`, `БезопасныйРежим` (Булево),
`Команды` (таблица с колонками `Идентификатор`, `Представление`, `Использование`,
`ПоказыватьОповещение`, …). Виды и типы команд берутся из
`ДополнительныеОтчетыИОбработкиКлиентСервер.ВидОбработки…()` /
`ТипКоманды…()` (все стабильные).

**Пример:**
```bsl
// Модуль объекта внешней обработки
Функция СведенияОВнешнейОбработке() Экспорт
    ПараметрыРегистрации = ДополнительныеОтчетыИОбработки.СведенияОВнешнейОбработке("3.1.11.0");

    ПараметрыРегистрации.Вид             = ДополнительныеОтчетыИОбработкиКлиентСервер.ВидОбработкиЗаполнениеОбъекта();
    ПараметрыРегистрации.Назначение       = Новый Массив;
    ПараметрыРегистрации.Назначение.Добавить("Документ.РеализацияТоваровУслуг");
    ПараметрыРегистрации.Наименование   = "Заполнение реализации по заказу";
    ПараметрыРегистрации.БезопасныйРежим  = Истина;
    ПараметрыРегистрации.Информация      = "Заполняет табличную часть «Товары» по данным заказа";

    Команда = ПараметрыРегистрации.Команды.Добавить();
    Команда.Идентификатор         = "ЗаполнитьПоЗаказу";
    Команда.Представление         = "Заполнить по заказу";
    Команда.Использование         = ДополнительныеОтчетыИОбработкиКлиентСервер.ТипКомандыВызовСерверногоМетода();
    Команда.ПоказыватьОповещение   = Истина;

    Возврат ПараметрыРегистрации;
КонецФункции

Процедура ВыполнитьКоманду(ИдентификаторКоманды, ПараметрыВыполнения) Экспорт
    // ...серверная логика заполнения...
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Хранить внешнюю обработку в макете конфигурации и подключать через
  `ПолучитьМакетОбработки`/`ОткрытьЗначение` — обход справочника ломает: загрузку/
  обновление без обновления конфигурации, безопасный режим, разрешения, разделитель
  данных в модели сервиса, регламентные задания. Только справочник
  `ДополнительныеОтчетыИОбработки` (загрузка через «Администрирование → Печатные
  формы, отчёты и обработки»).
- `БезопасныйРежим = Истина` запрещает внешней обработке внешние соединения, файловую
  систему и COM — ставьте `Истина`, если обработке не нужны опасные действия.
- Доступные виды: `ВидОбработкиПечатнаяФорма()`, `…ЗаполнениеОбъекта()`,
  `…СозданиеСвязанныхОбъектов()`, `…Отчет()`, `…ШаблонСообщения()`,
  `…ДополнительнаяОбработка()`, `…ДополнительныйОтчет()`. Типы команд:
  `ТипКомандыВызовСерверногоМетода()`, `…ВызовКлиентскогоМетода()`, `…ОткрытиеФормы()`,
  `…ЗаполнениеФормы()`, `…ЗагрузкаДанныхИзФайла()`.

### 5. Программно выполнить команду внешней обработки (из регламентного задания)

**Задача:** запустить команду назначаемой внешней обработки (напр. «Заполнение
объекта») из регламентного задания или серверного кода, минуя форму.

**Функция:**
`ДополнительныеОтчетыИОбработки.ВыполнитьКоманду(ПараметрыКоманды, АдресРезультата = Неопределено) Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ПараметрыКоманды` (Структура) — обязательные ключи:
  - `ДополнительнаяОбработкаСсылка` (СправочникСсылка.ДополнительныеОтчетыИОбработки) —
    ссылка на зарегистрированную обработку;
  - `ИдентификаторКоманды` (Строка) — идентификатор команды из `ПараметрыРегистрации.Команды`;
  - `ОбъектыНазначения` (Массив) — ссылки объектов-назначений; обязателен для
    назначаемых обработок (вид «Заполнение объекта», «Создание связанных объектов»).
- `АдресРезультата` (Строка) — адрес временного хранилища для результата; по умолчанию
  `Неопределено`.

**Пример:**
```bsl
// Модуль регламентного задания
Процедура ЗаполнитьДокументы() Экспорт
    Ссылка = Справочники.ДополнительныеОтчетыИОбработки.НайтиПоНаименованию("Заполнение документов");
    Если Ссылка.Пустая() Тогда
        Возврат;
    КонецЕсли;

    ПараметрыКоманды = Новый Структура;
    ПараметрыКоманды.Вставить("ДополнительнаяОбработкаСсылка", Ссылка);
    ПараметрыКоманды.Вставить("ИдентификаторКоманды",          "Заполнить");
    ПараметрыКоманды.Вставить("ОбъектыНазначения",              СсылкиНаДокументы());

    ДополнительныеОтчетыИОбработки.ВыполнитьКоманду(ПараметрыКоманды);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Использовать «самописный» реестр обработок (регистр сведений) вместо
  справочника `ДополнительныеОтчетыИОбработки` — нет безопасного режима, разделителей,
  регламентных заданий, перехватчика `УправлениеВнешнимиОбработками`.
- `ОбъектыНазначения` обязателен для назначаемых обработок; для глобальных
  (вид «Дополнительная обработка») — не передаётся.
- Перед запуском можно проверить `ДополнительныеОтчетыИОбработки.ИспользуютсяДополнительныеОтчетыИОбработки()`
  — ⚠️ метод служебный (регион `СлужебныйПрограммныйИнтерфейс`), обратная
  совместимость не гарантируется; в новом коде предпочтительно проверить наличие
  ссылки через `Справочники.ДополнительныеОтчетыИОбработки.НайтиПоНаименованию(...).Пустая()`.

### 6. Подключить внешнюю обработку и получить её объект

**Задача:** программно подключить epf/erf из справочника и вызвать его методы через
объект (напр. в серверной печати по внешнему источнику).

**Функции:**
`ДополнительныеОтчетыИОбработки.ПодключитьВнешнююОбработку(Ссылка) Экспорт` — Функция → Строка (имя подключённой обработки), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ДополнительныеОтчетыИОбработки.ОбъектВнешнейОбработки(Ссылка) Экспорт` — Функция → Объект (экземпляр внешней обработки/отчёта), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Ссылка` (СправочникСсылка.ДополнительныеОтчетыИОбработки) — запись справочника.

**Пример:**
```bsl
ИмяОбработки = ДополнительныеОтчетыИОбработки.ПодключитьВнешнююОбработку(Ссылка);
ВнешнийОбъект = ДополнительныеОтчетыИОбработки.ОбъектВнешнейОбработки(Ссылка);
// Дальше — вызов экспортных методов внешнего объекта:
ВнешнийОбъект.МояЭкспортнаяПроцедура(Параметры);
```

**Нюансы / антипаттерны:**
- ❌ Подключать epf через платформенный `ВнешниеОбработки.Подключить(...)` в обход
  БСП — теряются безопасный режим, разрешения, разделители. Только через
  `ДополнительныеОтчетыИОбработки.ПодключитьВнешнююОбработку`.
- `ОбъектВнешнейОбработки` под капотом сама подключает обработку при необходимости;
  явный `ПодключитьВнешнююОбработку` нужен, когда требуется имя подключения.

### 7. Сохранить и загрузить настройки внешней обработки

**Задача:** сохранить произвольные настройки обработки (параметры заполнения,
настройки отчёта) между запусками и прочитать их при следующем вызове.

**Функции:**
`ДополнительныеОтчетыИОбработки.СохранитьНастройки(Ссылка, Настройки) Экспорт` — Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ДополнительныеОтчетыИОбработки.ЗагрузитьНастройки(Ссылка) Экспорт` — Функция → Произвольный (сохранённые настройки), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Ссылка` (СправочникСсылка.ДополнительныеОтчетыИОбработки) — запись справочника.
- `Настройки` (Произвольный) — сериализуемое значение (Структура, ТаблицаЗначений …).

**Пример:**
```bsl
// Сохранить
Настройки = Новый Структура("Период, Склад", Период, Склад);
ДополнительныеОтчетыИОбработки.СохранитьНастройки(СсылкаОбработки, Настройки);

// Прочитать при следующем запуске
Настройки = ДополнительныеОтчетыИОбработки.ЗагрузитьНастройки(СсылкаОбработки);
Если Настройки <> Неопределено Тогда
    Период = Настройки.Период;
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Хранить настройки обработки в отдельном регистре сведений «по своему ключу» —
  теряется привязка к разделителю данных и модели сервиса. Используйте штатные
  `СохранитьНастройки`/`ЗагрузитьНастройки` (хранение привязано к ссылке обработки).
- `ЗагрузитьНастройки` возвращает `Неопределено`, если настройки ранее не
  сохранялись — проверяйте перед обращением через точку.

### 8. Открыть форму команд обработок и запустить команду в фоне

**Задача:** из клиентского кода открыть форму выбора команд дополнительных
отчётов/обработок выбранного вида; запустить длительную команду обработки в фоне
с обработчиком завершения.

**Функции:**
`ДополнительныеОтчетыИОбработкиКлиент.ОткрытьФормуКомандДополнительныхОтчетовИОбработок(ПараметрКоманды, ПараметрыВыполненияКоманды, Вид, ИмяРаздела = "") Экспорт` — Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.
`ДополнительныеОтчетыИОбработкиКлиент.ВыполнитьКомандуВФоне(Знач ИдентификаторКоманды, Знач ПараметрыКоманды, Знач Обработчик) Экспорт` — Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.
`ДополнительныеОтчетыИОбработкиКлиент.ПараметрыВыполненияКомандыВФоне(Ссылка) Экспорт` — Функция → Структура, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `ПараметрКоманды` (Произвольный) — объект/ссылка, для которого открывается список
  команд (напр. `Объект` формы).
- `ПараметрыВыполненияКоманды` (Структура) — параметры выполнения; см.
  `ПодключаемыеКомандыКлиент.ПараметрыВыполненияКоманды()`.
- `Вид` (Строка) — вид обработок, напр. результат
  `ДополнительныеОтчетыИОбработкиКлиентСервер.ВидОбработкиЗаполнениеОбъекта()`.
- `ИмяРаздела` (Строка) — раздел навигации для размещения; по умолчанию `""`.
- `ИдентификаторКоманды` (Строка) — идентификатор команды из `ПараметрыРегистрации.Команды`.
- `ПараметрыКоманды` (Структура) — ключи `ДополнительнаяОбработкаСсылка`,
  `ОбъектыНазначения` (см. сценарий 5).
- `Обработчик` (ОписаниеОповещения) — обработчик завершения фоновой команды.

**Пример:**
```bsl
&НаКлиенте
Процедура КомандаЗаполнить(Команда)
    ДополнительныеОтчетыИОбработкиКлиент.ОткрытьФормуКомандДополнительныхОтчетовИОбработок(
        Объект,
        ПодключаемыеКомандыКлиент.ПараметрыВыполненияКоманды(),
        ДополнительныеОтчетыИОбработкиКлиентСервер.ВидОбработкиЗаполнениеОбъекта(),
        "Справочник.Контрагенты");
КонецПроцедуры

&НаКлиенте
Процедура ЗапуститьВФоне(Идентификатор, ПараметрыКоманды)
    ПараметрыВФоне = ДополнительныеОтчетыИОбработкиКлиент.ПараметрыВыполненияКомандыВФоне(СсылкаОбработки);
    ДополнительныеОтчетыИОбработкиКлиент.ВыполнитьКомандуВФоне(
        Идентификатор, ПараметрыКоманды,
        Новый ОписаниеОповещения("ПослеВыполненияКоманды", ЭтотОбъект, ПараметрыВФоне));
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Открывать форму внешнего отчёта через `ОткрытьЗначение(Ссылка)` — обход
  подсистемы вариантов. Дополнительный отчёт открывается через
  `ДополнительныеОтчетыИОбработкиКлиент.ОткрытьВариантДополнительногоОтчета(ДополнительныйОтчет, КлючВарианта)`.
- `ОткрытьСписокДополнительныхОтчетовИОбработок()` — ⚠️ служебный (регион
  `СлужебныйПрограммныйИнтерфейс`); для администраторского UI допустимо, но обратная
  совместимость не гарантируется.

## Редкие методы

Конструкторы структур и вспомогательные методы (все стабильные, регион
`ПрограммныйИнтерфейс`, если не указано иное):

- `ПодключаемыеКоманды.ПараметрыРазмещения() Экспорт` — Функция → Структура
  (`Источники`, `КоманднаяПанель`, `ПрефиксГрупп`, `ВладелецКоманд`). Передаётся вторым
  параметром в `ПриСозданииНаСервере` для нестандартного размещения подменю.
- `ПодключаемыеКоманды.ПараметрыВыполненияКоманды() Экспорт` и
  `ПодключаемыеКомандыКлиент.ПараметрыВыполненияКоманды() Экспорт` — Функции →
  Структура параметров выполнения (`ОписаниеКоманды`, `Форма`, `ЭтоФормаОбъекта`,
  `Источник`). Серверный и клиентский варианты соответственно.
- `ПодключаемыеКомандыКлиентСервер.ОбновитьКоманды(Форма, Знач Источник = Неопределено) Экспорт`
  — Процедура (клиент + сервер): пересчитать видимость/доступность/пометки команд
  формы. Используется после программного изменения контекста.
- `ДополнительныеОтчетыИОбработки.ВыполнитьКомандуИзФормыВнешнегоОбъекта(ИдентификаторКоманды, ПараметрыКоманды, Форма) Экспорт`
  — Функция (сервер): запуск команды из модуля формы самой внешней обработки.
- `ДополнительныеОтчетыИОбработки.ПечатьПоВнешнемуИсточнику(ДополнительнаяОбработкаСсылка, ПараметрыИсточника, КоллекцияПечатныхФорм, ОбъектыПечати, ПараметрыВывода) Экспорт`
  — Процедура (сервер): связь внешней обработки-печатной формы с менеджером печати
  (вызывается инфраструктурой печати, см. `print-reports.md`).

Хуки переопределения (`*Переопределяемый`, регион `ПрограммныйИнтерфейс`): БСП
вызывает, прикладной код реализует в своём модуле-переопределителе — НЕ вызывает
напрямую:

- `ПодключаемыеКомандыПереопределяемый.ПриОпределенииВидовПодключаемыхКоманд(ВидыПодключаемыхКоманд)`
  — добавить собственные виды подключаемых команд.
- `ПодключаемыеКомандыПереопределяемый.ПриОпределенииСоставаНастроекПодключаемыхОбъектов(НастройкиПрограммногоИнтерфейса)`
  — настроить состав объектов с подключаемыми командами.
- `ПодключаемыеКомандыПереопределяемый.ПриОпределенииКомандПодключенныхКОбъекту(НастройкиФормы, Источники, ПодключенныеОтчетыИОбработки, Команды)`
  — реализовать произвольные команды, привязанные к объекту метаданных.

Для поиска сигнатур/регионов любого метода —
`python .claude/skills/bsp/scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`.
