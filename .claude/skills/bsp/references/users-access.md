# Пользователи и управление доступом БСП

Подсистемы **Пользователи**, **УправлениеДоступом** (и тесно связанная
**ВнешниеПользователи**). Покрывают: текущий пользователь сеанса, проверка
ролей и «полных прав», RLS-проверки чтения/изменения на уровне записей, права на
объект, профили групп доступа, внешний пользователь (B2B-портал).

## Модули

Суффиксная система (один корень + контекст):

- `Пользователи` — серверный стабильный API (текущий пользователь, роли,
  свойства пользователя ИБ, поиск).
- `ПользователиКлиент` — клиентский стабильный API (те же «текущий
  пользователь»/«полноправный» — только для текущего пользователя).
- `ПользователиКлиентСервер` — ⚠️ **устарел** целиком (регион
  `УстаревшиеПроцедурыИФункции`): `ТекущийПользователь`, `АвторизованныйПользователь`,
  `ТекущийВнешнийПользователь`, `ЭтоСеансВнешнегоПользователя`. Используйте
  серверный или клиентский вариант без суффикса `КлиентСервер`.
- `ВнешниеПользователи` — серверный стабильный API внешних пользователей.
- `УправлениеДоступом` — серверный стабильный API (RLS, права, профили, группы
  доступа, наборы значений доступа).
- `УправлениеДоступомПереопределяемый` — **хуки**: БСП вызывает, прикладной код
  реализует (копирует модуль-переопределитель и переопределяет тело). Не
  вызывается из прикладного кода напрямую.
- `ПользователиПереопределяемый` — **хуки** подсистемы Пользователи.

⚠️ **Не существуют:** `УправлениеДоступомКлиент` (без `Служебный`) — для клиентского
кода подсистемы «Управление доступом» служебного стабильного аналога нет;
`УправлениеДоступомСлужебныйКлиент` — ⚠️ служебный, без гарантий. Также
`Пользователи.СсылкаТекущегоПользователя`, `ПользователиКлиент.Авторизоваться`,
`ПользователиСлужебный.СоздатьПользователяИБ`, `УправлениеДоступом.НастройкиПрав` —
не существуют (типовые ошибки «по аналогии»).

## Сценарии

### 1. Получить текущего пользователя сеанса

**Задача:** в серверном коде получить ссылку пользователя для подстановки в
реквизит «Ответственный»/«Автор» и т. п., корректно работая и с внешними
пользователями.

**Функции:**
`Пользователи.АвторизованныйПользователь() Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`Пользователи.ТекущийПользователь() Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`Пользователи.ЭтоСеансВнешнегоПользователя() Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:** без параметров. `АвторизованныйПользователь` возвращает
`СправочникСсылка.Пользователи` или `СправочникСсылка.ВнешниеПользователи` в
зависимости от того, кто вошёл. `ТекущийПользователь` всегда возвращает
`СправочникСсылка.Пользователи` и **бросает исключение**, если вход выполнил
внешний пользователь.

**Пример:**
```bsl
// Универсально — для кода, поддерживающего внешних пользователей
ТекПользователь = Пользователи.АвторизованныйПользователь();
ДокументОбъект.Ответственный = ТекПользователь;

// Код, который НЕ поддерживает внешних пользователей — можно звать ТекущийПользователь
Если Не Пользователи.ЭтоСеансВнешнегоПользователя() Тогда
    Автор = Пользователи.ТекущийПользователь();
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ `ДокументОбъект.Ответственный = ИмяПользователя();` — платформенный метод
  возвращает строку; при смене имени ссылки «уезжают». Используйте
  `АвторизованныйПользователь()` — возвращает ссылку справочника.
- ❌ `Пользователи.СсылкаТекущегоПользователя()` — метод **не существует**
  (ошибка компиляции). `ТекущийПользователь()` сам возвращает ссылку.
- Для клиентского кода — `ПользователиКлиент.АвторизованныйПользователь()` или
  `ПользователиКлиент.ТекущийПользователь()` (стабильные, только текущий
  пользователь). ⚠️ `ПользователиКлиентСервер.ТекущийПользователь` — устарел.
- Кэшируйте результат в начале серверного вызова и не дёргайте функцию повторно.

### 2. Проверить роли и «полные права» пользователя

**Задача:** проверить, есть ли у пользователя роль конфигурации (или
полноправность) перед открытием административного интерфейса или выполнением
привилегированной операции.

**Функции:**
`Пользователи.РолиДоступны(ИменаРолей, Пользователь = Неопределено, УчитыватьПривилегированныйРежим = Истина) Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`Пользователи.ЭтоПолноправныйПользователь(Пользователь = Неопределено, ПроверятьПраваАдминистрированияСистемы = Ложь, УчитыватьПривилегированныйРежим = Истина) Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ПользователиКлиент.ЭтоПолноправныйПользователь(ПроверятьПраваАдминистрированияСистемы = Ложь) Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `ИменаРолей` (Строка) — имена ролей через запятую (не массив). Возвращает
  `Истина`, если доступна хотя бы одна; полноправному пользователю — `Истина`
  при `УчитыватьПривилегированныйРежим = Истина`.
- `Пользователь` (`СправочникСсылка.Пользователи` / `ВнешниеПользователи` /
  `ПользовательИнформационнойБазы` / `Неопределено` — текущий) — для
  `РолиДоступны` и серверного `ЭтоПолноправныйПользователь`. На клиенте
  проверяется только текущий.
- `ПроверятьПраваАдминистрированияСистемы` (Булево) — `Истина` — проверять не
  только `ПолныеПрава`, но и `АдминистраторСистемы`.

**Пример:**
```bsl
// Сервер: проверить несколько ролей у произвольного пользователя
Если Пользователи.РолиДоступны("ДобавлениеИзменениеСправочников,ЧтениеКадровыхДанных", ПользовательСсылка) Тогда
    // ...
КонецЕсли;

// Сервер: полные права + администратор системы
Если Пользователи.ЭтоПолноправныйПользователь(, Истина) Тогда
    ОткрытьФорму("Обработка.НастройкиПрограммы.Форма");
КонецЕсли;

// Клиент: только текущий, без указания пользователя
Если ПользователиКлиент.ЭтоПолноправныйПользователь(Истина) Тогда
    // открыть админский раздел
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ `Если РольДоступна("ПолныеПрава") Тогда` — платформенный метод, не учитывает
  привилегированный режим и полноправность. Используйте
  `Пользователи.ЭтоПолноправныйПользователь` или `РолиДоступны`.
- ❌ Передавать массив в `РолиДоступны` — метод ждёт **строку** через запятую.
- `ЭтоПолноправныйПользователь` на сервере принимает `Пользователь` (можно
  проверить произвольного), на клиенте — только текущего.

### 3. Проверить RLS-доступ к объекту (чтение/изменение)

**Задача:** до выполнения тяжёлого запроса или записи проверить на уровне
записей (RLS), что текущему пользователю разрешено чтение/изменение объекта, и
при запрете — сгенерировать исключение.

**Функции:**
`УправлениеДоступом.ЧтениеРазрешено(ОписаниеДанных, Пользователь = Неопределено) Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеДоступом.ИзменениеРазрешено(ОписаниеДанных, Пользователь = Неопределено) Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеДоступом.ПроверитьЧтениеРазрешено(ОписаниеДанных) Экспорт`
`УправлениеДоступом.ПроверитьИзменениеРазрешено(ОписаниеДанных) Экспорт`
— Процедуры, бросают исключение при запрете, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ОписаниеДанных` (`СправочникСсылка` / `ДокументСсылка` / `ПланВидовХарактеристикСсылка`
  / `ПланСчетовСсылка` / `ПланВидовРасчетаСсылка` / `БизнесПроцессСсылка` / ключ
  записи / набор записей / объект в памяти) — для `ИзменениеРазрешено` объект в
  памяти проверяется для нового объекта, объект БД — для ссылки.
- `Пользователь` (`СправочникСсылка.Пользователи` /
  `СправочникСсылка.ВнешниеПользователи` / `Неопределено` — текущий). В
  стандартном (не производительном) варианте при указании «не текущего»
  пользователя метод бросает исключение.

**Пример:**
```bsl
// Мягкая проверка — ветвление
Если Не УправлениеДоступом.ЧтениеРазрешено(СсылкаНаДокумент) Тогда
    ВызватьИсключение СтрШаблон("Чтение документа %1 запрещено", СсылкаНаДокумент);
КонецЕсли;

// Жёсткая проверка — автоматическое исключение при запрете
УправлениеДоступом.ПроверитьЧтениеРазрешено(СсылкаНаДокумент);

// Перед записью — проверка изменения
УправлениеДоступом.ПроверитьИзменениеРазрешено(ДокументОбъект);
```

**Нюансы / антипаттерны:**
- ❌ Писать собственные RLS-фильтры через `Если РольДоступна("ЧтениеДокументов")
  Тогда` — это не учитывает ограничения на уровне записей и расходится с моделью
  безопасности БСП. Делегируйте проверку `УправлениеДоступом`.
- В стандартном варианте (`ПроизводительныйВариант()` = `Ложь`) при указании
  пользователя, отличного от текущего, методы бросают исключение; проверка
  идёт только объекта в БД. Перед расширенным использованием проверяйте
  `УправлениеДоступом.ПроизводительныйВариант()`.
- `ИзменениеРазрешено` для ссылки проверяет право Чтение на уровне записей и
  Изменение на таблицу в целом; для нового объекта — только объект в памяти.
- Не путать с `ЕстьПраво` (права на объект, см. сценарий 4) — `ЧтениеРазрешено`/
  `ИзменениеРазрешено` про RLS на чтение/изменение.

### 4. Проверить право на объект и роль в профиле групп доступа

**Задача:** проверить, что у пользователя настроено «право на объект»
(например «УправлениеПравами», «Чтение», «ИзменениеПапок» для папки файлов) с
учётом иерархии, либо что у него есть роль в одном из профилей групп доступа.

**Функции:**
`УправлениеДоступом.ЕстьПраво(Право, СсылкаНаОбъект, Знач Пользователь = Неопределено) Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеДоступом.ЕстьРоль(Знач Роль, Знач СсылкаНаОбъект = Неопределено, Знач Пользователь = Неопределено) Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Право` (Строка) — имя права, как оно задано в хуке
  `УправлениеДоступомПереопределяемый.ПриЗаполненииВозможныхПравДляНастройкиПравОбъектов`.
- `СсылкаНаОбъект` (`СправочникСсылка` / `ПланВидовХарактеристикСсылка`) —
  ссылка на конкретный объект-владелец прав (например, папку файлов), **не**
  метаданные.
- `Роль` (Строка) — имя роли; `СсылкаНаОбъект` (`ЛюбаяСсылка` /
  `ТаблицаЗначений` наборов значений доступа / `Неопределено`) — для проверки
  права Чтение в группах доступа.
- `Пользователь` — для `ЕстьПраво` можно передать произвольного; `ЕстьРоль`
  проверяет по профилям групп доступа с учётом RLS на чтение.

**Пример:**
```bsl
// Право на конкретную папку файлов с учётом иерархии
Если УправлениеДоступом.ЕстьПраво("ИзменениеПапок", ПапкаФайлов) Тогда
    // ...
КонецЕсли;

// Роль в профиле групп доступа (для текущего пользователя)
Если УправлениеДоступом.ЕстьРоль("ДобавлениеИзменениеПапокФайлов", ПапкаФайлов) Тогда
    // ...
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ `УправлениеДоступом.ЕстьПраво("Чтение", Метаданные.Справочники.Файлы)` —
  ошибка компиляции: второй аргумент — ссылка на объект, не метаданные.
- ❌ `УправлениеДоступом.НастройкиПрав(...)` — метод **не существует**. Права
  настраиваются через хук `УправлениеДоступомПереопределяемый`.
- `ЕстьРоль` проверяет роль в профилях групп доступа с учётом RLS на чтение; для
  проверки «голой» роли конфигурации без учёта RLS используйте
  `Пользователи.РолиДоступны` (см. сценарий 2).

### 5. Обработать внешнего пользователя (B2B-портал)

**Задача:** в коде, рассчитанном на внешний портал, отличить вход внешнего
пользователя, получить его ссылку и объект-владелец авторизации (контрагент).

**Функции:**
`ВнешниеПользователи.ИспользоватьВнешнихПользователей() Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ВнешниеПользователи.ТекущийВнешнийПользователь() Экспорт`
— Функция → `СправочникСсылка.ВнешниеПользователи`, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ВнешниеПользователи.ПолучитьОбъектАвторизацииВнешнегоПользователя(ВнешнийПользователь = Неопределено) Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ВнешнийПользователь` (`СправочникСсылка.ВнешниеПользователи` /
  `Неопределено` — текущий) — для `ПолучитьОбъектАвторизацииВнешнегоПользователя`.

**Пример:**
```bsl
Если ВнешниеПользователи.ИспользоватьВнешнихПользователей()
   И Пользователи.ЭтоСеансВнешнегоПользователя() Тогда
    ТекВнешний = ВнешниеПользователи.ТекущийВнешнийПользователь();
    Контрагент  = ВнешниеПользователи.ПолучитьОбъектАвторизацииВнешнегоПользователя(ТекВнешний);
    // ... работаем с Контрагентом
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Звать `ВнешниеПользователи.ТекущийВнешнийПользователь()` без предварительной
  проверки `ЭтоСеансВнешнегоПользователя()` — метод бросает исключение, если вход
  выполнил обычный пользователь. Сначала проверка, потом вызов.
- `Пользователи.ТекущийПользователь()` в сеансе внешнего пользователя бросает
  исключение — используйте `АвторизованныйПользователь()` (см. сценарий 1), если
  код поддерживает оба варианта.

### 6. Назначить профиль групп доступа / установить права пользователя

**Задача:** программно включить пользователю профиль групп доступа (для
упрощённой настройки прав) или полностью переустановить его права по списку
групп доступа и групп пользователей.

**Функции:**
`УправлениеДоступом.ВключитьПрофильПользователю(Пользователь, Профиль) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеДоступом.ВыключитьПрофильПользователю(Пользователь, Профиль = Неопределено) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеДоступом.УстановитьПраваПользователя(Пользователь, ГруппыДоступа, ГруппыПользователей) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Пользователь` (`СправочникСсылка.Пользователи` /
  `СправочникСсылка.ВнешниеПользователи`).
- `Профиль` (`СправочникСсылка.ПрофилиГруппДоступа` / `УникальныйИдентификатор`
  поставляемого профиля / `Строка` — имя поставляемого профиля) — для
  `ВключитьПрофильПользователю` создаёт/находит персональную группу доступа и
  включает в неё пользователя. `Неопределено` в `ВыключитьПрофильПользователю`
  — отключить все профили.
- `ГруппыДоступа` (`Массив` из `ГруппыДоступа` / `ПрофилиГруппДоступа`),
  `ГруппыПользователей` (`Массив` из `ГруппыПользователей`) — полная
  переустановка прав.

**Пример:**
```bsl
// Включить профиль по имени поставляемого профиля
УправлениеДоступом.ВключитьПрофильПользователю(Пользователь, "ПрофильМенеджераПродаж");

// Полная установка прав: набор групп доступа и групп пользователей
МассивГрупп = Новый Массив;
МассивГрупп.Добавить(Справочники.ГруппыДоступа.НайтиПоНаименованию("Менеджеры"));
УправлениеДоступом.УстановитьПраваПользователя(Пользователь, МассивГрупп, Новый Массив);
```

**Нюансы / антипаттерны:**
- ❌ Искать несуществующий `УправлениеДоступом.ДобавлениеПользователейВГруппу` —
  метода нет. Назначение профиля — через `ВключитьПрофильПользователю`, полная
  переустановка — через `УстановитьПраваПользователя`.
- `ВключитьПрофильПользователю` работает в упрощённом режиме настройки прав
  (создаёт персональную группу доступа); для неупрощённого режима используйте
  `УстановитьПраваПользователя` с группами доступа.

### 7. Хуки переопределения доступа (реализует прикладной код)

**Задача:** внедрить в конфигурацию собственные виды доступа, права на объекты,
поставляемые профили — через модули-переопределители.

**Функции (хуки):**
`УправлениеДоступомПереопределяемый.ПриЗаполненииВидовДоступа(ВидыДоступа) Экспорт`
`УправлениеДоступомПереопределяемый.ПриЗаполненииВозможныхПравДляНастройкиПравОбъектов(ВозможныеПрава) Экспорт`
`УправлениеДоступомПереопределяемый.ПриЗаполнениеПоставляемыхПрофилейГруппДоступа(ОписанияПрофилей, ПараметрыОбновления) Экспорт`
`УправлениеДоступомПереопределяемый.ПриИзмененииНаборовЗначенийДоступа(Ссылка, СсылкиНаЗависимыеОбъекты) Экспорт`
— все Процедуры, регион `#Область ПрограммныйИнтерфейс`. **Хуки**: БСП вызывает
их в моменты заполнения видов доступа, прав, профилей и при изменении наборов
значений доступа; прикладной код копирует модуль-переопределитель в
конфигурацию и реализует тело.

**Пример (реализация в копии переопределителя):**
```bsl
// В модуле УправлениеДоступомПереопределяемый, скопированном в конфигурацию
Процедура ПриЗаполнениеВозможныхПравДляНастройкиПравОбъектов(ВозможныеПрава) Экспорт
    // Добавить право "ИзменениеПапок" для папок файлов
    Право = ВозможныеПрава.Строки.Добавить();
    Право.Имя = "ИзменениеПапок";
    Право.Описание = НСтр("ru = 'Изменение папок файлов'");
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать хуки напрямую из прикладного кода (`УправлениеДоступомПереопределяемый.
  ПриЗаполнениеВидовДоступа(...)`) — это хуки: их реализует прикладная
  конфигурация, БСП вызывает сама. Прямой вызов не имеет смысла.
- Аналогично `ПользователиПереопределяемый.ПриОпределенииНазначенияРолей`,
  `ПриОпределенииНастроек` и др. — хуки, реализуемые во внедрении.

## Редкие методы

Прочие стабильные методы (регион `ПрограммныйИнтерфейс`), полные сигнатуры —
через `python scripts/bsp_api.py method <Имя> [--module <М>] --src src/cf`:

- `Пользователи.НайтиПоИмени(ИмяДляВхода)` /
  `НайтиПоИдентификатору(ИдентификаторПользователяИБ)` /
  `НайтиПоСсылке(Пользователь)` — поиск пользователя ИБ.
- `Пользователи.СвойстваПользователяИБ(ИмяИлиИдентификатор)` /
  `УстановитьСвойстваПользователяИБ(...)` / `УдалитьПользователяИБ(...)` —
  свойства пользователя ИБ. ⚠️ `ПользователиСлужебный.СоздатьПользователяИБ`
  не существует; программное создание пользователя — задача уровня внедрения,
  через `ЗаписатьПользователяИБ(ПользовательОбъект, ПараметрыОбработки)` в
  служебном модуле (формат `ПараметрыОбработки` нестабилен).
- `УправлениеДоступом.ОграничиватьДоступНаУровнеЗаписей()` → Булево — включён ли
  RLS; `ПроизводительныйВариант()` → Булево — вариант RLS.
- `УправлениеДоступом.ОбновитьНаборыЗначенийДоступа(СсылкаИлиОбъект,
  ОбновлениеИБ = Ложь)` — пересчёт наборов значений доступа после изменения
  объекта.
- `УправлениеДоступом.ПраваДоступаКДанным(ОписаниеДанных,
  ДляВнешнихПользователей = Ложь, СоставПользователей = Неопределено)` —
  состав прав доступа к данным.
