# Классификаторы БСП (загрузка и обновление нормативно-справочной информации)

Подсистема **РаботаСКлассификаторами** — единый механизм загрузки и обновления
классификаторов из сервиса 1С (портал ИТС) или из кэша поставляемых данных:
адресный классификатор, классификатор банков (БИК), курсы валют, производственные
календари, страны мира. Прикладной код либо регистрирует свой классификатор через
хук и обрабатывает загруженные файлы, либо проверяет/обновляет существующие, либо
читает справочные данные (страны, регионы).

Сама **проверка адресов** по классификатору и **коды регионов/ОКТМО** — в
`contact-info.md` (модуль `АдресныйКлассификатор`); **расчёт дат по
производственному календарю** — в `currencies-banks.md` (модуль `КалендарныеГрафики`);
**поиск банка по БИК** — в `currencies-banks.md` (модуль `РаботаСБанками`). Здесь —
механизм обновления и регистрация новых классификаторов.

## Модули

- `РаботаСКлассификаторами` — серверный стабильный API: обновление, версия/дата,
  файлы, описание. регион `ПрограммныйИнтерфейс` (stable).
- `РаботаСКлассификаторамиКлиент` — UI: помощник обновления, параметры. Клиент.
- `РаботаСКлассификаторамиПереопределяемый` — хуки: БСП вызывает, прикладной код
  реализует (не вызывается напрямую).
- `РаботаСКлассификаторамиВызовСервера` / `РаботаСКлассификаторамиГлобальный` /
  `РаботаСКлассификаторамиКлиентСервер` / `РаботаСКлассификаторамиСлужебныйВМоделиСервиса` /
  `РаботаСКлассификаторамиВМоделиСервисаПереопределяемый` — ⚠️ служебные
  (`СлужебныйПрограммныйИнтерфейс` / `СлужебныеПроцедурыИФункции`); обратная
  совместимость не гарантируется. Методы `РаботаСКлассификаторамиВызовСервера`
  (`НастройкиОбновленияКлассификаторов`, `ЗаписатьРасписаниеОбновления`,
  `ВключитьАвтоматическоеОбновлениеКлассификаторовИзСервиса`) — все в регионе
  `СлужебныйПрограммныйИнтерфейс`, не для прикладного вызова.

⚠️ Не путать: справочник стран мира и методы `ДанныеКлассификатораСтранМира*` живут
в модуле `УправлениеКонтактнойИнформацией` (подсистема `КонтактнаяИнформация`), а не
в `РаботаСКлассификаторами` — см. сценарий 7.

### Какие классификаторы загружает механизм

| Идентификатор (строка сервиса) | Что обновляет | Кто регистрирует |
|---|---|---|
| `АдресныйКлассификатор` | КЛАДР/ФИАС (регионы, города, улицы) | `КалендарныеГрафики.ПриДобавленииКлассификаторов`, `РаботаСАдресами` |
| `КлассификаторБанков` | справочник банков по БИК | `РаботаСБанками` |
| `КурсыВалют` | периодический РС курсов | `РаботаСКурсамиВалют` |
| `ПроизводственныеКалендари` | производственные календари РФ | `КалендарныеГрафики.ПриДобавленииКлассификаторов` |
| `СтраныМира` | справочник стран | `УправлениеКонтактнойИнформацией` |

Регистрация — через хук `ПриДобавленииКлассификаторов` (см. сценарий 1).

## Сценарии

### 1. Зарегистрировать свой классификатор

**Задача:** подключить прикладной классификатор к механизму обновления БСП, чтобы
он обновлялся автоматически вместе с остальными.

**Функция (хук):**
`РаботаСКлассификаторамиПереопределяемый.ПриДобавленииКлассификаторов(Классификаторы) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс`. **Хук переопределения**: реализуется в
одноимённом модуле прикладной конфигурации, БСП вызывает при сборе классификаторов.

**Параметры:**
- `Классификаторы` (`Массив` из `Структура`) — каждая структура (из
  `РаботаСКлассификаторами.ОписаниеКлассификатора`) с ключами: `Наименование`
  (Строка, ≤150 символов), `Идентификатор` (Строка, ≤50, идентификатор в сервисе),
  `ОбновлятьАвтоматически` (Булево), `ОбщиеДанные` (Булево — `Ложь` загружать в каждую
  область данных; только в модели сервиса), `СохранятьФайлВКэш` (Булево).

**Пример:**
```bsl
// В модуле РаботаСКлассификаторамиПереопределяемый прикладной конфигурации
Процедура ПриДобавленииКлассификаторов(Классификаторы) Экспорт
    Описание = РаботаСКлассификаторами.ОписаниеКлассификатора();
    Описание.Наименование             = НСтр("ru = 'Свой справочник НСИ'");
    Описание.Идентификатор            = "MyAppClassifier";
    Описание.ОбновлятьАвтоматически   = Истина;
    Описание.СохранятьФайлВКэш         = Истина;
    Классификаторы.Добавить(Описание);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `ПриДобавленииКлассификаторов` из прикладного кода напрямую — это
  хук, БСП сам его вызывает при сборе. Только реализовать тело.
- `ОписаниеКлассификатора()` возвращает готовую структуру со всеми полями и
  значениями по умолчанию — не конструируйте структуру вручную.
- При `ОбщиеДанные = Ложь` в модели сервиса загрузка идёт в каждую область данных
  отдельно; `Истина` — один раз в общие данные.

### 2. Проверить наличие доступных обновлений

**Задача:** узнать, есть ли в сервисе 1С свежие версии классификаторов, без
выполнения самой загрузки.

**Функция:**
`РаботаСКлассификаторами.ДоступныеОбновленияКлассификаторов(Идентификаторы) Экспорт`
— Функция, регион `ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Идентификаторы` (`Массив` из `Строка`) — идентификаторы классификаторов в сервисе.

**Возвращаемое значение:** `Структура` с результатом: `КодОшибки` (Строка; пустая =
успех; `"НеверныйЛогинИлиПароль"`, `"ПревышеноКоличествоПопыток"` и т. д.) и
`ДоступныеВерсии` (`ТаблицаЗначений`) — пустая, если `СохранятьФайлВКэш = Ложь` у
классификатора.

**Пример:**
```bsl
Идентификаторы = Новый Массив;
Идентификаторы.Добавить("АдресныйКлассификатор");
Идентификаторы.Добавить("КлассификаторБанков");

Результат = РаботаСКлассификаторами.ДоступныеОбновленияКлассификаторов(Идентификаторы);
Если ПустаяСтрока(Результат.КодОшибки) Тогда
    Для Каждого СтрокаВерсии Из Результат.ДоступныеВерсии Цикл
        // СтрокаВерсии.Идентификатор, .Версия, .ИдентификаторФайла
    КонецЦикла;
КонецЕсли;
```

**Нюансы / антипаттерны:**
- Метод обращается к сервису 1С (или к кэшу поставляемых данных) — выполняйте в
  фоновом задании, если вызываете из UI, чтобы не блокировать форму.
- `ДоступныеВерсии` будет пустой для классификаторов с `СохранятьФайлВКэш = Ложь` —
  для них актуальные версии не кэшируются; используйте `ПолучитьФайлыКлассификаторов`.

### 3. Загрузить обновление классификатора

**Задача:** выполнить загрузку и обработку обновления классификатора (одним
вызовом или пошагово — проверить + обработать).

**Функции:**
`РаботаСКлассификаторами.ОбновитьКлассификаторы(Идентификаторы) Экспорт`
— Функция, регион `ПрограммныйИнтерфейс` (stable). Полный цикл: загрузка файла +
  обработка. Возвращает `Структура` с `КодОшибки` (`""`, `"ОбновлениеНеТребуется"`,
  `"НеверныйЛогинИлиПароль"`, `"ПревышеноКоличествоПопыток"` …).
`РаботаСКлассификаторами.ОбработатьОбновлениеКлассификатора(Идентификатор, Версия, ИдентификаторФайла) Экспорт`
— Функция, регион `ПрограммныйИнтерфейс` (stable). Загрузка конкретного файла +
  обработка; использовать совместно с `ДоступныеОбновленияКлассификаторов`.
`РаботаСКлассификаторами.ПолучитьФайлыКлассификаторов(Идентификаторы) Экспорт`
— Функция, регион `ПрограммныйИнтерфейс` (stable). Актуальные файлы из сервиса/кэша.

**Параметры:**
- `Идентификаторы` (`Массив` из `Строка`).
- `Идентификатор` (`Строка`), `Версия` (`Строка`), `ИдентификаторФайла` (`Строка`) —
  из строки результата `ДоступныеОбновленияКлассификаторов`.

**Пример:**
```bsl
// Вариант А — полный цикл одним вызовом
Результат = РаботаСКлассификаторами.ОбновитьКлассификаторы(Идентификаторы);
Если Результат.КодОшибки = "ОбновлениеНеТребуется" Тогда
    // уже актуально
КонецЕсли;

// Вариант Б — пошагово: сначала проверить, потом обработать
Доступно = РаботаСКлассификаторами.ДоступныеОбновленияКлассификаторов(Идентификаторы);
Если ПустаяСтрока(Доступно.КодОшибки) Тогда
    Для Каждого СтрокаВерсии Из Доступно.ДоступныеВерсии Цикл
        РаботаСКлассификаторами.ОбработатьОбновлениеКлассификатора(
            СтрокаВерсии.Идентификатор, СтрокаВерсии.Версия, СтрокаВерсии.ИдентификаторФайла);
    КонецЦикла;
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Игнорировать `КодОшибки` — при `"НеверныйЛогинИлиПароль"` данные не загрузятся,
  но исключение не выбрасывается. Проверяйте код и сообщайте пользователю.
- Загрузка из сервиса — сетевая операция; в регламентных заданиях БСП это уже
  обёрнуто. Из прикладного кода вызывайте в фоновом задании, если данных много.
- После успешной обработки БСП сама вызовет `УстановитьВерсиюКлассификатора` /
  `УстановитьДатуОбновленияКлассификатора` — вручную делать это нужно только при
  загрузке **не из сервиса** (напр. миграция данных из внешнего файла).

### 4. Узнать версию и дату загруженного классификатора

**Задача:** проверить, какая версия классификатора загружена и когда, без обращения
к сервису.

**Функции:**
`РаботаСКлассификаторами.ВерсияКлассификатора(Идентификатор, ВызыватьИсключение = Ложь) Экспорт`
— Функция → `Число` / `Неопределено`, регион `ПрограммныйИнтерфейс` (stable).
`РаботаСКлассификаторами.ДатаОбновленияКлассификатора(Идентификатор, ВызыватьИсключение = Ложь) Экспорт`
— Функция → `Дата` / `Неопределено`, регион `ПрограммныйИнтерфейс` (stable).
`РаботаСКлассификаторами.УстановитьВерсиюКлассификатора(Идентификатор, Версия) Экспорт`
`РаботаСКлассификаторами.УстановитьДатуОбновленияКлассификатора(Идентификатор, ДатаОбновления) Экспорт`
— Процедуры (stable). Только для загрузки не из сервиса.

**Параметры:**
- `Идентификатор` (`Строка`).
- `ВызыватьИсключение` (`Булево`) — `Истина` выбрасывает исключение, если
  идентификатор не найден; `Ложь` (по умолчанию) — возвращает `Неопределено`.

**Пример:**
```bsl
Версия = РаботаСКлассификаторами.ВерсияКлассификатора("АдресныйКлассификатор");
ДатаОбновления = РаботаСКлассификаторами.ДатаОбновленияКлассификатора("АдресныйКлассификатор");

Если Версия = Неопределено Тогда
    // классификатор ещё не загружался
КонецЕсли;

// После ручной загрузки данных (не из сервиса) — отметить версию
РаботаСКлассификаторами.УстановитьВерсиюКлассификатора("MyAppClassifier", 42);
РаботаСКлассификаторами.УстановитьДатуОбновленияКлассификатора("MyAppClassifier", ТекущаяДатаСеанса());
```

**Нюансы / антипаттерны:**
- Если идентификатор не найден, методы сами выполняют обновление данных регистра
  `ВерсииКлассификаторов` — не требует ручной синхронизации.
- `УстановитьДатуОбновленияКлассификатора` в монопольном обработчике обновления
  регистрирует информацию только **после** выполнения обновления подсистемы
  «Работа с классификаторами» — учитывайте порядок.

### 5. Обработать загруженный файл классификатора

**Задача:** прикладная конфигурация зарегистрировала свой классификатор — нужно
обработать бинарный файл, который БСП загрузил из сервиса.

**Функция (хук):**
`РаботаСКлассификаторамиПереопределяемый.ПриЗагрузкеКлассификатора(Идентификатор, Версия, Адрес, Обработан, ДополнительныеПараметры) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс`. **Хук переопределения**: реализуется в
одноимённом модуле прикладной конфигурации.

**Параметры:**
- `Идентификатор` (`Строка`) — идентификатор, заданный в `ПриДобавленииКлассификаторов`.
- `Версия` (`Число`) — номер загруженной версии.
- `Адрес` (`Строка`) — адрес двоичных данных файла во временном хранилище.
- `Обработан` (`Булево`, выходной) — `Ложь`, если при обработке были ошибки и файл
  надо загрузить повторно.
- `ДополнительныеПараметры` (`Структура`) — служебные параметры.

**Пример:**
```bsl
// В модуле РаботаСКлассификаторамиПереопределяемый прикладной конфигурации
Процедура ПриЗагрузкеКлассификатора(Идентификатор, Версия, Адрес, Обработан, ДополнительныеПараметры) Экспорт
    Если Идентификатор = "MyAppClassifier" Тогда
        ДвоичныеДанные = ПолучитьИзВременногоХранилища(Адрес);
        // ...разбор файла и заполнение своего справочника...
        Обработан = Истина;   // файл обработан успешно
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Удалять временное хранилище `Адрес` в обработчике — БСП сама сохраняет файл в
  кэш (при `СохранятьФайлВКэш = Истина`) для последующего использования в других
  областях данных.
- `Обработан = Ложь` (по умолчанию) заставит БСП повторить загрузку — обязательно
  выставляйте `Истина` при успешной обработке.
- Хук `ПриОпределенииНачальногоНомераВерсииКлассификатора(Идентификатор, НачальныйНомерВерсии)`
  задаёт стартовую версию, с которой начинается отсчёт «обновлений» для нового
  классификатора.

### 6. Запустить интерактивное обновление классификаторов с клиента

**Задача:** из формы показать пользователю помощник обновления классификаторов и
дождаться завершения загрузки.

**Функции:**
`РаботаСКлассификаторамиКлиент.НовыйПараметрыОбновленияКлассификаторов(Владелец = Неопределено, Идентификаторы = Неопределено, ОписаниеОповещения = Неопределено) Экспорт`
— Функция-конструктор параметров. Клиент, регион `ПрограммныйИнтерфейс` (stable).
`РаботаСКлассификаторамиКлиент.ОбновитьКлассификаторы(ПараметрыОбновления = Неопределено) Экспорт`
— Процедура, открывает помощник обновления. Клиент, stable.
`РаботаСКлассификаторамиКлиент.ИмяСобытияОповещенияОЗагрузки() Экспорт`
— Функция → `Строка` (имя события оповещения о завершении загрузки). Клиент, stable.

**Параметры:**
- `Владелец` (`ФормаКлиентскогоПриложения`) — форма-владелец.
- `Идентификаторы` (`Массив` из `Строка` / `Неопределено`) — какие обновлять; `Неопределено` = все зарегистрированные.
- `ОписаниеОповещения` (`ОписаниеОповещения`) — вызовется после закрытия помощника; `Результат` = массив обновлённых идентификаторов.

**Пример:**
```bsl
&НаКлиенте
Процедура ОбновитьКлассификаторы(Команда)
    Параметры = РаботаСКлассификаторамиКлиент.НовыйПараметрыОбновленияКлассификаторов(
        ЭтаФорма,
        ,   // все классификаторы
        Новый ОписаниеОповещения("ПослеОбновленияКлассификаторов", ЭтаФорма));
    РаботаСКлассификаторамиКлиент.ОбновитьКлассификаторы(Параметры);
КонецПроцедуры

&НаКлиенте
Процедура ПослеОбновленияКлассификаторов(ОбновленныеИдентификаторы, ДопПараметры) Экспорт
    // ОбновленныеИдентификаторы — массив строк
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать серверные `ОбновитьКлассификаторы`/`ОбработатьОбновлениеКлассификатора`
  напрямую из клиентского обработчика — это серверный API с сетевыми вызовами.
  Используйте клиентский помощник `ОбновитьКлассификаторы`.
- Перед вызовом можно проверить `РаботаСКлассификаторами.ИнтерактивнаяЗагрузкаКлассификаторовДоступна()`
  (сервер, stable) — возвращает `Истина`, если обработка обновления классификаторов
  разрешена в текущем режиме.

### 7. Получить данные страны мира по коду или наименованию

**Задача:** по цифровому/буквенному коду или наименованию получить структуру
страны (наименование, коды Альфа-2/Альфа-3, признак ЕАЭС).

**Функции:**
`УправлениеКонтактнойИнформацией.ДанныеКлассификатораСтранМираПоКоду(Знач Код, Знач ТипКода = "КодСтраны") Экспорт`
— Функция → `Структура` (`Наименование, Код, НаименованиеПолное, КодАльфа2, КодАльфа3, УчастникЕАЭС`), регион `ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеКонтактнойИнформацией.ДанныеКлассификатораСтранМираПоНаименованию(Знач Наименование) Экспорт`
— Функция → `Структура`. stable.
`УправлениеКонтактнойИнформацией.СтранаМираПоКодуИлиНаименованию(КодИлиНаименование, ДанныеЗаполнения = Неопределено) Экспорт`
— Функция → ссылка/данные страны. stable.
`УправлениеКонтактнойИнформацией.СтраныУчастникиЕАЭС() Экспорт` / `ЭтоСтранаУчастникЕАЭС(Страна) Экспорт`
— справочные функции по ЕАЭС. stable.

**Параметры:**
- `Код` (`Строка` / `Число`) — код страны.
- `ТипКода` (`Строка`) — `"КодСтраны"` (по умолчанию), `"Альфа2"`, `"Альфа3"`.
- `Наименование` (`Строка`) — наименование страны.

**Пример:**
```bsl
Россия  = УправлениеКонтактнойИнформацией.ДанныеКлассификатораСтранМираПоКоду("643");
Беларусь = УправлениеКонтактнойИнформацией.ДанныеКлассификатораСтранМираПоКоду("BY", "Альфа2");
// Россия.Наименование, Россия.КодАльфа2, Россия.КодАльфа3, Россия.УчастникЕАЭС

Если УправлениеКонтактнойИнформацией.ЭтоСтранаУчастникЕАЭС(Россия) Тогда
    // особый режим адреса ЕАЭС
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ⚠️ Методы живут в модуле `УправлениеКонтактнойИнформацией`, **не** в
  `РаботаСКлассификаторами` — типовая ошибка искать их в модуле классификаторов.
- `ДанныеКлассификатораСтранМираПоКоду` работает из загруженного справочника стран
  (обновляется через `СтраныМира` механизм) — если классификатор не загружен,
  данные будут неполными.

## Дополнительно

Прочие стабильные методы (регион `ПрограммныйИнтерфейс`), полные сигнатуры — через
`python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`:

- `РаботаСКлассификаторами.ИнтерактивнаяЗагрузкаКлассификаторовДоступна()` —
  признак доступности обработки обновления в текущем режиме (сервер).
- `РаботаСКлассификаторами.ОписаниеКлассификатора()` — конструктор структуры
  описания классификатора для `ПриДобавленииКлассификаторов`.
- `РаботаСКлассификаторамиКлиент.ИмяСобытияОповещенияОЗагрузки()` — имя события
  оповещения о завершении загрузки (для подписки через `ПодключитьОбработчикОповещения`).

Хук переопределения `РаботаСКлассификаторамиПереопределяемый.ПриОпределенииНастроекПользователя(Настройки)`
— настройка параметров пользователя (аутентификация портала ИТС) для классификаторов.
