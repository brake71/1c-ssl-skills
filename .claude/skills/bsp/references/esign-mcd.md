# Электронная подпись и машиночитаемые доверенности БСП

Подсистемы **ЭлектроннаяПодпись** (подписание, проверка, сертификаты) и
**МашиночитаемыеДоверенности** (МЧД — реестр ФНС). Покрывает стабильный
серверный/клиентский API, выбор сертификата, проверку подписи и сертификата,
усовершенствование подписи, работу с МЧД.

> ⚠️ **Две модели вызова.** Клиент (`ЭлектроннаяПодписьКлиент`) — асинхронный,
> через `ОписаниеОповещения`, открывает формы. Сервер (`ЭлектроннаяПодпись`) —
> синхронный, возвращает значение напрямую. Серверного `ЭлектроннаяПодпись.Подписать`
> **не существует** — подписание только клиентское; сервер только сохраняет
> результат (`ДобавитьПодпись`), читает подписи (`ПодписиОбъекта`) и проверяет
> (`ПроверитьПодпись`, `ПроверитьСертификат`).

## Модули

Семейство `ЭлектроннаяПодпись*` по суффиксной системе БСП:

- `ЭлектроннаяПодпись` — сервер: операции над ссылочными объектами, регистры
  подписей, проверка, менеджер криптографии. Стабильный API (60 экспортов в
  `ПрограммныйИнтерфейс`).
- `ЭлектроннаяПодписьКлиент` — клиент: интерактивные сценарии — форма подписания,
  выбор сертификата, диалоги, асинхронные операции через `ОписаниеОповещения`.
- `ЭлектроннаяПодписьКлиентСервер` — общие структуры: `НовыеСвойстваПодписи`,
  `РезультатПроверкиПодписи`.
- `ЭлектроннаяПодписьСлужебный` — ⚠️ служебный (`СлужебныйПрограммныйИнтерфейс`):
  `Зашифровать(Данные, Сертификат, МенеджерКриптографии)`, `ДоступнаЭлектроннаяПодпись`.
  Обратная совместимость не гарантируется. Ряд родственных служебных методов
  (`ПодписьВКодировкеDER`, `РасшифровкаДанных`) лежат в основном модуле
  `ЭлектроннаяПодпись` (тоже `СлужебныйПрограммныйИнтерфейс`) — не в Служебный.
- `ЭлектроннаяПодписьПереопределяемый` / `ЭлектроннаяПодписьКлиентПереопределяемый`
  — хуки (реализуйте, не вызывайте): `ПередНачаломОперации` и т. п.
- `ЭлектроннаяПодписьЛокализация` / `*КлиентЛокализация` / `*КлиентСерверЛокализация`
  — региональные переопределения (РФ-специфика по МЧД).
- `ЭлектроннаяПодписьВМоделиСервиса*` — разделение данных в SaaS.

МЧД — **отдельное семейство с суффиксом `ФНС`**:

- `МашиночитаемыеДоверенностиФНС` — серверный API МЧД: создание, подпись файла
  доверенности, проверка, результат проверки подписи по МЧД.
- `МашиночитаемыеДоверенностиФНСКлиент` — клиент: `ОткрытьСписокМЧД`,
  `СоздатьМЧД`, `ПроверитьДоверенность`.
- `МашиночитаемыеДоверенностиФНСПереопределяемый` /
  `*КлиентПереопределяемый` — хуки.
- ⚠️ Модуля `МашиночитаемыеДоверенности` (без `ФНС`) **не существует** —
  типовая ошибка по аналогии с `УправлениеДоступом`. Реальное имя — с суффиксом
  `ФНС`.

DSS (сервис цифровой подписи, КриптоПро DSS и аналоги) — отдельное семейство
модулей `СервисКриптографииDSS*` и `СервисМобильнойПодписи*` (подсистема
`ЭлектроннаяПодписьСервисаDSS`). Этот файл покрывает DSS кратко — основной путь
через `ЭлектроннаяПодписьКлиент.Подписать`; специфичные DSS-функции —
`СервисКриптографииDSS*` / `СервисМобильнойПодписи*` напрямую.

> ⚠️ **`ПроверитьПодпись` существует в 6 модулях с РАЗНЫМИ сигнатурами**:
> `ЭлектроннаяПодпись` (сервер, синхронный, по менеджеру криптографии),
> `ЭлектроннаяПодписьКлиент` (клиент, асинхронный, по `ОписаниеОповещения`),
> `СервисКриптографии` / `СервисКриптографииКлиент` (общий менеджер криптографии),
> `СервисКриптографииDSS` / `СервисКриптографииDSSКлиент` (DSS). Для прикладного
> кода канонический — `ЭлектроннаяПодпись.ПроверитьПодпись` (сервер) и
> `ЭлектроннаяПодписьКлиент.ПроверитьПодпись` (клиент). **Всегда уточняйте
> `--module`** при поиске сигнатуры.

## Сценарии

### 1. Подписать данные или объект (клиент, асинхронно)

**Задача:** подписать двоичные данные, файл или ссылочный объект выбранным
сертификатом через стандартную форму; при указании объекта — записать подпись
в ИБ.

**Функция:**
`ЭлектроннаяПодписьКлиент.Подписать(ОписаниеДанных, Форма = Неопределено, ОбработкаРезультата = Неопределено, ПараметрыПодписи = Неопределено) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент (Тонкий,
Толстый). Результат — в `ОбработкаРезультата` (асинхронно).

**Параметры:**
- `ОписаниеДанных` (Структура) — ключи:
  - `Операция` (Строка) — заголовок формы подписания (напр.
    `НСтр("ru = 'Подписание документа'")`).
  - `ЗаголовокДанных` (Строка) — заголовок данных (`"Документ"`, `"Файл"`).
  - `Данные` (`ДвоичныеДанные` / `Строка`-адрес / `ОписаниеОповещения` / Структура
    `ПараметрыCMS` + `Данные`) — данные для подписания.
  - `Объект` (`ЛюбаяСсылка`, необяз.) — ссылка, к которой добавить подпись; если
    указан, серверная часть сама вызовет `ДобавитьПодпись` и установит
    `ПодписанЭП = Истина`.
  - `ВерсияОбъекта` (Строка, необяз.) — версия для проверки/блокировки.
  - `ПоказатьКомментарий` (Булево, необяз.) — разрешить ввод комментария в форме.
  - `ОтборСертификатов` (Массив, необяз.) — ссылки на
    `СертификатыЭлектроннойПодписиИШифрования` для отбора выбора.
  - `ВыбраннаяДоверенность` (`СправочникСсылка.МашиночитаемыеДоверенности`,
    необяз.) — МЧД для подписи от имени представителя.
  - `ВыполнятьНаСервере` (Булево / `Неопределено`, необяз.) — `Неопределено` =
    сначала сервер, при неудаче клиент; `Истина` = только сервер; `Ложь` =
    только клиент.
- `Форма` (`ФормаКлиентскогоПриложения` / `УникальныйИдентификатор` /
  `Неопределено`) — форма для блокировки объекта; `Неопределено` = стандартная
  форма.
- `ОбработкаРезультата` (`ОписаниеОповещения`) — обработчик результата; в него
  приходит `ОписаниеДанных`, дополненное `Успех` (Булево), `Отказ` (Булево),
  `СвойстваПодписи` (Структура/адрес) и `ВыбранныйСертификат`.
- `ПараметрыПодписи` (см. `ЭлектроннаяПодписьКлиент.НовыйТипПодписи`) — тип подписи.

**Пример:**
```bsl
// &НаКлиенте — команда "Подписать"
ОписаниеДанных = Новый Структура;
ОписаниеДанных.Вставить("Операция",        НСтр("ru = 'Подписание документа'"));
ОписаниеДанных.Вставить("ЗаголовокДанных", НСтр("ru = 'Документ'"));
ОписаниеДанных.Вставить("Объект",          Объект.Ссылка);
ОписаниеДанных.Вставить("ВерсияОбъекта",   Объект.ВерсияДанных);
ОписаниеДанных.Вставить("ПоказатьКомментарий", Истина);

ОбработкаРезультата = Новый ОписаниеОповещения("ПослеПодписания", ЭтотОбъект);
ЭлектроннаяПодписьКлиент.Подписать(ОписаниеДанных, ЭтаФорма, ОбработкаРезультата);

// &НаКлиенте
Процедура ПослеПодписания(Результат, ДопПараметры) Экспорт
    Если Результат.Свойство("Успех") И Результат.Успех Тогда
        ЭтаФорма.Прочитать();  // серверная часть уже записала подпись к объекту
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ `ЭлектроннаяПодпись.Подписать(...)` из серверного кода — метод не существует
  на сервере (ошибка компиляции). Подписание инициирует клиент; сервер только
  сохраняет результат через `ДобавитьПодпись` (вызывается внутри `Подписать`,
  когда указан `Объект`).
- ❌ Вызывать клиентское API из `&НаСервере` — `ЭлектроннаяПодписьКлиент` не
  компилируется на сервере.
- Перед вызовом проверяйте доступность подсистемы:
  `ОбщегоНазначения.ПодсистемаСуществует("СтандартныеПодсистемы.ЭлектроннаяПодпись")`
  и `ЭлектроннаяПодпись.ИспользоватьЭлектронныеПодписи()`.

### 2. Проверить подпись на сервере (фон/обработка)

**Задача:** программно проверить действительность подписи и сертификата на
сервере (регламентное задание, обработка), с захватом описания ошибки.

**Функции:**
`ЭлектроннаяПодпись.МенеджерКриптографии(Операция, ПоказатьОшибку = Истина, ОписаниеОшибки = "", Программа = Неопределено) Экспорт`
— Функция → `МенеджерКриптографии` / `Неопределено`, регион `ПрограммныйИнтерфейс`
(stable). Сервер.
`ЭлектроннаяПодпись.ПроверитьПодпись(МенеджерКриптографии, ИсходныеДанные, Подпись, ОписаниеОшибки = Null, НаДату = Неопределено, РезультатСтруктура = Неопределено) Экспорт`
— Функция → `Булево`, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ЭлектроннаяПодпись.ПроверитьСертификат(МенеджерКриптографии, Сертификат, ОписаниеОшибки = Null, НаДату = Неопределено, ПараметрыПроверки = Неопределено) Экспорт`
— Функция → `Булево`, регион `ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Операция` (Строка) — `"Подписание"`, `"ПроверкаПодписи"`, `"Шифрование"`,
  `"Расшифровка"`, `"ПроверкаСертификата"`, `"ПолучениеСертификатов"` (вставляется
  в текст ошибки).
- `ПоказатьОшибку` (Булево) — `Истина` = вызвать исключение при неудаче; `Ложь` =
  вернуть `Неопределено` и заполнить `ОписаниеОшибки`.
- `ОписаниеОшибки` (Строка) — выходной, заполняется при возврате `Неопределено`.
- `Программа` (`СправочникСсылка.ПрограммыЭлектроннойПодписиИШифрования` /
  `Неопределено`) — `Неопределено` = первая программа из справочника.
- `ИсходныеДанные` (`ДвоичныеДанные` / `Строка`-адрес / Структура конверта) —
  подписанные данные.
- `Подпись` (`ДвоичныеДанные` / `Строка`-адрес) — проверяемая подпись.
- `ОписаниеОшибки` (Строка, выходной, по умолчанию `Null`) — заполняется только
  при неуспехе.
- `НаДату` (`Дата` / `Неопределено`) — дата проверки сертификата; `Неопределено`
  = дата из подписи, иначе дата сеанса.
- `РезультатСтруктура` (Структура, необяз.) — если передать структуру (из
  `ЭлектроннаяПодписьКлиентСервер.РезультатПроверкиПодписи()`), результат
  заполняется детально (категории ошибок, статусы).

**Пример:**
```bsl
// Серверный код (регламентное задание / обработка)
ОписаниеОшибки = "";
Менеджер = ЭлектроннаяПодпись.МенеджерКриптографии("ПроверкаПодписи", Ложь, ОписаниеОшибки);
Если Менеджер = Неопределено Тогда
    ЗаписьЖурналаРегистрации(НСтр("ru = 'ЭП.Проверка подписи'"),
        УровеньЖурналаРегистрации.Предупреждение, , , ОписаниеОшибки);
    Возврат;
КонецЕсли;

Подписи = ЭлектроннаяПодпись.ПодписиОбъекта(ДокументСсылка);
Для Каждого СвойстваПодписи Из Подписи Цикл
    ОписаниеОшибки = "";
    Верна = ЭлектроннаяПодпись.ПроверитьПодпись(Менеджер,
        ДвоичныеДанныеОбъекта, СвойстваПодписи.Подпись, ОписаниеОшибки);
    Если Не Верна Тогда
        ЗаписьЖурналаРегистрации(НСтр("ru = 'ЭП.Проверка подписи'"),
            УровеньЖурналаРегистрации.Предупреждение, , , ОписаниеОшибки);
    КонецЕсли;
КонецЦикла;
```

**Нюансы / антипаттерны:**
- ❌ Игнорировать `ОписаниеОшибки` (4-й параметр) — булево не даёт причины для
  пользователя/журнала. Всегда передавайте выходную строку и логируйте при неуспехе.
- ❌ Создавать `Новый МенеджерКриптографии("Crypto-Pro GOST R 34.10-2012", "", 75)`
  в обход `ЭлектроннаяПодпись.МенеджерКриптографии` — ломает интеграцию с подсистемой
  (настройки программ, логирование, оповещения о сертификатах). Используйте
  обёртку БСП.
- Сертификат всегда проверяется на сервере, если администратор настроил проверку
  ЭП на сервере (`ЭлектроннаяПодпись.ПроверятьЭлектронныеПодписиНаСервере()`).
- Для детального результата — передайте структуру из
  `ЭлектроннаяПодписьКлиентСервер.РезультатПроверкиПодписи()` в `РезультатСтруктура`.

### 3. Записать, обновить и удалить подпись объекта

**Задача:** сохранить подпись к ссылочному объекту (с установкой `ПодписанЭП`),
обновить свойства уже записанной подписи, удалить подпись.

**Функции:**
`ЭлектроннаяПодпись.ДобавитьПодпись(Объект, Знач СвойстваПодписи, ИдентификаторФормы = Неопределено, ВерсияОбъекта = Неопределено, ЗаписанныйОбъект = Неопределено) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ЭлектроннаяПодпись.ОбновитьПодпись(Объект, Знач СвойстваПодписи, ОбновитьПоПорядковомуНомеру = Ложь) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ЭлектроннаяПодпись.УдалитьПодпись(Объект, ПорядковыйНомер, ИдентификаторФормы = Неопределено, ВерсияОбъекта = Неопределено, ЗаписанныйОбъект = Неопределено) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Объект` (`ОпределяемыйТип.ПодписанныйОбъект` — ссылка или объект) — должен
  иметь реквизит `ПодписанЭП`. По ссылке: объект блокируется, изменяется,
  записывается. Если передать **объект** (а не ссылку) — изменяется без блокировки
  и без записи (вызывающий записывает сам).
- `СвойстваПодписи` (`Строка`-адрес / `Структура` / Массив) — структура из
  `ЭлектроннаяПодписьКлиентСервер.НовыеСвойстваПодписи()` (отпечаток, дата,
  сертификат, статус, тип подписи, подпись).
- `ИдентификаторФормы` (`УникальныйИдентификатор`) — для блокировки объекта;
  из формы — `ЭтаФорма.УникальныйИдентификатор`.
- `ВерсияОбъекта` (Строка, необяз.) — версия данных для проверки и блокировки.
- `ПорядковыйНомер` (Число) — порядковый номер подписи (для `УдалитьПодпись` и
  `ОбновитьПодпись` при `ОбновитьПоПорядковомуНомеру = Истина`).
- `ЗаписанныйОбъект` (объект, необяз.) — уже записанный объект (чтобы не читать
  повторно).

**Пример:**
```bsl
// В собственной серверной обработке (вне клиентского Подписать)
СвойстваПодписи = ЭлектроннаяПодписьКлиентСервер.НовыеСвойстваПодписи();
СвойстваПодписи.Отпечаток = ОтпечатокСертификата;
СвойстваПодписи.Подпись   = ДвоичныеДанныеПодписи;
СвойстваПодписи.Сертификат = ДвоичныеДанныеСертификата;
// ... остальные свойства ...

ЭлектроннаяПодпись.ДобавитьПодпись(Объект.Ссылка, СвойстваПодписи, УИДФормы);

// Обновить свойства подписи (напр. после усовершенствования) по порядковому номеру
ЭлектроннаяПодпись.ОбновитьПодпись(Объект.Ссылка, НовыеСвойства, Истина);

// Удалить подпись № 2
ЭлектроннаяПодпись.УдалитьПодпись(Объект.Ссылка, 2, УИДФормы);
```

**Нюансы / антипаттерны:**
- ❌ Писать подпись прямо в `РегистрСведений.ЭлектронныеПодписи` через набор
  записей — обходит логику `ПодписанЭП`, блокировки и оповещения. Только через
  `ДобавитьПодпись` / `ОбновитьПодпись` / `УдалитьПодпись`.
- `ДобавитьПодпись` сама блокирует, изменяет и записывает объект по ссылке. Если
  вы уже держите объект и записываете сами — передайте объект (не ссылку), тогда
  БСП изменит без записи.
- ❌ `ЭлектроннаяПодпись.УстановленныеПодписи(...)` — **deprecated**
  (`УстаревшиеПроцедурыИФункции`), помечена `// Устарела. Следует использовать
  ПодписиОбъекта`. Не использовать в новом коде.

### 4. Получить подписи объекта и их свойства

**Задача:** прочитать список установленных подписей объекта (свойства,
отпечатки, статусы) для UI-формы или отчёта.

**Функции:**
`ЭлектроннаяПодпись.ПодписиОбъекта(Объект, ДополнительныеПараметры = Неопределено) Экспорт`
— Функция → `Массив` из структуры свойств подписи, регион `ПрограммныйИнтерфейс`
(stable). Сервер.
`ЭлектроннаяПодпись.СвойстваПодписи(Подпись, ПрочитатьСертификаты = Истина) Экспорт`
— Функция → Структура, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ЭлектроннаяПодпись.ДатаПодписания(Подпись, ПривестиКЧасовомуПоясуСеанса = Истина) Экспорт`
— Функция → `Дата` / `Неопределено`, регион `ПрограммныйИнтерфейс` (stable).
Сервер.
`ЭлектроннаяПодписьКлиентСервер.НовыеСвойстваПодписи() Экспорт` — Функция →
Структура-конструктор свойств подписи, регион `ПрограммныйИнтерфейс` (stable).
Клиент + Сервер.

**Параметры:**
- `Объект` (`ОпределяемыйТип.ПодписанныйОбъект`) — ссылка; должен иметь реквизит
  `ПодписанЭП`.
- `ДополнительныеПараметры` (см. `ЭлектроннаяПодпись.НовыйПараметрыПолученияПодписейОбъекта()`)
  — необязательный фильтр/настройки выборки.
- `Подпись` (`ДвоичныеДанные` / `Строка`-адрес) — подпись для разбора свойств /
  даты.
- `ПрочитатьСертификаты` (Булево) — `Истина` = прочитать сертификаты из подписи.

**Пример:**
```bsl
Подписи = ЭлектроннаяПодпись.ПодписиОбъекта(ДокументСсылка);
Для Каждого СвойстваПодписи Из Подписи Цикл
    // СвойстваПодписи.Отпечаток, .ДатаПодписи, .Сертификат, .ТипПодписи, .СтатусПроверки
    Сообщить(СвойстваПодписи.Отпечаток + " от " + СвойстваПодписи.ДатаПодписи);
КонецЦикла;

// Извлечь свойства и дату прямо из двоичных данных подписи
Св = ЭлектроннаяПодпись.СвойстваПодписи(ДвоичныеДанныеПодписи);
ДатаПодписи = ЭлектроннаяПодпись.ДатаПодписания(ДвоичныеДанныеПодписи);
```

**Нюансы / антипаттерны:**
- ⚠️ `ДатаПодписания` существует в **двух** модулях с разной сигнатурой:
  `ЭлектроннаяПодпись.ДатаПодписания(Подпись, ПривестиКЧасовомуПоясуСеанса = Истина)`
  — сервер, Функция (возвращает `Дата`); и
  `ЭлектроннаяПодписьКлиент.ДатаПодписания(Оповещение, Подпись,
  ПривестиКЧасовомуПоясуСеанса = Истина)` — клиент, Процедура (результат в
  `Оповещение`). Уточняйте модуль через `--module`.
- ❌ `УстановленныеПодписи(...)` — deprecated, используйте `ПодписиОбъекта`.

### 5. Усовершенствовать подпись до квалифицированной/архивной

**Задача:** повысить тип подписи (напр. до CAdES-T/A) — отдельно взятую подпись
или подпись, уже записанную к объекту.

**Функции:**
`ЭлектроннаяПодпись.УсовершенствоватьПодпись(Подпись, ТипПодписи, ДобавитьАрхивнуюМеткуВремени = Ложь, ДополнительныеПараметры = Неопределено) Экспорт`
— Функция → Структура изменившихся свойств, регион `ПрограммныйИнтерфейс`
(stable). Сервер.
`ЭлектроннаяПодпись.УсовершенствоватьПодписьОбъекта(ПодписанныйОбъект, ПорядковыйНомер, ТипПодписи, ДобавитьАрхивнуюМеткуВремени = Ложь, ИдентификаторФормы = Неопределено, ДополнительныеПараметры = Неопределено) Экспорт`
— Функция, регион `ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Подпись` (`ДвоичныеДанные`) — подпись для усовершенствования.
- `ТипПодписи` (`ПеречислениеСсылка.ТипыПодписиКриптографии`) — целевой тип. Если
  фактический тип уже выше — действий не будет.
- `ДобавитьАрхивнуюМеткуВремени` (Булево) — `Истина` = добавить метку времени в
  архивную подпись (CAdES-A).
- `ПодписанныйОбъект` (`ОпределяемыйТип.ПодписанныйОбъект`) — объект, чью подпись
  усовершенствуем.
- `ПорядковыйНомер` (Число) — порядковый номер подписи объекта.
- `ИдентификаторФормы` (`УникальныйИдентификатор`) — для блокировки.
- `ДополнительныеПараметры` (Структура) — `МенеджерКриптографии`
  (`Неопределено` / `МенеджерКриптографии`), `ИгнорироватьСрокДействияСертификата`
  (Булево).

**Пример:**
```bsl
// Усовершенствовать подпись объекта № 1 до архивной
ДопПараметры = Новый Структура;
ДопПараметры.Вставить("ИгнорироватьСрокДействияСертификата", Ложь);
ЭлектроннаяПодпись.УсовершенствоватьПодписьОбъекта(
    ДокументСсылка, 1, Перечисления.ТипыПодписиКриптографии.Архивная, Истина, , ДопПараметры);

// Усовершенствовать «голую» подпись (двоичные данные)
ИзмененныеСвойства = ЭлектроннаяПодпись.УсовершенствоватьПодпись(
    ДвоичныеПодписи, Перечисления.ТипыПодписиКриптографии.Усиленная);
```

**Нюансы / антипаттерны:**
- ❌ `ЭлектроннаяПодпись.ДоступнаУсовершенствованнаяПодпись()` — **deprecated**
  (`УстаревшиеПроцедурыИФункции`). Не использовать в новом коде; целевой тип
  подписи задаётся параметром `ТипПодписи`.
- `УсовершенствоватьПодпись` возвращает только **изменившиеся** свойства — не
  весь набор. Если тип уже выше целевого — структура пустая, действий нет.

### 6. Зарегистрировать сертификат и установить пароль (клиент)

**Задача:** открыть форму добавления сертификата пользователя в справочник
`СертификатыКлючейЭлектроннойПодписиИШифрования`; установить пароль сертификата
на сеанс.

**Функции:**
`ЭлектроннаяПодписьКлиент.ДобавитьСертификат(ОбработчикЗавершения = Неопределено, ПараметрыДобавления = Неопределено) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Клиент.
`ЭлектроннаяПодписьКлиент.ПараметрыДобавленияСертификата() Экспорт` — Функция →
Структура, регион `ПрограммныйИнтерфейс` (stable). Клиент.
`ЭлектроннаяПодписьКлиент.УстановитьПарольСертификата(СертификатСсылка, Пароль, ПояснениеПароля = Неопределено) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Клиент.
`ЭлектроннаяПодписьКлиент.ПолучитьОтпечаткиСертификатов(Оповещение, ТолькоЛичные, ПараметрыПолучения = Истина) Экспорт`
— Процедура (асинхронно), регион `ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `ОбработчикЗавершения` (`ОписаниеОповещения`) — вызывается после добавления.
- `ПараметрыДобавления` (Структура из `ПараметрыДобавленияСертификата`) — `Комментарий`,
  `ДляШифрования` и т. п.
- `СертификатСсылка` (`СправочникСсылка.СертификатыКлючейЭлектроннойПодписиИШифрования`)
  — сертификат, для которого запоминается пароль.
- `Пароль` (Строка) — пароль сертификата (хранится только на сеанс).
- `Оповещение` (`ОписаниеОповещения`) — возвращает массив строк-отпечатков.
- `ТолькоЛичные` (Булево) — `Истина` = личные сертификаты пользователя, `Ложь` = все.
- `ПараметрыПолучения` (Булево / Структура из `ПараметрыПолученияОтпечатковСертификатов`)
  — `Истина` = значения по умолчанию (клиентские + серверные + сервис).

**Пример:**
```bsl
// &НаКлиенте — добавление сертификата
Параметры = ЭлектроннаяПодписьКлиент.ПараметрыДобавленияСертификата();
Параметры.Комментарий = НСтр("ru = 'Сертификат для подписания ЭП'");
Обработчик = Новый ОписаниеОповещения("ПослеДобавленияСертификата", ЭтотОбъект);
ЭлектроннаяПодписьКлиент.ДобавитьСертификат(Обработчик, Параметры);

// Запомнить пароль сертификата на время сеанса (чтобы не спрашивать при каждом подписании)
ЭлектроннаяПодписьКлиент.УстановитьПарольСертификата(СертификатСсылка, ВведенныйПароль);
```

**Нюансы / антипаттерны:**
- ❌ Хранить пароль сертификата в реквизитах/константах в открытом виде. Если
  нужно долговременное хранение — только через безопасное хранилище
  (`ОбщегоНазначения.ЗаписатьДанныеВБезопасноеХранилище` — см. `base-common.md`),
  и читать перед подписанием, без записи в ИБ.
- `УстановитьПарольСертификата` действует **только на текущий сеанс** — при
  следующем запуске пароль нужно вводить заново (или хранить в безопасном хранилище).

### 7. Создать МЧД, проверить доверенность и подпись по МЧД

**Задача:** открыть форму создания машиночитаемой доверенности; проверить
доверенность (в т.ч. в реестре ФНС); проверить записанную подпись объекта на
соответствие МЧД (подписант = представитель, полномочия, дата).

**Функции:**
`МашиночитаемыеДоверенностиФНСКлиент.СоздатьМЧД(ПараметрыФормы, ОповещениеОЗавершении = Неопределено) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Клиент (Тонкий, Толстый).
`МашиночитаемыеДоверенностиФНСКлиент.ПроверитьДоверенность(Оповещение, Доверенность, ИдентификаторФормы = Неопределено) Экспорт`
— Процедура (асинхронно), регион `ПрограммныйИнтерфейс` (stable). Клиент.
`МашиночитаемыеДоверенностиФНСКлиент.ОткрытьСписокМЧД(Отборы = Неопределено, ОповещениеОЗакрытии = Неопределено, Владелец = Неопределено) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Клиент.
`МашиночитаемыеДоверенностиФНС.РезультатПроверкиДоверенности(Доверенность, ПроверятьВРеестреФНС = Неопределено) Экспорт`
— Функция → Структура, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`МашиночитаемыеДоверенностиФНС.РезультатПроверкиПодписиПоМЧД(ПодписанныйОбъект, ИдентификаторПодписи, СертификатПодписи, НаДату) Экспорт`
— Функция → `Массив` из Структура, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`МашиночитаемыеДоверенностиФНС.ДобавитьПодписьКФайлуДоверенности(ФайлДоверенности, Знач Подпись) Экспорт`
— Функция → `Булево` (Истина) или Строка (текст ошибки), регион
`ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ПараметрыФормы` (см. `МашиночитаемыеДоверенностиФНСКлиент.ПараметрыСозданияМЧД()`)
  — параметры формы создания.
- `Оповещение` / `ОповещениеОЗавершении` (`ОписаниеОповещения`) — обработчик
  результата.
- `Доверенность` (`СправочникСсылка.МашиночитаемыеДоверенности`) — проверяемая
  доверенность.
- `ПроверятьВРеестреФНС` (Булево / `Неопределено`) — `Неопределено` = в
  зависимости от признака `РегистрироватьВРеестре`.
- `ПодписанныйОбъект` (`ОпределяемыйТип.ПодписанныйОбъект`) — объект, чью подпись
  проверяем по МЧД.
- `ИдентификаторПодписи` (`УникальныйИдентификатор`) — идентификатор подписи из
  `НовыеСвойстваПодписи.ИдентификаторПодписи`.
- `СертификатПодписи` (`СертификатКриптографии` / `ДвоичныеДанные` / `Строка`-адрес)
  — сертификат подписавшего.
- `НаДату` (`Дата`) — дата подписи; если не заполнена — проверка на дату сеанса.
- `ФайлДоверенности` (ссылка на присоединённый файл МЧД) — файл, к которому
  добавляется подпись.
- `Подпись` (Структура свойств подписи) — подпись доверителя.

**Пример:**
```bsl
// Клиент: открыть форму создания МЧД
Параметры = МашиночитаемыеДоверенностиФНСКлиент.ПараметрыСозданияМЧД();
МашиночитаемыеДоверенностиФНСКлиент.СоздатьМЧД(Параметры,
    Новый ОписаниеОповещения("ПослеСозданияМЧД", ЭтотОбъект));

// Сервер: проверить доверенность (в т.ч. в реестре ФНС)
Результат = МашиночитаемыеДоверенностиФНС.РезультатПроверкиДоверенности(МЧДСсылка);
Если Результат.Верна Тогда
    // подписи доверенности верны и соответствуют доверителям
Иначе
    Сообщить(Результат.ТекстОшибки);
КонецЕсли;

// Сервер: проверить подпись объекта на соответствие МЧД (массив!)
РезультатМЧД = МашиночитаемыеДоверенностиФНС.РезультатПроверкиПодписиПоМЧД(
    ДокументСсылка, ИдентификаторПодписи, СертификатПодписи, ДатаПодписи);
Для Каждого СтрокаРезультата Из РезультатМЧД Цикл
    Если СтрокаРезультата.Верна Тогда
        // подпись валидна + полномочия соответствуют
    Иначе
        // СтрокаРезультата.ПротоколПроверки — детальный разбор
    КонецЕсли;
КонецЦикла;
```

**Нюансы / антипаттерны:**
- ❌ `РезультатПроверкиПодписиПоМЧД(...).Верна` напрямую (как в старых примерах) —
  метод возвращает **Массив** из Структура, а не одну структуру. Перебирайте
  массив: `Для Каждого СтрокаРезультата Из РезультатМЧД`. Каждая строка содержит
  `МашиночитаемаяДоверенность`, `Верна`, `ТребуетсяПроверка`,
  `ПодписантСоответствуетПредставителю`, `СовместныеПолномочия`,
  `ПротоколПроверки`.
- ❌ Модуль `МашиночитаемыеДоверенности` (без `ФНС`) — не существует. Реальные
  модули — `МашиночитаемыеДоверенностиФНС*` (с суффиксом `ФНС`).
- `ДобавитьПодписьКФайлуДоверенности` возвращает `Истина` при успехе либо
  **строку** с текстом ошибки при неудаче — проверяйте `ТипЗнч(Результат) =
  Тип("Булево")`, а не просто `Если Результат Тогда`.

## Редкие методы

Дополнительные стабильные методы, полные сигнатуры — через
`python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`:

- `ЭлектроннаяПодпись.СертификатИзДвоичныхДанныхПодписи(Подпись)` — извлечь
  сертификат из подписи (ДвоичныеДанные).
- `ЭлектроннаяПодпись.ОтпечаткиСертификатов(ТолькоЛичные, ОписаниеОшибки = Null,
  Сервис = Истина)` — массив отпечатков доступных сертификатов (сервер).
- `ЭлектроннаяПодпись.ПолучитьСертификатПоОтпечатку(Отпечаток, ТолькоВЛичномХранилище)`
  — `СертификатКриптографии` по отпечатку.
- `ЭлектроннаяПодпись.ЗаписатьСертификатВСправочник(Знач Сертификат,
  ДополнительныеПараметры = Неопределено)` — серверная запись сертификата в
  справочник (клиентский аналог — `ЭлектроннаяПодписьКлиент.ЗаписатьСертификатВСправочник`).
- `ЭлектроннаяПодпись.СсылкаНаСертификат(Знач Сертификат)` /
  `ЭлектроннаяПодпись.СсылкиНаСертификаты(Знач Сертификаты,
  Знач ВозвращатьНесуществующие = Ложь)` — ссылка/ссылки на справочник
  сертификатов по двоичным данным.
- `ЭлектроннаяПодпись.ШтампВизуализацииЭлектроннойПодписи(Знач Подпись,
  Знач ДатаПодписи = Неопределено, Знач ТекстОтметки = "",
  Знач ЛоготипОрганизации = Неопределено)` — штамп-визуализация подписи для
  печатной формы.
- `ЭлектроннаяПодпись.ПроверитьУстановкуПрограммКриптографии(
  ПараметрыПроверки = Неопределено)` — проверка установки программ ЭП.
- `ЭлектроннаяПодпись.РезультатПроверкиУдостоверяющегоЦентраСертификата(
  Сертификат, НаДату = Неопределено, ПараметрыПроверки = Неопределено)` —
  результат проверки УЦ сертификата.
- `ЭлектроннаяПодписьКлиент.ОтправитьНаПодписание(ОписаниеДанных, Форма =
  Неопределено, ОбработкаРезультата = Неопределено, ПараметрыПодписи =
  Неопределено)` — отправить на удалённое подписание (DSS).
- `ЭлектроннаяПодписьКлиент.ПроверитьСертификат(Оповещение, Сертификат,
  МенеджерКриптографии = Неопределено, НаДату = Неопределено,
  ПараметрыПроверки = Неопределено)` — асинхронная проверка сертификата (клиент).
- `ЭлектроннаяПодписьКлиентСервер.РезультатПроверкиПодписи()` — конструктор
  структуры детального результата проверки; передавайте в
  `ЭлектроннаяПодпись.ПроверитьПодпись(..., РезультатСтруктура)`.
- `МашиночитаемыеДоверенностиФНС.СоздатьИзменитьМашиночитаемуюДоверенность(
  Доверенность, ДанныеЗаполнения)` — серверное создание/изменение МЧД
  (программно, без формы).
- `МашиночитаемыеДоверенностиФНС.ФайлыДоверенности(Знач Доверенность,
  Знач ДляНалоговыхОрганов)` — файлы доверенности (для ФНС / для представления).
- `МашиночитаемыеДоверенностиФНС.УстановитьСтатусРегистрации(Доверенность,
  ИдентификаторТранзакции = Неопределено, ЭтоОтмена = Ложь)` — установить статус
  регистрации в реестре ФНС.
- `МашиночитаемыеДоверенностиФНС.ПрочитатьСостояниеМЧД(Доверенность)` —
  состояние МЧД (статус, ошибки).

⚠️ Служебные методы `ЭлектроннаяПодписьСлужебный` (регион
`СлужебныйПрограммныйИнтерфейс`, обратная совместимость не гарантируется):
- `Зашифровать(Данные, Сертификат, МенеджерКриптографии)` — серверное
  шифрование; третий параметр — **менеджер криптографии**, а не строка алгоритма.
  ⚠️ Коллизия: в основном модуле `ЭлектроннаяПодпись` есть одноимённый
  `Зашифровать(Данные, Сертификат, АлгоритмШифрования = "")` — там третий параметр
  строка; различайте по модулю при вызове (`--module` в `bsp_api.py`).
- `ДоступнаЭлектроннаяПодпись(ТипОбъекта)` — `Булево`, проверить наличие
  реквизита `ПодписанЭП` у типа (через `ОпределяемыйТип.ПодписанныйОбъект`).

⚠️ Родственные служебные методы в основном модуле `ЭлектроннаяПодпись` (регион
`СлужебныйПрограммныйИнтерфейс` — не в `…Служебный`):
- `ПодписьВКодировкеDER(ДанныеПодписи)` — подпись из двоичных данных в кодировке
  DER (`ДвоичныеДанные` / `Строка`-адрес).
- `РасшифровкаДанных(Данные, Сертификат, МенеджерКриптографии)` — расшифровка;
  возвращает `ДвоичныеДанные` / `Неопределено`.

Для поиска сигнатуры/региона любого метода —
`python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`.
