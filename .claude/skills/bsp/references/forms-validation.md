# Защита реквизитов, дополнительные свойства и даты запрета изменения

Три смежные подсистемы БСП, связанные с валидацией и защитой данных на управляемых
формах: **ЗапретРедактированияРеквизитовОбъектов** (блокировка «ключевых» реквизитов
после записи), **Свойства** (дополнительные реквизиты и сведения через ПВХ
`ДополнительныеРеквизитыИСведения`, модули `УправлениеСвойствами*`) и
**ДатыЗапретаИзменения** (период закрытия учёта — запрет записи/удаления объектов и
наборов записей в закрытом периоде). Когда нужно защитить реквизит от случайного
изменения, подключить к форме пользовательские свойства или учесть дату запрета при
записи.

## Модули

**ЗапретРедактированияРеквизитовОбъектов:**

- `ЗапретРедактированияРеквизитовОбъектов` — сервер, стабильный API:
  `ЗаблокироватьРеквизиты`, `БлокируемыеРеквизитыОбъекта`, `НовыйБлокируемыйРеквизит`,
  `ОписаниеБлокируемогоРеквизита`.
- `ЗапретРедактированияРеквизитовОбъектовКлиент` — клиент, стабильный API:
  `РазрешитьРедактированиеРеквизитовОбъекта`, `УстановитьДоступностьЭлементовФормы`,
  `УстановитьРазрешенностьРедактированияРеквизитов`, `Реквизиты`.
- `ЗапретРедактированияРеквизитовОбъектовСлужебный` /
  `…СлужебныйКлиент` — ⚠️ служебные (служебные реквизиты формы, кеш).
- `ЗапретРедактированияРеквизитовОбъектовПереопределяемый` — **хук**:
  `ПриОпределенииОбъектовСЗаблокированнымиРеквизитами` (регистрация объектов),
  `ПриОпределенииЗаблокированныхРеквизитов` (точечная донастройка).

> ⚠️ Не существует модуля `ЗапретРедактированияРеквизитовОбъектовКлиентСервер` и метода
> `СнятьЗапрет` — частые ошибки по аналогии. Снятие блокировки — только через
> пользовательский UI (команда «Разрешить редактирование»), серверного «разблокировать»
> намеренно нет.

**Свойства (УправлениеСвойствами):**

- `УправлениеСвойствами` — сервер, стабильный API: `ПриСозданииНаСервере`,
  `ПриЧтенииНаСервере`, `ПередЗаписьюНаСервере`, `ОбработкаПроверкиЗаполнения`,
  `ЗначениеСвойства`, `ЗначенияСвойств`, `СвойстваОбъекта`, `ЗаписатьСвойстваУОбъекта`,
  `УстановитьСвойстваУОбъекта`, `ОбновитьЭлементыДополнительныхРеквизитов`.
- `УправлениеСвойствамиКлиент` — клиент, стабильный API:
  `ОбрабатыватьОповещения`, `ОбновитьЗависимостиДополнительныхРеквизитов`,
  `ПослеЗагрузкиДополнительныхРеквизитов`, `ВыполнитьКоманду`, `РедактироватьМетки`.
- `УправлениеСвойствамиПереопределяемый` — **хуки**:
  `ЗаполнитьНаборыСвойствОбъекта`, `ПриПолученииПредопределенныхНаборовСвойств`,
  `ПриПолученииНаименованийНаборовСвойств`.
- `УправлениеСвойствамиСлужебный` — ⚠️ служебный.
- ⚠️ `УправлениеСвойствами.ПолучитьЗначенияСвойств`, `ПолучитьСписокСвойств`,
  `ПолучитьСписокЗначенийСвойств` — **устаревшие** (регион `УстаревшиеПроцедурыИФункции`);
  не использовать в новом коде, альтернативы — `ЗначенияСвойств`, `СвойстваОбъекта`.

> ⚠️ Не существует модуля `УправлениеСвойствамиКлиентСервер`. Клиентский и серверный
> код разделены на `УправлениеСвойствами` и `УправлениеСвойствамиКлиент` (как в
> большинстве подсистем БСП).

**ДатыЗапретаИзменения:**

- `ДатыЗапретаИзменения` — сервер, стабильный API: `ИзменениеЗапрещено`,
  `НайденЗапретИзмененияДанных`, `ОбъектПриЧтенииНаСервере`,
  `ПроверитьДатыЗапретаЗагрузкиДанных`, `ПараметрыСообщенияОЗапрете`,
  `ШаблонДанныхДляПроверки`, `ОтключитьПроверкуДатЗапрета`,
  `ПроверкаДатЗапретаОтключена`, и серия подписок на события
  (`ПроверитьДатуЗапретаИзмененияПередЗаписью`,
  `…ПередЗаписьюДокумента`, `…ПередЗаписьюНабораЗаписей`,
  `…ПередЗаписьюНабораЗаписейРегистраБухгалтерии`,
  `…ПередЗаписьюНабораЗаписейРегистраРасчета`, `…ПередУдалением`).
- `ДатыЗапретаИзмененияПереопределяемый` — **хук** регистрации разделов/объектов с
  датами запрета (состав проверок).
- ⚠️ `ДатыЗапретаИзменения.ОбновитьРазделыДатЗапретаИзменения` — **устаревший**
  (регион `УстаревшиеПроцедурыИФункции`).

> Проверка заполнения в формах с дополнительными реквизитами — платформенный контракт
> (обработчики `ОбработкаПроверкиЗаполненияНаСервере` формы и `ОбработкаПроверкиЗаполнения`
> объекта); БСП расширяет его через `УправлениеСвойствами.ОбработкаПроверкиЗаполнения`.
> Отдельного модуля `ПроверкаЗаполненияСлужебный` в БСП нет — это не ошибка, а реальность
> выгрузки.

## Сценарии

### 1. Заблокировать реквизиты объекта в форме

**Задача:** после записи объекта сделать «ключевые» реквизиты (валюта кассы,
организация договора) доступными только для чтения; дать пользователю с ролью
`ПолныеПрава`/`РедактированиеРеквизитовОбъектов` команду «Разрешить редактирование».

**Функции:**
`ЗапретРедактированияРеквизитовОбъектов.ЗаблокироватьРеквизиты(Форма, ГруппаДляКнопкиЗапрета = Неопределено, ЗаголовокКнопкиЗапрета = "", Объект = Неопределено) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ЗапретРедактированияРеквизитовОбъектовКлиент.РазрешитьРедактированиеРеквизитовОбъекта(Знач Форма, ОбработкаПродолжения = Неопределено, ТолькоВидимые = Истина) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Клиент.
`ЗапретРедактированияРеквизитовОбъектовКлиент.Реквизиты(Знач Форма, Знач ТолькоЗаблокированные = Истина, ТолькоВидимые = Истина) Экспорт` — Функция → Массив, `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `Форма` (УправляемаяФорма) — форма объекта.
- `ГруппаДляКнопкиЗапрета` (ГруппаФормы / Неопределено) — группа, куда поместить
  кнопку «Разрешить редактирование»; `Неопределено` → стандартное размещение.
- `ЗаголовокКнопкиЗапрета` (Строка) — текст кнопки; `""` → стандартный.
- `Объект` (ДанныеФормыСтруктура / Неопределено) — объект формы; `Неопределено` →
  `Форма.Объект`.
- `ОбработкаПродолжения` (ОписаниеОповещения / Неопределено) — обработчик после
  разблокировки; вызывается с `Булево` (результат разблокировки).
- `ТолькоВидимые` (Булево) — `Истина` → разблокировать только видимые реквизиты.

**Пример:**
```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    Если Объект.Ссылка.Пустая() Тогда
        Возврат;  // новый объект — блокировать не нужно
    КонецЕсли;
    ЗапретРедактированияРеквизитовОбъектов.ЗаблокироватьРеквизиты(ЭтаФорма);
КонецПроцедуры

&НаКлиенте
Процедура Подключаемый_РазрешитьРедактированиеРеквизитовОбъекта(Команда)
    ЗапретРедактированияРеквизитовОбъектовКлиент.РазрешитьРедактированиеРеквизитовОбъекта(
        ЭтаФорма, Новый ОписаниеОповещения("ПослеРазблокировки", ЭтотОбъект));
КонецПроцедуры

&НаКлиенте
Процедура ПослеРазблокировки(Результат, ДополнительныеПараметры) Экспорт
    Если Результат = Истина Тогда
        // реквизиты разблокированы
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вручную перебирать реквизиты и ставить `Элементы.Валюта.ТолькоПросмотр = Истина` —
  теряются: проверка прав, кнопка разблокировки, условная блокировка (для новых
  объектов), интеграция с групповым изменением реквизитов. Один вызов
  `ЗаблокироватьРеквизиты` делает всё это.
- ❌ Вызывать несуществующий `ЗапретРедактированияРеквизитовОбъектов.СнятьЗапрет(Форма)`
  — метод отсутствует (намеренно, чтобы избежать случайного снятия кодом). Снятие —
  только через UI (команда формы → общая форма `РазблокированиеРеквизитов`).
- Кнопка `Подключаемый_РазрешитьРедактированиеРеквизитовОбъекта` и её обработчик
  добавляются в форму автоматически при `ЗаблокироватьРеквизиты` (если у пользователя
  есть право и есть право на редактирование объекта).
- Список блокируемых реквизитов на сервере —
  `ЗапретРедактированияРеквизитовОбъектов.БлокируемыеРеквизитыОбъекта(ИмяОбъекта)`
  (возвращает массив имён из `ПолучитьБлокируемыеРеквизитыОбъекта` модуля менеджера).

### 2. Объявить блокируемые реквизиты в модуле менеджера

**Задача:** в модуле менеджера объекта вернуть список блокируемых реквизитов
(простых строк и расширенных описаний с предупреждением).

**Функции:**
`ЗапретРедактированияРеквизитовОбъектов.НовыйБлокируемыйРеквизит() Экспорт` — Функция → Структура (расширенное описание: `Имя`, `ЭлементыФормы` (Массив из Строка), `Предупреждение`, `Группа`), `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ЗапретРедактированияРеквизитовОбъектов.ОписаниеБлокируемогоРеквизита() Экспорт` — Функция → `Строка` (формат `"ИмяРеквизита[;ИмяЭлементаФормы,...][;ИмяГруппы]"` — простой вариант описания для массива из `ПолучитьБлокируемыеРеквизитыОбъекта`; для расширенного описания с полями используйте `НовыйБлокируемыйРеквизит`), `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:** нет; методы-конструкторы возвращают пустые структуры для заполнения.

**Пример:**
```bsl
// Модуль менеджера справочника "Кассы"
Функция ПолучитьБлокируемыеРеквизитыОбъекта() Экспорт
    БлокируемыеРеквизиты = Новый Массив;
    БлокируемыеРеквизиты.Добавить("Валюта");
    БлокируемыеРеквизиты.Добавить("Организация");

    // Расширенное описание с предупреждением при разблокировке
    Реквизит = ЗапретРедактированияРеквизитовОбъектов.НовыйБлокируемыйРеквизит();
    Реквизит.Имя = "Ответственный";
    Реквизит.Предупреждение = НСтр("ru = 'Смена ответственного влияет на все ранее введённые документы.'");
    БлокируемыеРеквизиты.Добавить(Реквизит);

    Возврат БлокируемыеРеквизиты;
КонецФункции
```

**Нюансы / антипаттерны:**
- Регистрация объекта как участника подсистемы — в хуке
  `ЗапретРедактированияРеквизитовОбъектовПереопределяемый.ПриОпределенииОбъектовСЗаблокированнымиРеквизитами(Объекты)`
  (прикладной код реализует, БСП вызывает). Без регистрации
  `ЗаблокироватьРеквизиты` не сработает.
- Формат `"Валюта"` (строка) — простая блокировка; `"Партнер;Партнер"` — после `;`
  перечисляются имена элементов формы, блокируемые вместе с реквизитом.
- ❌ Хранить список блокируемых реквизитов в коде формы, а не в модуле менеджера —
  нарушает единый источник истины; `БлокируемыеРеквизитыОбъекта` читает именно модуль
  менеджера.

### 3. Подключить подсистему «Свойства» к форме объекта

**Задача:** в форме объекта отрисовать дополнительные реквизиты из набора
`НаборыДополнительныхРеквизитовИСведений`, корректно прочитать их при открытии и
сохранить перед записью.

**Функции:**
`УправлениеСвойствами.ПриСозданииНаСервере(Форма, ДополнительныеПараметры = Неопределено) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеСвойствами.ПриЧтенииНаСервере(Форма, ТекущийОбъект) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеСвойствами.ПередЗаписьюНаСервере(Форма, ТекущийОбъект) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеСвойствами.ОбновитьЭлементыДополнительныхРеквизитов(Форма, Объект = Неопределено, СкрытьУдаленные = Неопределено) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Форма` (УправляемаяФорма) — форма объекта со свойствами (должна иметь `Объект`
  с `Ссылка`).
- `ДополнительныеПараметры` (Неопределено / Структура) — необязательные свойства:
  `Объект` (ДанныеФормыСтруктура), `ИмяЭлементаДляРазмещения` (Строка) — группа для
  размещения полей, и др.
- `ТекущийОбъект` (ДанныеФормыСтруктура) — объект из обработчика `ПриЧтенииНаСервере`.

**Пример:**
```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    УправлениеСвойствами.ПриСозданииНаСервере(ЭтаФорма);
КонецПроцедуры

&НаСервере
Процедура ПриЧтенииНаСервере(ТекущийОбъект)
    УправлениеСвойствами.ПриЧтенииНаСервере(ЭтаФорма, ТекущийОбъект);
КонецПроцедуры

&НаСервере
Процедура ПередЗаписьюНаСервере(Отказ, ТекущийОбъект, ПараметрыЗаписи)
    УправлениеСвойствами.ПередЗаписьюНаСервере(ЭтаФорма, ТекущийОбъект);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Забыть `ПриЧтенииНаСервере` при открытии формы из списка (минуя
  `ПриСозданииНаСервере`) — значения доп. реквизитов не перечитаются.
- ❌ Дублировать служебные реквизиты формы (`Свойства_ИспользоватьСвойства`,
  `Свойства_ОписаниеДополнительныхРеквизитов`, `ПараметрыСвойств`) своими — БСП сама
  создаёт их в `ПриСозданииНаСервере`.
- `ОбновитьЭлементыДополнительныхРеквизитов` — перестроить поля после изменения
  состава набора администратором (добавил/удалил реквизит в
  `НаборыДополнительныхРеквизитовИСведений`).

### 4. Проверить заполнение обязательных дополнительных реквизитов

**Задача:** при записи объекта проверить обязательные дополнительные реквизиты и
привязать сообщения об ошибках к полям формы; добавить собственные прикладные
проверки.

**Функция:**
`УправлениеСвойствами.ОбработкаПроверкиЗаполнения(Форма, Отказ, ПроверяемыеРеквизиты, Объект = Неопределено) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Форма` (УправляемаяФорма) — форма объекта.
- `Отказ` (Булево) — выходной: `Истина` прерывает запись.
- `ПроверяемыеРеквизиты` (Массив) — из платформенного обработчика
  `ОбработкаПроверкиЗаполненияНаСервере(Отказ, ПроверяемыеРеквизиты)`.
- `Объект` (ДанныеФормыСтруктура / Неопределено) — объект; `Неопределено` → `Форма.Объект`.

**Пример:**
```bsl
&НаСервере
Процедура ОбработкаПроверкиЗаполненияНаСервере(Отказ, ПроверяемыеРеквизиты)
    // Первая строка — БСП-мост: расширяет проверку обязательными доп. реквизитами
    УправлениеСвойствами.ОбработкаПроверкиЗаполнения(ЭтаФорма, Отказ, ПроверяемыеРеквизиты);

    // Прикладная проверка с привязкой к реквизиту формы
    Если Объект.Сумма <= 0 Тогда
        ОбщегоНазначения.СообщитьПользователю(
            НСтр("ru = 'Сумма должна быть больше нуля.'"),
            , "Объект.Сумма", , Отказ);
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Опустить вызов `УправлениеСвойствами.ОбработкаПроверкиЗаполнения` — флаг
  `ЗаполнятьОбязательно` в ПВХ `ДополнительныеРеквизитыИСведения` игнорируется,
  пользователь запишет объект с пустым обязательным доп. реквизитом.
- Платформа уже выполнила проверку «по умолчанию» (`АвтоОтметкаНезаполненного`,
  `ПроверкаЗаполнения`); прикладной код только **дополняет** её и выводит сообщения
  через `ОбщегоНазначения.СообщитьПользователю` (см. `base-common.md`).
- ❌ Вызывать несуществующий `ПроверкаЗаполненияСлужебный.ПроверитьЗаполнениеРеквизитовОбъекта`
  — такого модуля в БСП нет; валидация доп. реквизитов делается только через
  `УправлениеСвойствами.ОбработкаПроверкиЗаполнения` из обработчика формы.

### 5. Программно прочитать и записать свойства объекта

**Задача:** прочитать значения дополнительных реквизитов/сведений объекта по ссылке
и программно записать новые значения (напр. при обмене/загрузке).

**Функции:**
`УправлениеСвойствами.ЗначениеСвойства(Объект, Свойство, КодЯзыка = "") Экспорт` — Функция → Произвольный, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеСвойствами.ЗначенияСвойств(ОбъектыСоСвойствами, ПолучатьДопРеквизиты = Истина, ПолучатьДопСведения = Истина, Свойства = Неопределено, КодЯзыка = "") Экспорт` — Функция → ТаблицаЗначений, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеСвойствами.СвойстваОбъекта(ВладелецСвойств, ПолучатьДопРеквизиты = Истина, ПолучатьДопСведения = Истина) Экспорт` — Функция → Массив, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеСвойствами.ЗаписатьСвойстваУОбъекта(ВладелецСвойств, ТаблицаСвойствИЗначений) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеСвойствами.УстановитьСвойстваУОбъекта(Владелец, Свойства) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Объект` / `ВладелецСвойств` (ЛюбаяСсылка) — объект-владелец свойств.
- `Свойство` (СправочникСсылка.ДополнительныеРеквизиты / Сведения) — свойство.
- `ОбъектыСоСвойствами` (Массив / ЛюбаяСсылка) — массив ссылок для `ЗначенияСвойств`.
- `ПолучатьДопРеквизиты` (Булево) — `Истина` → включить доп. реквизиты.
- `ПолучатьДопСведения` (Булево) — `Истина` → включить доп. сведения.
- `Свойства` (Массив / Неопределено) — отбор конкретных свойств.
- `ТаблицаСвойствИЗначений` (ТаблицаЗначений) — колонки `Свойство`, `Значение`.
- `Свойства` (Структура / Массив) — для `УстановитьСвойстваУОбъекта`: соответствие
  свойство → значение.

**Пример:**
```bsl
// Прочитать одно свойство
Значение = УправлениеСвойствами.ЗначениеСвойства(Ссылка, СвойствоИнн);

// Пакетно прочитать реквизиты и сведения нескольких объектов
ТаблицаСвойств = УправлениеСвойствами.ЗначенияСвойств(МассивСсылок);

// Записать несколько свойств разом
Таблица = Новый ТаблицаЗначений("Свойство,Значение");
Таблица.Добавить().Свойство = СвойствоИнн;     Таблица[0].Значение = "7701234567";
Таблица.Добавить().Свойство = СвойствоОтдел;  Таблица[1].Значение = Справочники.Отделы.Продажи;
УправлениеСвойствами.ЗаписатьСвойстваУОбъекта(Ссылка, Таблица);
```

**Нюансы / антипаттерны:**
- ❌ Использовать устаревшие `УправлениеСвойствами.ПолучитьЗначенияСвойств` /
  `ПолучитьСписокСвойств` / `ПолучитьСписокЗначенийСвойств` (регион
  `УстаревшиеПроцедурыИФункции`) — в новом коде альтернативы `ЗначенияСвойств`,
  `СвойстваОбъекта`, `ПредставленияЗначенийСвойств`.
- `ЗначенияСвойств` пакетным вызовом эффективнее нескольких `ЗначениеСвойства` —
  один запрос к БД.
- ❌ Прямая запись в табличную часть `ДополнительныеРеквизиты` объекта в обход
  `ЗаписатьСвойстваУОбъекта` — теряется контроль типов и обязательности. Только
  через API `УправлениеСвойствами`.

### 6. Обновить зависимые реквизиты и обрабатывать оповещения свойств

**Задача:** после изменения управляющего реквизита пересчитать
видимость/доступность/обязательность зависимых доп. реквизитов; реагировать на
изменение набора свойств администратором.

**Функции:**
`УправлениеСвойствамиКлиент.ОбновитьЗависимостиДополнительныхРеквизитов(Форма, Объект = Неопределено) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Клиент.
`УправлениеСвойствамиКлиент.ОбрабатыватьОповещения(Форма, ИмяСобытия, Параметр) Экспорт` — Функция → Булево, `#Область ПрограммныйИнтерфейс` (stable). Клиент.
`УправлениеСвойствамиКлиент.ПослеЗагрузкиДополнительныхРеквизитов(Форма) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `Форма` (УправляемаяФорма) — форма объекта со свойствами.
- `Объект` (ДанныеФормыСтруктура / Неопределено) — объект; `Неопределено` → `Форма.Объект`.
- `ИмяСобытия` (Строка) — имя события оповещения.
- `Параметр` (Произвольный) — параметр оповещения.

**Пример:**
```bsl
&НаКлиенте
Процедура ПриИзмененииУправляющегоРеквизита(Элемент)
    УправлениеСвойствамиКлиент.ОбновитьЗависимостиДополнительныхРеквизитов(ЭтаФорма);
КонецПроцедуры

&НаКлиенте
Процедура ОбработкаОповещения(ИмяСобытия, Параметр, Источник)
    Если УправлениеСвойствамиКлиент.ОбрабатыватьОповещения(ЭтаФорма, ИмяСобытия, Параметр) Тогда
        УправлениеСвойствами.ОбновитьЭлементыДополнительныхРеквизитов(ЭтаФорма);
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать несуществующий `УправлениеСвойствамиКлиентСервер.ОбновитьЗависимости…`
  — модуля нет; клиентский код целиком в `УправлениеСвойствамиКлиент`, серверный — в
  `УправлениеСвойствами`.
- `ОбрабатыватьОповещения` возвращает `Истина`, если оповещение относится к набору
  свойств/доп. реквизиту/сведению формы — тогда форма обновляет элементы; иначе
  оповещение чужое, не реагируйте.
- `УправлениеСвойствамиКлиент.ОткрытьСписокСвойств(ИмяКоманды)` — ⚠️ служебный (регион
  `СлужебныйПрограммныйИнтерфейс`), обратная совместимость не гарантируется;
  используйте только для администраторских команд, понимая риск.

### 7. Заблокировать форму при чтении объекта в закрытом периоде

**Задача:** при открытии объекта, дата которого попадает в закрытый период
(дата запрета изменения), заблокировать форму на чтение — пользователь видит, что
редактирование запрещено, без отдельных проверок в каждом обработчике.

**Функция:**
`ДатыЗапретаИзменения.ОбъектПриЧтенииНаСервере(Форма, ТекущийОбъект) Экспорт` — Функция, `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Форма` (УправляемаяФорма) — форма элемента объекта или записи регистра.
- `ТекущийОбъект` (СправочникОбъект / ДокументОбъект / … / РегистрСведенийМенеджерЗаписи)
  — объект из обработчика `ПриЧтенииНаСервере`.

**Пример:**
```bsl
&НаСервере
Процедура ПриЧтенииНаСервере(ТекущийОбъект)
    ДатыЗапретаИзменения.ОбъектПриЧтенииНаСервере(ЭтаФорма, ТекущийОбъект);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- Метод — обработчик `ПриЧтенииНаСервере`; встраивается в формы элементов
  справочников, документов, записей регистров. При срабатывании запрета форма
  переводится в режим «только просмотр».
- Сами подписки на запись/удаление (`ПроверитьДатуЗапретаИзмененияПередЗаписью*`)
  БСП подключает через подписки на события — прикладной код обычно не вызывает их
  напрямую; они срабатывают автоматически для объектов, зарегистрированных в
  подсистеме (через `ДатыЗапретаИзмененияПереопределяемый`).
- Для программных операций, минующих формы (обмен, регламентные задания),
  используйте явную проверку `ИзменениеЗапрещено` / `НайденЗапретИзмененияДанных`
  (сценарий 8) — подписки не покрывают все пути записи.

### 8. Программно проверить запрет изменения данных

**Задача:** перед программной записью/удалением объекта или набора записей в
закрытом периоде проверить, разрешено ли изменение, и при запрете получить
описание ошибки для пользователя.

**Функции:**
`ДатыЗапретаИзменения.ИзменениеЗапрещено(ДанныеИлиПолноеИмя, ИдентификаторДанных = Неопределено, ОписаниеОшибки = Null, УзелПроверкиЗапретаЗагрузки = Неопределено) Экспорт` — Функция → Булево, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ДатыЗапретаИзменения.НайденЗапретИзмененияДанных(Знач ДанныеДляПроверки, ПараметрыСообщенияОЗапрете = Неопределено, ОписаниеОшибки = Null, УзелПроверкиЗапретаЗагрузки = Неопределено) Экспорт` — Функция → Булево, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ДатыЗапретаИзменения.ШаблонДанныхДляПроверки() Экспорт` — Функция → `ТаблицаЗначений` (колонки `Дата` — Дата, `Раздел` — Строка, `Объект` — ЛюбаяСсылка) для заполнения и передачи в `НайденЗапретИзмененияДанных`, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ДатыЗапретаИзменения.ПараметрыСообщенияОЗапрете() Экспорт` — Функция → Структура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ДанныеИлиПолноеИмя` (Объект / НаборЗаписей / Строка) — проверяемый объект данных
  (справочник/документ/… объект, набор записей) или полное имя метаданных.
- `ИдентификаторДанных` (Произвольный / Неопределено) — уточняющий идентификатор
  (напр. ссылка записи регистра).
- `ОписаниеОшибки` (Null / Строка / Структура) — `Null` → описание не нужно; `Строка`
  → вернуть текст; `Структура` → вернуть структурное описание (`ПредставлениеДанных`,
  `ЗаголовокОшибки`, `Запреты`).
- `УзелПроверкиЗапретаЗагрузки` (ПланОбменаСсылка / Неопределено) — узел для проверки
  запрета **загрузки** данных при обмене; `Неопределено` → проверка запрета изменения.
- `ДанныеДляПроверки` (`ТаблицаЗначений`) — из `ШаблонДанныхДляПроверки()`
  (колонки `Дата`, `Раздел`, `Объект`); заполните строки проверяемыми данными.
- `ПараметрыСообщенияОЗапрете` (Структура / Неопределено) — из
  `ПараметрыСообщенияОЗапрете()`; `Неопределено` → текст сообщения не формируется.

**Пример:**
```bsl
// Проверка перед программной записью объекта
ОписаниеОшибки = "";
Если ДатыЗапретаИзменения.ИзменениеЗапрещено(ДокументОбъект, , ОписаниеОшибки) Тогда
    ОбщегоНазначения.СообщитьПользователю(ОписаниеОшибки, , "Объект.Дата", , Отказ);
    Возврат;
КонецЕсли;

// Проверка запрета загрузки из узла обмена
Если ДатыЗапретаИзменения.ИзменениеЗапрещено(НаборЗаписей, , , УзелОбмена) Тогда
    // пропустить запись, записать в журнал регистрации
КонецЕсли;

// Детальная проверка с шаблоном данных
ДанныеДляПроверки = ДатыЗапретаИзменения.ШаблонДанныхДляПроверки();
Строка = ДанныеДляПроверки.Добавить();
Строка.Дата   = ДокументОбъект.Дата;
Строка.Объект = ДокументОбъект.Ссылка;
Строка.Раздел = "Документы";  // раздел из ПриЗаполненииРазделовДатЗапретаИзменения
ОписаниеОшибки = "";
Если ДатыЗапретаИзменения.НайденЗапретИзмененияДанных(ДанныеДляПроверки, , ОписаниеОшибки) Тогда
    // ОписаниеОшибки — структура с ПредставлениеДанных, ЗаголовокОшибки, Запреты
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Игнорировать `ИзменениеЗапрещено` при программной записи в регламентном
  задании/обмене — нарушается период закрытия учёта, данные «проталкиваются» мимо
  запрета. Всегда проверяйте перед записью объектов в закрытых периодах.
- Дата запрета изменения **не распространяется** на обмен данными (синхронизацию) —
  для защиты от изменений при обмене используйте отдельную дату запрета загрузки и
  передавайте `УзелПроверкиЗапретаЗагрузки`.
- `ОтключитьПроверкуДатЗапрета(Истина)` / `ПроверкаДатЗапретаОтключена()` —
  программное отключение проверки для привилегированного кода (напр. обработчиков
  обновления); используйте крайне аккуратно и только в обёртке
  `УстановитьПривилегированныйРежим`.
- ❌ Вызывать устаревший `ДатыЗапретаИзменения.ОбновитьРазделыДатЗапретаИзменения()`
  (регион `УстаревшиеПроцедурыИФункции`) — в новом коде не использовать.

  Функция (сервер): проверка наличия свойства у объекта.

  — Процедура (сервер): заполнение таблицы данных для проверки запрета.

Хуки переопределения (`*Переопределяемый`, регион `ПрограммныйИнтерфейс`) — БСП

  — точечная донастройка состава блокируемых реквизитов.
- `УправлениеСвойствамиПереопределяемый.ЗаполнитьНаборыСвойствОбъекта(Знач Объект, ТипСсылки, НаборыСвойств, СтандартнаяОбработка, КлючНазначения)`
  — привязка типа владельца к наборам свойств.
- `УправлениеСвойствамиПереопределяемый.ПриПолученииПредопределенныхНаборовСвойств(Наборы)`
  — задание предопределённых наборов свойств.
- `УправлениеСвойствамиПереопределяемый.ПриПолученииНаименованийНаборовСвойств(...)`
  — наименования наборов.
- `ДатыЗапретаИзмененияПереопределяемый.ПриЗаполненииРазделовДатЗапретаИзменения(Разделы)`
  — состав разделов дат запрета (имена + объекты каждого раздела). Без этого хука
  доступна только настройка общей даты запрета. Соседние хуки:
  `ЗаполнитьИсточникиДанныхДляПроверкиЗапретаИзменения` (источники для проверки),
  `ПередПроверкойЗапретаИзменения` (точечная донастройка проверки).
