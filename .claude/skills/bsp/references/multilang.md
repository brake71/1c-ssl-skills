# Мультиязычность БСП

Подсистема **Мультиязычность** — штатный механизм БСП для хранения и
отображения значений строковых реквизитов прикладных объектов на нескольких
языках. Два способа хранения переводов: в **шапке** объекта дублирующими
реквизитами с суффиксом `ЯзыкN` (`Наименование`, `НаименованиеЯзык1`,
`НаименованиеЯзык2`) или в **табличной части `Представления`** (реквизиты
`КодЯзыка` + локализуемые реквизиты). Способы взаимоисключающие на уровне
одного объекта.

Здесь — только API подсистемы `Мультиязычность`. Перевод интерфейсных строк
через `НСтр("ru = '...'; en = '...'")` — платформенный механизм, к БСП не
относится. Печатные формы на разных языках — `УправлениеПечатьюМультиязычность*`
(скил печати). Машинный перевод — подсистема «Перевод текста», не покрыта.

## Модули

- `МультиязычностьСервер` — серверный API: обработчики формы, преобразование
  запросов, информация о языках.
- `МультиязычностьКлиентСервер` — общий код (клиент + сервер): суффикс по
  номеру, обработчики представления объекта/ссылки.
- `МультиязычностьКлиент` — клиентский API: обработчик `ПриОткрытии` поля
  формы, открывающий `ОбщаяФорма.ВводНаРазныхЯзыках`.
- `МультиязычностьПереопределяемый` — хук `ПриОпределенииНастроек(Настройки)`
  (БСП вызывает, прикладной код реализует; не вызывается напрямую).
- `МультиязычностьПовтИсп` — кешируемые сведения о языках; в прикладном коде
  **напрямую не вызывать** — только через обёртки `МультиязычностьСервер` /
  `МультиязычностьКлиентСервер`.

Код основного языка — `ОбщегоНазначения.КодОсновногоЯзыка()` (stable,
`ПрограммныйИнтерфейс`; также есть `ОбщегоНазначенияКлиент.КодОсновногоЯзыка`).
Суффикс для основного языка — пустая строка: реквизит называется просто
`Наименование` (без `Язык0`). Дополнительные языки нумеруются по порядку
включения: `Язык1`, `Язык2`, … Код дополнительного языка (`"en"`, `"de"`) —
BCP-47-совместимый код из метаданных `Языки.КодЯзыка`.

## Сценарии

### 1. Форма объекта с мультиязычными реквизитами

**Задача:** в форме объекта подключить кнопку «ввод на разных языках» и
автоматическое отображение значения на текущем языке пользователя при чтении
и запись правильной языковой колонки при сохранении.

**Функции:**
`МультиязычностьСервер.ПриСозданииНаСервере(Форма, Объект = Неопределено, ИмяОбъекта = Неопределено) Экспорт`
— подключает кнопку открытия и обработчик к полям формы с локализуемыми
реквизитами; для форм списка модифицирует текст запроса динамического списка.
`МультиязычностьСервер.ПриЧтенииНаСервере(Форма, ТекущийОбъект, ИмяОбъекта = Неопределено) Экспорт`
— заполняет реквизиты формы значениями на текущем языке.
`МультиязычностьСервер.ПередЗаписьюНаСервере(ТекущийОбъект) Экспорт`
— разбирает значения на текущем языке в реквизиты `ЯзыкN` или строки ТЧ
  `Представления`.
— все Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Форма` (УправляемаяФорма) — форма объекта.
- `Объект` (Произвольный) — опц. основной реквизит формы.
- `ИмяОбъекта` (Строка) — имя основного реквизита формы: `"Объект"` (по
  умолчанию для форм объектов), `"Запись"` для регистров, `"Список"` для
  списков. Передавайте явно, если имя нестандартное.
- `ТекущийОбъект` (Произвольный) — объект/запись формы.

**Пример:**
```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    МультиязычностьСервер.ПриСозданииНаСервере(ЭтотОбъект, Объект, "Объект");
КонецПроцедуры

&НаСервере
Процедура ПриЧтенииНаСервере(ТекущийОбъект)
    МультиязычностьСервер.ПриЧтенииНаСервере(ЭтотОбъект, ТекущийОбъект, "Объект");
КонецПроцедуры

&НаСервере
Процедура ПередЗаписьюНаСервере(Отказ, ТекущийОбъект, ПараметрыЗаписи)
    МультиязычностьСервер.ПередЗаписьюНаСервере(ТекущийОбъект);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Не вызывать `МультиязычностьСервер.ПриСозданииНаСервере(ЭтотОбъект)` в
  форме списка — список будет показывать значения на основном языке для всех
  пользователей. Метод модифицирует `ТекстЗапроса` динамического списка, и
  вызывать его надо **первым** в обработчике, иначе последующий код может
  перезаписать текст.
- Fallback на основной язык встроен в `ПриЧтенииНаСервере`: если значение на
  текущем языке пусто — подставляется основной язык. Не реализуйте fallback
  вручную.
- `ИмяОбъекта = "Объект"` — стандартное имя основного реквизита. Для форм
  записей регистров передавайте `"Запись"`, для форм списков — `"Список"`.

### 2. Модуль менеджера: представление ссылки и ввод по строке

**Задача:** чтобы ссылка на элемент отображалась на текущем языке
пользователя, а ввод по строке искал по всем языковым представлениям объекта.

**Функции:**
`МультиязычностьКлиентСервер.ОбработкаПолученияПредставления(Данные, Представление, СтандартнаяОбработка, ИмяРеквизита = "Наименование") Экспорт`
— подменяет представление ссылки на значение реквизита на текущем языке.
  Клиент + Сервер.
`МультиязычностьКлиентСервер.ОбработкаПолученияПолейПредставления(Поля, СтандартнаяОбработка, ИмяРеквизита = "Наименование") Экспорт`
— добавляет к полям представления языковые реквизиты. Клиент + Сервер.
`МультиязычностьСервер.ОбработкаПолученияДанныхВыбора(ДанныеВыбора, Знач Параметры, СтандартнаяОбработка, ОбъектМетаданных) Экспорт`
— подменяет стандартный выбор по строке: ищет по всем языковым представлениям.
  Сервер.
— все Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable).

**Параметры:**
- `Данные` (Произвольный) — данные объекта из обработчика модуля менеджера.
- `Представление` (Строка) — **выходной** параметр: формируемое представление.
- `СтандартнаяОбработка` (Булево) — **выходной** параметр; метод ставит
  `Ложь`, когда подменяет представление.
- `ИмяРеквизита` (Строка) — имя локализуемого реквизита; по умолчанию
  `"Наименование"`. Передавайте явно для нестандартных реквизитов
  (напр. `"НаименованиеДляПечати"`).
- `Поля` (Массив) — **выходной** параметр: поля, по которым формируется
  представление.
- `ДанныеВыбора` (ДанныеВыбора) — **выходной** параметр: список данных выбора.
- `ОбъектМетаданных` (Метаданные) — метаданные объекта (из `Метаданные()`).

**Пример:**
```bsl
// Модуль менеджера справочника
Процедура ОбработкаПолученияПредставления(Данные, Представление, СтандартнаяОбработка) Экспорт
    МультиязычностьКлиентСервер.ОбработкаПолученияПредставления(
        Данные, Представление, СтандартнаяОбработка, "Наименование");
КонецПроцедуры

Процедура ОбработкаПолученияПолейПредставления(Поля, СтандартнаяОбработка) Экспорт
    МультиязычностьКлиентСервер.ОбработкаПолученияПолейПредставления(
        Поля, СтандартнаяОбработка, "Наименование");
КонецПроцедуры

Процедура ОбработкаПолученияДанныхВыбора(ДанныеВыбора, Параметры, СтандартнаяОбработка) Экспорт
    МультиязычностьСервер.ОбработкаПолученияДанныхВыбора(
        ДанныеВыбора, Параметры, СтандартнаяОбработка, Метаданные());
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ `ОбработкаПолученияПолейПредставления(Поля, СтандартнаяОбработка)` без
  третьего параметра — `ИмяРеквизита` по умолчанию `"Наименование"`. Если
  локализуемый реквизит называется иначе (напр. `"Заголовок"`), без явной
  передачи метод подставит не то поле. Уточняйте `ИмяРеквизита` для
  нестандартных реквизитов.
- Без этих трёх обработчиков ссылка на элемент будет отображаться только на
  основном языке, а ввод по строке не найдёт элементы по переводам.
- Fallback на основной язык встроен в `ОбработкаПолученияПредставления`: если
  на текущем языке пусто — возвращается основной; если и он пуст — стандартная
  обработка не отключается.

### 3. Динамический список с переключением языка

**Задача:** в форме списка динамический список должен показывать значения на
текущем языке пользователя; для программно построенного запроса — дописать
суффикс `ЯзыкN` к нужному полю.

**Функции:**
`МультиязычностьСервер.ПриСозданииНаСервере(Форма, Объект = Неопределено, ИмяОбъекта = Неопределено) Экспорт`
— для формы списка без явного `Объект` модифицирует `ТекстЗапроса`
  динамического списка `Список`. Сервер.
`МультиязычностьСервер.ИзменитьПолеЗапросаПодТекущийЯзык(ТекстЗапроса, ИмяПоля) Экспорт`
— дописывает суффикс `ЯзыкN` к указанному полю в тексте запроса (поддерживает
  `ИмяПоля КАК Псевдоним`). Ничего не делает, если текущий язык — основной.
  Сервер.
— обе Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable).

**Параметры:**
- `ТекстЗапроса` (Строка) — **выходной** параметр: текст запроса с
  модифицированным полем.
- `ИмяПоля` (Строка) — имя поля в тексте запроса, напр.
  `"СправочникНоменклатура.Наименование"`.

**Пример:**
```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    // Автоматически модифицирует ТекстЗапроса динамического списка "Список"
    МультиязычностьСервер.ПриСозданииНаСервере(ЭтотОбъект);
КонецПроцедуры

// Ручное управление (когда текст запроса строится программно)
ТекстЗапроса = "ВЫБРАТЬ СправочникНоменклатура.Наименование КАК Наименование, "
    + "СправочникНоменклатура.Артикул КАК Артикул ИЗ Справочник.Номенклатура КАК СправочникНоменклатура";
МультиязычностьСервер.ИзменитьПолеЗапросаПодТекущийЯзык(ТекстЗапроса, "СправочникНоменклатура.Наименование");
// На первом доп. языке Наименование превратится в НаименованиеЯзык1; на основном — без изменений
```

**Нюансы / антипаттерны:**
- ❌ Хардкодить `"Наименование" + "Язык1"` в запросе — сломается при смене
  порядка языков (пользователь включил `en` первым, а в конфигурации первым
  идёт `de`). Используйте `ИзменитьПолеЗапросаПодТекущийЯзык` или
  `СуффиксТекущегоЯзыка()`.
- `ИзменитьПолеЗапросаПодТекущийЯзык` ничего не делает на основном языке —
  поле остаётся `Наименование`. Это норма: основной язык хранится в базовом
  реквизите без суффикса.

### 4. Начальное заполнение предопределённых значений

**Задача:** при начальном заполнении предопределённых элементов заполнить
колонки `ИмяРеквизита_КодЯзыка` из строки в формате `НСтр` сразу для всех
языков, с fallback на основной.

**Функция:**
`МультиязычностьСервер.ЗаполнитьМультиязычныйРеквизит(Элемент, ИмяРеквизита, ИсходнаяСтрока, КодыЯзыков = Неопределено) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Элемент` (СтрокаТаблицыЗначений) — заполняемая строка таблицы с колонками
  `ИмяРеквизита_КодЯзыка`.
- `ИмяРеквизита` (Строка) — имя реквизита, напр. `"Наименование"`.
- `ИсходнаяСтрока` (Строка) — строка в формате `НСтр`, напр.
  `"ru = 'Пример'; en = 'Example'"`.
- `КодыЯзыков` (Массив) — коды языков, на которых нужно заполнить строки;
  `Неопределено` — все языки конфигурации.

**Пример:**
```bsl
Процедура ПриНачальномЗаполненииЭлементов(Элементы) Экспорт
    КодыЯзыков = Новый Массив;
    КодыЯзыков.Добавить("en");
    КодыЯзыков.Добавить("de");

    Для Каждого Элемент Из Элементы Цикл
        МультиязычностьСервер.ЗаполнитьМультиязычныйРеквизит(
            Элемент,
            "Наименование",
            "ru = 'Демо: пример'; en = 'Demo: example'; de = 'Demo: Beispiel'",
            КодыЯзыков);
    КонецЦикла;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Заполнять реквизит объекта через `НСтр` напрямую:
  `Объект.НаименованиеЯзык1 = НСтр("en = 'Example'")` — `НСтр` зависит от
  текущего языка **сеанса**, а не от языка, на котором редактируется объект;
  у разных пользователей будет разное содержимое. Используйте
  `ЗаполнитьМультиязычныйРеквизит` с явным списком кодов — он не зависит от
  сеанса и заполняет все колонки сразу.
- Метод сам обеспечивает fallback на основной язык для языков, отсутствующих
  в `ИсходнаяСтрока`: при пустом `НСтр(ИсходнаяСтрока, КодЯзыка)` подставляется
  `НСтр(ИсходнаяСтрока, КодОсновногоЯзыка())`.
- ❌ Хранить переводы в одном строковом реквизите с разделителем
  (`"Пример||Example||Beispiel"`) — ломает ввод по строке, индексирование и
  поиск. Используйте реквизиты `ЯзыкN` (шапка) или ТЧ `Представления`.

### 5. Узнать суффикс и код текущего языка

**Задача:** программно получить суффикс текущего языка (`Язык1` / `""`) или
суффикс по коду языка для построения имени реквизита.

**Функции:**
`МультиязычностьСервер.СуффиксТекущегоЯзыка() Экспорт`
— суффикс текущего языка сеанса (`""` для основного, `"Язык1"`, `"Язык2"`).
  Сервер.
`МультиязычностьСервер.СуффиксЯзыка(КодЯзыка) Экспорт`
— суффикс по коду языка (`"en"` → `"Язык1"`); `""` если язык не используется.
  Сервер.
`МультиязычностьКлиентСервер.СуффиксЯзыка(ПорядковыйНомерЯзыка = Неопределено) Экспорт`
— суффикс по порядковому номеру (`1` → `"Язык1"`); без параметра — базовая
  строка `"Язык"`. Клиент + Сервер.
`МультиязычностьСервер.ИспользуетсяДополнительныйЯзык(ПорядковыйНомерЯзыка) Экспорт`
— `Булево`: включён ли доп. язык с данным номером. Сервер.
`МультиязычностьСервер.КоличествоДополнительныхЯзыков() Экспорт`
— число включённых дополнительных языков. Сервер.
`МультиязычностьСервер.СведенияОЯзыках() Экспорт`
— структура сведений о языках конфигурации. Сервер.
— все Функции, регион `#Область ПрограммныйИнтерфейс` (stable).

**Параметры:**
- `КодЯзыка` (Строка) — BCP-47-совместимый код в нижнем регистре (`"en"`).
- `ПорядковыйНомерЯзыка` (Число) — номер доп. языка (`1`, `2`, …).

**Пример:**
```bsl
// Суффикс текущего языка
ИмяРеквизита = "Наименование" + МультиязычностьСервер.СуффиксТекущегоЯзыка();
// На основном языке -> "Наименование"; на первом доп. -> "НаименованиеЯзык1"

// Суффикс по коду языка
Суффикс = МультиязычностьСервер.СуффиксЯзыка("en");  // "Язык1" или ""
Если ПустаяСтрока(Суффикс) Тогда
    // "en" не включён в конфигурацию — берём основной реквизит
КонецЕсли;

// Суффикс по номеру (клиент + сервер)
Суффикс = МультиязычностьКлиентСервер.СуффиксЯзыка(1);  // "Язык1"

// Проверить, включён ли второй доп. язык
Если МультиязычностьСервер.ИспользуетсяДополнительныйЯзык(2) Тогда
    // ...
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Хардкодить `"Язык1"` в прикладном коде — сломается при смене порядка
  языков. Всегда получайте суффикс программно через `СуффиксТекущегоЯзыка()`
  или `СуффиксЯзыка(КодЯзыка)`.
- ❌ Вызывать `МультиязычностьПовтИсп.КоличествоДополнительныхЯзыков()` напрямую
  — `ПовтИсп` для внутреннего использования БСП. Обёртка
  `МультиязычностьСервер.КоличествоДополнительныхЯзыков()` читает из `ПовтИсп`
  и не сломается при изменении внутреннего контракта.
- Нормализуйте ввод кода языка к нижнему регистру (`НРег(КодЯзыка)`) и не
  полагайтесь на региональный суффикс: БСП не различает `"en-US"` и `"en"` —
  оба мапятся на один суффикс.

### 6. Программно прочитать мультиязычные реквизиты объекта

**Задача:** при программном чтении объекта вне формы (в обработке заполнения,
фоновом задании) заполнить мультиязычные реквизиты на текущем языке, как это
делает форма.

**Функция:**
`МультиязычностьСервер.ПриЧтенииПредставленийНаСервере(Объект) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Объект` (СправочникОбъект / ДокументОбъект) — уже полученный объект
  (через `ПолучитьОбъект()` или запрос с `Разрешенные`).

**Пример:**
```bsl
// В обработке заполнения/фоновом задании
Объект = Ссылка.ПолучитьОбъект();
МультиязычностьСервер.ПриЧтенииПредставленийНаСервере(Объект);
// Объект.Наименование теперь содержит значение на текущем языке сеанса
// (с fallback на основной, если перевод пуст)
```

**Нюансы / антипаттерны:**
- В отличие от `ПриЧтенииНаСервере(Форма, …)` (для форм), этот метод принимает
  уже полученный `*Объект`, без контекста формы. Используйте его для
  программного чтения вне UI.
- Перед записью объекта, изменённого программно, вызовите
  `МультиязычностьСервер.ПередЗаписьюНаСервере(ТекущийОбъект)` — он разберёт
  значение на текущем языке в нужные языковые колонки.

### 7. Расширение настроек подсистемы (хук переопределения)

**Задача:** прикладная конфигурация подстраивает поведение мультиязычности
(напр., отключает ввод на разных языках для отдельных ролей).

**Хук:**
`МультиязычностьПереопределяемый.ПриОпределенииНастроек(Настройки) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (хук переопределения).
  БСП вызывает этот метод; прикладной код **реализует** его в одноимённом
  модуле прикладной конфигурации. Не вызывается из прикладного кода напрямую.

**Параметры:**
- `Настройки` (Структура) — настройки подсистемы; заполняется/изменяется в хуке.

**Пример:**
```bsl
// МультиязычностьПереопределяемый (модуль прикладной конфигурации)
Процедура ПриОпределенииНастроек(Настройки) Экспорт
    // Напр., отключить ввод на разных языках для роли "Только чтение"
    Если Пользователи.РолиДоступны("ТолькоЧтение") Тогда
        Настройки.ВводНаРазныхЯзыкахДоступен = Ложь;
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Реализовывать хук в основном модуле `МультиязычностьСервер` — нельзя.
  Хук должен быть в `МультиязычностьПереопределяемый` **прикладной**
  конфигурации (копируется модуль-переопределитель, переопределяется тело).
- ❌ Вызывать `МультиязычностьПереопределяемый.ПриОпределенииНастроек(…)` из
  прикладного кода — это хук, его вызывает БСП в нужный момент.

## Дополнительно

Прочие стабильные методы (регион `ПрограммныйИнтерфейс`), полные сигнатуры —
через `python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`:

- `МультиязычностьСервер.ИменаРеквизитовСУчетомКодаЯзыка(ИменаРеквизитов, КодЯзыка = "")`
  — массив имён реквизитов с дописанным суффиксом языка.
- `МультиязычностьСервер.СведенияОЯзыках()` — структура сведений о языках
  конфигурации (коды, порядковые номера, основной язык).
- `МультиязычностьКлиент.ПриОткрытии(Форма, Объект, Элемент, СтандартнаяОбработка)`
  — клиентский обработчик `ПриОткрытии` поля формы, открывающий
  `ОбщаяФорма.ВводНаРазныхЯзыках` (вызывается из подключаемого обработчика
  формы, не напрямую).
- `ОбщегоНазначения.КодОсновногоЯзыка()` / `ОбщегоНазначенияКлиент.КодОсновногоЯзыка()`
  — код основного языка ИБ (напр. `"ru"`), stable. ⚠️ Служебная обёртка
  `МультиязычностьСервер.КодОсновногоЯзыка()` тоже есть, но в
  `СлужебныйПрограммныйИнтерфейс` — предпочтителен стабильный
  `ОбщегоНазначения.КодОсновногоЯзыка()`.
- `МультиязычностьКлиент.ОткрытьФормуРегиональныхНастроек(ОписаниеОповещения = Неопределено, Параметры = Неопределено)`
  — ⚠️ `СлужебныйПрограммныйИнтерфейс`: открытие формы региональных
  настроек (административный сценарий, не для прикладного вызова).

Для поиска сигнатуры/региона любого из этих методов —
`python scripts/bsp_api.py method <Имя> --src src/cf`.
