# Префиксация объектов: префиксы номеров и кодов

Подсистема **ПрефиксацияОбъектов** — автоматическое назначение и разбор
префиксов в номерах документов и кодах справочников/ПВХ/бизнес-процессов.
Префиксы нужны в двух разрезах: **информационной базы** (РИБ, чтобы различать
узлы) и **организации** (сквозная нумерация по организациям). Покрывает
программный разбор номера, формирование номера на печать и переопределение
логики префиксации через модули `*Переопределяемый`. Обмен данными
(планы обмена, регистрация изменений) здесь НЕ описан — это отдельная
подсистема.

## Модули

Суффиксная система имён (корень `ПрефиксацияОбъектов` + контекст):

- `ПрефиксацияОбъектовКлиентСервер` — **стабильный API для прикладного кода**:
  разбор номера, формирование номера на печать, извлечение пользовательского
  префикса. Клиент + сервер + внешнее соединение.
- `ПрефиксацияОбъектовКлиентСерверПереопределяемый` — **хук**: замещение
  логики `НомерНаПечать`. БСП вызывает, прикладной код реализует.
- `ПрефиксацияОбъектовПереопределяемый` — **хуки**: вычисление базового
  номера/кода при нестандартных форматах, карта префиксообразующих реквизитов.
- `ПрефиксацияОбъектовПовтИсп` — кэш соответствия «объект метаданных → имя
  реквизита организации»; внутренний, кэшируется.
- `ПрефиксацияОбъектовСлужебный` — ⚠️ служебный: добавление вычисляемого поля
  в СКД, изменение префикса ИБ с транзакционной перенумерацией.
- `ПрефиксацияОбъектовСобытия` — обработчики подписок на события
  (`УстановитьПрефиксОрганизации`, `УстановитьПрефиксИнформационнойБазы`,
  `ПроверитьНомерДокументаПоДате…`) и событийные `ПриОпределенииПрефиксаИнформационнойБазы` /
  `ПриОпределенииПрефиксаОрганизации`. Стабильные, но **не вызывать
  напрямую** — платформа дёргает их через подписки.

⚠️ **Модуль `ПрефиксацияОбъектов` (без суффикса) НЕ существует** как общий
модуль — в дереве конфигурации есть подсистема с таким именем, но не общий
модуль. Стабильный API лежит в `ПрефиксацияОбъектовКлиентСервер`. Типовая
ошибка — `ПрефиксацияОбъектов.НомерНаПечать(...)` (ошибка компиляции).

Формат номера: `[ОО][ИБ]-[ПП]ННН…`, где `ОО` — префикс организации (2 символа,
из справочника организаций), `ИБ` — префикс ИБ (2 символа, константа
`ПрефиксУзлаРаспределеннойИнформацийБазы`), `ПП` — пользовательский префикс
(опционально, 2 символа), `ННН…` — собственно номер. Пример: `ОРИБ-АВ0000123`.
На печать выводится без префикса организации и лидирующих нулей: `ИБ-АВ123`.

## Сценарии

### 1. Получить номер объекта на печать

**Задача:** преобразовать номер документа/кода в строку для печати — убрать
префикс организации, опционально префикс ИБ и пользовательский префикс, снять
лидирующие нули.

**Функция:**
`ПрефиксацияОбъектовКлиентСервер.НомерНаПечать(Знач НомерОбъекта, УдалитьПрефиксИнформационнойБазы = Ложь, УдалитьПользовательскийПрефикс = Ложь) Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент + сервер +
внешнее соединение.

**Параметры:**
- `НомерОбъекта` (Строка) — номер или код объекта для вывода на печать.
- `УдалитьПрефиксИнформационнойБазы` (Булево) — `Истина` удаляет префикс ИБ
  (два символа после префикса организации). По умолчанию `Ложь`.
- `УдалитьПользовательскийПрефикс` (Булево) — `Истина` удаляет пользовательский
  префикс (часть между стандартным префиксом и цифрами номера). По умолчанию
  `Ложь`.

**Пример:**
```bsl
// Номер "ОРИБ-АВ0000123" -> на печать без префикса организации и нулей
ПечатныйНомер = ПрефиксацияОбъектовКлиентСервер.НомерНаПечать(НомерДокумента);
// "ИБ-АВ123"

// Убрать и префикс ИБ, и пользовательский
ПечатныйНомер = ПрефиксацияОбъектовКлиентСервер.НомерНаПечать(НомерДокумента, Истина, Истина);
// "АВ123" (или "123" если пользовательский префикс тоже убрать — зависит от флагов)
```

**Нюансы / антипаттерны:**
- ❌ `ПрефиксацияОбъектов.НомерНаПечать(Номер)` — модуль без суффикса не
  существует. Только `ПрефиксацияОбъектовКлиентСервер`.
- ❌ `Сред(Номер, 6)` / `Лев(Номер, …)` — хрупко: не учитывает незначащие нули,
  разное положение дефиса (3 или 5), пользовательский префикс. Делегируйте
  подсистеме.
- Перед стандартной обработкой `НомерНаПечать` вызывает хук
  `ПрефиксацияОбъектовКлиентСерверПереопределяемый.ПриПолученииНомераНаПечать`;
  если в прикладном коде реализован этот обработчик и
  `СтандартнаяОбработка = Ложь`, функция вернёт то, что задано в `НомерОбъекта`.

### 2. Удалить указанные префиксы из номера (без снятия нулей)

**Задача:** убрать префикс организации и/или префикс ИБ из номера, сохраняя
лидирующие нули и пользовательский префикс на месте.

**Функция:**
`ПрефиксацияОбъектовКлиентСервер.УдалитьПрефиксыИзНомераОбъекта(Знач НомерОбъекта, УдалитьПрефиксОрганизации = Ложь, УдалитьПрефиксИнформационнойБазы = Ложь) Экспорт`
— Функция (stable). Клиент + сервер + внешнее соединение.

**Параметры:**
- `НомерОбъекта` (Строка) — номер/код объекта.
- `УдалитьПрефиксОрганизации` (Булево) — `Истина` удаляет префикс организации
  (первые 2 символа). По умолчанию `Ложь`.
- `УдалитьПрефиксИнформационнойБазы` (Булево) — `Истина` удаляет префикс ИБ
  (символы 3–4). По умолчанию `Ложь`.

**Пример:**
```bsl
// Удалить оба стандартных префикса, нули и пользовательский префикс сохранить
Номер = ПрефиксацияОбъектовКлиентСервер.УдалитьПрефиксыИзНомераОбъекта(
    "0ФГЛ-000001234", Истина, Истина);
// "000001234" — незначащие нули префиксов убраны, лидирующие нули номера сохранены
```

**Нюансы / антипаттерны:**
- Незначащие символы префиксов («0») также удаляются. Отличие от `НомерНаПечать`
  — лидирующие нули самого номера **не** снимаются.
- Чтобы отдельно снять лидирующие нули, сохранив префиксы —
  `УдалитьЛидирующиеНулиИзНомераОбъекта`; чтобы убрать только пользовательский
  префикс — `УдалитьПользовательскиеПрефиксыИзНомераОбъекта`.

### 3. Извлечь пользовательский префикс и снять лидирующие нули

**Задача:** получить пользовательский префикс из номера (часть между стандартным
префиксом и цифрами) и отдельно — убрать лидирующие нули, не трогая префиксы.

**Функции:**
`ПрефиксацияОбъектовКлиентСервер.ПользовательскийПрефикс(Знач НомерОбъекта) Экспорт`
— Функция (stable).
`ПрефиксацияОбъектовКлиентСервер.УдалитьЛидирующиеНулиИзНомераОбъекта(Знач НомерОбъекта) Экспорт`
— Функция (stable).
`ПрефиксацияОбъектовКлиентСервер.УдалитьПользовательскиеПрефиксыИзНомераОбъекта(Знач НомерОбъекта) Экспорт`
— Функция (stable).

**Параметры:**
- `НомерОбъекта` (Строка) — номер/код объекта. Шаблон для `ПользовательскийПрефикс`:
  `ООГГ-ААХ…Х` или `ГГ-ААХ…Х`, где `ОО` — префикс организации, `ГГ` — префикс
  ИБ, `АА` — пользовательский префикс, `Х…Х` — номер.

**Пример:**
```bsl
// Номер "ОР00-АВ00777" — пользовательский префикс "АВ"
Префикс = ПрефиксацияОбъектовКлиентСервер.ПользовательскийПрефикс("ОР00-АВ00777");
// "АВ"

// Снять лидирующие нули, сохранив стандартный и пользовательский префиксы
НомерБезНулей = ПрефиксацияОбъектовКлиентСервер.УдалитьЛидирующиеНулиИзНомераОбъекта("00ИБ-000001234");
// "ИБ-1234"

// Убрать только пользовательский префикс, стандартный сохранить
Номер = ПрефиксацияОбъектовКлиентСервер.УдалитьПользовательскиеПрефиксыИзНомераОбъекта("ОР00-АВ00777");
// "ОР00-00777"
```

**Нюансы / антипаттерны:**
- `ПользовательскийПрефикс` возвращает пустую строку, если пользовательского
  префикса в номере нет.
- ❌ Собственный парсинг через `СтрНайти`/`Сред` — пропустит случаи
  нестандартного формата и незначащих нулей; стандартные функции уже учитывают
  положение дефиса (позиция 3 для `ГГ-…` или 5 для `ООГГ-…`).

### 4. Переопределить «базовый номер» для нестандартного формата

**Задача:** для документа с нестандартным форматом номера (буквы-префиксы без
дефиса, напр. `АБВГ0012345`) корректно вычислить базовый номер при смене
префикса ИБ.

**Функция (хук):**
`ПрефиксацияОбъектовПереопределяемый.ПриИзмененииНомера(Объект, Знач Номер, БазовыйНомер, СтандартнаяОбработка) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` модуля `*Переопределяемый`.
**Хук**: БСП вызывает при обработке номера, прикладной код реализует тело в
своём модуле-переопределителе. Не вызывать напрямую.
Аналог для кодов: `ПриИзмененииКода(Объект, Знач Код, БазовыйКод, СтандартнаяОбработка) Экспорт`.

**Параметры:**
- `Объект` (ЛюбаяСсылка) — объект, чей номер обрабатывается.
- `Номер` (Строка) — текущий номер.
- `БазовыйНомер` (Строка) — выходной: базовая часть номера для продолжения
  нумерации.
- `СтандартнаяОбработка` (Булево) — выходной: `Ложь` отключает стандартный
  алгоритм (по умолчанию ищет первую нецифровую позицию справа и оставляет
  цифровой хвост).

**Пример:**
```bsl
// В прикладном модуле ПрефиксацияОбъектовПереопределяемый:
Процедура ПриИзмененииНомера(Объект, Знач Номер, БазовыйНомер, СтандартнаяОбработка) Экспорт
    Если ТипЗнч(Объект) = Тип("ДокументОбъект.РеализацияТоваров") Тогда
        Если Сред(Номер, 5, 1) <> "-" Тогда
            // Нестандартный формат "АБВГ0012345": 3 старых префикса по 1 символу
            СтандартнаяОбработка = Ложь;
            БазовыйНомер = Сред(Номер, 4); // пропускаем 3 префикса
        КонецЕсли;
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Игнорировать хук при нестандартном формате — стандартный алгоритм даст
  неверный базовый номер, и `ПрефиксацияОбъектовСлужебный.ОбработатьДанныеДляПродолженияНумерации`
  перенумерует документы неправильно при смене префикса ИБ.
- Без `СтандартнаяОбработка = Ложь` стандартный алгоритм выполнится
  поверх прикладного результата.

### 5. Зарегистрировать префиксообразующий реквизит с нестандартным именем

**Задача:** сообщить БСП, что в прикладном документе организация хранится не в
реквизите `Организация`, а в `ГоловнаяОрганизация`, чтобы подписка
`УстановитьПрефиксОрганизации` нашла реквизит и сформировала префикс.

**Функция (хук):**
`ПрефиксацияОбъектовПереопределяемый.ПолучитьПрефиксообразующиеРеквизиты(Объекты) Экспорт`
— Процедура (хук), модуль `*Переопределяемый`. БСП вызывает, прикладной код
заполняет таблицу `Объекты`.

**Параметры:**
- `Объекты` (ТаблицаЗначений) — заранее создана БСП; колонки `Объект`
  (ОбъектМетаданных) и `Реквизит` (Строка, имя реквизита организации). Достаточно
  добавить строки. Результат кэшируется в `ПрефиксацияОбъектовПовтИсп`.

**Пример:**
```bsl
// В прикладном модуле ПрефиксацияОбъектовПереопределяемый:
Процедура ПолучитьПрефиксообразующиеРеквизиты(Объекты) Экспорт
    Строка = Объекты.Добавить();
    Строка.Объект   = Метаданные.Документы.РеализацияТоваров;
    Строка.Реквизит = "ГоловнаяОрганизация";
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `ПрефиксацияОбъектовСобытия.УстановитьПрефиксОрганизации(Источник, Истина, "")`
  напрямую из прикладного кода — обработчик подключён как подписка на событие,
  платформа дёргает его автоматически. Прямой вызов нарушает контракт подписки.
- По умолчанию БСП ищет реквизит `Организация`; без регистрации нестандартного
  имени префикс организации не сформируется.

### 6. Изменить префикс ИБ программно и добавить поле «Номер с префиксом» в СКД

**Задача:** программно сменить префикс ИБ с продолжением нумерации (в фоне,
транзакционно) и добавить вычисляемое поле `Номер.СПрефиксом` в схему
компоновки данных отчёта.

**Функции (⚠️ служебный регион):**
`ПрефиксацияОбъектовСлужебный.ИзменитьПрефиксИБ(Параметры, АдресРезультата = "") Экспорт`
— Процедура, регион `СлужебныйПрограммныйИнтерфейс`. Сервер. Записывает новое
значение константы `ПрефиксУзлаРаспределеннойИнформационнойБазы` и (если
`ПродолжитьНумерацию = Истина`) перенумеровывает последние в периоде элементы.
Требует подключённой подсистемы «Обмен данными».
`ПрефиксацияОбъектовСлужебный.ДобавитьРасширениеПоляНомер(СхемаКомпоновкиДанных) Экспорт`
— Процедура (служебный), добавляет в СКД вычисляемое поле `Номер.СПрефиксом`.

**Параметры:**
- `Параметры` (Структура) — `НовыйПрефиксИБ` (Строка), `ПродолжитьНумерацию`
  (Булево).
- `АдресРезультата` (Строка) — адрес временного хранилища для результата.
- `СхемаКомпоновкиДанных` (СхемаКомпоновкиДанных) — схема, где есть поле `Номер`.

**Пример:**
```bsl
// Смена префикса ИБ с продолжением нумерации (фоновое задание)
ПараметрыВыполнения = Новый Структура;
ПараметрыВыполнения.Вставить("НовыйПрефиксИБ", "НБ");
ПараметрыВыполнения.Вставить("ПродолжитьНумерацию", Истина);
АдресРезультата = ПоместитьВоВременноеХранилище(Неопределено, УникальныйИдентификатор);
ПрефиксацияОбъектовСлужебный.ИзменитьПрефиксИБ(ПараметрыВыполнения, АдресРезультата);

// Поле "Номер.СПрефиксом" в СКД отчёта
ПрефиксацияОбъектовСлужебный.ДобавитьРасширениеПоляНомер(СхемаКомпоновкиДанных);
```

**Нюансы / антипаттерны:**
- ⚠️ Оба метода — служебный регион (`СлужебныйПрограммныйИнтерфейс`), обратная
  совместимость не гарантируется. В обычном сценарии префикс ИБ меняется через
  интерфейс подсистемы «Обмен данными» (Администрирование → Синхронизация
  данных); программная установка — только для обработок миграции.
- ❌ `Константы.ПрефиксУзлаРаспределеннойИнформационнойБазы.Установить("НБ")`
  напрямую — не обновляет префикс в подсистеме обмена, ломает нумерацию РИБ.
  Только через `ИзменитьПрефиксИБ` (или интерактивно).
- `ДобавитьРасширениеПоляНомер` создаёт вычисляемое поле `Номер.СПрефиксом` —
  повторно подставлять `НомерНаПечать` в выражении поля не нужно.

## Дополнительно

Событийные методы модуля `ПрефиксацияОбъектовСобытия` (регион
`ПрограммныйИнтерфейс`) — обработчики подписок, **не вызывать напрямую**
(платформа дёргает их через подписки на события):

- `УстановитьПрефиксОрганизации(Источник, СтандартнаяОбработка, Префикс)` —
  назначает префикс организации в `ПриУстановкеНовогоКода`/`ПриУстановкеНомера`.
- `УстановитьПрефиксИнформационнойБазы(Источник, СтандартнаяОбработка, Префикс)` /
  `УстановитьПрефиксИнформационнойБазыИОрганизации(...)` — то же для ИБ и обоих.
- `ПроверитьНомерДокументаПоДате(Источник, Отказ, РежимЗаписи, РежимПроведения)` /
  `ПроверитьНомерДокументаПоДатеИОрганизации(...)` /
  `ПроверитьКодСправочникаПоОрганизации(Источник, Отказ)` /
  `ПроверитьНомерБизнесПроцессаПоДате(...)` — обнуляют номер/код при изменении
  даты/организации, чтобы платформа сформировала новый номер в правильном периоде.
- `ПриОпределенииПрефиксаИнформационнойБазы(ПрефиксИнформационнойБазы)` /
  `ПриОпределенииПрефиксаОрганизации(Знач Организация, ПрефиксОрганизации)` —
  событийные методы получения префиксов; переопределять логику можно в
  прикладном модуле `ПрефиксацияОбъектовСобытия` (скопировав его в конфигурацию).

Хук переопределения `ПрефиксацияОбъектовКлиентСерверПереопределяемый.ПриПолученииНомераНаПечать(НомерОбъекта, СтандартнаяОбработка, УдалитьПрефиксИнформационнойБазы, УдалитьПользовательскийПрефикс)`
— БСП вызывает перед стандартной обработкой `НомерНаПечать`; прикладной код
реализует тело и может выставить `СтандартнаяОбработка = Ложь` и задать
`НомерОбъекта` напрямую. Не вызывать из прикладного кода.
