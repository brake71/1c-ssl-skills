# Валюты, банки и графики работы

Три смежные подсистемы нормативно-справочной информации: **Валюты** (курсы,
пересчёт, сумма прописью), **Банки** (классификатор БИК) и **ГрафикиРаботы** /
**КалендарныеГрафики** (производственные календари, расчёт рабочих дат,
расписание работы). Загрузка справочников (банки, курсы, календари) выполняется
механизмом `РаботаСКлассификаторами` — см. `classifiers.md`; здесь — чтение и расчёты.

## Модули

Валюты — имена модулей **не совпадают** с подсистемой (нет модуля `Валюты`):

- `РаботаСКурсамиВалют` — серверный стабильный API: курс, пересчёт, сумма
  прописью, добавление валют. регион `ПрограммныйИнтерфейс` (stable).
- `РаботаСКурсамиВалютКлиентСервер` — `ПересчитатьПоКурсу` (без серверного вызова,
  по готовым параметрам курса). Клиент + Сервер, stable.
- `РаботаСКурсамиВалютКлиентЛокализация` — `ПоказатьЗагрузкуКурсовВалют`
  (интерактивная загрузка курсов). Клиент, stable.
- `РаботаСКурсамиВалютКлиент` — ⚠️ `ПоказатьЗагрузкуКурсовВалют` здесь в регионе
  `СлужебныйПрограммныйИнтерфейс` (не stable) — используйте вариант из
  `…КлиентЛокализация`.
- `РаботаСКурсамиВалютСлужебный` / `…ВызовСервера` / `…Глобальный` — ⚠️ служебные.
- `РаботаСКурсамиВалютПереопределяемый` — хуки (не вызывать напрямую).

Банки:

- `РаботаСБанками` — серверный стабильный API: `СведенияБИК`,
  `ПояснениеНедействительногоБанка`. `ПолучитьДанныеКлассификатора` — ⚠️ устаревший
  (регион `УстаревшиеПроцедурыИФункции`).
- `РаботаСБанкамиКлиент` — `ВыбратьИзСправочникаБИК` (UI выбора). Клиент, stable.
- `РаботаСБанкамиСлужебный` / `…ВызовСервера` / `…Глобальный` — ⚠️ служебные.
- `РаботаСБанкамиПереопределяемый` — хуки.

Графики и календари:

- `КалендарныеГрафики` — основной серверный API: расчёт дат, ближайшие рабочие,
  расписание, заполнение формы, основной календарь. Принимает как
  `СправочникСсылка.ПроизводственныеКалендари`, так и `СправочникСсылка.Календари`
  (внутри сам переключается на `ГрафикиРаботы`). stable.
- `ГрафикиРаботы` — API по графикам работы (`СправочникСсылка.Календари`):
  `ДатаПоГрафику`, `БлижайшиеДатыВключенныеВГрафик`, `РасписанияРаботыНаПериод`.
  ⚠️ `ГрафикиРаботы.РазностьДатПоКалендарю` — в `СлужебныйПрограммныйИнтерфейс`
  (не stable); для разности дат используйте `КалендарныеГрафики.РазностьДатПоКалендарю`.
- `ГрафикиРаботыКлиент` — утилиты коллекций строк графика (не для расчёта дат).
- `КалендарныеГрафикиСлужебный` / `КалендарныеГрафикиПереопределяемый` — ⚠️ служебный
  / хуки.

⚠️ Типовая ловушка — выдумывать модули `Валюты`, `Банки`, `Организации`. Реальные
имена — `РаботаСКурсамиВалют`, `РаботаСБанками`, `ОрганизацииСервер`.

## Сценарии

### 1. Получить курс валюты и пересчитать сумму

**Задача:** прочитать курс валюты на дату; пересчитать сумму из одной валюты в
другую по курсу на дату; пересчитать на клиенте по заранее полученным курсам.

**Функции:**
`РаботаСКурсамиВалют.ПолучитьКурсВалюты(Валюта, ДатаКурса) Экспорт`
— Функция → `Структура` (`Курс, Кратность, Валюта, ДатаКурса`) / `Неопределено`, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`РаботаСКурсамиВалют.ПересчитатьВВалюту(Сумма, ИсходнаяВалюта, НоваяВалюта, Дата) Экспорт`
— Функция → `Число`, регион `ПрограммныйИнтерфейс` (stable). Сервер. Сам получает оба курса внутри.
`РаботаСКурсамиВалютКлиентСервер.ПересчитатьПоКурсу(Сумма, ПараметрыТекущегоКурса, ПараметрыНовогоКурса) Экспорт`
— Функция → `Число`, регион `ПрограммныйИнтерфейс` (stable). Клиент + Сервер. Пересчёт по готовым структурам курса.

**Параметры:**
- `Валюта` (`СправочникСсылка.Валюты`), `ДатаКурса` (`Дата`).
- `Сумма` (`Число`), `ИсходнаяВалюта` / `НоваяВалюта` (`СправочникСсылка.Валюты`), `Дата` (`Дата`).
- `ПараметрыТекущегоКурса` / `ПараметрыНовогоКурса` (`Структура`) — из `ПолучитьКурсВалюты`: `Валюта, Курс, Кратность`.

**Пример:**
```bsl
// Сервер: получить курс и пересчитать
Курс = РаботаСКурсамиВалют.ПолучитьКурсВалюты(Объект.Валюта, Объект.Дата);
Если Курс = Неопределено Тогда
    ОбщегоНазначения.СообщитьПользователю(
        НСтр("ru = 'Курс валюты на дату не задан.'"), , , "Объект", Отказ);
    Возврат;
КонецЕсли;

СуммаРуб = РаботаСКурсамиВалют.ПересчитатьВВалюту(
    Объект.Сумма, Объект.Валюта,
    Справочники.Валюты.НайтиПоКоду("643"), Объект.Дата);

// Клиент: пересчёт по заранее полученным курсам (без серверного вызова)
СуммаНовая = РаботаСКурсамиВалютКлиентСервер.ПересчитатьПоКурсу(Сумма, КурсИсходный, КурсНовый);
```

**Нюансы / антипаттерны:**
- ❌ Прямой запрос к `РегистрыСведений.КурсыВалют` — обходит обёртку, ломает
  кратность и кэширование. Только `ПолучитьКурсВалюты`.
- `ПолучитьКурсВалюты` возвращает `Неопределено`, если записи курса нет —
  обязательно проверяйте перед использованием.
- `ПересчитатьВВалюту` сам получает оба курса; явный `ПолучитьКурсВалюты` нужен
  только чтобы показать курс пользователю или валидировать.
- `ПересчитатьПоКурсу` удобен в клиентском коде: один серверный вызов за курсами,
  затем пересчёты на клиенте без повторных обращений.

### 2. Сумма прописью и добавление валют по коду

**Задача:** сформировать сумму прописью на нужном языке; при начальном
заполнении добавить валюты в справочник по цифровому коду ОКВ.

**Функции:**
`РаботаСКурсамиВалют.СформироватьСуммуПрописью(СуммаЧислом, Валюта, БезДробнойЧасти = Ложь, Знач КодЯзыка = Неопределено, ДробнаяЧастьПрописью = Ложь) Экспорт`
— Функция → `Строка`, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`РаботаСКурсамиВалют.ДобавитьВалютыПоКоду(Знач КодыВалют) Экспорт`
— Функция → `Массив` из `СправочникСсылка.Валюты`, регион `ПрограммныйИнтерфейс` (stable). Сервер. Для обработчиков начального заполнения.
`РаботаСКурсамиВалют.ПодключитьИсточникДанныхПечатиЧислоПрописью(ИсточникиДанныхПечати) Экспорт`
— Процедура, stable. Подключает источник данных для макета печати числа прописью.

**Параметры:**
- `СуммаЧислом` (`Число`), `Валюта` (`СправочникСсылка.Валюты`).
- `БезДробнойЧасти` (`Булево`) — `Истина` без копеек.
- `КодЯзыка` (`Строка`) — код по ISO 639-1 (+ опц. ISO 3166-1 через `_`): `"ru"`,
  `"ru_RU"`, `"en"`, `"en_US"`. По умолчанию язык конфигурации.
- `ДробнаяЧастьПрописью` (`Булево`).
- `КодыВалют` (`Массив` из `Строка`) — цифровые коды (840, 978, 643 …).

**Пример:**
```bsl
Текст = РаботаСКурсамиВалют.СформироватьСуммуПрописью(1234.56, ВалютаRUB, , "ru");
// "Одна тысяча двести тридцать четыре рубля 56 копеек"

// Начальное заполнение справочника валют
Коды = Новый Массив; Коды.Добавить("840"); Коды.Добавить("978"); Коды.Добавить("643");
Ссылки = РаботаСКурсамиВалют.ДобавитьВалютыПоКоду(Коды);
```

**Нюансы / антипаттерны:**
- ❌ Формировать число прописью вручную для печатной формы — используйте
  `СформироватьСуммуПрописью` или `ПодключитьИсточникДанныхПечатиЧислоПрописью`
  (источник данных для макета).
- При отсутствии классификатора валют `ДобавитьВалютыПоКоду` создаёт элементы с
  наименованием `"Валюта"`, символьный код равен цифровому.

### 3. Найти банк по БИК и пояснить недействительный банк

**Задача:** по БИК (опционально с корр. счётом) получить реквизиты банка;
пояснить пользователю, почему банк помечен недействительным.

**Функции:**
`РаботаСБанками.СведенияБИК(Знач БИК, Знач КоррСчет = Неопределено, ТолькоАктуальные = Истина) Экспорт`
— Функция → `ТаблицаЗначений` (`Ссылка, БИК, КоррСчет, Наименование, Город, Адрес` …), регион `ПрограммныйИнтерфейс` (stable). Сервер.
`РаботаСБанками.ПояснениеНедействительногоБанка(Банк) Экспорт`
— Функция → `ФорматированнаяСтрока`, регион `ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `БИК` (`Строка`), `КоррСчет` (`Строка` / `Неопределено`).
- `ТолькоАктуальные` (`Булево`) — `Истина` (по умолчанию) только действующие банки.
- `Банк` (`СправочникСсылка.КлассификаторБанков`).

**Пример:**
```bsl
Таблица = РаботаСБанками.СведенияБИК("044525225", , Ложь);   // включая недействующие
Если Таблица.Количество() > 0 Тогда
    БанкСсылка = Таблица[0].Ссылка;
    Наименование = Таблица[0].Наименование;
КонецЕсли;

// Пояснение для реквизита формы банка
Пояснение = РаботаСБанками.ПояснениеНедействительногоБанка(БанкСсылка);
// ФорматированнаяСтрока с гиперссылкой на новый БИК, если он нашёлся
```

**Нюансы / антипаттерны:**
- ❌ `РаботаСБанками.ПолучитьДанныеКлассификатора(БИК = "", КоррСчет = "", ЗаписьОБанке = "")` —
  ⚠️ устаревший (регион `УстаревшиеПроцедурыИФункции`): использует выходной
  параметр `ЗаписьОБанке` вместо возврата значения. В новом коде — `СведенияБИК`.
- `ПояснениеНедействительногоБанка` возвращает `ФорматированнаяСтрока` для
  отображения в реквизите формы, а не сообщение пользователю — для сообщений
  используйте `ОбщегоНазначения.СообщитьПользователю` (`base-common.md`).

### 4. Выбор БИК из формы

**Задача:** из поля ввода БИК на форме объекта открыть форму выбора с фильтром и
получить выбранный банк в обработчик оповещения.

**Функция:**
`РаботаСБанкамиКлиент.ВыбратьИзСправочникаБИК(БИК, Форма, ОбработчикОповещения = Неопределено) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Тонкий/Веб-клиент.

**Параметры:**
- `БИК` (`Строка`) — фильтр выбора.
- `Форма` (`ФормаКлиентскогоПриложения`) — форма-источник.
- `ОбработчикОповещения` (`ОписаниеОповещения`) — `Результат` = `СправочникСсылка.КлассификаторБанков`
  (выбранный элемент) или `Неопределено`; при отсутствии — стандартный обработчик выбора.

**Пример:**
```bsl
&НаКлиенте
Процедура БИКНачалоВыбора(Элемент, ДанныеВыбора, СтандартнаяОбработка)
    СтандартнаяОбработка = Ложь;
    РаботаСБанкамиКлиент.ВыбратьИзСправочникаБИК(
        Объект.БИК, ЭтотОбъект,
        Новый ОписаниеОповещения("ОбработкаВыбораБИК", ЭтотОбъект));
КонецПроцедуры

&НаКлиенте
Процедура ОбработкаВыбораБИК(Результат, ДопПараметры) Экспорт
    Если Результат <> Неопределено Тогда
        Объект.БИК = Результат.БИК;                  // если Результат — ссылка, читать реквизиты
        Объект.НаименованиеБанка = Результат.Наименование;
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Забыть `СтандартнаяОбработка = Ложь` — отработает штатный выбор из справочника
  вместо формы БИК.
- Если в выборке единственная запись, выбор выполняется автоматически (без показа формы).

### 5. Расчёт даты по производственному календарю / графику

**Задача:** плановая дата `ДатаОт + N рабочих дней`; цепочка связанных дат; сколько
рабочих дней между двумя датами.

**Функции:**
`КалендарныеГрафики.ДатаПоКалендарю(Знач ГрафикРаботы, Знач ДатаОт, Знач КоличествоДней, ВызыватьИсключение = Истина) Экспорт`
— Функция → `Дата` / `Неопределено`, регион `ПрограммныйИнтерфейс` (stable). Сервер. `ГрафикРаботы` — `СправочникСсылка.ПроизводственныеКалендари` или `СправочникСсылка.Календари`.
`КалендарныеГрафики.ДатыПоКалендарю(Знач ГрафикРаботы, Знач ДатаОт, Знач МассивДней, Знач РассчитыватьСледующуюДатуОтПредыдущей = Ложь, ВызыватьИсключение = Истина) Экспорт`
— Функция → `Массив` из `Дата`, stable.
`КалендарныеГрафики.РазностьДатПоКалендарю(Знач ГрафикРаботы, Знач ДатаНачала, Знач ДатаОкончания, ВызыватьИсключение = Истина) Экспорт`
— Функция → `Число`, stable.
`ГрафикиРаботы.ДатаПоГрафику(Знач ГрафикРаботы, Знач ДатаОт, Знач КоличествоДней, ВызыватьИсключение = Истина) Экспорт`
— Функция → `Дата` / `Неопределено`, stable. Только `СправочникСсылка.Календари`.

**Параметры:**
- `ГрафикРаботы` — календарь/график.
- `ДатаОт` (`Дата`), `КоличествоДней` (`Число`).
- `МассивДней` (`Массив` из `Число`) — сдвиги для `ДатыПоКалендарю`.
- `РассчитыватьСледующуюДатуОтПредыдущей` (`Булево`) — `Истина` = цепной сдвиг (каждая следующая от предыдущей).
- `ВызыватьИсключение` (`Булево`) — `Истина` (по умолчанию) бросает исключение при незаполненном календаре; `Ложь` → `Неопределено`.

**Пример:**
```bsl
Календарь = КалендарныеГрафики.ОсновнойПроизводственныйКалендарь();
Если Календарь = Неопределено Тогда
    ПлановаяДата = ТекущаяДатаСеанса() + 5 * 86400;   // fallback: календарные дни
Иначе
    ПлановаяДата = КалендарныеГрафики.ДатаПоКалендарю(Календарь, ТекущаяДатаСеанса(), 5);
КонецЕсли;

// Цепочка дат: каждая от предыдущей (стадии согласования)
МассивДней = Новый Массив; МассивДней.Добавить(3); МассивДней.Добавить(5); МассивДней.Добавить(7);
МассивДат = КалендарныеГрафики.ДатыПоКалендарю(Календарь, ДатаСтарта, МассивДней, Истина);

// Сколько рабочих дней между датами
ДнейПросрочки = КалендарныеГрафики.РазностьДатПоКалендарю(Календарь, ДатаОтгрузки, ТекущаяДатаСеанса());
```

**Нюансы / антипаттерны:**
- ❌ Считать рабочие дни циклом `Пока ... Если ДеньНедели < 6` — не учитывает
  праздники, переносы выходных, нерабочие периоды по указам. Только
  `ДатаПоКалендарю` / `РазностьДатПоКалендарю`.
- `ДатаПоКалендарю` принимает оба типа (календарь и график); `ДатаПоГрафику`
  (модуль `ГрафикиРаботы`) — только `СправочникСсылка.Календари`. Результат
  одинаковый — выбирайте по тому, что у вас есть.
- `РазностьДатПоКалендарю` всегда возвращает положительное число (знак
  нормализуется внутри).

### 6. Ближайший рабочий день с учётом нерабочих периодов

**Задача:** для набора дат найти ближайшие рабочие дни (вперёд/назад), учитывая
особые нерабочие периоды (указы президента).

**Функции:**
`КалендарныеГрафики.ПараметрыПолученияБлижайшихРабочихДат(ПроизводственныйКалендарь = Неопределено) Экспорт`
— Функция → `Структура`, регион `ПрограммныйИнтерфейс` (stable). Конструктор параметров.
`КалендарныеГрафики.БлижайшиеРабочиеДаты(ПроизводственныйКалендарь, НачальныеДаты, ПараметрыПолучения = Неопределено) Экспорт`
— Функция → `Соответствие` (`Ключ` — исходная `Дата`, `Значение` — ближайшая рабочая `Дата`), stable. Только `СправочникСсылка.ПроизводственныеКалендари`.
`ГрафикиРаботы.БлижайшиеДатыВключенныеВГрафик(ГрафикРаботы, НачальныеДаты, ПараметрыПолучения = Неопределено) Экспорт`
— Функция → `Соответствие`, stable. Только `СправочникСсылка.Календари` — ближайшая дата, **включённая в график**.

**Параметры:**
- `ПроизводственныйКалендарь` (`СправочникСсылка.ПроизводственныеКалендари`).
- `НачальныеДаты` (`Массив` из `Дата`).
- `ПараметрыПолучения` (`Структура` из `ПараметрыПолученияБлижайшихРабочихДат`): `ПолучатьПредшествующие` (`Булево` — назад), `УчитыватьНерабочиеПериоды` (`Булево`), `НерабочиеПериоды`, `ВызыватьИсключение`, `ПолучатьДатыЕслиКалендарьНеЗаполнен`.

**Пример:**
```bsl
Параметры = КалендарныеГрафики.ПараметрыПолученияБлижайшихРабочихДат(Календарь);
Параметры.ПолучатьПредшествующие = Ложь;        // вперёд
Параметры.УчитыватьНерабочиеПериоды = Истина;   // учесть особые нерабочие дни

Даты = Новый Массив; Даты.Добавить(ДатаСдачи);
Соотв = КалендарныеГрафики.БлижайшиеРабочиеДаты(Календарь, Даты, Параметры);
НоваяДатаСдачи = Соотв[ДатаСдачи];   // если исходная рабочая — вернёт её же
```

**Нюансы / антипаттерны:**
- ❌ `БлижайшиеРабочиеДаты(ГрафикРаботы, ...)` с `СправочникСсылка.Календари` —
  метод бросит исключение: принимает **только** `ПроизводственныеКалендари`. Для
  графика работы — `ГрафикиРаботы.БлижайшиеДатыВключенныеВГрафик`.
- ⚠️ `КалендарныеГрафики.ДатыБлижайшихРабочихДней(График, НачальныеДаты, ПолучатьПредшествующие = Ложь, ВызыватьИсключение = Истина, ИгнорироватьНезаполненностьГрафика = Ложь)`
  — устаревший (регион `УстаревшиеПроцедурыИФункции`), 5 параметров вместо 3. В
  новом коде — `БлижайшиеРабочиеДаты` / `БлижайшиеДатыВключенныеВГрафик`.

### 7. Расписание работы на период и заполнение календаря в форме

**Задача:** получить время начала/окончания работы по графикам на период;
заполнить реквизит «ПроизводственныйКалендарь» в форме с учётом региона (КПП).

**Функции:**
`КалендарныеГрафики.ОсновнойПроизводственныйКалендарь() Экспорт`
— Функция → `СправочникСсылка.ПроизводственныеКалендари` / `Неопределено`, stable. Сервер.
`КалендарныеГрафики.ЗаполнитьПроизводственныйКалендарьВФорме(Форма, ПутьРеквизита, КПП = Неопределено) Экспорт`
— Процедура, stable. Сервер. Учитывает функциональную опцию `ИспользоватьНесколькоПроизводственныхКалендарей`.
`ГрафикиРаботы.РасписанияРаботыНаПериод(Графики, ДатаНачала, ДатаОкончания) Экспорт`
— Функция → `ТаблицаЗначений` (`ГрафикРаботы, ДатаГрафика, ВремяНачала, ВремяОкончания`), stable. Сервер. Требует подключённой подсистемы `ГрафикиРаботы`.
`КалендарныеГрафики.РасписанияРаботыНаПериод(Графики, ДатаНачала, ДатаОкончания) Экспорт`
— Функция, stable. Делегирует в `ГрафикиРаботы`; бросает исключение, если подсистема отсутствует.

**Параметры:**
- `Форма` (`ФормаКлиентскогоПриложения`), `ПутьРеквизита` (`Строка`, напр. `"Объект.ПроизводственныйКалендарь"`).
- `КПП` (`Строка`) — для регионального календаря при включённой опции нескольких календарей.
- `Графики` (`Массив` из `СправочникСсылка.Календари`), `ДатаНачала` / `ДатаОкончания` (`Дата`).

**Пример:**
```bsl
// Заполнить реквизит в форме (сервер, ПриСозданииНаСервере)
КалендарныеГрафики.ЗаполнитьПроизводственныйКалендарьВФорме(
    ЭтаФорма, "Объект.ПроизводственныйКалендарь", Объект.КПП);

// Расписание работы склада на неделю
Графики = Новый Массив; Графики.Добавить(ГрафикСклада);
ТаблицаРасписаний = ГрафикиРаботы.РасписанияРаботыНаПериод(
    Графики, ДатаНачала, ДатаОкончания);
// Колонки: ГрафикРаботы, ДатаГрафика, ВремяНачала, ВремяОкончания
```

**Нюансы / антипаттерны:**
- ❌ `ЭтаФорма.Объект.ПроизводственныйКалендарь = КалендарныеГрафики.ОсновнойПроизводственныйКалендарь()`
  без учёта КПП — при включённой опции нескольких календарей для обособленного
  подразделения будет подставлен «основной», а не региональный. Используйте
  `ЗаполнитьПроизводственныйКалендарьВФорме` с КПП.
- ❌ Хранить ссылку на календарь в константе и считать его «единственным» — при
  опции нескольких календарей константы может не быть; `ОсновнойПроизводственныйКалендарь`
  возвращает «первый попавшийся».
- Методы `КалендарныеГрафики` и `ГрафикиРаботы` — серверные (Сервер, Толстый клиент,
  Внешнее соединение); с тонкого клиента — через `&НаСервере`. Отдельного модуля
  `КалендарныеГрафикиКлиент` нет.

## Дополнительно

Прочие стабильные методы (регион `ПрограммныйИнтерфейс`), полные сигнатуры — через
`python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`:

- `РаботаСКурсамиВалют.ОписаниеТипаДенежногоПоля(Знач ДопустимыйЗнакПоля = Неопределено)` —
  описание типа денежного поля (для конструирования реквизитов).
- `РаботаСКурсамиВалютКлиентЛокализация.ПоказатьЗагрузкуКурсовВалют(ПараметрыЗагрузки)` —
  интерактивная загрузка курсов (стабильный вариант; `ПараметрыЗагрузки.ОткрытиеИзСписка`).
- `КалендарныеГрафики.ПериодыНерабочихДней(ПроизводственныйКалендарь, ПериодОтбор)` —
  периоды нерабочих дней календаря.
- `КалендарныеГрафики.СоздатьВТРасписанияРаботыНаПериод(МенеджерВременныхТаблиц, Графики, ДатаНачала, ДатаОкончания)` /
  `ГрафикиРаботы.СоздатьВТРасписанияРаботыНаПериод(...)` — вариант расписания во
  временную таблицу (для запросов).
- `ГрафикиРаботы.ДатыПоГрафику(Знач ГрафикРаботы, Знач ДатаОт, Знач МассивДней, Знач РассчитыватьСледующуюДатуОтПредыдущей = Ложь, ВызыватьИсключение = Истина)` —
  цепочка дат по графику (аналог `ДатыПоКалендарю`).

⚠️ Служебные методы (не использовать в новом коде):
- `РаботаСКурсамиВалютСлужебный.ЗагрузитьКурсы()` — ⚠️ служебный
  (`СлужебныйПрограммныйИнтерфейс`). Ручная загрузка курсов; обратная совместимость
  не гарантируется. Запуск через интерактивную форму
  `РаботаСКурсамиВалютКлиентЛокализация.ПоказатьЗагрузкуКурсовВалют`.
- `РаботаСБанками.ПолучитьДанныеКлассификатора(...)` — ⚠️ устаревший, заменён на `СведенияБИК`.
- `КалендарныеГрафики.ДатыБлижайшихРабочихДней(...)` — ⚠️ устаревший, заменён на
  `БлижайшиеРабочиеДаты` / `БлижайшиеДатыВключенныеВГрафик`.
- `ГрафикиРаботы.РазностьДатПоКалендарю(...)` — ⚠️ служебный регион; используйте
  `КалендарныеГрафики.РазностьДатПоКалендарю`.

Хук переопределения `КалендарныеГрафикиПереопределяемый.ПриОбновленииПроизводственныхКалендарей`
— реакция прикладной конфигурации на обновление производственных календарей
(реализуется в одноимённом модуле, не вызывается напрямую).
