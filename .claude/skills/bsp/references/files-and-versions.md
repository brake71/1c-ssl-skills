# Файлы и версионирование объектов БСП

Подсистемы **РаботаСФайлами** (присоединённые файлы — сканы, изображения, PDF,
хранение в ИБ и томах), **ВерсионированиеОбъектов** (история изменений
справочников и документов, отчёты по версиям) и **ВыгрузкаОбъектовВФайлы**
(сохранение объектов/печатных форм в файлы XML/JSON/Excel). Все три прорастают
в формы объектов через типовые команды и гиперссылки, поэтому их общие
модули встречаются в каждом документе и справочнике типовой конфигурации.

## Модули

Суффиксная система имён БSP (один корень + контекст исполнения):

- `РаботаСФайлами` — серверный API: создание, обновление, копирование,
  чтение присоединённых файлов.
- `РаботаСФайламиКлиент` — клиентский API: диалоги, открытие форм, сканирование.
- `РаботаСФайламиКлиентСервер` — общий код (структуры параметров, имена команд).
- `РаботаСФайламиПереопределяемый` / `РаботаСФайламиКлиентПереопределяемый` —
  хуки переопределения (БСП вызывает, прикладной код реализует; не вызывается
  напрямую).
- Служебные: `РаботаСФайламиСлужебный*` (⚠️ обратная совместимость не
  гарантируется) — не использовать как основной способ.
- `ВерсионированиеОбъектов` — серверный API: настройки, чтение версий,
  формирование отчётов.
- `ВерсионированиеОбъектовКлиент` — клиентский API: отчёты по версии/
  сравнению, открытие истории.
- `ВерсионированиеОбъектовПереопределяемый` — хуки (не вызов).
- `ВерсионированиеОбъектовСобытия` — обработчики `ПриЗаписи`/`ПередУдалением`,
  вызываются **подписками БСП автоматически**; напрямую из прикладного кода
  не вызывать.
- `ВыгрузкаОбъектовВФайлы` — серверный API сохранения объектов в файлы;
  подключается к формам через подсистему `ПодключаемыеКоманды`.

⚠️ Типовые ошибки выбора модуля:
- `ВерсионированиеОбъектовСлужебный` (без суффикса `ВызовСервера`) **не
  существует**. Единственный служебный модуль подсистемы —
  `ВерсионированиеОбъектовСлужебныйВызовСервера`. Все нужные операции — в
  основном модуле `ВерсионированиеОбъектов`, разнесённые по областям.
- `РаботаСФайламиСлужебный` — существует, но ⚠️ служебный; стабильные
  аналоги — в `РаботаСФайлами`.

Присоединённый файл — это **элемент справочника**, подчинённый «владельцу»
(определяемый тип `ВладелецПрисоединенныхФайлов`). Имя справочника хранения
зависит от владельца (напр. для `Справочник.ДоговорыКонтрагентов` —
`Справочник.ДоговорыКонтрагентовПрисоединенныеФайлы`).

## Сценарии

### 1. Прикрепить файл к объекту из серверного кода

**Задача:** создать присоединённый файл (PDF-счёт, скан) по двоичным данным во
временном хранилище и привязать к документу/справочнику.

**Функции:**
`РаботаСФайлами.ПараметрыДобавленияФайла(ДополнительныеРеквизиты = Неопределено) Экспорт`
— возвращает структуру параметров (заполнить `ВладелецФайлов`, `ИмяБезРасширения`,
`Расширение`, `ДатаМодификацииУниверсальная`).
`РаботаСФайлами.ДобавитьФайл(ПараметрыФайла, Знач АдресФайлаВоВременномХранилище, Знач АдресВременногоХранилищаТекста = "", Знач Описание = "", Знач НоваяСсылкаНаФайл = Неопределено) Экспорт`
— создаёт элемент справочника хранения, возвращает ссылку.
— обе Функции, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ПараметрыФайла` (Структура) — параметры файла; обязательное поле
  `ВладелецФайлов` (ссылка владельца). Заполняется через
  `ПараметрыДобавленияФайла()`.
- `АдресФайлаВоВременномХранилище` (Строка) — адрес двоичных данных во временном
  хранилище (из `ПоместитьВоВременноеХранилище`).
- `АдресВременногоХранилищаТекста` (Строка) — адрес извлечённого текста; `""` —
  не извлекать.
- `Описание` (Строка) — комментарий/описание файла.
- `НоваяСсылкаНаФайл` (ЛюбаяСсылка) — предзаполненная ссылка на новый файл
  (`Неопределено` — сгенерировать).

**Пример:**
```bsl
// Двоичные данные уже помещены во временное хранилище (АдресДвоичныхДанныхВХ)
ПараметрыФайла = РаботаСФайлами.ПараметрыДобавленияФайла();
ПараметрыФайла.ВладелецФайлов      = ЗаказПоставщику;
ПараметрыФайла.ИмяБезРасширения    = "Счёт_" + ЗаказПоставщику.Номер;
ПараметрыФайла.Расширение          = "pdf";
ПараметрыФайла.ДатаМодификацииУниверсальная = ТекущаяДатаСеанса();

ПрисоединенныйФайл = РаботаСФайлами.ДобавитьФайл(
    ПараметрыФайла,
    АдресДвоичныхДанныхВХ,
    "",                       // без извлечения текста
    "Загружен из EDI");       // описание
```

**Нюансы / антипаттерны:**
- ❌ `РаботаСФайлами.ПолучитьДанныеФайла(Файл)` — метод **не существует**
  (ошибка компиляции). Реальное имя — `ДанныеФайла` (без префикса `Получить`).
- ❌ `РаботаСФайлами.УдалитьФайл(Файл)` — метод **не существует**. Удаление
  намеренно не вынесено в стабильный API (сайд-эффекты: очистка версий, ЭП,
  регистров). Используйте платформенную пометку удаления
  `Файл.ПолучитьОбъект().УстановитьПометкуУдаления(Истина)`.
- ❌ Передавать `ВладелецФайлов` первым позиционным аргументом `ДобавитьФайл` —
  исключение «Не задано значение параметра ПараметрыФайла.ВладелецФайлов».
  Владелец берётся из структуры `ПараметрыФайла`.
- Для файлов с диска есть упрощённая обёртка
  `РаботаСФайлами.ДобавитьФайлСДиска(ВладелецФайлов, ПутьКФайлуНаДиске)` —
  без ручного формирования параметров.

### 2. Прочитать и обновить двоичные данные файла

**Задача:** получить реквизиты и двоичные данные присоединённого файла,
перезаписать его содержимое без создания новой версии.

**Функции:**
`РаботаСФайлами.ДанныеФайла(Знач ПрисоединенныйФайл, Знач ДополнительныеПараметры = Неопределено, Знач УдалитьПолучатьСсылкуНаДвоичныеДанные = Истина, Знач УдалитьДляРедактирования = Ложь) Экспорт`
— реквизиты + опционально адрес двоичных данных во временном хранилище.
`РаботаСФайлами.ДвоичныеДанныеФайла(Знач ПрисоединенныйФайл, Знач ВызыватьИсключение = Истина) Экспорт`
— двоичные данные файла напрямую.
`РаботаСФайлами.ОбновитьФайл(Знач ПрисоединенныйФайл, Знач ИнформацияОФайле) Экспорт`
— перезапись двоичных данных/реквизитов **без** новой версии.
— все Функция/Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ПрисоединенныйФайл` (СправочникСсылка.ПрисоединенныеФайлы) — ссылка на файл.
- `ДополнительныеПараметры` (Структура) — опц. параметры; `Неопределено` — по
  умолчанию.
- `УдалитьПолучатьСсылкуНаДвоичныеДанные` (Булево) — `Истина` → поместить
  двоичные данные во временное хранилище и вернуть адрес в
  `ДанныеФайла.СсылкаНаДвоичныеДанныеФайла`.
- `УдалитьДляРедактирования` (Булево) — `Истина` → разместить в рабочем каталоге
  для редактирования.
- `ИнформацияОФайле` (Структура) — для `ОбновитьФайл`: новые свойства файла.
  Ключи: `АдресФайлаВоВременномХранилище` (Строка — новые двоичные данные),
  `АдресВременногоХранилищаТекста` (Строка — извлечённый текст),
  `ИмяБезРасширения` / `Расширение` (Строка — необязательные),
  `ДатаМодификацииУниверсальная` (Дата — необязательное, иначе текущая дата сеанса).

**Пример:**
```bsl
// Прочитать реквизиты и адрес двоичных данных
Данные = РаботаСФайлами.ДанныеФайла(Файл, Неопределено);
АдресДвоичных = Данные.СсылкаНаДвоичныеДанныеФайла;  // во временном хранилище

// Получить двоичные данные напрямую (без временного хранилища)
ДвоичныеДанные = РаботаСФайлами.ДвоичныеДанныеФайла(Файл);

// Обновить содержимое без новой версии (только для файлов БЕЗ хранения версий)
АдресВХ = ПоместитьВоВременноеХранилище(НовыеДвоичныеДанные);
ИнформацияОФайле = Новый Структура;
ИнформацияОФайле.Вставить("АдресФайлаВоВременномХранилище", АдресВХ);
ИнформацияОФайле.Вставить("ДатаМодификацииУниверсальная", ТекущаяДатаСеанса());
РаботаСФайлами.ОбновитьФайл(Файл, ИнформацияОФайле);
```

**Нюансы / антипаттерны:**
- ❌ `РаботаСФайлами.ОбновитьФайл` для файла, у которого **включено хранение
  версий** — тихая потеря старой версии. Сначала проверьте
  `ВерсионированиеОбъектов.ВключеноВерсионированиеОбъекта("Справочник." +
  Файл.Метаданные().Имя)`; для версионируемых файлов создавайте новую версию
  через механизм «Занять/Освободить файл для редактирования» в форме файла.
- `ДанныеФайла` сама переходит в привилегированный режим для чтения
  двоичных данных; вызывающему коду отдельно поднимать его не нужно.
- Для пакетного чтения двоичных данных массива файлов есть
  `РаботаСФайлами.ДвоичныеДанныеФайлов(Знач ПрисоединенныеФайлы, Знач ВызыватьИсключение = Истина)`.

### 3. Открыть форму файла и диалог добавления из клиентского кода

**Задача:** из команды формы открыть карточку присоединённого файла или
диалог множественного добавления файлов с фильтром по расширению.

**Функции:**
`РаботаСФайламиКлиент.ОткрытьФормуФайла(Знач ПрисоединенныйФайл, СтандартнаяОбработка = Ложь, ДополнительныеПараметры = Неопределено, ОписаниеОповещенияОЗакрытии = Неопределено) Экспорт`
— открыть карточку файла (используется и как обработчик команды «Открыть»).
`РаботаСФайламиКлиент.ДобавитьФайлы(Знач ВладелецФайла, Знач ИдентификаторФормы, Знач Фильтр = "", ГруппаФайлов = Неопределено, ОбработчикРезультата = Неопределено) Экспорт`
— открыть диалог множественного добавления с фильтром по расширению.
`РаботаСФайламиКлиент.ОткрытьФормуВыбораФайлов(Знач ВладелецФайлов, Знач ЭлементФормы, СтандартнаяОбработка = Ложь, ОписаниеОповещенияОВыборе = Неопределено) Экспорт`
— открыть форму выбора файлов владельца.
— все Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `ПрисоединенныйФайл` (СправочникСсылка.ПрисоединенныеФайлы) — файл для открытия.
- `СтандартнаяОбработка` (Булево) — **выходной** параметр; метод всегда ставит
  `Ложь`, чтобы стандартная платформенная обработка не срабатывала.
- `ВладелецФайла` (ЛюбаяСсылка) — объект-владелец файлов.
- `ИдентификаторФормы` (УникальныйИдентификатор) — UID формы-владельца (для
  возврата результата).
- `Фильтр` (Строка) — фильтр расширений через запятую, напр. `"pdf,docx"`.

**Пример:**
```bsl
&НаКлиенте
Процедура ОткрытьСканДоговора(Команда)
    Если Объект.Ссылка.Пустая() Тогда
        Возврат;  // файлы нельзя открыть у незаписанного объекта
    КонецЕсли;

    МассивФайлов = Новый Массив;
    РаботаСФайлами.ЗаполнитьПрисоединенныеФайлыКОбъекту(Объект.Ссылка, МассивФайлов);

    Если МассивФайлов.Количество() = 1 Тогда
        РаботаСФайламиКлиент.ОткрытьФормуФайла(МассивФайлов[0], Ложь);
    Иначе
        РаботаСФайламиКлиент.ОткрытьФормуВыбораФайлов(Объект.Ссылка, Элементы.СканДоговора, Ложь);
    КонецЕсли;
КонецПроцедуры

// Диалог пакетного добавления с фильтром
&НаКлиенте
Процедура ДобавитьДокументы(Команда)
    РаботаСФайламиКлиент.ДобавитьФайлы(Объект.Ссылка, УникальныйИдентификатор, "pdf,docx");
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Открывать файлы незаписанного объекта (пустая ссылка владельца) —
  справочник хранения подчинён владельцу, файл «повиснет».
- `ОткрытьФормуФайла` совмещает две роли: самостоятельный метод и обработчик
  команды формы. Как обработчик — передавайте `СтандартнаяОбработка = Ложь`
  (метод сам установит его в `Ложь`).
- Для формы списка файлов владельца —
  `РаботаСФайламиКлиент.ОткрытьФормуСпискаФайлов(ВладелецФайлов, ПараметрыФормы, ВладелецФормы, ОповещениеОЗакрытии)`.

### 4. Скопировать файлы и проверить возможность прикрепления

**Задача:** при «Создать на основании» / «Скопировать» перенести все
присоединённые файлы с источника на получатель; проверить, что к типу
объекта вообще можно прикреплять файлы.

**Функции:**
`РаботаСФайлами.КОбъектуМожноПрисоединятьФайлы(ВладелецФайлов, ИмяСправочника = "") Экспорт`
— `Булево`: есть ли у владельца справочник хранения.
`РаботаСФайлами.СкопироватьПрисоединенныеФайлы(Знач Источник, Знач Получатель) Экспорт`
— скопировать все файлы источника на получатель (одного типа).
`РаботаСФайлами.ЗаполнитьПрисоединенныеФайлыКОбъекту(Знач ВладелецФайла, Знач Файлы) Экспорт`
— заполнить переданный массив ссылками на все файлы владельца.
— все Функция/Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ВладелецФайлов` (ЛюбаяСсылка) — объект-владелец (или его метаданные).
- `ИмяСправочника` (Строка) — имя справочника хранения; `""` — любой.
- `Источник` / `Получатель` (ЛюбаяСсылка) — объекты одного типа.
- `Файлы` (Массив) — заполняется ссылками на файлы; очистите перед вызовом.

**Пример:**
```bsl
// Проверка перед прикреплением
Если Не РаботаСФайлами.КОбъектуМожноПрисоединятьФайлы(Документ) Тогда
    Возврат;  // у типа объекта нет справочника хранения файлов
КонецЕсли;

// Собрать все файлы объекта в массив
МассивФайлов = Новый Массив;
РаботаСФайлами.ЗаполнитьПрисоединенныеФайлыКОбъекту(ДокументИсточник, МассивФайлов);

// Перенести файлы при копировании объекта
РаботаСФайлами.СкопироватьПрисоединенныеФайлы(ДокументИсточник, ДокументПолучатель);
```

**Нюансы / антипаттерны:**
- `СкопироватьПрисоединенныеФайлы` требует, чтобы `Источник` и `Получатель`
  были одного типа (один справочник хранения). При несовпадении типов
  поведение не определено.
- `КОбъектуМожноПрисоединятьФайлы` возвращает `Ложь`, если у владельца нет ни
  одного справочника хранения — использовать как guard перед динамическим
  прикреплением.

### 5. Включить и настроить версионирование объекта

**Задача:** при начальном заполнении ИБ включить запись истории изменений
для своего справочника/документа и задать вариант и срок хранения версий.

**Функции:**
`ВерсионированиеОбъектов.ВключеноВерсионированиеОбъекта(ИмяОбъекта) Экспорт`
— `Булево`: включено ли версионирование для объекта метаданных.
`ВерсионированиеОбъектов.ВключитьВерсионированиеОбъекта(ИмяОбъекта, Знач ВариантВерсионирования = Неопределено) Экспорт`
— включить версионирование для объекта.
`ВерсионированиеОбъектов.ВключитьВерсионированиеОбъектов(Объекты) Экспорт`
— пакетное включение для списка объектов. `Объекты` — **`Соответствие`**
(ключ — полный путь метаданных `"Справочник.Номенклатура"`, значение —
`Перечисления.ВариантыВерсионированияОбъектов.*`), не массив.
`ВерсионированиеОбъектов.ЗаписатьНастройкуВерсионированияПоОбъекту(Знач ТипОбъекта, Знач ВариантВерсионирования, Знач СрокХраненияВерсий = Неопределено) Экспорт`
— тонкая настройка: вариант (`Перечисления.ВариантыВерсионированияОбъектов.*`)
  и срок хранения (`Перечисления.СрокиХраненияВерсий.*`).
`ВерсионированиеОбъектов.ЗначениеФлажкаХранитьИсторию() Экспорт`
— текущее значение функциональной опции «Использовать версионирование объектов».
— все Функции/Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ИмяОбъекта` (Строка) — полное имя метаданных, напр.
  `"Справочник.ДоговорыКонтрагентов"`.
- `ВариантВерсионирования` (ПеречислениеСсылка.ВариантыВерсионированияОбъектов) —
  `Неопределено` = значение по умолчанию (`ПриЗаписи`).
- `ТипОбъекта` (Тип) — тип ссылки объекта, напр.
  `Тип("СправочникСсылка.ДоговорыКонтрагентов")`.
- `СрокХраненияВерсий` (ПеречислениеСсылка.СрокиХраненияВерсий) — `Неопределено`
  = без ограничения срока.

**Пример:**
```bsl
ИмяОбъекта = "Справочник.ДоговорыКонтрагентов";

Если Не ВерсионированиеОбъектов.ВключеноВерсионированиеОбъекта(ИмяОбъекта) Тогда
    ВерсионированиеОбъектов.ВключитьВерсионированиеОбъекта(ИмяОбъекта);
КонецЕсли;

// Тонкая настройка: вариант «При записи» + срок «Последний год»
ТипОбъекта = Тип("СправочникСсылка.ДоговорыКонтрагентов");
ВерсионированиеОбъектов.ЗаписатьНастройкуВерсионированияПоОбъекту(
    ТипОбъекта,
    Перечисления.ВариантыВерсионированияОбъектов.ПриЗаписи,
    Перечисления.СрокиХраненияВерсий.ПоследнийГод);
```

**Нюансы / антипаттерны:**
- ❌ Дублировать логику записи версии в `ПриЗаписи` модуля объекта — БСП
  подключает свои подписки (`ВерсионированиеОбъектовСобытия.ЗаписатьВерсиюОбъекта`)
  автоматически ко всем версионируемым объектам. Включить/выключить — через
  `ВключитьВерсионированиеОбъекта`; запись в регистр `ВерсииОбъектов`
  произойдёт сама.
- Функциональная опция `ИспользоватьВерсионированиеОбъектов` глобально
  отключает запись версий; проверяется внутри методов записи. Значение —
  через `ЗначениеФлажкаХранитьИсторию()`.
- Для пакетного включения сразу нескольких объектов —
  `ВключитьВерсионированиеОбъектов(Объекты)`, где `Объекты` — **Соответствие**
  (ключ — полный путь метаданных, значение — `ВариантыВерсионированияОбъектов`),
  не массив.

### 6. Прочитать версию объекта и сформировать отчёт

**Задача:** по ссылке и номеру версии получить реквизиты объекта на момент
версии и вывести печатную форму отчёта по версии.

**Функции:**
`ВерсионированиеОбъектов.СведенияОВерсииОбъекта(Знач Ссылка, Знач НомерВерсии) Экспорт`
— структура: `ВерсияОбъекта` (`ДвоичныеДанные` — сериализованная версия объекта),
  `АвторВерсии` (`СправочникСсылка.Пользователи` / `.ВнешниеПользователи`),
  `ДатаВерсии` (`Дата`).
`ВерсионированиеОбъектов.ОтчетПоВерсииОбъекта(СсылкаНаОбъект, Знач ВерсияОбъекта = Неопределено, ПользовательскийНомерВерсии = Неопределено) Экспорт`
— `ТабличныйДокумент` с реквизитами указанной версии.
`ВерсионированиеОбъектов.НомерПоследнейВерсии(Ссылка, ИзмененныеПользователем = Ложь) Экспорт`
— номер последней версии объекта.
— ⚠️ **все три метода в `#Область СлужебныеПроцедурыИФункции`** — обратная
совместимость **не гарантируется**. Стабильного публичного аналога чтения
версий нет намеренно. Сервер.

**Параметры:**
- `Ссылка` (ЛюбаяСсылка) — объект.
- `НомерВерсии` (Число) — номер версии (с `1`).
- `ВерсияОбъекта` (Число) — `Неопределено` → последняя версия.
- `ПользовательскийНомерВерсии` (Число) — отображаемый номер (при
  переиндексации может отличаться от внутреннего).

**Пример:**
```bsl
// Прочитать данные версии
Сведения = ВерсионированиеОбъектов.СведенияОВерсииОбъекта(СсылкаНаОбъект, 3);
// Сведения.ВерсияОбъекта — ДвоичныеДанные сериализованной версии объекта
// Сведения.АвторВерсии, Сведения.ДатаВерсии — автор и дата записи версии

// Печатная форма отчёта по версии
ТабДок = ВерсионированиеОбъектов.ОтчетПоВерсииОбъекта(СсылкаНаОбъект, 3);
ТабДок.Показать("Версия №" + 3);

// Узнать последнюю версию
Номер = ВерсионированиеОбъектов.НомерПоследнейВерсии(СсылкаНаОбъект);
```

**Нюансы / антипаттерны:**
- ⚠️ `СведенияОВерсииОбъекта` сама устанавливает привилегированный режим и
  проверяет право `ЧтениеДанныхВерсийОбъектов` (через
  `ЕстьПравоЧтенияДанныхВерсийОбъектов`). При отсутствии роли выбрасывает
  исключение «Не удалось получить предыдущую версию объекта».
- ❌ `ВерсионированиеОбъектовСлужебный.ЗаписатьВерсиюОбъекта(...)` — модуль
  **не существует** (нет суффикса `ВызовСервера`). Принудительная запись
  версии — `ВерсионированиеОбъектов.ЗаписатьВерсиюОбъекта(Знач Источник, РежимЗаписи = Неопределено)`
  (⚠️ `СлужебныйПрограммныйИнтерфейс`), но в прикладном коде её вызывать не
  нужно — подписки БСП делают это сами.
- ❌ `ВерсионированиеОбъектовКлиент.ОткрытьИсторию(Ссылка)` — метод **не
  существует**. Реальное имя — `ПоказатьИсториюИзменений(Ссылка, ФормаВладелец)`
  (⚠️ `СлужебныйПрограммныйИнтерфейс`). Для отчёта по версии из клиента —
  `ВерсионированиеОбъектовКлиент.ОткрытьОтчетПоВерсииОбъекта(Ссылка, АдресСериализованногоОбъекта)`.

### 7. Подключить файлы и версионирование к форме объекта

**Задача:** в нетиповой форме (расширение/своя форма) вывести гиперссылку
присоединённых файлов и кнопку «История изменений».

**Функции:**
`РаботаСФайлами.ПриСозданииНаСервере(Форма, ДобавляемыеЭлементы = Неопределено, НастройкиРаботыСФайламиВФорме = Неопределено) Экспорт`
— добавляет гиперссылку, команды и поле предпросмотра файлов. Сервер.
`РаботаСФайламиКлиент.ПриОткрытии(Форма, Отказ) Экспорт`
— клиентская инициализация при открытии формы. Клиент.
`РаботаСФайламиКлиент.ОбработкаОповещения(Форма, ИмяСобытия) Экспорт`
— клиентская обработка оповещений (обновление списка файлов). Клиент.
`ВерсионированиеОбъектов.ПриСозданииНаСервере(Форма) Экспорт`
— устанавливает параметр функциональной опции `ТипВерсионируемогоОбъекта`;
  без вызова кнопка «История изменений» не появится. Сервер.
— все в `#Область ПрограммныйИнтерфейс` (stable).

**Параметры:**
- `Форма` (УправляемаяФорма) — форма объекта.
- `ДобавляемыеЭлементы` (Массив) — массив добавляемых элементов формы (из
  обработчика `ПриСозданииНаСервере`).
- `НастройкиРаботыСФайламиВФорме` (Структура) — опц. преднастройки.

**Пример:**
```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    // Присоединённые файлы — гиперссылка, команды, поле предпросмотра
    РаботаСФайлами.ПриСозданииНаСервере(ЭтаФорма, ДобавляемыеЭлементы);

    // Версионирование — кнопка "История изменений"
    ВерсионированиеОбъектов.ПриСозданииНаСервере(ЭтаФорма);
КонецПроцедуры

&НаКлиенте
Процедура ПриОткрытии(Отказ)
    РаботаСФайламиКлиент.ПриОткрытии(ЭтаФорма, Отказ);
КонецПроцедуры

&НаКлиенте
Процедура ОбработкаОповещения(ИмяСобытия, Параметр, Источник)
    РаботаСФайламиКлиент.ОбработкаОповещения(ЭтаФорма, ИмяСобытия);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- Эти вызовы **опциональны**, если БСП подключена через стандартный механизм
  «присоединённых файлов» — она подцепится автоматически. Нужны только для
  нетиповых форм (расширение, своя форма).
- Без `ВерсионированиеОбъектов.ПриСозданииНаСервере` кнопка «История
  изменений» не появится, даже если версионирование глобально включено и в
  `НастройкиВерсионированияОбъектов` есть записи для объекта.

### 8. Сохранить объекты в файлы (XML/JSON/Excel/печать)

**Задача:** программно выгрузить массив объектов в файлы заданного формата
(напр., пакет XML для обмена, Excel-отчёт, PDF-печатная форма) и получить
двоичные данные файлов.

**Функция:**
`ВыгрузкаОбъектовВФайлы.СохранитьПоФорматуВФайл(КомандыВыгрузки, СписокОбъектов, НастройкиСохранения) Экспорт`
— Функция → `ТаблицаЗначений` (колонки `ИмяФайла`, `ДвоичныеДанные`),
  регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `КомандыВыгрузки` (Структура / Массив) — команда или команды выгрузки;
  структура совместима с `УправлениеПечатью.КомандыПечатиФормы` (формат
  выгрузки берётся из команды).
- `СписокОбъектов` (Массив из СправочникСсылка / ДокументСсылка) — ссылки на
  сохраняемые объекты.
- `НастройкиСохранения` (Структура) — настройки сохранения; структура
  совместима с `УправлениеПечатью.НастройкиСохранения` (каталог, способ
  именования файлов и т. п.).

**Пример:**
```bsl
// Команда выгрузки формируется по аналогии с командами печати
КомандыВыгрузки = Новый Массив;
КомандаВыгрузки = Новый Структура("Формат, Идентификатор",
    "XML", "ВыгрузкаВXML");
КомандыВыгрузки.Добавить(КомандаВыгрузки);

СписокОбъектов = Новый Массив;
СписокОбъектов.Добавить(СсылкаНаДокумент);

НастройкиСохранения = Новый Структура;  // см. УправлениеПечатью.НастройкиСохранения

Результат = ВыгрузкаОбъектовВФайлы.СохранитьПоФорматуВФайл(
    КомандыВыгрузки, СписокОбъектов, НастройкиСохранения);
// Результат — ТаблицаЗначений: ИмяФайла, ДвоичныеДанные
Для Каждого СтрокаФайла Из Результат Цикл
    СтрокаФайла.ДвоичныеДанные.Записать(КаталогВыгрузки + СтрокаФайла.ИмяФайла);
КонецЦикла;
```

**Нюансы / антипаттерны:**
- Подсистема `ВыгрузкаОбъектовВФайлы` — надстройка над `УправлениеПечатью`:
  форматы выгрузки (`XML`, `JSON`, табличные форматы) переиспользуют инфраструктуру
  команд печати. Структуры `КомандыВыгрузки` / `НастройкиСохранения` —
  совместимы с аналогами модуля печати.
- В интерактивном режиме выгрузка подключается к формам через подсистему
  `ПодключаемыеКоманды` (методы `ПриОпределенииКомандПодключенныхКОбъекту` и
  `ПриОпределенииВидовПодключаемыхКоманд` модуля — ⚠️
  `СлужебныйПрограммныйИнтерфейс`, вызываются БСП автоматически). Для
  программной выгрузки используйте только `СохранитьПоФорматуВФайл`.
- Остальные методы модуля (`ВыполнитьВыгрузкуВXML`, `ВыполнитьВыгрузкуВJSON`,
  `СформироватьСтруктуруДляВыгрузки`) — ⚠️ `СлужебныйПрограммныйИнтерфейс`;
  напрямую их вызывать не нужно — `СохранитьПоФорматуВФайл` вызывает их сам по
  формату команды.

## Редкие методы

Прочие стабильные методы (регион `ПрограммныйИнтерфейс`), полные сигнатуры —
через `python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`:

- `РаботаСФайлами.НоваяСсылкаНаФайл(ВладелецФайлов, ИмяСправочника = Неопределено)`
  — предзаполненная ссылка на новый файл (для `ДобавитьФайл`).
- `РаботаСФайлами.ИмяФормыОбъектаФайловПоВладельцу(Знач ВладелецФайлов)` — имя
  формы списка файлов по владельцу.
- `РаботаСФайлами.ЕстьТомаХраненияФайлов()` — `Булево`: настроены ли тома
  хранения файлов (внешние каталоги).
- `РаботаСФайлами.НастройкиРаботыСФайлами()` / `СохранитьНастройкиРаботыСФайлами(Настройки)`
  — чтение/запись общих настроек работы с файлами.
- `РаботаСФайлами.МаксимальныйРазмерФайла()` / `МаксимальныйРазмерФайлаОбщий()`
  — ограничения размера файла (одного и суммарного).
- `РаботаСФайлами.ПеренестиФайлыМеждуСправочникамиХранения(Знач ВладелецФайлов, Знач Источник = Неопределено, Знач Приемник = Неопределено)`
  — миграция файлов при смене справочника хранения.
- `ВерсионированиеОбъектов.ВключеноВерсионированиеОбъектов(СписокОбъектов)` —
  пакетная проверка включения для списка объектов.
- `ВерсионированиеОбъектов.ЕстьПравоЧтенияИнформацииОВерсияхОбъектов()` /
  `ЕстьПравоЧтенияДанныхВерсийОбъектов()` — ⚠️ `СлужебныйПрограммныйИнтерфейс`:
  проверки прав на чтение информации о версиях и данных версий.

Хуки переопределения (модули `*Переопределяемый` — БСП вызывает, прикладной
код реализует; не вызывается напрямую):
- `РаботаСФайламиПереопределяемый` — настройка поведения файлов (какие файлы
  «ненужные», фильтры прикрепления).
- `ВерсионированиеОбъектовПереопределяемый` — какие объекты версионируются,
  прикладные правила версионирования.

Для поиска сигнатуры/региона любого из этих методов —
`python scripts/bsp_api.py method <Имя> --src src/cf`.
