# Длительные операции и регламентные задания

Два связанных инфраструктурных механизма БСП: **ДлительныеОперации***
(функциональный блок внутри `БазоваяФункциональность` — запуск серверного
кода в фоновом задании с UI-ожиданием, прогрессом и отменой) и подсистема
**РегламентныеЗадания** (программное управление расписанием и состоянием
регламентных заданий). Нужны, когда серверный вызов длится > 1 сек и форма не
должна «виснуть», либо когда требуется запланировать/найти/изменить
регламентное задание из кода.

## Модули

Суффиксная система имён (один корень + контекст исполнения):

- `ДлительныеОперации` — сервер, внешнее соединение: запуск фоновых заданий,
  прогресс, отмена, чтение состояния. Основной стабильный API.
- `ДлительныеОперацииКлиент` — тонкий/толстый клиент: ожидание завершения,
  форма прогресса, параметры ожидания.
- `ДлительныеОперацииВызовСервера` — ⚠️ содержит только `ФоновоеЗаданиеЗавершено`
  в `СлужебныеПроцедурыИФункции`; не для прямого вызова из прикладного кода.
- `РегламентныеЗаданияСервер` — сервер, внешнее соединение: стабильный CRUD
  регламентных заданий (`НайтиЗадания`, `ДобавитьЗадание`, `ИзменитьЗадание`,
  `УдалитьЗадание`, `УстановитьРасписаниеРегламентногоЗадания` и т. д.).
- `РегламентныеЗаданияКлиент` — клиент: переход к настройке, блокировка
  внешних ресурсов.
- `РегламентныеЗаданияПереопределяемый` — **хук**: БСП вызывает, прикладной код
  реализует (копирует модуль-переопределитель и переопределяет тело). Не
  вызывать напрямую.
- `РегламентныеЗаданияСлужебный` — ⚠️ служебный API, обратная совместимость не
  гарантируется.

⚠️ **Модуль `РегламентныеЗадания` (без суффикса) НЕ существует** как общий
модуль. CRUD-операции лежат в `РегламентныеЗаданияСервер`. Конвертация
расписания — в `ОбщегоНазначенияКлиентСервер.РасписаниеВСтруктуру` /
`СтруктураВРасписание` (регион `ПрограммныйИнтерфейс`, область
`РегламентныеЗадания`).
⚠️ **Модуль `ДлительныеОперацииСлужебный` НЕ существует** — служебные методы
встроены в сам `ДлительныеОперации` в `СлужебныйПрограммныйИнтерфейс`.

## Сценарии

### 1. Запустить функцию в фоне с возвратом значения

**Задача:** выполнить тяжёлую серверную обработку в фоновом задании, вернуть
результат на клиент через временное хранилище, не блокируя форму.

**Функции:**
`ДлительныеОперации.ВыполнитьФункцию(Знач ПараметрыВыполнения, ИмяФункции, Знач Параметр1 = Неопределено, Знач Параметр2 = Неопределено, Знач Параметр3 = Неопределено, Знач Параметр4 = Неопределено, Знач Параметр5 = Неопределено, Знач Параметр6 = Неопределено, Знач Параметр7 = Неопределено) Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, внешнее
соединение.
`ДлительныеОперации.ПараметрыВыполненияФункции(Знач ИдентификаторФормы) Экспорт`
— Функция (stable), конструктор структуры параметров.
`ДлительныеОперацииКлиент.ОжидатьЗавершение(Знач ДлительнаяОперация, Знач ОповещениеОЗавершении = Неопределено, Знач ПараметрыОжидания = Неопределено) Экспорт`
— Процедура (stable), клиент.

**Параметры:**
- `ПараметрыВыполнения` (ФормаКлиентскогоПриложения / УникальныйИдентификатор /
  Структура) — для `ВыполнитьФункцию`: форма-владелец, её идентификатор или
  структура из `ПараметрыВыполненияФункции`.
- `ИмяФункции` (Строка) — имя экспортной функции общего модуля, модуля
  менеджера или обработки, напр. `"Обработка.МояОбработка.ПодготовитьДанные"`.
- `Параметр1…7` (Произвольный) — параметры вызываемой функции; значения и
  возвращаемое значение должны быть сериализуемыми. Параметры не должны быть
  возвращаемыми.
- `ДлительнаяОперация` (Структура) — результат `ВыполнитьФункцию`: `Статус`,
  `ИдентификаторЗадания`, `АдресРезультата`.
- `ОповещениеОЗавершении` (ОписаниеОповещения) — обработчик
  `<ИмяПроцедуры>(Результат, ДопПараметры) Экспорт`.
- `ПараметрыОжидания` (Структура) — из `ДлительныеОперацииКлиент.ПараметрыОжидания`.

**Пример:**
```bsl
// Серверная функция — будет выполнена в фоне.
Функция ПодготовитьДанныеОтчёта(Знач Параметр1, Знач Параметр2) Экспорт
    ДлительныеОперации.СообщитьПрогресс(10, "Загрузка справочников");
    // ...тяжёлая работа...
    ДлительныеОперации.СообщитьПрогресс(90, "Формирование таблицы");
    Возврат Результат;  // попадёт в АдресРезультата
КонецФункции

&НаСервере
Функция ЗапуститьПодготовку()
    ПараметрыВыполнения = ДлительныеОперации.ПараметрыВыполненияФункции(УникальныйИдентификатор);
    Возврат ДлительныеОперации.ВыполнитьФункцию(
        ПараметрыВыполнения,
        "Обработка.МояОбработка.ПодготовитьДанныеОтчёта",
        Параметр1, Параметр2);
КонецФункции

&НаКлиенте
Процедура Запустить(Команда)
    ДлительнаяОперация = ЗапуститьПодготовку();
    Оповещение = Новый ОписаниеОповещения("ОбработатьРезультат", ЭтотОбъект);
    ДлительныеОперацииКлиент.ОжидатьЗавершение(ДлительнаяОперация, Оповещение,
        ДлительныеОперацииКлиент.ПараметрыОжидания(ЭтотОбъект));
КонецПроцедуры

&НаКлиенте
Процедура ОбработатьРезультат(Результат, ДопПараметры) Экспорт
    Если Результат = Неопределено Тогда Возврат; КонецЕсли;
    Если Результат.Статус = "Ошибка" Тогда
        СтандартныеПодсистемыКлиент.ВывестиИнформациюОбОшибке(Результат.ИнформацияОбОшибке);
        Возврат;
    КонецЕсли;
    Данные = ПолучитьИзВременногоХранилища(Результат.АдресРезультата);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Синхронный вызов тяжёлой функции в `&НаКлиенте` — форма «виснет» на
  минуты. Всегда `ВыполнитьФункцию` + `ОжидатьЗавершение`.
- ❌ `ФоновыеЗадания.Выполнить(...)` напрямую — обходит журнал регистрации,
  замеры производительности, обработку ошибок БСП. Только через
  `ДлительныеОперации.ВыполнитьФункцию` / `ВыполнитьПроцедуру`.
- Фоновое задание выполняется **вне транзакции** формы; сообщения
  `СообщитьПользователю` из фона накапливаются и доставляются на клиент через
  `ДлительныеОперации.СообщенияПользователю`.

### 2. Запустить процедуру в фоне без возвращаемого значения

**Задача:** выполнить фоновую обработку (рассылка, обмен, загрузка), не
ожидая результата.

**Функции:**
`ДлительныеОперации.ВыполнитьПроцедуру(Знач ПараметрыВыполнения = Неопределено, ИмяПроцедуры, Знач Параметр1 = Неопределено, Знач Параметр2 = Неопределено, Знач Параметр3 = Неопределено, Знач Параметр4 = Неопределено, Знач Параметр5 = Неопределено, Знач Параметр6 = Неопределено, Знач Параметр7 = Неопределено) Экспорт`
— Функция (stable). Сервер, внешнее соединение.
`ДлительныеОперации.ПараметрыВыполненияПроцедуры() Экспорт` — Функция (stable),
конструктор параметров (без аргументов).

**Параметры:**
- `ПараметрыВыполнения` (Структура) — из `ПараметрыВыполненияПроцедуры`; можно
  задать `НаименованиеФоновогоЗадания` для заголовка формы ожидания.
- `ИмяПроцедуры` (Строка) — имя экспортной процедуры (без возвращаемого
  значения).
- `Параметр1…7` (Произвольный) — параметры процедуры, сериализуемые.

**Пример:**
```bsl
&НаСервере
Процедура ЗапуститьРассылку()
    ПараметрыВыполнения = ДлительныеОперации.ПараметрыВыполненияПроцедуры();
    ПараметрыВыполнения.НаименованиеФоновогоЗадания = "Рассылка уведомлений";
    ДлительныеОперации.ВыполнитьПроцедуру(
        ПараметрыВыполнения,
        "Обработка.РассылкаУведомлений.ВыполнитьРассылку",
        СписокПолучателей);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ `ВыполнитьПроцедуру(Параметры, П1, "Обработка.Моя.Метод")` — `ИмяПроцедуры`
  это **второй** параметр, а не последний. Порядок: `ПараметрыВыполнения,
  ИмяПроцедуры, Параметр1…`.
- Для процедуры без параметров можно передать `ПараметрыВыполнения = Неопределено`
  (по умолчанию) — будет создана временная форма-владелец.

### 3. Показать прогресс и получить сообщения пользователя

**Задача:** выводить процент/текст выполнения длительной операции в
стандартной форме ожидания и забрать накопленные сообщения пользователя.

**Функции:**
`ДлительныеОперации.СообщитьПрогресс(Знач Процент = Неопределено, Знач Текст = Неопределено, Знач ДополнительныеПараметры = Неопределено) Экспорт`
— Процедура (stable). Вызывается **в теле фоновой процедуры/функции**.
`ДлительныеОперации.ПрочитатьПрогресс(Знач ИдентификаторЗадания) Экспорт` —
Функция (stable), сервер.
`ДлительныеОперации.СообщенияПользователю(УдалятьПолученные = Ложь, ИдентификаторЗадания = Неопределено) Экспорт`
— Функция (stable), сервер.
`ДлительныеОперацииКлиент.ПараметрыОжидания(ФормаВладелец) Экспорт` — Функция
(stable), клиент; возвращает структуру со свойствами `ВыводитьОкноОжидания`,
`ВыводитьПрогрессВыполнения`, `ОповещениеОПрогрессеВыполнения`, `ВыводитьСообщения`,
`Интервал`.

**Параметры:**
- `Процент` (Число) — 0…100; `Неопределено` — только текст без процента.
- `Текст` (Строка) — описание текущего шага.
- `ДополнительныеПараметры` (Произвольный) — произвольные данные, передаются в
  `ОповещениеОПрогрессеВыполнения`.
- `ИдентификаторЗадания` (УникальныйИдентификатор) — идентификатор фонового
  задания.
- `УдалятьПолученные` (Булево) — `Истина` удаляет прочитанные сообщения из
  очереди.

**Пример:**
```bsl
// В теле фоновой процедуры
Процедура ОбработатьНаборДанных(Параметры) Экспорт
    Для Сч = 1 По Параметры.КоличествоЦикл Цикл
        // ...обработка порции...
        ДлительныеОперации.СообщитьПрогресс(Сч * 100 / Параметры.КоличествоЦикл,
            "Обработано " + Сч + " из " + Параметры.КоличествоЦикл);
    КонецЦикла;
КонецПроцедуры

// На клиенте — включить вывод прогресса в форме ожидания
&НаКлиенте
Процедура Запустить()
    ПараметрыОжидания = ДлительныеОперацииКлиент.ПараметрыОжидания(ЭтотОбъект);
    ПараметрыОжидания.ВыводитьПрогрессВыполнения = Истина;
    ДлительныеОперацииКлиент.ОжидатьЗавершение(ДлительнаяОперация, Оповещение, ПараметрыОжидания);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ⚠️ Не сообщать прогресс чаще 100 раз за операцию — избыточное потребление
  памяти и утечки. Сообщения чаще 3 сек заменяют предыдущее.
- ❌ Использовать `СообщитьПрогресс` для передачи результата по частям — это
  не его назначение; результат возвращается через `АдресРезультата`.
- Прогресс виден пользователю, только если `ВыводитьПрогрессВыполнения = Истина`
  в `ПараметрыОжидания`.

### 4. Отменить фоновое задание и проверить статус

**Задача:** отменить длительную операцию по команде пользователя и узнать
состояние задания без подключения формы ожидания.

**Функции:**
`ДлительныеОперации.ОтменитьВыполнениеЗадания(Знач ИдентификаторЗадания) Экспорт`
— Процедура (stable). Сервер.
`ДлительныеОперации.ЗаданиеВыполнено(Знач ИдентификаторЗадания, РасширенныйРезультат = Ложь) Экспорт`
— Функция (stable). Сервер. Возвращает `Булево` или `Структуру` (при
`РасширенныйРезультат = Истина`) со свойством `Статус`:
`"Выполняется"`/`"Выполнено"`/`"Ошибка"`/`"Отменено"`.

**Параметры:**
- `ИдентификаторЗадания` (УникальныйИдентификатор) — из
  `ДлительнаяОперация.ИдентификаторЗадания`.
- `РасширенныйРезультат` (Булево) — `Истина` → структура с деталями; `Ложь`
  (по умолчанию) → просто `Булево` (завершено или нет).

**Пример:**
```bsl
// Отмена по команде пользователя
&НаСервере
Процедура ОтменитьОперацию(ИдентификаторЗадания)
    ДлительныеОперации.ОтменитьВыполнениеЗадания(ИдентификаторЗадания);
КонецПроцедуры

// Поллинг состояния без формы ожидания (напр., из внешнего соединения)
Статус = ДлительныеОперации.ЗаданиеВыполнено(ИдентификаторЗадания, Истина).Статус;
Если Статус = "Отменено" Тогда
    // пользователь отменил
ИначеЕсли Статус = "Ошибка" Тогда
    // ЗаданиеВыполнено уже вызвало исключение с текстом ошибки
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ⚠️ При аварийном завершении `ЗаданиеВыполнено` **вызывает исключение** с
  текстом ошибки из фонового задания. Оборачивайте в `Попытка…Исключение`,
  если нужна мягкая обработка.
- `ОтменитьВыполнениеЗадания` только инициирует отмену; фактически задание
  остановится не мгновенно — проверяйте статус через `ЗаданиеВыполнено`.
- ❌ `ДлительныеОперацииВызовСервера.ФоновоеЗаданиеЗавершено` — служебный
  (регион `СлужебныеПроцедурыИФункции`); используйте стабильный
  `ДлительныеОперации.ЗаданиеВыполнено`.

### 5. Создать регламентное задание с расписанием

**Задача:** программно создать регламентное задание (при первичном
заполнении ИБ, обновлении конфигурации) с расписанием и прикладным ключом.

**Функции:**
`РегламентныеЗаданияСервер.ДобавитьЗадание(Параметры) Экспорт` — Функция
(stable). Сервер, внешнее соединение. Возвращает созданное задание.
`ОбщегоНазначенияКлиентСервер.РасписаниеВСтруктуру(Знач Расписание) Экспорт` —
Функция (stable), `РасписаниеРегламентногоЗадания` → `Структура`.
`ОбщегоНазначенияКлиентСервер.СтруктураВРасписание(Знач СтруктураРасписания) Экспорт`
— Функция (stable), `Структура` → `РасписаниеРегламентногоЗадания`.

**Параметры:**
- `Параметры` (Структура) — для `ДобавитьЗадание`: `Метаданные`
  (ОбъектМетаданныхРегламентноеЗадание, обязательно), `Расписание`
  (РасписаниеРегламентногоЗадания), `Использование` (Булево), `Ключ` (Строка,
  прикладной идентификатор), `Параметры` (Массив — параметры метода обработчика),
  `ИнтервалПовтораПриАварийномЗавершении` (Число, сек),
  `КоличествоПовторовПриАварийномЗавершении` (Число).
- `Расписание` (РасписаниеРегламентногоЗадания) — исходное расписание.
- `СтруктураРасписания` (Структура) — поля расписания: `ПериодПовтораДней`,
  `ПериодПовтораВТечениеДня`, `ДниНедели`, `ВремяНачала`, `ВремяКонца`,
  `ДатаНачала`, `ДатаКонца` и т. п.

**Пример:**
```bsl
// Расписание удобнее собирать в структуре (на клиенте), затем конвертировать
РасписаниеСтруктурой = ОбщегоНазначенияКлиентСервер.РасписаниеВСтруктуру(РасписаниеРегламентногоЗадания);
РасписаниеСтруктурой.ПериодПовтораДней = 1;
РасписаниеСтруктурой.ПериодПовтораВТечениеДня = 3600;
РасписаниеОбъектом = ОбщегоНазначенияКлиентСервер.СтруктураВРасписание(РасписаниеСтруктурой);

ПараметрыЗадания = Новый Структура;
ПараметрыЗадания.Вставить("Метаданные",   Метаданные.РегламентныеЗадания.МояЗадача);
ПараметрыЗадания.Вставить("Расписание",   РасписаниеОбъектом);
ПараметрыЗадания.Вставить("Использование", Истина);
ПараметрыЗадания.Вставить("Ключ",         "МояЗадача_Основная");
Задание = РегламентныеЗаданияСервер.ДобавитьЗадание(ПараметрыЗадания);
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `РегламентныеЗаданияСервер.ДобавитьЗадание` из клиентского кода —
  модуль серверный, упадёт на тонком клиенте. Только через `&НаСервере` /
  `&НаСервереБезКонтекста`.
- ❌ `РегламентныеЗадания.ДобавитьЗадание(...)` — модуль без суффикса не
  существует. Только `РегламентныеЗаданияСервер`.
- В модели сервиса `ДобавитьЗадание` создаёт запись в справочнике
  `ОчередьЗаданий`, а не платформенное регламентное задание.

### 6. Найти, изменить и удалить регламентное задание

**Задача:** найти задание по прикладному ключу, сменить расписание или
включить/выключить его, удалить дубли.

**Функции:**
`РегламентныеЗаданияСервер.НайтиЗадания(Отбор) Экспорт` — Функция (stable).
Возвращает массив заданий (локальный режим) или таблицу значений (модель
сервиса).
`РегламентныеЗаданияСервер.ИзменитьЗадание(Знач Идентификатор, Знач Параметры) Экспорт`
— Процедура (stable).
`РегламентныеЗаданияСервер.УдалитьЗадание(Знач Идентификатор) Экспорт` —
Процедура (stable).
`РегламентныеЗаданияСервер.УстановитьРасписаниеРегламентногоЗадания(Знач Идентификатор, Знач Расписание) Экспорт`
— Процедура (stable).
`РегламентныеЗаданияСервер.УстановитьИспользованиеРегламентногоЗадания(Знач Идентификатор, Знач Использование) Экспорт`
— Процедура (stable).

**Параметры:**
- `Отбор` (Структура) — свойства: `Метаданные`
  (ОбъектМетаданныхРегламентноеЗадание), `Ключ` (Строка), `Использование`
  (Булево), `УникальныйИдентификатор` (УникальныйИдентификатор / Строка /
  `СправочникСсылка.ОчередьЗаданий`).
- `Идентификатор` (УникальныйИдентификатор / Строка / ОбъектМетаданных) — для
  `ИзменитьЗадание`/`УдалитьЗадание`/`Установить*`.
- `Параметры` (Структура) — изменяемые свойства для `ИзменитьЗадание` (те же,
  что в `ДобавитьЗадание`).
- `Расписание` (РасписаниеРегламентногоЗадания) — новое расписание.
- `Использование` (Булево) — `Истина` включает выполнение по расписанию.

**Пример:**
```bsl
// Найти по ключу и сменить расписание, удалить дубли
Отбор = Новый Структура("Ключ", "МояЗадача_Основная");
Задания = РегламентныеЗаданияСервер.НайтиЗадания(Отбор);
Если Задания.Количество() > 0 Тогда
    Идентификатор = Задания[0].УникальныйИдентификатор;
    РегламентныеЗаданияСервер.УстановитьРасписаниеРегламентногоЗадания(Идентификатор, НовоеРасписание);
    // Удалить случайные дубли — оставить только первое
    Для Сч = 1 По Задания.ВГраница() Цикл
        РегламентныеЗаданияСервер.УдалитьЗадание(Задания[Сч].УникальныйИдентификатор);
    КонецЦикла;
КонецЕсли;

// Выключить задание при ошибке
РегламентныеЗаданияСервер.УстановитьИспользованиеРегламентногоЗадания(Идентификатор, Ложь);
```

**Нюансы / антипаттерны:**
- ❌ Плодить задания с одним `Метаданные` и разными ключами для каждой
  организации — конфликтуют при запуске. Лучше одно задание, итерация внутри
  обработчика; либо разные `Метаданные`.
- `НайтиЗадания` без отбора возвращает все задания — дорого; всегда передавайте
  отбор.
- В обработчике регламентного задания проверяйте, что оно ещё существует
  (`НайтиЗадания(Отбор).Количество() > 0`), прежде чем выполнять работу —
  задание могли удалить.

### 7. Многопоточное выполнение и корректная обработка ошибок

**Задача:** распараллелить обработку большого набора данных в несколько
фоновых потоков и безопасно обработать ошибку в обработчике регламентного
задания.

**Функции:**
`ДлительныеОперации.ВыполнитьФункциюВНесколькоПотоков(ИмяФункции, ПараметрыВыполнения, НаборПараметровФункции = Неопределено) Экспорт`
— Функция (stable). Сервер.
`ДлительныеОперации.ВыполнитьПроцедуруВНесколькоПотоков(ИмяПроцедуры, ПараметрыВыполнения, НаборПараметровПроцедуры = Неопределено) Экспорт`
— Функция (stable). Сервер.
`ДлительныеОперации.ДопустимоеКоличествоПотоков() Экспорт` — Функция
(⚠️ регион `СлужебныйПрограммныйИнтерфейс`), возвращает максимально
допустимое число потоков.

**Параметры:**
- `ИмяФункции` / `ИмяПроцедуры` (Строка) — экспортная функция/процедура общего
  модуля или модуля менеджера.
- `ПараметрыВыполнения` (Структура) — из `ПараметрыВыполненияФункции` /
  `ПараметрыВыполненияПроцедуры`.
- `НаборПараметровФункции` (Массив из Структура / `Неопределено`) — массив
  структур параметров; по одному элементу на поток. `Неопределено` —
  параметр один на все потоки.

**Пример:**
```bsl
// Многопоточная обработка порций
ПараметрыВыполнения = ДлительныеОперации.ПараметрыВыполненияФункции(УникальныйИдентификатор);
НаборПараметров = Новый Массив;
Для Каждого Порция Из ПорцииДанных Цикл
    НаборПараметров.Добавить(Новый Структура("Данные", Порция));
КонецЦикла;
ДлительнаяОперация = ДлительныеОперации.ВыполнитьФункциюВНесколькоПотоков(
    "Обработка.МояОбработка.ОбработатьПорцию", ПараметрыВыполнения, НаборПараметров);

// Обработчик регламентного задания с защитой от ошибок
Процедура МояЗадача() Экспорт
    Ключ = "МояЗадача_Основная";
    Если РегламентныеЗаданияСервер.НайтиЗадания(Новый Структура("Ключ", Ключ)).Количество() = 0 Тогда
        Возврат; // задание удалили
    КонецЕсли;
    Попытка
        // ...основная работа...
    Исключение
        Ид = РегламентныеЗаданияСервер.НайтиЗадания(Новый Структура("Ключ", Ключ))[0].УникальныйИдентификатор;
        РегламентныеЗаданияСервер.УстановитьИспользованиеРегламентногоЗадания(Ид, Ложь);
        ОбщегоНазначения.ЗаписатьВЖурналРегистрации(УровеньЖурналаРегистрации.Ошибка,, , ,
            "МояЗадача", ИнформацияОбОшибке());
        ВызватьИсключение;
    КонецПопытки;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ⚠️ `ДопустимоеКоличествоПотоков` — служебный регион; для прикладного кода
  ориентируйтесь на `ДлительныеОперации.ПараметрыВыполненияФункции` и не
  превышайте платформенное ограничение числа фоновых заданий.
- ❌ Транзакционная запись внутри потока без `Попытка…Исключение` — ошибка в
  одном потоке прерывает всю операцию. Каждый поток — независимая обработка
  своей порции.
- `ВыполнитьВФоне(ИмяПроцедуры, ПараметрыПроцедуры, ПараметрыВыполнения)` —
  стабильный (регион `ПрограммныйИнтерфейс`), но doc-комментарий рекомендует
  `ВыполнитьФункцию`/`ВыполнитьПроцедуру` (произвольное число параметров до 7,
  без обёртки «Параметры»/«АдресРезультата»). В новом коде — `ВыполнитьФункцию`.
- ⚠️ `ЗапуститьВыполнениеВФоне` — регион `УстаревшиеПроцедурыИФункции`
  (deprecated), не использовать в новом коде; альтернатива — `ВыполнитьФункцию`.

## Дополнительно

Прочие стабильные методы (регион `ПрограммныйИнтерфейс`), полные сигнатуры —
через `python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`:

- `РегламентныеЗаданияСервер.РасписаниеРегламентногоЗадания(Знач Идентификатор, Знач ВСтруктуре = Ложь)` — получить расписание (объектом или структурой).
- `РегламентныеЗаданияСервер.РегламентноеЗаданиеИспользуется(Знач Идентификатор)` — Булево, включено ли задание.
- `РегламентныеЗаданияСервер.ПолучитьРегламентноеЗадание(Знач Идентификатор)` — объект задания.
- `РегламентныеЗаданияСервер.СвойстваПоследнегоЗадания(Знач Задание)` — свойства последнего фонового выполнения.
- `РегламентныеЗаданияСервер.РаботаСВнешнимиРесурсамиЗаблокирована()` /
  `ЗаблокироватьРаботуСВнешнимиРесурсами()` /
  `РазблокироватьРаботуСВнешнимиРесурсами()` — управление блокировкой внешних
  ресурсов на время обновления.
- `ДлительныеОперацииКлиент.НовыйРезультатДлительнойОперации()` /
  `НовоеСостояниеДлительнойОперации()` — конструкторы служебных структур.
- Хук переопределения:
  `РегламентныеЗаданияПереопределяемый.ПриОпределенииНастроекРегламентныхЗаданий(Настройки)`
  — БСП вызывает, прикладной код реализует (модуль `*Переопределяемый`);
  настройка блокировки регламентных заданий в модели сервиса. Не вызывать
  напрямую.
- ⚠️ Служебные (регион `СлужебныйПрограммныйИнтерфейс` /
  `СлужебныеПроцедурыИФункции`), не использовать в прикладном коде:
  `РегламентныеЗаданияСлужебный.ВыполнитьРегламентноеЗаданиеВручную(Знач Задание)`,
  `РегламентныеЗаданияСлужебный.ПолучитьСвойстваФоновогоЗадания(Идентификатор, ИменаСвойств = "")`,
  `ДлительныеОперации.ОперацияВыполнена(Знач ИдентификаторЗадания, Задание = Неопределено)`.
