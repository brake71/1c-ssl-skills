# Печать и варианты отчётов

Подсистемы **Печать** и **ВариантыОтчетов** — инфраструктура печатных форм (табличные
и офисные документы, подменю «Печать» в формах) и вариантов отчётов (справочник
`ВариантыОтчетов`, подключение вариантов к подсистемам, программное формирование
отчёта на СКД). Когда нужно сформировать печатную форму по команде «Печать» или
открыть/сформировать отчёт через БСП-инфраструктуру вариантов.

## Модули

Семейство **УправлениеПечатью** (подсистема `Печать`):

- `УправлениеПечатью` — сервер, стабильный API: `СоздатьКоллекциюКомандПечати`,
  `КомандыПечатиФормы`, `ВывестиТабличныйДокументВКоллекцию`, `ЗадатьОбластьПечатиДокумента`,
  `МакетПечатнойФормы`, `МакетыИДанныеОбъектовДляПечати`, `СведенияОПечатнойФорме`,
  `НужноПечататьМакет`, `ДобавитьУсловиеВидимостиКоманды`.
- `УправлениеПечатьюКлиент` — клиент, стабильный API: `ВыполнитьКомандуПечати`,
  `ВыполнитьКомандуПечатиНаПринтер`, `ПечатьДокументов`, `ПараметрыПечати`.
- `УправлениеПечатьюВызовСервера` — серверный вызов с клиента без контекста формы.
- `УправлениеПечатьюКлиентСервер` — общий: константы имён полей, настройки сохранения.
- `УправлениеПечатьюПереопределяемый` — **хуки**: `ПриОпределенииНастроекПечати`,
  `ПриПечати`, `ПриПолученииКомандПечати`, `ПередПечатью`. БСП вызывает, прикладной
  код реализует, не вызывается напрямую.
- `УправлениеПечатьюСлужебный`, `УправлениеПечатьюСлужебныйКлиент` — ⚠️ служебные.

Семейство **ВариантыОтчетов** (подсистема `ВариантыОтчетов`):

- `ВариантыОтчетов` — сервер, стабильный API: `ВариантОтчета`, `НастроитьОтчетВМодулеМенеджера`,
  `ОписаниеОтчета`, `ОписаниеВарианта`, `КлючиВариантовОтчета`.
- `ВариантыОтчетовКлиент` — клиент, стабильный API: `ОткрытьФормуОтчета`,
  `ПоказатьПанельОтчетов`, `ОбновитьОткрытыеФормы`.
- `ВариантыОтчетовВызовСервера` — серверный вызов с клиента.
- `ВариантыОтчетовПереопределяемый` — **хуки**: `НастроитьВариантыОтчетов`,
  `ОпределитьРазделыСВариантамиОтчетов`, `ОпределитьОбъектыСКомандамиОтчетов`,
  `ПриОпределенииНастроек`.
- `ОтчетыКлиент` — клиент, стабильный: `СформироватьОтчет(ФормаОтчета, ОбработчикЗавершения)`
  (формирование отчёта из его формы).

> ⚠️ **`Печать` — это подсистема (объект метаданных), а не общий модуль.** Модуль
  подсистемы называется `УправлениеПечатью`. `python … module Печать` сообщит
  «Module 'Печать' not found» — это не ошибка, используйте `module УправлениеПечатью`.
>
> ⚠️ `УправлениеПечатью.СформироватьПечатныеФормы` — регион
> `СлужебныеПроцедурыИФункции` (не `ПрограммныйИнтерфейс`); `ВариантыОтчетов.СформироватьОтчет`
> — `СлужебныйПрограммныйИнтерфейс`. Оба могут измениться в минорной версии БСП —
> для нового кода предпочтительны клиентские стабильные пути (`ВыполнитьКомандуПечати`,
> `ОтчетыКлиент.СформироватьОтчет`).

## Сценарии

### 1. Объявить печатную форму в подменю «Печать» объекта

**Задача:** в модуле менеджера объекта (документа/справочника) зарегистрировать
команду печати, чтобы БСП автоматически поместила её в подменю «Печать» формы.

**Функция:**
`УправлениеПечатью.СоздатьКоллекциюКомандПечати() Экспорт`
— Функция → ТаблицаЗначений (пустая коллекция команд печати), регион
`#Область ПрограммныйИнтерфейс` (stable). Сервер, ВнешнееСоединение.

**Параметры:** нет. Возвращает пустую таблицу с колонками `Идентификатор`,
`Представление`, `МенеджерПечати`, `Картинка`, `Порядок`, `ПроверкаПроведенияПередПечатью`,
`ДополнительныеПараметры` и др.

**Пример:**
```bsl
// Модуль менеджера документа
Процедура ДобавитьКомандыПечати(КомандыПечати, Параметры) Экспорт
    // КомандыПечати приходит уже инициализированной — СоздатьКоллекциюКомандПечати
    // вызывать не нужно. Достаточно добавить строку.
    КомандаПечати = КомандыПечати.Добавить();
    КомандаПечати.Идентификатор = "СписаниеТоваров";
    КомандаПечати.Представление = НСтр("ru = 'Списание товаров'");
    КомандаПечати.МенеджерПечати = "Документ._ДемоСписаниеТоваров";
    КомандаПечати.Картинка = БиблиотекаКартинок.ПечатьMXL;
    КомандаПечати.Порядок = 10;
    КомандаПечати.ПроверкаПроведенияПередПечатью = Истина;
    КомандаПечати.ДополнительныеПараметры = Новый Структура("ПоказыватьЦены", Истина);

    // Видимость команды — только для проведённых
    УправлениеПечатью.ДобавитьУсловиеВидимостиКоманды(
        КомандаПечати, "Проведен", Истина, ВидСравненияКомпоновкиДанных.Равно);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Реализовывать команды печати вне модуля менеджера объекта (напр. в общем
  модуле «ПечатныеФормы»). `УправлениеПечатью.КомандыПечатиФормы` ищет команды именно
  в `ДобавитьКомандыПечати` модуля менеджера — подменю останется пустым.
- ❌ Создавать собственный общий модуль `ПечатьМоейПодсистемы` и формировать табличные
  документы в обход `УправлениеПечатью` — ломает подменю «Печать», замеры
  производительности, контроль прав, переключение языка печатных форм, рассылку.
- `УправлениеПечатью.ДобавитьУсловиеВидимостиКоманды(КомандаПечати, Реквизит, Значение, Знач МетодСравнения = Неопределено)`
  — 4-й параметр называется `МетодСравнения` (не `ВидСравнения`, как у
  `ПодключаемыеКоманды.ДобавитьУсловиеВидимостиКоманды` из `commands-external.md`).

### 2. Реализовать менеджер печати (сформировать табличный документ)

**Задача:** в модуле менеджера объекта реализовать экспортную `Печать(...)`,
которую БСП-инфраструктура вызовет при выборе команды печати; собрать табличный
документ и поместить его в коллекцию печатных форм.

**Функции:**
`УправлениеПечатью.НужноПечататьМакет(КоллекцияПечатныхФорм, ИмяМакета) Экспорт` — Функция → Булево, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеПечатью.ЗадатьОбластьПечатиДокумента(ТабличныйДокумент, НомерСтрокиНачало, ОбъектыПечати, Ссылка) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеПечатью.ВывестиТабличныйДокументВКоллекцию(КоллекцияПечатныхФорм, ИмяМакета, СинонимМакета, ТабличныйДокумент, Картинка = Неопределено, ПолныйПутьКМакету = "", ИмяФайлаПечатнойФормы = Неопределено) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`УправлениеПечатью.МакетПечатнойФормы(ПутьКМакету, Знач КодЯзыка = Неопределено) Экспорт` — Функция → ТабличныйДокумент, `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `КоллекцияПечатныхФорм` (ТаблицаЗначений) — коллекция из `Печать(...)`; колонки
  `ИмяМакета`, `СинонимМакета`, `ТабличныйДокумент`, `Картинка`, …
- `ИмяМакета` (Строка) — идентификатор макета, для которого формируется документ.
- `ТабличныйДокумент` (ТабличныйДокумент) — готовый документ для вывода.
- `ОбъектыПечати` (ТаблицаЗначений) — таблица «объект → область» для связи
  распечатанных областей с исходными ссылками (используется при печати комплекта).
- `НомерСтрокиНачало` (Число) — номер строки начала области одного объекта.
- `Ссылка` (ЛюбаяСсылка) — объект-владелец области.
- `ПутьКМакету` (Строка) — путь вида `"Документ._ДемоСписаниеТоваров.ПФ_MXL_СписаниеТоваров"`.
- `КодЯзыка` (Строка) — код языка макета; `Неопределено` → текущий.

**Пример:**
```bsl
// Модуль менеджера документа
Процедура Печать(МассивОбъектов, ПараметрыПечати, КоллекцияПечатныхФорм, ОбъектыПечати, ПараметрыВывода) Экспорт
    Если УправлениеПечатью.НужноПечататьМакет(КоллекцияПечатныхФорм, "СписаниеТоваров") Тогда
        ТабДок = Новый ТабличныйДокумент;
        Для Каждого Ссылка Из МассивОбъектов Цикл
            НачалоОбласти = ТабДок.ВысотаТаблицы + 1;
            // ...заполнение строк ТабДок по данным Ссылка...
            УправлениеПечатью.ЗадатьОбластьПечатиДокумента(ТабДок, НачалоОбласти, ОбъектыПечати, Ссылка);
        КонецЦикла;
        УправлениеПечатью.ВывестиТабличныйДокументВКоллекцию(
            КоллекцияПечатныхФорм, "СписаниеТоваров", НСтр("ru = 'Списание товаров'"), ТабДок);
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Читать макет через `ПолучитьОбщийМакет(...)` / `МакетОбъекта()` в обход
  `УправлениеПечатью.МакетПечатнойФормы` — теряется учёт пользовательских макетов
  (администратор мог переопределить макет) и кода языка.
- `МассивОбъектов`, `КоллекцияПечатныхФорм`, `ОбъектыПечати`, `ПараметрыВывода` —
  стандартные параметры менеджера печати; `ПараметрыПечати` — структура, переданная
  из `УправлениеПечатьюКлиент.ВыполнитьКомандуПечати` (содержит `ДополнительныеПараметры`
  команды, `ЗаголовокФормы` и т. п.).
- Постобработка печатной формы (добавить дату/подпись) — в хуке
  `УправлениеПечатьюПереопределяемый.ПриПечати(МассивОбъектов, ПараметрыПечати, КоллекцияПечатныхФорм, ОбъектыПечати, ПараметрыВывода)`
  (переопределение, не вызов).

### 3. Запустить печать из формы объекта (клиент → форма ПечатьДокументов)

**Задача:** по клику команды «Печать» в форме открыть форму `ПечатьДокументов` с
готовыми табличными документами; или напечатать сразу на принтер.

**Функции:**
`УправлениеПечатьюКлиент.ВыполнитьКомандуПечати(ИмяМенеджераПечати, ИменаМакетов, МассивОбъектов, ВладелецФормы, ПараметрыПечати = Неопределено) Экспорт` — Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.
`УправлениеПечатьюКлиент.ВыполнитьКомандуПечатиНаПринтер(ИмяМенеджераПечати, ИменаМакетов, МассивОбъектов, ПараметрыПечати = Неопределено) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `ИмяМенеджераПечати` (Строка) — полное имя модуля менеджера печати, напр.
  `"Документ._ДемоСписаниеТоваров"`.
- `ИменаМакетов` (Строка) — идентификаторы макетов через запятую, напр.
  `"СписаниеТоваров,АктСписания"`.
- `МассивОбъектов` (Массив) — ссылки объектов для печати.
- `ВладелецФормы` (УправляемаяФорма) — форма-владелец (для `ВыполнитьКомандуПечати`).
- `ПараметрыПечати` (Структура / Неопределено) — структура параметров; по умолчанию
  `Неопределено` (БСП соберёт из `ПараметрыПечати()` и `ДополнительныеПараметры` команды).

**Пример:**
```bsl
&НаКлиенте
Процедура КомандаПечати(Команда)
    УправлениеПечатьюКлиент.ВыполнитьКомандуПечати(
        "Документ._ДемоСписаниеТоваров",
        "СписаниеТоваров,АктСписания",
        ДокументыНаПечать,
        ЭтотОбъект);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `УправлениеПечатью.СформироватьПечатныеФормы` синхронно из обработчика
  команды формы — форма «висит» на тяжёлой печати. Для интерактивной печати —
  `ВыполнитьКомандуПечати` (открывает `ПечатьДокументов` с формированием в фоне);
  для тяжёлого серверного формирования — `ДлительныеОперации.ВыполнитьФункцию`
  (см. сценарий 6).
- `ВыполнитьКомандуПечатиНаПринтер` минует предпросмотр; используйте для
  регламентной/пакетной печати, не для ручных команд.

### 4. Программно сформировать пакет печатных форм (сервер)

**Задача:** сформировать пакет печатных форм из серверного кода (напр. для
отправки по e-mail, рассылки отчётов) и получить готовые табличные документы.

**Функция:**
`УправлениеПечатью.СформироватьПечатныеФормы(Знач ИмяМенеджераПечати, Знач ИменаМакетов, Знач МассивОбъектов, Знач ПараметрыПечати, ДопустимыеТипыОбъектовПечати = Неопределено, Знач КодЯзыка = Неопределено, Знач ОбъектыПечати = Неопределено) Экспорт`
— Функция → Структура (`КоллекцияПечатныхФорм`, `ОбъектыПечати`, `ПараметрыВывода`),
регион `#Область СлужебныеПроцедурыИФункции` (⚠️ служебный, не `ПрограммныйИнтерфейс`).
Сервер.

**Параметры:**
- `ИмяМенеджераПечати` (Строка) — полное имя модуля менеджера печати.
- `ИменаМакетов` (Строка) — идентификаторы макетов через запятую.
- `МассивОбъектов` (Массив) — ссылки объектов для печати.
- `ПараметрыПечати` (Структура) — параметры; минимум пустая `Новый Структура`.
- `ДопустимыеТипыОбъектовПечати` (ОписаниеТипов / Неопределено) — ограничение типов
  объектов; `Неопределено` → без проверки.
- `КодЯзыка` (Строка) — код языка; `Неопределено` → текущий.
- `ОбъектыПечати` (ТаблицаЗначений / Неопределено) — для возврата связи областей.

**Пример:**
```bsl
&НаСервере
Функция ПодготовитьПечатныеФормыНаСервере(МассивСсылок)
    ПараметрыПечати = Новый Структура;
    ПараметрыПечати.Вставить("ЗаголовокФормы", НСтр("ru = 'Печатные формы документов'"));
    Возврат УправлениеПечатью.СформироватьПечатныеФормы(
        "Документ._ДемоСписаниеТоваров",
        "СписаниеТоваров,АктСписания",
        МассивСсылок,
        ПараметрыПечати);
КонецФункции

// Результат.КоллекцияПечатныхФорм — таблица с колонками ИмяМакета, СинонимМакета,
// ТабличныйДокумент; Результат.ОбъектыПечати, Результат.ПараметрыВывода.
```

**Нюансы / антипаттерны:**
- ⚠️ Метод в регионе `СлужебныеПроцедурыИФункции` — обратная совместимость не
  гарантируется. Для интерактивной печати предпочитайте клиентский
  `ВыполнитьКомандуПечати`; для серверной «тяжёлой» печати — фоновое формирование
  через `ДлительныеОперации.ВыполнитьФункцию` (сценарий 6).
- У метода есть варианты в `УправлениеПечатьюВызовСервера.СформироватьПечатныеФормы(МассивОбъектов, Команды)`
  и `УправлениеПечатьюСлужебный.СформироватьПечатныеФормы(...)` — все служебные, с
  разными сигнатурами. Прикладной код использует только серверный
  `УправлениеПечатью.СформироватьПечатныеФормы` с 7-параметрной сигнатурой выше.

### 5. Открыть форму отчёта по варианту

**Задача:** программно открыть форму отчёта (внутреннего или дополнительного) по
ссылке на вариант из справочника `ВариантыОтчетов`.

**Функции:**
`ВариантыОтчетов.ВариантОтчета(Отчет, КлючВарианта) Экспорт` — Функция → СправочникСсылка.ВариантыОтчетов / Неопределено, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ВариантыОтчетовКлиент.ОткрытьФормуОтчета(Знач ФормаВладелец, Знач Вариант, Знач ДополнительныеПараметры = Неопределено) Экспорт` — Процедура, `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `Отчет` (СправочникСсылка.ИдентификаторыОбъектовРасширений /
  ИдентификаторыОбъектовМетаданных / ДополнительныеОтчетыИОбработки / Строка) — отчёт
  или полное имя внешнего отчёта.
- `КлючВарианта` (Строка) — имя варианта отчёта (напр. `"Основной"`).
- `ФормаВладелец` (УправляемаяФорма) — форма-владелец (для модального открытия).
- `Вариант` (СправочникСсылка.ВариантыОтчетов / Структура) — ссылка варианта или
  структура-описание.
- `ДополнительныеПараметры` (Структура / Неопределено) — доп. параметры открытия.

**Пример:**
```bsl
&НаКлиенте
Процедура ОткрытьОтчет(Команда)
    Вариант = ПолучитьСсылкуВариантаНаСервере();
    Если Вариант <> Неопределено Тогда
        ВариантыОтчетовКлиент.ОткрытьФормуОтчета(ЭтотОбъект, Вариант);
    КонецЕсли;
КонецПроцедуры

&НаСервереБезКонтекста
Функция ПолучитьСсылкуВариантаНаСервере()
    Возврат ВариантыОтчетов.ВариантОтчета("Отчет.ЗависимостиПодсистем", "Основной");
КонецФункции
```

**Нюансы / антипаттерны:**
- ❌ `ОткрытьЗначение(СсылкаВарианта)` или `ПолучитьФорму(...).Открыть()` — обход
  подсистемы вариантов: ломает кэширование настроек, панель отчётов, доступность
  по правам. Только `ВариантыОтчетовКлиент.ОткрытьФормуОтчета`.
- `ВариантОтчета` возвращает `Неопределено`, если отчёт отсутствует или недоступен
  по правам — проверяйте перед открытием.
- Для внешнего (дополнительного) отчёта тот же `ОткрытьФормуОтчета` открывает форму
  дополнительного отчёта; ссылку на вариант даёт `ВариантОтчета` с `Отчет` типа
  `СправочникСсылка.ДополнительныеОтчетыИОбработки`.

### 6. Сформировать отчёт программно (тяжёлый — в фоне)

**Задача:** программно сформировать отчёт на СКД и получить табличный документ;
тяжёлое формирование не блокировать форму — вынести в фоновое задание.

**Функции:**
`ВариантыОтчетов.СформироватьОтчет(Знач Параметры, Знач ПроверятьЗаполнение, Знач ПолучатьФлажокПустой) Экспорт` — Функция → Структура (`ТабличныйДокумент`, `Успех`, `ТекстОшибки`, `Расшифровка`, `НастройкиКД`), регион `#Область СлужебныйПрограммныйИнтерфейс` (⚠️ служебный). Сервер.
`ВариантыОтчетов.ПараметрыФормированияОтчета() Экспорт` — Функция → Структура, `#Область СлужебныйПрограммныйИнтерфейс`. Сервер.
`ДлительныеОперации.ВыполнитьФункцию(Знач ПараметрыВыполнения, ИмяФункции, Знач Параметр1 = Неопределено, Знач Параметр2 = Неопределено, Знач Параметр3 = Неопределено, Знач Параметр4 = Неопределено, Знач Параметр5 = Неопределено, Знач Параметр6 = Неопределено, Знач Параметр7 = Неопределено) Экспорт` — Функция, `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры (СформироватьОтчет):**
- `Параметры` (Структура) — из `ПараметрыФормированияОтчета()`, заполненный:
  `Отчет` (Метаданные / строка), `КлючВарианта` (Строка), пользовательские настройки.
- `ПроверятьЗаполнение` (Булево) — `Истина` → проверять заполнение параметров.
- `ПолучатьФлажокПустой` (Булево) — `Истина` → вернуть флаг «отчёт пустой».

**Пример:**
```bsl
&НаСервере
Функция СформироватьОтчетСервер(ПараметрыОтчета)
    ПараметрыФормирования = ВариантыОтчетов.ПараметрыФормированияОтчета();
    ПараметрыФормирования.Отчет = Метаданные.Отчеты.ЗависимостиПодсистем;
    ПараметрыФормирования.КлючВарианта = "Основной";
    ЗаполнитьЗначенияСвойств(ПараметрыФормирования, ПараметрыОтчета);
    Возврат ВариантыОтчетов.СформироватьОтчет(ПараметрыФормирования, Истина, Истина);
КонецФункции

// Тяжёлый отчёт — в фоне (предпочтительный путь для нового кода):
&НаСервере
Функция ЗапуститьФормирование(ПараметрыОтчета)
    ПараметрыВыполнения = ДлительныеОперации.ПараметрыВыполненияФункции(УникальныйИдентификатор);
    Возврат ДлительныеОперации.ВыполнитьФункцию(
        ПараметрыВыполнения,
        "Отчет.ЗависимостиПодсистем.СформироватьТяжелыйОтчет",
        ПараметрыОтчета);
КонецФункции

&НаКлиенте
Процедура Сформировать(Команда)
    ДлительнаяОперация = ЗапуститьФормирование(ПараметрыОтчета);
    Оповещение = Новый ОписаниеОповещения("ПослеФормирования", ЭтотОбъект);
    ДлительныеОперацииКлиент.ОжидатьЗавершение(
        ДлительнаяОперация, Оповещение,
        ДлительныеОперацииКлиент.ПараметрыОжидания(ЭтотОбъект));
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ⚠️ `ВариантыОтчетов.СформироватьОтчет` — служебный (регион
  `СлужебныйПрограммныйИнтерфейс`), обратная совместимость не гарантируется. Для
  формирования отчёта **из его формы** есть стабильный
  `ОтчетыКлиент.СформироватьОтчет(ФормаОтчета, ОбработчикЗавершения = Неопределено)`
  (клиент, `ПрограммныйИнтерфейс`) — предпочтителен, когда отчёт уже открыт формой.
- ❌ `ДлительныеОперации.ВыполнитьВФоне(ИмяПроцедуры, ПараметрыПроцедуры, ПараметрыВыполнения)`
  в новом коде — не рекомендуется (метод стабильный, `ПрограммныйИнтерфейс`, но
  doc-комментарий БСП прямо советует `ВыполнитьФункцию`/`ВыполнитьПроцедуру`;
  требует упаковки параметров в `Структуру` и явного `АдресХранилища` в
  обработчике). Используйте `ВыполнитьФункцию` / `ВыполнитьПроцедуру` (до 7
  параметров напрямую, без обёртки). Подробности — в `longs-and-jobs.md`.
- ❌ Синхронный `СформироватьОтчет` из `&НаКлиенте`-обработчика блокирует форму —
  выносите тяжёлое формирование в `ВыполнитьФункцию` + `ОжидатьЗавершение`.

### 7. Зарегистрировать варианты отчётов в переопределяемом модуле

**Задача:** при внедрении БСП зарегистрировать варианты отчётов прикладной
конфигурации (вызвать `НастроитьОтчетВМодулеМенеджера` для каждого отчёта).

**Функция:**
`ВариантыОтчетов.НастроитьОтчетВМодулеМенеджера(Настройки, ОтчетМетаданные) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Настройки` (Структура) — настройки вариантов (передаются из хука
  `НастроитьВариантыОтчетов`).
- `ОтчетМетаданные` (ОбъектМетаданных) — метаданные отчёта, в модуле менеджера
  которого определена `НастроитьВариантыОтчета(Настройки, НастройкиОтчета)`.

**Пример:**
```bsl
// В своём ВариантыОтчетовПереопределяемый (хук — БСП вызывает, прикладной код реализует)
Процедура НастроитьВариантыОтчетов(Настройки) Экспорт
    ВариантыОтчетов.НастроитьОтчетВМодулеМенеджера(Настройки, Метаданные.Отчеты.ЗависимостиПодсистем);
    ВариантыОтчетов.НастроитьОтчетВМодулеМенеджера(Настройки, Метаданные.Отчеты.СтатистикаВыполненияОбработчиковОбновления);
КонецПроцедуры

// В модуле менеджера отчёта:
Процедура НастроитьВариантыОтчета(Настройки, НастройкиОтчета) Экспорт
    Вариант = НастройкиОтчета.Варианты.Добавить();
    Вариант.КлючВарианта = "Основной";
    Вариант.Представление = НСтр("ru = 'Основной'");
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- `ВариантыОтчетовПереопределяемый.НастроитьВариантыОтчетов(Настройки)` — **хук**
  переопределения (БСП вызывает, прикладной код реализует). Не вызывайте его из
  прикладного кода напрямую.
- ❌ Формировать варианты отчёта «в обход» справочника `ВариантыОтчетов` (своим
  регистром) — ломает панель отчётов, кэширование настроек, доступность по правам,
  пользовательские варианты.

## Дополнительно

Прочие стабильные методы (регион `ПрограммныйИнтерфейс`, если не указано иное):

- `УправлениеПечатью.КомандыПечатиФормы(Форма, СписокОбъектов = Неопределено) Экспорт`
  — Функция (сервер): собранный список команд печати для произвольной формы
  (журналы, общие формы).
- `УправлениеПечатью.СведенияОПечатнойФорме(КоллекцияПечатныхФорм, Идентификатор) Экспорт`
  — Функция (сервер): строка коллекции по `ИмяМакета`.
- `УправлениеПечатью.МакетыИДанныеОбъектовДляПечати(Знач ИмяМенеджераПечати, Знач ИменаМакетов, Знач СоставДокументов) Экспорт`
  — Функция (сервер): одним вызовом — двоичные данные макетов + данные объектов
  (для печати офисных документов с клиента).
- `УправлениеПечатью.СписокПечатныхФормИзВнешнихИсточников(ПолноеИмяОбъектаМетаданных) Экспорт`
  — Функция (сервер): печатные формы из внешних обработок-печатных форм.
- `ВариантыОтчетов.ОписаниеОтчета(Настройки, Отчет) Экспорт`,
  `ВариантыОтчетов.ОписаниеВарианта(Настройки, Отчет, КлючВарианта) Экспорт` — Функции
  (сервер): описание отчёта/варианта.
- `ВариантыОтчетов.КлючиВариантовОтчета(КлючОтчета, Знач Пользователь = Неопределено) Экспорт`
  — Функция (сервер): ключи вариантов отчёта пользователя.
- `ВариантыОтчетовКлиент.ОбновитьОткрытыеФормы(Знач КлючВарианта = "", Знач Источник = Неопределено) Экспорт`
  — Процедура (клиент): обновить открытые формы отчётов после изменения варианта.

Хуки переопределения (`УправлениеПечатьюПереопределяемый`, `ВариантыОтчетовПереопределяемый`,
регион `ПрограммныйИнтерфейс`) — БСП вызывает, прикладной код реализует:

- `УправлениеПечатьюПереопределяемый.ПриОпределенииНастроекПечати(Настройки)` — общие
  настройки подсистемы печати (`ИспользоватьПодписиИПечати`, список объектов с командами).
- `УправлениеПечатьюПереопределяемый.ПриПечати(МассивОбъектов, ПараметрыПечати, КоллекцияПечатныхФорм, ОбъектыПечати, ПараметрыВывода)`
  — постобработка печатных форм после менеджера печати.
- `УправлениеПечатьюПереопределяемый.ПриПолученииКомандПечати(Знач ПолноеИмяОбъектаМетаданных, КомандыПечати)`
  — модификация коллекции команд печати объекта.
- ⚠️ `УправлениеПечатьюПереопределяемый.ПриОпределенииОбъектовСКомандамиПечати(СписокОбъектов)`
  — **устаревший** (регион `УстаревшиеПроцедурыИФункции`); не использовать в новом
  коде, альтернатива — `ПриОпределенииНастроекПечати`.
- `ВариантыОтчетовПереопределяемый.НастроитьВариантыОтчетов(Настройки)` —
  регистрация вариантов (см. сценарий 7).
- `ВариантыОтчетовПереопределяемый.ОпределитьРазделыСВариантамиОтчетов(Разделы)` —
  состав разделов с вариантами отчётов.
- `ВариантыОтчетовПереопределяемый.ОпределитьОбъектыСКомандамиОтчетов(Объекты)` —
  объекты с командами отчётов в формах.
- `ВариантыОтчетовПереопределяемый.ПриОпределенииНастроек(Настройки)` — общие
  настройки подсистемы вариантов отчётов.
