# Обновление версии ИБ и конфигурации

Подсистемы **ОбновлениеВерсииИБ** (в блоке `БазоваяФункциональность`) и
**ОбновлениеКонфигурации**. Покрывает обработчики обновления (монопольно /
оперативно / отложенно), миграцию данных при смене версии конфигурации или
библиотеки, управление версиями ИБ через регистр `ВерсииПодсистем`, а также
интерактивную установку обновлений и патчей. Главная задача — научить
правильно писать код **внутри** обработчика обновления (обёртки записи),
регистрировать обработчики и не путать модули `*Переопределяемый` (хуки —
реализуются) с модулями, которые нужно **вызывать**.

## Модули

Семейство `ОбновлениеИнформационнойБазы*` (корень + суффикс контекста):

- `ОбновлениеИнформационнойБазы` — сервер, внешнее соединение, толстый клиент.
  **Стабильный API**: запуск обновления, версии ИБ, таблицы обработчиков,
  обёртки записи, логирование.
- `ОбновлениеИнформационнойБазыКлиент` — клиент: формы отложенных обработчиков,
  индикация хода, инициация интерактивного обновления.
- `ОбновлениеИнформационнойБазыГлобальный` — глобальный: ⚠️
  `ПроверитьСтатусОтложенногоОбновления` (регион
  `СлужебныеПроцедурыИФункции`, обратная совместимость не гарантируется),
  вызывается по имени без префикса.
- `ОбновлениеИнформационнойБазыВызовСервера` — серверный вызов с клиента без
  контекста формы.
- `ОбновлениеИнформационнойБазыПереопределяемый` — **хук**: серверные
  «крючки» прикладной конфигурации (`ПередОбновлениемИнформационнойБазы`,
  `ПослеОбновленияИнформационнойБазы`, `ПриОпределенииНастроек` и др.).
  БСП вызывает, прикладной код реализует.
- `ОбновлениеИнформационнойБазыКлиентПереопределяемый` — **хук**: клиентские
  `ПриОпределенииВозможностиОбновления`,
  `ПриНажатииНаГиперссылкуВДокументеОписанияОбновлений`.
- `ОбновлениеИнформационнойБазыСлужебный` — ⚠️ служебный API (регион
  `СлужебныйПрограммныйИнтерфейс`): внутренняя логика итераций, блокировка ИБ,
  фоновое обновление. Обратная совместимость не гарантируется.
- `ОбновлениеИнформационнойБазыСлужебныйВызовСервера` /
  `ОбновлениеИнформационнойБазыСлужебныйПовтИсп` — ⚠️ служебные, не вызывать.

Подсистема **ОбновлениеКонфигурации** (отдельная): интерактивная установка
обновлений и патчи. Модули `ОбновлениеКонфигурации` (сервер),
`ОбновлениеКонфигурацииКлиент`, `ОбновлениеКонфигурацииГлобальный`,
`ОбновлениеКонфигурацииВызовСервера`.

⚠️ **Модуль `ОбновлениеИнформационнойБазыСервер` НЕ существует** — серверный
модуль называется `ОбновлениеИнформационнойБазы` (без суффикса). Сравните: для
регламентных заданий, наоборот, серверный — `РегламентныеЗаданияСервер`
(без суффикса `РегламентныеЗадания` не существует). Перед вызовом
сверяйтесь с реальным каталогом общих модулей.

## Сценарии

### 1. Зарегистрировать обработчик обновления

**Задача:** описать обработчик миграции данных для конкретной версии и
зарегистрировать его в таблице обработчиков обновления.

**Функции:**
`ОбновлениеИнформационнойБазы.НоваяТаблицаОбработчиковОбновления() Экспорт` —
Функция (stable), возвращает `ТаблицаЗначений` с полным набором колонок для
всех режимов выполнения.

**Параметры (колонки таблицы обработчиков):**
- `Версия` (Строка) — номер версии, при переходе на которую выполняется
  обработчик, напр. `"2.4.1.5"`. `"*"` — обязательный обработчик (при каждом
  обновлении). Пустая строка — обработчик только для начального заполнения
  (тогда `НачальноеЗаполнение = Истина`).
- `Процедура` (Строка) — полное имя экспортной процедуры, напр.
  `"ОбновлениеИнформационнойБазыУПП.ЗаполнитьНовыйРеквизит"`.
- `РежимВыполнения` (Строка) — `"Монопольно"` (по умолчанию, тяжёлая миграция),
  `"Оперативно"` (без блокировки ИБ, лёгкая миграция), `"Отложенно"` (в фоне
  после основного цикла; требует `Идентификатор`, `БлокируемыеОбъекты`,
  `ПроцедураПроверки`).
- `НачальноеЗаполнение` (Булево) — `Истина`, обработчик срабатывает на «пустой» базе.
- `Идентификатор` (УникальныйИдентификатор) — для отложенного обработчика.
- `БлокируемыеОбъекты` (Строка) — для отложенного, напр. `"Справочник.Контрагенты"`.
- `ПроцедураПроверки` (Строка) — для отложенного, функция проверки обработанности.
- `Комментарий` (Строка) — описание.

**Пример:**
```bsl
// В прикладном модуле ОбновлениеИнформационнойБазыУПП:
Процедура ПриДобавленииОбработчиковОбновления(Обработчики) Экспорт
    Обработчик = Обработчики.Добавить();
    Обработчик.Версия          = "2.4.1.5";
    Обработчик.Процедура       = "ОбновлениеИнформационнойБазыУПП.ЗаполнитьНовыйРеквизит";
    Обработчик.РежимВыполнения = "Монопольно";

    Обработчик = Обработчики.Добавить();
    Обработчик.Версия          = "*";  // при каждом обновлении
    Обработчик.Процедура       = "ОбновлениеИнформационнойБазыУПП.ОбновитьКонтактнуюИнформацию";
    Обработчик.РежимВыполнения = "Отложенно";
    Обработчик.Идентификатор    = Новый УникальныйИдентификатор("a1b2c3d4-...");
    Обработчик.БлокируемыеОбъекты = "Справочник.Контрагенты";
    Обработчик.ПроцедураПроверки = "ОбновлениеИнформационнойБазыУПП.КонтрагентОбработан";
    Обработчик.Комментарий     = "Обновление КИ по новому формату";
КонецПроцедуры

Процедура ЗаполнитьНовыйРеквизит() Экспорт
    Запрос = Новый Запрос;
    Запрос.Текст = "ВЫБРАТЬ Ссылка ИЗ Справочник.Контрагенты ГДЕ НовыйРеквизит = """"";
    Выборка = Запрос.Выполнить().Выбрать();
    Пока Выборка.Следующий() Цикл
        Контрагент = Выборка.Ссылка.ПолучитьОбъект();
        Контрагент.НовыйРеквизит = "ЗначениеПоУмолчанию";
        ОбновлениеИнформационнойБазы.ЗаписатьДанные(Контрагент);
    КонецЦикла;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ⚠️ Регистрация обработчиков выполняется в `ПриДобавленииОбработчиковОбновления`
  **прикладного** модуля `ОбновлениеИнформационнойБазы<Префикс>` (напр.
  `…УПП`, `…БП`). Модуль `ОбновлениеИнформационнойБазыСлужебный` БСП содержит
  свой `ПриДобавленииОбработчиковОбновления` — это внутренний крючок БСП, не
  место для прикладных обработчиков.
- `Версия = "*"` — обязательный обработчик; выполняется при каждом обновлении,
  не зависит от предыдущей версии ИБ.
- Версия конфигурации хранится в `Метаданные.Версия` и регистре `ВерсииПодсистем`.

### 2. Безопасно записать данные из обработчика обновления

**Задача:** записать объект / набор записей / константу в обработчике
обновления без бизнес-логики и без регистрации на планах обмена (миграция не
должна разлетаться по узлам РИБ и не должна триггерить ещё не готовую
бизнес-логику).

**Функции:**
`ОбновлениеИнформационнойБазы.ЗаписатьДанные(Знач Данные, Знач РегистрироватьНаУзлахПлановОбмена = Неопределено, Знач ВключитьБизнесЛогику = Ложь) Экспорт`
— Процедура (stable). Запись объекта, набора записей или менеджера константы.
`ОбновлениеИнформационнойБазы.ЗаписатьОбъект(Знач Объект, Знач РегистрироватьНаУзлахПлановОбмена = Неопределено, Знач ВключитьБизнесЛогику = Ложь, ДокументРежимЗаписи = Неопределено) Экспорт`
— Процедура (stable). Запись объекта с возможностью проведения документа.
`ОбновлениеИнформационнойБазы.ЗаписатьНаборЗаписей(Знач НаборЗаписей, Замещать = Истина, Знач РегистрироватьНаУзлахПлановОбмена = Неопределено, Знач ВключитьБизнесЛогику = Ложь) Экспорт`
— Процедура (stable).
`ОбновлениеИнформационнойБазы.УдалитьДанные(Знач Данные, Знач РегистрироватьНаУзлахПлановОбмена = Неопределено, Знач ВключитьБизнесЛогику = Ложь) Экспорт`
— Процедура (stable).

**Параметры:**
- `Данные` / `Объект` / `НаборЗаписей` (Произвольный) — записываемые данные.
- `РегистрироватьНаУзлахПлановОбмена` (Булево / `Неопределено`) — `Неопределено`
  (по умолчанию) — стандартное поведение подсистемы обновления (обычно **не**
  регистрировать). `Истина` — принудительно регистрировать (миграция уходит в
  обмен).
- `ВключитьБизнесЛогику` (Булево) — `Ложь` (по умолчанию) отключает обработчики
  модуля объекта и подписки на события. `Истина` — включить (напр. для
  проведения документов, где миграция не нарушает логику).
- `ДокументРежимЗаписи` (РежимЗаписиДокумента) — для `ЗаписатьОбъект`;
  `Неопределено` — обычная запись, `РежимЗаписиДокумента.Проведение` —
  проведение.
- `Замещать` (Булево) — для `ЗаписатьНаборЗаписей`; `Истина` (по умолчанию) —
  замещение.

**Пример:**
```bsl
Процедура ОбновитьРеквизитДокумента() Экспорт
    Выборка = Документы.Заказ.Выбрать();
    Пока Выборка.Следующий() Цикл
        ДокументОбъект = Выборка.ПолучитьОбъект();
        ДокументОбъект.НовыйРеквизит = "...";
        // Бизнес-логика ОТКЛЮЧЕНА, регистрация на планах обмена НЕ выполняется
        ОбновлениеИнформационнойБазы.ЗаписатьДанные(ДокументОбъект);
    КонецЦикла;
КонецПроцедуры

// Запись с проведением (бизнес-логика включена)
ОбновлениеИнформационнойБазы.ЗаписатьОбъект(ДокументОбъект, , Истина, РежимЗаписиДокумента.Проведение);
```

**Нюансы / антипаттерны:**
- ❌ `Объект.Записать()` напрямую в обработчике — триггерит бизнес-логику и
  регистрацию обмена. Только обёртки `ОбновлениеИнформационнойБазы.Записать*`.
- ❌ Использовать `ЗаписатьДанные` в обычной прикладной логике вне обработчика
  обновления — обёртки предназначены только для кода миграции. Для
  пользовательской записи — `Объект.Записать()` со всей бизнес-логикой.

### 3. Проверить блокировку объекта в форме

**Задача:** заблокировать редактирование объекта в форме, пока отложенное
обновление не обработало его (стандартная проверка БСП).

**Функции:**
`ОбновлениеИнформационнойБазы.МетаданныеИОтборПоДанным(Данные, ДополнительныеПараметры = Неопределено) Экспорт`
— Функция (stable), нормализует данные для проверки блокировки.
`ОбновлениеИнформационнойБазы.ДанныеОбновленыНаНовуюВерсиюПрограммы(МетаданныеИОтбор) Экспорт`
— Функция (stable), возвращает `Булево`: `Истина` — объект обновлён и доступен
для изменения.

**Параметры:**
- `Данные` (СправочникОбъект / ДокументОбъект / … / ЛюбаяСсылка /
  ДанныеФормыСтруктура) — объект или ссылка для нормализации.
- `ДополнительныеПараметры` (Структура / `Неопределено`) — доп. параметры
  отбора.
- `МетаданныеИОтбор` (Структура) — результат `МетаданныеИОтборПоДанным`.

**Пример:**
```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    МетаданныеИОтбор = ОбновлениеИнформационнойБазы.МетаданныеИОтборПоДанным(Объект);
    Если Не ОбновлениеИнформационнойБазы.ДанныеОбновленыНаНовуюВерсиюПрограммы(МетаданныеИОтбор) Тогда
        Текст = НСтр("ru = 'Объект заблокирован для редактирования до завершения обновления.'");
        ОбщегоНазначения.СообщитьПользователю(Текст, , , , Отказ);
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- `ДанныеОбновленыНаНовуюВерсиюПрограммы` — стандартная функция проверки по
  умолчанию; заблокированные объекты зарегистрированы на узлах плана обмена
  `ОбновлениеИнформационнойБазы`.
- Для нестандартной проверки можно зарегистрировать собственную через хук
  `ПриВыполненииПроверкиОбъектОбработан` модуля
  `ОбновлениеИнформационнойБазыПереопределяемый`.

### 4. Программно запустить обновление (пакетный режим)

**Задача:** запустить неинтерактивное обновление из внешнего соединения или
обработки, предварительно очистив устаревшие патчи.

**Функции:**
`ОбновлениеКонфигурации.ИсправленияИзменены(ТолькоПроверка = Ложь) Экспорт` —
Функция (stable). Удаляет устаревшие патчи и применяет новые. Возвращает
`ЕстьИзменения` (Булево), `ОписаниеИзменений`.
`ОбновлениеИнформационнойБазы.ВыполнитьОбновлениеИнформационнойБазы(ВыполнятьОтложенныеОбработчики = Ложь) Экспорт`
— Функция (stable). Возвращает строку: `"Успешно"` / `"НеТребуется"` /
`"ОшибкаУстановкиМонопольногоРежима"`.

**Параметры:**
- `ТолькоПроверка` (Булево) — для `ИсправленияИзменены`: `Истина` только
  проверить, не применять.
- `ВыполнятьОтложенныеОбработчики` (Булево) — `Истина` — отложенное обновление
  выполняется в основном цикле (только клиент-серверный режим).

**Пример:**
```bsl
// В обработке, вызываемой из внешнего соединения
ОбновлениеКонфигурации.ИсправленияИзменены();  // удалить устаревшие патчи
Результат = ОбновлениеИнформационнойБазы.ВыполнитьОбновлениеИнформационнойБазы();
Если Результат = "ОшибкаУстановкиМонопольногоРежима" Тогда
    // повторить позже или сообщить администратору
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Запускать `ВыполнитьОбновлениеИнформационнойБазы` без `ИсправленияИзменены`
  — накопление «мёртвых» исправлений и ошибки совместимости. Сначала патчи.
- ❌ `ОбновлениеИнформационнойБазыВызовСервера.ВыполнитьОбновлениеИнформационнойБазы(Истина)`
  из клиентского кода при работающих пользователях — интерактивное обновление
  нужно через `ОбновлениеКонфигурацииКлиент.ПоказатьПоискИУстановкуОбновлений()`.
- При вызове с подключенными расширениями, модифицирующими роли, метод
  вызовет исключение.

### 5. Прочитать и записать версию ИБ

**Задача:** узнать текущую версию подсистемы/конфигурации, зафиксировать версию
без выполнения обработчиков (напр. для отмены штатного перехода с другой
программы), зарегистрировать новую подсистему.

**Функции:**
`ОбновлениеИнформационнойБазы.ВерсияИБ(Знач ИдентификаторБиблиотеки) Экспорт` —
Функция (stable), возвращает сохранённую версию (Строка).
`ОбновлениеИнформационнойБазы.УстановитьВерсиюИБ(Знач ИдентификаторБиблиотеки, Знач НомерВерсии, Знач ЭтоОсновнаяКонфигурация) Экспорт`
— Процедура (stable).
`ОбновлениеИнформационнойБазы.ВерсииПодсистем() Экспорт` — Функция (stable),
возвращает таблицу версий.
`ОбновлениеИнформационнойБазы.УстановитьВерсииПодсистем(ВерсииПодсистем) Экспорт`
— Процедура (stable).
`ОбновлениеИнформационнойБазы.ЗарегистрироватьНовуюПодсистему(ИмяПодсистемы, НомерВерсии = "") Экспорт`
— Процедура (stable). Регистрирует новую подсистему **без** выполнения
обработчиков начального заполнения. Вызывать из `ПередОбновлениемИнформационнойБазы`.

**Параметры:**
- `ИдентификаторБиблиотеки` (Строка) — имя конфигурации или библиотеки; для
  основной конфигурации — `Метаданные.Имя`.
- `НомерВерсии` (Строка) — номер версии, напр. `Метаданные.Версия`.
- `ЭтоОсновнаяКонфигурация` (Булево) — `Истина` для основной конфигурации,
  `Ложь` для библиотеки. Обязательный.

**Пример:**
```bsl
// Прочитать текущую версию основной конфигурации
ТекущаяВерсия = ОбновлениеИнформационнойБазы.ВерсияИБ(Метаданные.Имя);

// Записать версию без выполнения обработчиков (отмена штатного обновления)
ОбновлениеИнформационнойБазы.УстановитьВерсиюИБ(Метаданные.Имя, Метаданные.Версия, Истина);
```

**Нюансы / антипаттерны:**
- ❌ `УстановитьВерсиюИБ(Метаданные.Имя, "1.0.0.0")` без третьего параметра —
  версия запишется в библиотечную ветку. Всегда передавайте
  `ЭтоОсновнаяКонфигурация` (`Истина` для основной конфигурации).
- `ЗарегистрироватьНовуюПодсистему` не выполняет обработчики — используется,
  чтобы пометить подсистему как уже актуальной версии (напр. при переходе с
  другой программы, когда миграция не нужна).

### 6. Проверить необходимость и ход обновления

**Задача:** из прикладного кода узнать, требуется ли обновление, выполняется
ли оно сейчас, и завершено ли отложенное обновление (чтобы безопасно отказать
в записи или показать предупреждение).

**Функции:**
`ОбновлениеИнформационнойБазы.НеобходимоОбновлениеИнформационнойБазы() Экспорт` —
Функция (stable), `Булево`.
`ОбновлениеИнформационнойБазы.ВыполняетсяОбновлениеИнформационнойБазы() Экспорт`
— Функция (stable), `Булево`.
`ОбновлениеИнформационнойБазы.ОтложенноеОбновлениеЗавершено(Знач ИменаПодсистем = Неопределено) Экспорт`
— Функция (stable), `Булево`.
`ОбновлениеИнформационнойБазы.ПерезапуститьОтложенноеОбновление(Отбор = Неопределено) Экспорт`
— Процедура (stable).

**Параметры:**
- `ИменаПодсистем` (Строка / Массив / `Неопределено`) — имена подсистем для
  проверки отложенного обновления; `Неопределено` — все подсистемы.
- `Отбор` (Структура / `Неопределено`) — для `ПерезапуститьОтложенноеОбновление`,
  напр. `Новый Структура("ИмяОбработчика", "МойОбработчик")`.

**Пример:**
```bsl
// Отказать в записи, если обновление выполняется
Если ОбновлениеИнформационнойБазы.ВыполняетсяОбновлениеИнформационнойБазы() Тогда
    Отказ = Истина;
    ОбщегоНазначения.СообщитьПользователю(НСтр("ru = 'Запись невозможна: выполняется обновление.'"));
КонецЕсли;

// Перезапустить ранее отложенный обработчик после исправления данных
ОбновлениеИнформационнойБазы.ПерезапуститьОтложенноеОбновление(
    Новый Структура("ИмяОбработчика", "МойОбработчик"));
```

**Нюансы / антипаттерны:**
- `НеобходимоОбновлениеИнформационнойБазы` сравнивает `Метаданные.Версия` с
  версией в ИБ; `Истина` — есть обработчики с версией больше сохранённой.
- `ОтложенноеОбновлениеЗавершено` удобно для блокировки тяжёлых операций,
  зависящих от полной миграции данных.

### 7. Реализовать хуки переопределения обновления

**Задача:** в прикладной конфигурации реализовать хуки БСП: действия перед
обновлением, после обновления, переопределение настроек и проверок.

**Функции (хуки, модуль `*Переопределяемый`):**
`ОбновлениеИнформационнойБазыПереопределяемый.ПередОбновлениемИнформационнойБазы() Экспорт`
— Процедура (хук), сервер. Вызывается **перед** запуском обработчиков.
`ОбновлениеИнформационнойБазыПереопределяемый.ПослеОбновленияИнформационнойБазы(Знач ПредыдущаяВерсияИБ, Знач ТекущаяВерсияИБ, Знач ИтерацииОбновления, ВыводитьОписаниеОбновлений, Знач МонопольныйРежим) Экспорт`
— Процедура (хук), сервер. Вызывается **после** завершения обновления.
`ОбновлениеИнформационнойБазыПереопределяемый.ПриОпределенииНастроек(Параметры) Экспорт`
— Процедура (хук), переопределение общих настроек подсистемы обновления.
`ОбновлениеИнформационнойБазыПереопределяемый.ПриВыполненииПроверкиОбъектОбработан(ПолноеИмяОбъекта, БлокироватьИзменение, ТекстСообщения) Экспорт`
— Процедура (хук), вызывается при проверке «обработан ли объект» при
обновлении; прикладной код может задать `БлокироватьИзменение = Истина` и
текст сообщения, чтобы заблокировать изменение объекта до отработки
обработчика обновления.
`ОбновлениеИнформационнойБазыКлиентПереопределяемый.ПриОпределенииВозможностиОбновления(Знач ВерсияДанных) Экспорт`
— Процедура (хук), клиент. Проверка возможности обновления в
`ПередНачаломРаботыСистемы`.

**Параметры:**
- `ПредыдущаяВерсияИБ` / `ТекущаяВерсияИБ` (Строка) — версии до/после.
- `ИтерацииОбновления` (Структура) — данные об итерациях.
- `ВыводитьОписаниеОбновлений` (Булево) — выходной: `Ложь` отключает показ формы
  описания изменений.
- `МонопольныйРежим` (Булево) — признак монопольного режима.
- `Параметры` (Структура) — настройки подсистемы обновления.
- `ГруппыПроверок` / `Проверки` (Массив) — группы и проверки блокировки.
- `ВерсияДанных` (Строка) — версия данных ИБ для клиентской проверки.

**Пример:**
```bsl
// В прикладном модуле ОбновлениеИнформационнойБазыПереопределяемый:
Процедура ПередОбновлениемИнформационнойБазы() Экспорт
    ВерсииПодсистем = ОбновлениеИнформационнойБазы.ВерсииПодсистем();
    Если ВерсииПодсистем.Количество() > 0
        И ВерсииПодсистем.Найти(Метаданные.Имя, "ИмяПодсистемы") = Неопределено Тогда
        // Отмена штатного перехода с другой программы — регистрируем как актуальную
        ОбновлениеИнформационнойБазы.ЗарегистрироватьНовуюПодсистему(Метаданные.Имя, Метаданные.Версия);
    КонецЕсли;
КонецПроцедуры

Процедура ПослеОбновленияИнформационнойБазы(Знач ПредыдущаяВерсияИБ, Знач ТекущаяВерсияИБ,
    Знач ИтерацииОбновления, ВыводитьОписаниеОбновлений, Знач МонопольныйРежим) Экспорт
    ВыводитьОписаниеОбновлений = Ложь; // отключить форму описания изменений
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `ОбновлениеИнформационнойБазыПереопределяемый.ПередОбновлениемИнформационнойБазы()`
  из прикладного кода — модуль `*Переопределяемый` только реализуется, БСП
  вызывает хуки сама. Для фиксации версии до обновления используйте прямой API
  `УстановитьВерсиюИБ`.
- ❌ Использовать `ОбновлениеИнформационнойБазыСлужебный` для прикладных задач —
  служебный, обратная совместимость не гарантируется. Стабильные аналоги —
  `ВерсияИБ`, `УстановитьВерсиюИБ`, `ВерсииПодсистем`.

## Редкие методы

Методы, встречающиеся реже (из компаньона `*-key-methods.md`), без полного
сценария:

- `ОбновлениеИнформационнойБазы.НоваяТаблицаОбработчиковОбновления()` — см.
  сценарий 1; возвращает `ТаблицаЗначений` с полным набором колонок для всех
  режимов (`Монопольно` / `Оперативно` / `Отложенно` / `Параллельно`).
- `ОбновлениеИнформационнойБазы.ДанныеОбновленыНаНовуюВерсиюПрограммы(МетаданныеИОтбор)`
  — см. сценарий 3; стандартная функция проверки блокировки для
  `ПриСозданииНаСервере` формы объекта.
- `ОбновлениеИнформационнойБазы.ЗаписатьОшибкуВЖурналРегистрации(СсылкаМетаданные, Знач Представление, ИнформацияОбОшибке = Неопределено, Уровень = Неопределено)`
— Процедура (stable), запись ошибки в журнал с привязкой к событию
  обновления. Для информационных событий — `ЗаписатьСобытиеВЖурналРегистрации`
  (возвращает событие журнала через `СобытиеЖурналаРегистрации()`).
- `ОбновлениеИнформационнойБазы.ЭтоВызовИзОбработчикаОбновления(РежимВыполненияОбработчика = "")`
— Функция (stable), `Булево` — проверить, что код выполняется в контексте
  обработчика обновления (чтобы отключать лишнюю логику).
- `ОбновлениеИнформационнойБазы.ОбработчикиОбновления(Отбор = Неопределено)` /
  `ОбновляемыеОбъекты()` — чтение зарегистрированных обработчиков и
  обновляемых объектов отложенного обновления.
- `ОбновлениеКонфигурацииКлиент.ПоддерживаетсяУстановкаОбновлений()` — Функция
  (stable), клиент. Возвращает структуру с `Поддерживается` — можно ли
  интерактивно установить обновление (ОС Windows + Конфигуратор + права
  администратора; не модель сервиса).
- `ОбновлениеКонфигурацииКлиент.ПоказатьПоискИУстановкуОбновлений(ПараметрыУстановкиОбновлений = Неопределено)`
— Процедура (stable), клиент. Открывает форму интерактивной установки
  обновления (с бэкапом и блокировками).
- ⚠️ `ОбновлениеИнформационнойБазыСлужебный.ПараметрыОбновления()` — Функция,
  регион `СлужебныеПроцедурыИФункции` (⚠️ служебный). Внутренняя структура
  параметров обновления; только для отладки и редких системных сценариев, не
  для прикладного кода — сломается при обновлении БСП.
