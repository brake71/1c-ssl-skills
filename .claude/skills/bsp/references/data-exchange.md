# Обмен данными БСП: планы обмена, регистрация изменений, синхронизация

Подсистема **ОбменДанными** (с вложенными `ОбменДанными.Регистрация`,
`ОбменДанными.ТранспортСообщенийОбмена`) + **СинхронизацияДанныхЧерезУниверсальныйФормат**
(EnterpriseData) + **ОбменДаннымиВМоделиСервиса** (SaaS). Покрывает регистрацию
изменений, узлы планов обмена, разрешение коллизий, запуск обмена, обмен между
областями данных.

> ⚠️ **Главная особенность.** В отличие от `Печать` или `ВариантыОтчетов`, у
> подсистем обмена **нет «толстого» стабильного API для прямого вызова**.
> Основной путь — **платформенный** `ПланыОбмена[ИмяПлана].*` + **переопределение**
> модулей `*Переопределяемый`. Стабильный серверный API БСП (`ОбменДаннымиСервер`)
> в основном вспомогательный (узлы, версия, состояние); основную логику обмена
> реализует платформа по зарегистрированным изменениям. Переопределяйте, не
> вызывайте служебное.

## Модули

Семейство `ОбменДанными*` по суффиксной системе БСП:

- `ОбменДаннымиСервер` — сервер, основной стабильный API
  (`УзлыОбменаБСП`, `УзелПланаОбменаПоКоду`, `НастройкаСинхронизацииЗавершена`,
  `ЗавершитьНастройкуСинхронизацииДанных`, `ПрефиксИнформационнойБазы`,
  `ВерсияКорреспондента`, `ЭтоАвтономноеРабочееМесто`). Остальные ~290
  экспортов — `СлужебныйПрограммныйИнтерфейс` / `СлужебныеПроцедурыИФункции` (⚠️).
- `ОбменДаннымиВызовСервера` — серверный вызов с клиента без контекста формы.
  Стабильный: `СброситьКэшМеханизмаРегистрацииОбъектов`. Тяжёлый
  `ВыполнитьОбменДаннымиПоСценариюОбменаДанными` — ⚠️ `СлужебныеПроцедурыИФункции`.
- `ОбменДаннымиКлиент` — клиентский UI обменов
  (`ОткрытьПанельСинхронизацииДанных`, `ОткрытьСоставОтправляемыхДанных`,
  `УдалитьНастройкуСинхронизации`).
- `ОбменДаннымиПереопределяемый` — **хуки** (реализуйте, не вызывайте):
  `ПолучитьПланыОбмена`, `ПриКоллизииИзмененийДанных`, `ПриВыгрузкеДанных`,
  `ПриЗагрузкеДанных`, `ПриПолученииДоступныхВерсийФормата`.
- `ОбменДаннымиСобытия` — **событийные обработчики**, БСП подключает их через
  подписки на события (`МеханизмРегистрацииОбъектовПередЗаписьюДокумента` и т. п.).
  **Не вызывайте** — настройте правила регистрации объекта в плане обмена.
- `ОбменДаннымиXDTOСервер` — ⚠️ служебный, но огромный: правила конвертации
  EnterpriseData. Входная точка для модулей менеджера обмена через универсальный
  формат.
- `ТрансляцияXDTOПереопределяемый` — **хук**: `ЗаполнитьОбработчикиТрансляцииСообщений`.
- `ОбменДаннымиВМоделиСервиса` — стабильный API для SaaS (обмен между областями
  данных): `ПриОтключенииСинхронизацииДанных`, `ИзменитьПризнакНеобходимостиОбменаДаннымиВМоделиСервиса`.
- ⚠️ `ОбменДанными` (без суффикса) **не существует** как общий модуль — типовая
  ошибка: `ОбменДанными.ЧтоТо(...)`. Реальные модули — с суффиксами выше.
- ⚠️ `ОбменДаннымиПовтИсп` содержит **только deprecated** методы
  (`ЭтоАвтономноеРабочееМесто`, `НайтиУзелПланаОбменаПоКоду`) — заменены на
  методы `ОбменДаннымиСервер.*` того же имени.

## Сценарии

### 1. Зарегистрировать / снять изменения объекта для узла

**Задача:** программно зарегистрировать изменения объекта (или всех данных) для
узла плана обмена, либо снять регистрацию.

**Функции:**
`ПланыОбмена[ИмяПлана].ЗарегистрироватьИзменения(Узел, Данные)` — платформенный
метод менеджера плана обмена (БСП работает поверх него, не оборачивает).
`ПланыОбмена[ИмяПлана].УдалитьРегистрациюИзменений(Узел, Данные = Неопределено)` —
снять регистрацию конкретных данных либо всей выборки узла.
`ПланыОбмена[ИмяПлана].ВыбратьИзменения(Узел, НомерСообщения, ФильтрВыборки = Неопределено)` —
выборка зарегистрированных изменений для отправки.
Сервер, Внешнее соединение. Регион: платформенный API (метка не нужна).

**Параметры:**
- `Узел` (`ПланОбменаСсылка`) — узел-получатель, для которого регистрируются
  изменения.
- `Данные` (Произвольный) — ссылка/объект/ключ записи ИБ. Для
  `УдалитьРегистрациюИзменений` `Неопределено` = снять всю регистрацию узла.
- `НомерСообщения` (Число) — номер сообщения обмена, начиная с которого
  выбирать изменения (обычно `Узел.НомерОтправлено + 1`).
- `ФильтрВыборки` (Произвольный) — платформенный фильтр выборки изменений.

**Пример:**
```bsl
// Регистрация изменения документа для узла (сервер)
ИмяПлана = ОбменДаннымиСервер.УзлыОбменаБСП()[0].ИмяПланаОбмена;  // колонка таблицы
ПланыОбмена[ИмяПлана].ЗарегистрироватьИзменения(УзелОбмена, ДокументОбъект);

// Снять регистрацию конкретного объекта
ПланыОбмена[ИмяПлана].УдалитьРегистрациюИзменений(УзелОбмена, ДокументОбъект);

// Снять всю регистрацию узла (следующая выгрузка начнёт «с нуля»)
ПланыОбмена[ИмяПлана].УдалитьРегистрациюИзменений(УзелОбмена);

// Выборка изменений для отправки сообщения
Выборка = ПланыОбмена[ИмяПлана].ВыбратьИзменения(УзелОбмена, УзелОбмена.НомерОтправлено + 1);
Пока Выборка.Следующий() Цикл
    // Выборка.ПолучитьОбъект(), Выборка.Метаданные() и т. п.
КонецЦикла;
```

**Нюансы / антипаттерны:**
- ❌ Записывать в таблицы `РегистрацияИзмененийПланаОбмена` вручную через
  `НаборЗаписей` — обходит платформенный механизм, ломает авторегистрацию и
  приводит к расхождениям. Только `ПланыОбмена[ИмяПлана].ЗарегистрироватьИзменения`.
- ❌ Искать метод `ОбменДанными.ЗарегистрироватьИзменения` — такого модуля/метода
  БСП не предоставляет. Регистрация — платформенный API.
- Авторегистрация изменений по подпискам на события делает `ОбменДаннымиСобытия`
  автоматически; ручная регистрация нужна для нестандартных случаев (напр.
  принудительная перерегистрация после исправления данных).

### 2. Подключить свой план обмена к подсистеме

**Задача:** зарегистрировать прикладной план обмена в подсистеме обмена БСП,
чтобы он попал в панель синхронизации, правила регистрации и сценарии.

**Функция:**
`ОбменДаннымиПереопределяемый.ПолучитьПланыОбмена(ПланыОбменаПодсистемы) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` в модуле `*Переопределяемый`
(хук). БСП вызывает, прикладной код **реализует** тело; не вызывается напрямую.
Сервер, Внешнее соединение.

**Параметры:**
- `ПланыОбменаПодсистемы` (Массив) — массив метаданных планов обмена, входящих в
  подсистему. В теле хука добавляйте свои.

**Пример:**
```bsl
// В общем модуле ОбменДаннымиПереопределяемый (копируется в конфигурацию)
Процедура ПолучитьПланыОбмена(ПланыОбменаПодсистемы) Экспорт
    ПланыОбменаПодсистемы.Добавить(Метаданные.ПланыОбмена.МойПланОбмена);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `ОбменДаннымиПереопределяемый.ПолучитьПланыОбмена(Массив)` из
  прикладного кода «чтобы получить список» — метод переопределяемый, тело пустое
  по умолчанию, ваш вызов ничего не вернёт. Список узлов — через
  `ОбменДаннымиСервер.УзлыОбменаБСП()`.
- План обмена должен быть корректно описан в конфигурации (состав,
  реквизиты узла) — БСП проверяет метаданные, а не состояние ИБ.

### 3. Разрешить коллизии изменений при обмене

**Задача:** определить, чья версия объекта побеждает при коллизии (главный vs
подчинённый узел РИБ), с учётом типа объекта.

**Функция:**
`ОбменДаннымиПереопределяемый.ПриКоллизииИзмененийДанных(Знач ЭлементДанных, ПолучениеЭлемента, Знач Отправитель, Знач ПолучениеОтГлавного) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` в модуле `*Переопределяемый`
(хук). БСП вызывает при коллизии; прикладной код реализует логику.
Сервер, Внешнее соединение.

**Параметры:**
- `ЭлементДанных` (Произвольный) — объект/запись, по которому коллизия.
- `ПолучениеЭлемента` (`ПолучениеЭлементаДанных`) — **выходной**: как поступить с
  входящим элементом. Установите в теле: `Принять` или `Игнорировать`.
- `Отправитель` (`ПланОбменаСсылка`) — узел-отправитель.
- `ПолучениеОтГлавного` (Булево) — `Истина`, если входящее пришло от главного узла.

**Пример:**
```bsl
// В ОбменДаннымиПереопределяемый
Процедура ПриКоллизииИзмененийДанных(Знач ЭлементДанных, ПолучениеЭлемента, Знач Отправитель, Знач ПолучениеОтГлавного) Экспорт
    Если ТипЗнч(ЭлементДанных) = Тип("СправочникОбъект.Контрагенты") Тогда
        // В распределёнке подчинённый узел приоритетнее своих данных
        ПолучениеЭлемента = ?(ПолучениеОтГлавного,
            ПолучениеЭлементаДанных.Игнорировать,
            ПолучениеЭлементаДанных.Принять);
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `ПриКоллизииИзмененийДанных(...)` из прикладного кода — метод
  переопределяемый, БСП сам зовёт его при разборе сообщения. Реализуйте логику
  **в теле**, не вызывайте.
- `ПолучениеЭлемента` — это выходной параметр (по ссылке), его надо
  **присваивать**, а не возвращать. Значение по умолчанию БСП устанавливает до
  вызова хука.

### 4. Найти узел, проверить состояние и завершить настройку синхронизации

**Задача:** по коду найти узел плана обмена; проверить, завершена ли настройка
синхронизации; завершить создание начального образа узла РИБ.

**Функции:**
`ОбменДаннымиСервер.УзлыОбменаБСП() Экспорт` — Функция → `ТаблицаЗначений`
(колонки `УзелИнформационнойБазы` — `ПланОбменаСсылка`, `Наименование` — `Строка`,
`ИмяПланаОбмена` — `Строка`), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ОбменДаннымиСервер.УзелПланаОбменаПоКоду(ИмяПланаОбмена, КодУзла) Экспорт` —
Функция → `ПланОбменаСсылка` / `Неопределено`, регион `ПрограммныйИнтерфейс`
(stable). Сервер.
`ОбменДаннымиСервер.НастройкаСинхронизацииЗавершена(УзелОбмена) Экспорт` —
Функция → `Булево`, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ОбменДаннымиСервер.ЗавершитьНастройкуСинхронизацииДанных(УзелОбмена) Экспорт` —
Процедура, регион `ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ИмяПланаОбмена` (Строка) — имя плана обмена, как в конфигураторе.
- `КодУзла` (Строка) — код узла; `УзелПланаОбменаПоКоду` внутри зовёт
  `ПланыОбмена[ИмяПлана].НайтиПоКоду(КодУзла)`.
- `УзелОбмена` (`ПланОбменаСсылка`) — узел обмена.

**Пример:**
```bsl
// Все узлы обменов, зарегистрированные в подсистеме
Для Каждого Узел Из ОбменДаннымиСервер.УзлыОбменаБСП() Цикл
    // ...
КонецЦикла;

// Найти узел по коду в конкретном плане
Узел = ОбменДаннымиСервер.УзелПланаОбменаПоКоду("МойПланОбмена", "001");
Если Узел <> Неопределено И ОбменДаннымиСервер.НастройкаСинхронизацииЗавершена(Узел) Тогда
    // синхронизация готова к работе
КонецЕсли;

// После создания начального образа подчинённого узла РИБ
ОбменДаннымиСервер.ЗавершитьНастройкуСинхронизацииДанных(Узел);
```

**Нюансы / антипаттерны:**
- ❌ `ОбменДаннымиПовтИсп.НайтиУзелПланаОбменаПоКоду(...)` — метод **deprecated**,
  помечен `// Устарела. Следует использовать ОбменДаннымиСервер.УзелПланаОбменаПоКоду`.
  Используйте `ОбменДаннымиСервер`.
- `УзелПланаОбменаПоКоду` возвращает `Неопределено` (а не пустую ссылку), если узел
  не найден — проверяйте `<> Неопределено`, а не проверку пустой ссылки.

### 5. Узнать параметры ИБ, признак АРМ РИБ и версию корреспондента

**Задача:** получить префикс ИБ, признак автономного рабочего места, версию
формата корреспондента — для ветвления логики обмена.

**Функции:**
`ОбменДаннымиСервер.ПрефиксИнформационнойБазы() Экспорт` — Функция → Строка,
регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ОбменДаннымиСервер.УстановитьПрефиксИнформационнойБазы(Знач Префикс) Экспорт` —
Процедура, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ОбменДаннымиСервер.ЭтоАвтономноеРабочееМесто() Экспорт` — Функция → `Булево`,
регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ОбменДаннымиСервер.ВерсияКорреспондента(Знач Корреспондент) Экспорт` —
Функция → Строка (напр. `"1.7"`), регион `ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Префикс` (Строка) — новый префикс ИБ (для `УстановитьПрефиксИнформационнойБазы`).
- `Корреспондент` (`ПланОбменаСсылка`) — узел-корреспондент.

**Пример:**
```bsl
Если ОбменДаннымиСервер.ЭтоАвтономноеРабочееМесто() Тогда
    // АРМ РИБ — ограниченная логика, нет прямого обмена с главным
    Возврат;
КонецЕсли;

Префикс = ОбменДаннымиСервер.ПрефиксИнформационнойБазы();
Версия = ОбменДаннымиСервер.ВерсияКорреспондента(УзелКорреспондента);
Если Версия >= "1.7" Тогда
    // используем расширенный состав EnterpriseData 1.7
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ `ОбменДаннымиПовтИсп.ЭтоАвтономноеРабочееМесто()` — **deprecated**, заменён на
  `ОбменДаннымиСервер.ЭтоАвтономноеРабочееМесто()`.
- `ВерсияКорреспондента` возвращает версию формата **корреспондента** (с которым
  обмениваемся), а не собственную версию ИБ.

### 6. Запустить обмен в фоне и показать прогресс

**Задача:** запустить тяжёлый сценарий обмена (выгрузка/загрузка) не блокируя UI,
с показом прогресса и прерыванием.

**Функции:**
`ОбменДаннымиВызовСервера.ВыполнитьОбменДаннымиПоСценариюОбменаДанными(Отказ, НастройкаВыполненияОбмена, НомерСтроки = Неопределено) Экспорт`
— Процедура, регион `#Область СлужебныеПроцедурыИФункции` (⚠️ служебный).
Сервер (вызов с клиента без контекста формы).
`ОбменДаннымиВызовСервера.СостояниеЗадания(Знач ИдентификаторЗадания) Экспорт` —
Функция, регион `СлужебныйПрограммныйИнтерфейс` (⚠️). Сервер.
Для запуска в фоне — подсистема длительных операций: на сервере
`ДлительныеОперации.ПараметрыВыполненияПроцедуры()` (конструктор) +
`ДлительныеОперации.ВыполнитьПроцедуру(ПараметрыВыполнения, ИмяПроцедуры, Параметр1, …)`
(возвращает структуру длительной операции), на клиенте —
`ДлительныеОперацииКлиент.ОжидатьЗавершение(ДлительнаяОперация, ОписаниеОповещения)`.

**Параметры:**
- `Отказ` (Булево) — выходной: `Истина` прерывает обмен.
- `НастройкаВыполненияОбмена` (Произвольный) — параметры сценария обмена
  (узел, действие, расписание).
- `НомерСтроки` (Число / `Неопределено`) — номер строки сценария в списке
  настроек обмена.
- `ИдентификаторЗадания` (УникальныйИдентификатор) — идентификатор фонового
  задания для опроса состояния.

**Пример:**
```bsl
// ❌ НЕЛЬЗЯ — синхронный вызов с формы зависает
// ОбменДаннымиВызовСервера.ВыполнитьОбменДаннымиПоСценариюОбменаДанными(Отказ, Настройка);

// ✅ Запуск через длительную операцию: серверная часть запускает фоновое задание,
// клиентская — ожидает результат через оповещение

    Возврат ДлительныеОперации.ВыполнитьПроцедуру(
        ПараметрыВыполнения,
        "ОбменДаннымиВызовСервера.ВыполнитьОбменДаннымиПоСценариюОбменаДанными",
        Ложь,       // Отказ
        Настройка,  // НастройкаВыполненияОбмена
        1);         // НомерСтроки
КонецФункции

&НаКлиенте

    ДлительнаяОперация = ЗапуститьОбменНаСервере(НастройкаВыполненияОбмена);
    ДлительныеОперацииКлиент.ОжидатьЗавершение(
        ДлительнаяОперация,
        Новый ОписаниеОповещения("ОбменЗавершен", ЭтотОбъект));
КонецПроцедуры

&НаКлиенте
Процедура ОбменЗавершен(Результат, ДопПараметры) Экспорт
    // Результат.Статус = "Выполнено" / "Ошибка" / "Отменено"
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Дёргать `ВыполнитьОбменДаннымиПоСценариюОбменаДанными` синхронно с
  управляемой формы — UI зависает, пользователь не может прервать. Только через
  `ДлительныеОперации.ВыполнитьПроцедуру` + `ОжидатьЗавершение`.
- ⚠️ Метод в `СлужебныеПроцедурыИФункции` — обратная совместимость не
  гарантируется. Зафиксируйте зависимость комментарием с версией БСП, если
  используете напрямую.

### 7. Обмен между областями данных в модели сервиса (SaaS)

**Задача:** в модели сервиса (разделение данных по областям) управлять признаком
необходимости обмена и реагировать на отключение синхронизации.

**Функции:**
`ОбменДаннымиВМоделиСервиса.ИзменитьПризнакНеобходимостиОбменаДаннымиВМоделиСервиса(НеобходимоВыполнитьОбмен, ДополнительныеПараметры = Неопределено) Экспорт`
— Процедура, регион `#Область СлужебныйПрограммныйИнтерфейс` (⚠️ служебный).
Сервер.
`ОбменДаннымиВМоделиСервиса.ПриОтключенииСинхронизацииДанных(Отказ) Экспорт` —
Процедура, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ОбменДаннымиВМоделиСервиса.ПередЗаписьюОбщихДанных(Объект, Отказ) Экспорт` — Процедура,
регион `ПрограммныйИнтерфейс` (stable). Сервер.

**Пример:**
```bsl
// При изменении состава областей данных — пересчитать признак обмена
ОбменДаннымиВМоделиСервиса.ИзменитьПризнакНеобходимостиОбменаДаннымиВМоделиСервиса(Истина);

// Перед записью общих (неразделённых) данных
Отказ = Ложь;
ОбменДаннымиВМоделиСервиса.ПередЗаписьюОбщихДанных(Объект, Отказ);
```

**Нюансы / антипаттерны:**
- В модели сервиса обмен идёт **между областями данных** через общие узлы;
  обычная регистрация изменений `ПланыОбмена[Имя].ЗарегистрироватьИзменения`
  работает, но с учётом разделителя области данных — проверяйте, что объект
  принадлежит нужной области.
- `ОбменДаннымиВМоделиСервисаПереопределяемый` — хуки SaaS-специфики (реализуйте,
  не вызывайте).

## Редкие методы

Стабильные методы `ОбменДаннымиСервер` (регион `ПрограммныйИнтерфейс`), полные
сигнатуры — через `python scripts/bsp_api.py module ОбменДаннымиСервер --src src/cf`:

- `ЗавершитьСозданиеНачальногоОбраза(УзелОбмена)` — установить признак создания
  начального образа подчинённого узла РИБ.
- `ЭтоУзелАвтономногоРабочегоМеста(Знач УзелИнформационнойБазы)` — `Булево`,
  проверка, что узел является узлом АРМ.
- `ЕстьПраваНаАдминистрированиеОбменов()` — `Булево`, проверка прав на
  администрирование обменов.
- `ПрофильДоступаСинхронизацияДанныхСДругимиПрограммами()` — профиль доступа
  для роли синхронизации.
- `ИспользуютсяТиповыеПравила(ИмяПланаОбмена)` — `Булево`, используются ли
  поставляемые типовые правила конвертации.
- `ПолучитьWSПрокси(СтруктураНастроек, СтрокаСообщенияОбОшибке = "",
  СообщениеПользователю = "")` — `WSПрокси`, веб-сервис корреспондента. Есть
  версии `ПолучитьWSПрокси_2_1_1_7` / `_3_0_1_1` / `_3_0_2_2` под версии формата.
- `КоличествоЭлементовВТранзакцииЗагрузкиДанных()` /
  `КоличествоЭлементовВТранзакцииВыгрузкиДанных()` — настройки пакета обмена.

Хуки `ОбменДаннымиПереопределяемый` (реализуйте, не вызывайте):
- `ПриВыгрузкеДанных(СтандартнаяОбработка, Получатель, ИмяФайлаСообщения,
  ДанныеСообщения, КоличествоЭлементовВТранзакции, ИмяСобытияЖурналаРегистрации,
  КоличествоОтправленныхОбъектов)` — обработать выгрузку сообщения.
- `ПриЗагрузкеДанных(СтандартнаяОбработка, Отправитель, ИмяФайлаСообщения,
  ДанныеСообщения, КоличествоЭлементовВТранзакции, ИмяСобытияЖурналаРегистрации,
  КоличествоПолученныхОбъектов)` — обработать загрузку сообщения.
- `ПриПолученииДоступныхВерсийФормата(ВерсииФормата)` — заявить поддерживаемые
  версии EnterpriseData (ключ = версия, значение = модуль правил).
- `ПриОпределенииПрефиксаИнформационнойБазыПоУмолчанию(Префикс)` — рассчитать
  префикс ИБ по умолчанию.

Хук `ТрансляцияXDTOПереопределяемый.ЗаполнитьОбработчикиТрансляцииСообщений(МассивОбработчиков)`
— зарегистрировать модули-трансляторы между версиями EnterpriseData
(напр. 1.6 → 1.7).

Для поиска сигнатуры/региона любого метода —
`python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`.
