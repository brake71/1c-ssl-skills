# Мониторинг производительности и бизнес-статистики БСП

Три связанные подсистемы: **ОценкаПроизводительности** (APDEX-замеры ключевых
операций в регистр `ЗамерыВремени`), **ЦентрМониторинга** (обезличенная
бизнес-статистика и технологические сведения, отправляемые регламентным
заданием в сервис 1С или сторонний сервис) и **КонтрольРаботыПользователей**
(управление регистрацией событий доступа к данным в журнал регистрации).
Учит обрамлять бизнес-операции замерами и регистрировать количественные
показатели, не создавая собственных обёрток.

## Модули

Семейство `ОценкаПроизводительности` (суффиксная логика):

- `ОценкаПроизводительности` — серверные замеры: `НачатьЗамерВремени`,
  `ЗакончитьЗамерВремени`, `ЗакончитьЗамерВремениТехнологический`,
  `НачатьЗамерДлительнойОперации`, `ЗафиксироватьЗамерДлительнойОперации`,
  `ЗакончитьЗамерДлительнойОперации`, `СоздатьКлючевыеОперации`,
  `УстановитьЦелевоеВремя`, `ИзменитьКлючевыеОперации`.
- `ОценкаПроизводительностиКлиент` — клиентские замеры: `ЗамерВремени`
  (однострочный, авто-завершение), `ЗавершитьЗамерВремени`,
  `НачатьЗамерВремениТехнологический`, `УстановитьПараметрыЗамера` и др.
  ⚠️ `ОценкаПроизводительностиКлиент.НачатьЗамерВремени` — **устарел**
  (`УстаревшиеПроцедурыИФункции`); используйте `ЗамерВремени`.
- `ОценкаПроизводительностиВызовСервера` — пакетная запись замеров.
  ⚠️ `ЗафиксироватьДлительностьКлючевыхОпераций(ЗамерыДляЗаписи)` — регион
  `СлужебныеПроцедурыИФункции` (служебный, обратная совместимость не
  гарантируется); не использовать как основной API.
- `ОценкаПроизводительностиВызовСервераПовтИсп` — кешированная проверка
  `ВыполнятьЗамерыПроизводительности` (гейтинг замеров).
- ⚠️ `ОценкаПроизводительностиСлужебный` — служебный API; в `src/cf` у
  корневого модуля также есть методы в `СлужебныеПроцедурыИФункции`
  (`СоздатьКлючевуюОперацию`, `ЭкспортОценкиПроизводительности`, …) — не
  вызывать из прикладного кода.

Семейство `ЦентрМониторинга`:

- `ЦентрМониторинга` — стабильный серверный API: `ЦентрМониторингаВключен`,
  `ВключитьПодсистему`, `ОтключитьПодсистему`, `ИдентификаторИнформационнойБазы`,
  `ЗаписатьОперациюБизнесСтатистики` (+ `…Час`/`…Сутки`),
  `ЗаписыватьОперацииБизнесСтатистики`, `ЗаписатьСтатистикуКонфигурации`,
  `ЗаписатьСтатистикуОбъектаКонфигурации`.
- `ЦентрМониторингаКлиент` — клиентская запись статистики. ⚠️ Сигнатуры
  **отличаются** от серверных: `ЗаписатьОперациюБизнесСтатистики(ИмяОперации,
  Значение)` (без `Комментарий`/`Разделитель`), а `…Час`/`…Сутки` имеют иной
  порядок параметров (`Значение` перед `КлючУникальности`, `КлючУникальности`
  опционален). Не копируйте серверную сигнатуру в клиентский код.
- ⚠️ `ЦентрМониторингаСлужебный` — служебный API; в т.ч.
  `ЗаписатьОперациюБизнесСтатистикиСлужебная(ПараметрыЗаписи)`,
  `ПолучитьПараметрыЦентраМониторинга` — обратная совместимость не
  гарантируется.

`КонтрольРаботыПользователей` — один серверный модуль, 4 стабильных метода
управления регистрацией событий доступа к данным (отдельная подсистема от
`ОценкаПроизводительности`, но относится к мониторингу доступа).

## Сценарии

### 1. Замерить серверную ключевую операцию

**Задача:** обернуть серверный метод (проведение документа, формирование
отчёта) APDEX-замером и записать результат в регистр `ЗамерыВремени`.

**Функции:**
`ОценкаПроизводительности.НачатьЗамерВремени() Экспорт`
— Функция → `Число` (UTC, мс, 14 символов; `0`, если замеры выключены), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`ОценкаПроизводительности.ЗакончитьЗамерВремени(КлючеваяОперация, ВремяНачала, ВесЗамера = 1, Комментарий = Неопределено, ВыполненСОшибкой = Ложь) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.

**Параметры:**
- `КлючеваяОперация` (`СправочникСсылка.КлючевыеОперации` / `Строка`) —
  ключевая операция; по строке БСП сама найдёт/создаст элемент справочника
  `КлючевыеОперации` при записи.
- `ВремяНачала` (`Число`) — значение, возвращённое `НачатьЗамерВремени`.
  Если передан `0` (замеры были выключены в момент старта) — запись
  пропускается молча, проверки на стороне вызывающего кода не требуется.
- `ВесЗамера` (`Число`) — количественный показатель замера (напр. количество
  строк в документе); по умолчанию `1`.
- `Комментарий` (`Строка` / `Соответствие`) — произвольная информация замера;
  по умолчанию `Неопределено`.
- `ВыполненСОшибкой` (`Булево`) — признак того, что замер не доведён до конца;
  по умолчанию `Ложь`.

**Пример:**
```bsl
// В обработчике ОбработкаПроведения модуля документа
ВремяНачала = ОценкаПроизводительности.НачатьЗамерВремени();

Попытка
    // ...штатная логика проведения...
    Отказ = Ложь;
Исключение
    Отказ = Истина;
    ОценкаПроизводительности.ЗакончитьЗамерВремени(
        "Документы.ЗаказПокупателя.Проведение",
        ВремяНачала, 1, , Истина);   // ВыполненСОшибкой = Истина
    ВызватьИсключение;
КонецПопытки;

ОценкаПроизводительности.ЗакончитьЗамерВремени(
    "Документы.ЗаказПокупателя.Проведение", ВремяНачала);
```

**Нюансы / антипаттерны:**
- ❌ Замер через `ТекущаяДата()` / `ТекущаяДатаСеанса()` — низкая точность
  (секунды), нет связи с ключевой операцией и регистром. Только
  `ОценкаПроизводительности.НачатьЗамерВремени` (точность до мс, APDEX).
- ❌ Вызывать `ОценкаПроизводительности.НачатьЗамерВремени` на тонком клиенте
  — модуль серверный. Для клиента — `ОценкаПроизводительностиКлиент.ЗамерВремени`.
- ❌ `ОценкаПроизводительности.ЗакончитьЗамерВремени("Операция", 0)` без
  предшествующего `НачатьЗамерВремени` — длительность посчитается от «эпохи»;
  всегда передавайте значение из `НачатьЗамерВремени`.
- `НачатьЗамерВремени` сам гейтит запись через `ВыполнятьЗамерыПроизводительности`
  (повт.-исп. кеш) — обёртка в `Если … Тогда` не нужна.

### 2. Замерить длительную операцию с вложенными шагами

**Задача:** замерить многошаговую операцию (загрузка → разбор → запись в ИБ) с
детализацией по каждому шагу.

**Функции:**
`ОценкаПроизводительности.НачатьЗамерДлительнойОперации(КлючеваяОперация) Экспорт`
— Функция → `Соответствие` (контекст замера; ключи `КлючеваяОперация`, `ВремяНачала`, `ВремяПоследнегоЗамера`, `ВесЗамера`, `ВложенныеЗамеры`), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`ОценкаПроизводительности.ЗафиксироватьЗамерДлительнойОперации(ОписаниеЗамера, КоличествоДанных, ИмяШага, Комментарий = "") Экспорт`
— Процедура (промежуточный шаг), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`ОценкаПроизводительности.ЗакончитьЗамерДлительнойОперации(ОписаниеЗамера, КоличествоДанных, ИмяШага = "", Комментарий = "") Экспорт`
— Процедура (завершение), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.

**Параметры:**
- `КлючеваяОперация` (`Строка`) — имя ключевой операции.
- `ОписаниеЗамера` (`Соответствие`) — значение, возвращённое
  `НачатьЗамерДлительнойОперации`; **обязательно** из той же пары вызовов.
- `КоличествоДанных` (`Число`) — объём, обработанный на шаге (напр. число
  строк).
- `ИмяШага` (`Строка`) — произвольное имя вложенного шага.
- `Комментарий` (`Строка`) — произвольное описание; по умолчанию `""`.

**Пример:**
```bsl
Замер = ОценкаПроизводительности.НачатьЗамерДлительнойОперации("МассоваяЗагрузкаНоменклатуры");

ДанныеФайла = ПрочитатьФайл(ИмяФайла);
ОценкаПроизводительности.ЗафиксироватьЗамерДлительнойОперации(
    Замер, ДанныеФайла.КоличествоСтрок(), "ЧтениеФайла");

Строки = РазобратьДанные(ДанныеФайла);
ОценкаПроизводительности.ЗафиксироватьЗамерДлительнойОперации(
    Замер, Строки.Количество(), "РазборДанных");

Для Каждого Строка Из Строки Цикл ЗаписатьЭлемент(Строка); КонецЦикла;
ОценкаПроизводительности.ЗакончитьЗамерДлительнойОперации(
    Замер, Строки.Количество(), "ЗаписьВБД", "");
```

**Нюансы / антипаттерны:**
- ❌ Передавать в `ОписаниеЗамера` `Соответствие` от другой операции или
  `Неопределено` — замер будет некорректным; контекст обязан быть из того же
  `НачатьЗамерДлительнойОперации`.
- ❌ `ОценкаПроизводительности.ЗафиксироватьДлительностьКлючевойОперации(100)`
  — такого метода **не существует**. Одиночная запись —
  `ЗакончитьЗамерВремени`; пакетная —
  `ОценкаПроизводительностиВызовСервера.ЗафиксироватьДлительностьКлючевыхОпераций`
  (⚠️ служебный, см. «Редкие методы»).
- Для длительных операций в фоне (фоновое задание) — отдельная подсистема
  «ДлительныеОперации»; здесь только замер **внутри** такой операции.

### 3. Замерить клиентскую операцию

**Задача:** замерить время открытия формы / клиентской обработки с записью в
тот же регистр `ЗамерыВремени` через клиентский буфер.

**Функции:**
`ОценкаПроизводительностиКлиент.ЗамерВремени(КлючеваяОперация = Неопределено, ФиксироватьСОшибкой = Ложь, АвтоЗавершение = Истина) Экспорт`
— Функция → `УникальныйИдентификатор` (идентификатор замера), регион `#Область ПрограммныйИнтерфейс` (stable). Тонкий клиент, Толстый клиент.
`ОценкаПроизводительностиКлиент.ЗавершитьЗамерВремени(УИДЗамера, ВыполненСОшибкой = Ложь) Экспорт`
— Процедура (явное завершение при `АвтоЗавершение = Ложь`), регион `#Область ПрограммныйИнтерфейс` (stable). Тонкий клиент, Толстый клиент.
`ОценкаПроизводительностиКлиент.УстановитьКлючевуюОперациюЗамера(УИДЗамера, КлючеваяОперация) Экспорт`
`ОценкаПроизводительностиКлиент.УстановитьВесЗамера(УИДЗамера, ВесЗамера) Экспорт`
`ОценкаПроизводительностиКлиент.УстановитьКомментарийЗамера(УИДЗамера, Комментарий) Экспорт`
`ОценкаПроизводительностиКлиент.УстановитьПризнакОшибкиЗамера(УИДЗамера, Признак) Экспорт`
— все Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Тонкий клиент, Толстый клиент.

**Параметры:**
- `КлючеваяОперация` (`Строка` / `Неопределено`) — имя ключевой операции; если
  `Неопределено` — задаётся позже через `УстановитьКлючевуюОперациюЗамера`.
- `ФиксироватьСОшибкой` (`Булево`) — `Истина` → при автозавершении замер
  запишется с признаком «выполнен с ошибкой»; `Ложь` → корректный.
- `АвтоЗавершение` (`Булево`) — `Истина` (по умолчанию) → завершение через
  глобальный обработчик ожидания; `Ложь` → явный вызов `ЗавершитьЗамерВремени`.
- `УИДЗамера` (`УникальныйИдентификатор`) — значение, возвращённое `ЗамерВремени`.

**Пример:**
```bsl
// Однострочный замер с авто-завершением
УИД = ОценкаПроизводительностиКлиент.ЗамерВремени("ОткрытиеФормыДокумента");

// Явный цикл: точные границы + вес + признак ошибки
УИД = ОценкаПроизводительностиКлиент.ЗамерВремени(, Ложь, Ложь);
Попытка
    // ...длительная клиентская обработка...
    ОценкаПроизводительностиКлиент.УстановитьВесЗамера(УИД, КоличествоСтрок);
Исключение
    ОценкаПроизводительностиКлиент.УстановитьПризнакОшибкиЗамера(УИД, Истина);
КонецПопытки;
ОценкаПроизводительностиКлиент.ЗавершитьЗамерВремени(УИД);
```

**Нюансы / антипаттерны:**
- ⚠️ Клиентские замеры хранятся в клиентском буфере и записываются с
  периодичностью константы `ОценкаПроизводительностиПериодЗаписи` (по
  умолчанию — каждую минуту); при аварийном завершении сеанса часть замеров
  может быть потеряна.
- ❌ `ОценкаПроизводительностиКлиент.НачатьЗамерВремени(...)` — **устарел**
  (`УстаревшиеПроцедурыИФункции`). Используйте `ЗамерВремени`.
- Если `КлючеваяОперация = Неопределено` — обязательно задайте её через
  `УстановитьКлючевуюОперациюЗамера`, иначе замер не привяжется к операции.

### 4. Зарегистрировать бизнес-статистику

**Задача:** записать количественной показатель («сколько документов
проведено», «средний размер вложения») в буфер `БуферОперацийСтатистики` для
регламентной отправки в сервис.

**Функции:**
`ЦентрМониторинга.ЗаписатьОперациюБизнесСтатистики(ИмяОперации, Значение, Комментарий = Неопределено, Разделитель = ".") Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`ЦентрМониторинга.ЗаписатьОперациюБизнесСтатистикиЧас(ИмяОперации, КлючУникальности, Значение, Замещать = Ложь) Экспорт`
`ЦентрМониторинга.ЗаписатьОперациюБизнесСтатистикиСутки(ИмяОперации, КлючУникальности, Значение, Замещать = Ложь) Экспорт`
— Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`ЦентрМониторинга.ЗаписыватьОперацииБизнесСтатистики() Экспорт`
— Функция → `Булево` (геттер состояния регистрации), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ИмяОперации` (`Строка`) — имя операции статистики; при отсутствии
  создаётся новое. Иерархия разделяется `Разделитель` (по умолчанию `"."`).
- `Значение` (`Число`) — количественное значение.
- `Комментарий` (`Строка`) — произвольный комментарий; по умолчанию
  `Неопределено`.
- `Разделитель` (`Строка`) — разделитель значений в `ИмяОперации`, если не
  точка; по умолчанию `"."`.
- `КлючУникальности` (произвольный) — для `…Час`/`…Сутки`: ключ, по которому
  за период (час/сутки) сохраняется только одна запись.
- `Замещать` (`Булево`) — `Истина` → замещает прежнее значение за период;
  `Ложь` (по умолчанию) → накапливает.

**Пример:**
```bsl
// Простой счётчик
ЦентрМониторинга.ЗаписатьОперациюБизнесСтатистики(
    "Документы.ЗаказПокупателя.Проведение.Количество", 1, , ".");

// Счётчик с уникальностью за час (одна запись на документ в час)
Для Каждого Строка Из ТаблицаДокументов Цикл
    ПровестиДокумент(Строка.Ссылка);
    ЦентрМониторинга.ЗаписатьОперациюБизнесСтатистикиЧас(
        "Документы.ЗаказПокупателя.Проведение.Факт",
        Строка.Ссылка.УникальныйИдентификатор(), 1, Ложь);
КонецЦикла;
```

**Нюансы / антипаттерны:**
- ❌ Оборачивать запись в `Если ЦентрМониторинга.ЦентрМониторингаВключен()`
  — избыточно: `ЗаписатьОперациюБизнесСтатистики` сам гейтит запись через
  `ЗаписыватьОперацииБизнесСтатистики()`. Проверка уместна только если при
  выключенном центре нужна отдельная логика (напр. запись в локальный лог).
- ❌ `ЦентрМониторинга.ЗаписатьОперацию("Сервис.Тест", 1)` — такого метода
  **нет**; стабильное имя — `ЗаписатьОперациюБизнесСтатистики`.
- ❌ Прямая запись в регистр `РегистрыСведений.БуферОперацийСтатистики.СоздатьНаборЗаписей()`
  — обходит служебные проверки и категорирование. Только через
  `ЦентрМониторинга.*`.
- ⚠️ На клиенте сигнатуры иные: `ЦентрМониторингаКлиент.ЗаписатьОперациюБизнесСтатистики(ИмяОперации, Значение)`
  (без `Комментарий`/`Разделитель`), а `…Час`/`…Сутки` —
  `(ИмяОперации, Значение, Замещать = Ложь, КлючУникальности = Неопределено)`
  (другой порядок параметров). Не подставляйте серверскую сигнатуру в
  клиентский код.

### 5. Управлять подсистемой «Центр мониторинга»

**Задача:** программно проверить/включить/отключить центр мониторинга и
получить идентификатор информационной базы для привязки статистики.

**Функции:**
`ЦентрМониторинга.ЦентрМониторингаВключен() Экспорт`
— Функция → `Булево`, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`ЦентрМониторинга.ВключитьПодсистему() Экспорт`
`ЦентрМониторинга.ОтключитьПодсистему() Экспорт`
— Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`ЦентрМониторинга.ИдентификаторИнформационнойБазы() Экспорт`
— Функция → идентификатор ИБ, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.

**Параметры:** без параметров.

**Пример:**
```bsl
Если Не ЦентрМониторинга.ЦентрМониторингаВключен() Тогда
    ЦентрМониторинга.ВключитьПодсистему();   // включит и регламент. задание сбора/отправки
КонецЕсли;

ИдентификаторИБ = ЦентрМониторинга.ИдентификаторИнформационнойБазы();
// Используется для привязки пакетов статистики к конкретной ИБ
```

**Нюансы / антипаттерны:**
- ❌ `ЦентрМониторингаКлиент.ПоказатьНастройкиЦентраМониторинга` — это
  UI-метод (регион `СлужебныйПрограммныйИнтерфейс`), не серверный API; для
  программного управления используйте серверные `ВключитьПодсистему` /
  `ОтключитьПодсистему`.
- `ВключитьПодсистему` задействует регламентное задание `СборИОтправкаСтатистики`
  (через служебный модуль) — отдельная регистрация задания не нужна.

### 6. Создать и настроить ключевую операцию программно

**Задача:** программно зарегистрировать ключевую операцию в справочнике
`КлючевыеОперации` с целевым временем и признаком «длительная».

**Функции:**
`ОценкаПроизводительности.СоздатьКлючевыеОперации(КлючевыеОперации) Экспорт`
`ОценкаПроизводительности.УстановитьЦелевоеВремя(КлючевыеОперации) Экспорт`
`ОценкаПроизводительности.ИзменитьКлючевыеОперации(КлючевыеОперации) Экспорт`
— Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`ОценкаПроизводительности.СоздатьКлючевуюОперацию(ИмяКлючевойОперации, ЦелевоеВремя = 1, Длительная = Ложь) Экспорт`
— Функция → `СправочникСсылка.КлючевыеОперации`, ⚠️ регион `#Область СлужебныеПроцедурыИФункции` (служебный).

**Параметры:**
- `КлючевыеОперации` (`Массив` из `Структура`) — для пакетных
  `СоздатьКлючевыеОперации`/`УстановитьЦелевоеВремя`/`ИзменитьКлючевыеОперации`:
  элементы-структуры с полями `ИмяКлючевойОперации`, `ЦелевоеВремя`,
  `Длительная` и др.
- `ИмяКлючевойОперации` (`Строка`) — имя операции.
- `ЦелевоеВремя` (`Число`) — целевое время в секундах; по умолчанию `1`.
- `Длительная` (`Булево`) — признак длительной операции; по умолчанию `Ложь`.

**Пример:**
```bsl
// Пакетная регистрация (стабильный API)
КлючевыеОперации = Новый Массив;
Операция = Новый Структура;
Операция.Вставить("ИмяКлючевойОперации", "Документы.ЗаказПокупателя.Проведение");
Операция.Вставить("ЦелевоеВремя", 2);
Операция.Вставить("Длительная", Ложь);
КлючевыеОперации.Добавить(Операция);
ОценкаПроизводительности.СоздатьКлючевыеОперации(КлючевыеОперации);

// Позже — поменять целевое время
ОценкаПроизводительности.УстановитьЦелевоеВремя(КлючевыеОперации);
```

**Нюансы / антипаттерны:**
- ⚠️ `СоздатьКлючевуюОперацию` (единичное) — **служебный** регион
  (`СлужебныеПроцедурыИФункции`); для нового кода предпочитайте стабильный
  пакетный `СоздатьКлючевыеОперации`.
- ❌ Выдуманное `ОценкаПроизводительности.ЗафиксироватьДлительностьКлючевойОперации(...)`
  — не существует. См. сценарий 2 и «Редкие методы».
- Передавать строку в `ЗакончитьЗамерВремени` вместо ссылки — допустимо:
  БСП сама найдёт/создаст элемент справочника `КлючевыеОперации` при записи
  замера; отдельная регистрация операции не обязательна для разовых замеров.

### 7. Включить регистрацию событий доступа к данным

**Задача:** программно управлять регистрацией событий «Доступ.Доступ» к
данным в журнал регистрации (требование 152-ФЗ для ПДн) — глобальный
выключатель и детальные настройки.

**Функции:**
`КонтрольРаботыПользователей.РегистрироватьДоступКДанным() Экспорт`
— Функция → `Булево` (геттер настройки панели «НастройкиПользователейИПрав»), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`КонтрольРаботыПользователей.УстановитьРегистрациюДоступаКДанным(РегистрироватьДоступКДанным) Экспорт`
— Процедура (сеттер), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`КонтрольРаботыПользователей.НастройкиРегистрацииСобытийДоступаКДанным() Экспорт`
— Функция → `Структура` (`Состав` — массив описаний, `Комментарии` — `Соответствие`, `ОбщийКомментарий` — `Строка`), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`КонтрольРаботыПользователей.УстановитьНастройкиРегистрацииСобытийДоступаКДанным(Настройки) Экспорт`
— Процедура (сеттер настроек), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.

**Параметры:**
- `РегистрироватьДоступКДанным` (`Булево`) — глобальный признак регистрации
  событий доступа.
- `Настройки` (`Структура`, см. `НастройкиРегистрацииСобытийДоступаКДанным`) —
  состав регистрируемых событий, комментарии по полям, общий комментарий.

**Пример:**
```bsl
// Включить регистрацию, если выключена
Если Не КонтрольРаботыПользователей.РегистрироватьДоступКДанным() Тогда
    КонтрольРаботыПользователей.УстановитьРегистрациюДоступаКДанным(Истина);
КонецЕсли;

// Прочитать текущие настройки и добавить комментарий по полю
Настройки = КонтрольРаботыПользователей.НастройкиРегистрацииСобытийДоступаКДанным();
Настройки.Комментарии.Вставить("Справочник.ФизическиеЛица.НомерДокумента",
    НСтр("ru = 'Серия и номер паспорта'"));
КонтрольРаботыПользователей.УстановитьНастройкиРегистрацииСобытийДоступаКДанным(Настройки);
```

**Нюансы / антипаттерны:**
- Это подсистема **Контроль работы пользователей** (общий журнал доступа),
  отдельная от `ЗащитаПерсональныхДанных` (которая через
  `УстановитьИспользованиеСобытияДоступ` включает «Доступ.Доступ» для
  конкретных категорий ПДн). Связка: сначала глобальный рубильник
  `КонтрольРаботыПользователей.УстановитьРегистрациюДоступаКДанным(Истина)`,
  затем детализация по категориям через `ЗащитаПерсональныхДанных`.
- ❌ Прямая запись в журнал регистрации `ЗаписьЖурналаРегистрации(...)` вместо
  настроек подсистемы — обходит структурированный учёт событий доступа и
  категории ПДн.
- Настройки — это структура, модифицируемая через сеттер; не пытайтесь
  записать её напрямую в константу/регистр.

## Редкие методы

- `ОценкаПроизводительности.ЗакончитьЗамерВремениТехнологический(КлючеваяОперация, ВремяНачала, ВесЗамера = 1, Комментарий = Неопределено) Экспорт`
  — стабильный (`ПрограммныйИнтерфейс`); технологический замер (без признака
  ошибки). Используется реже, чем `ЗакончитьЗамерВремени`.
- `ОценкаПроизводительностиКлиент.НачатьЗамерВремениТехнологический(АвтоЗавершение = Истина, КлючеваяОперация = Неопределено) Экспорт`
  — стабильный; клиентский технологический замер.
- `ОценкаПроизводительности.УстановитьПризнакЗавершенияСОшибкой(КлючевыеОперации) Экспорт`
  — ⚠️ **устарел** (`УстаревшиеПроцедурыИФункции`): «устаревший, не
  использовать в новом коде; альтернатива —
  `ЗакончитьЗамерВремени(…, ВыполненСОшибкой = Истина)`».
- `ОценкаПроизводительностиВызовСервера.ЗафиксироватьДлительностьКлючевыхОпераций(ЗамерыДляЗаписи) Экспорт`
  — ⚠️ **служебный** (`СлужебныеПроцедурыИФункции`): пакетная запись массива
  замеров (`Структура` с `ЗамерыЗавершенные` — `Соответствие` из
  `УникальныйИдентификатор` → `Соответствие`, и `ИнформацияПрограммыПросмотра`
  — `Строка`). Возвращает период записи замеров на сервере (секунды).
  Обратная совместимость не гарантируется — не использовать как основной
  способ; для одиночной записи применяйте `ЗакончитьЗамерВремени`.
- `ЦентрМониторинга.ЗаписатьСтатистикуКонфигурации(СоответствиеИменМетаданных) Экспорт`
  и `ЦентрМониторинга.ЗаписатьСтатистикуОбъектаКонфигурации(ИмяОбъекта, Значение) Экспорт`
  — стабильные (`ПрограммныйИнтерфейс`); запись технологической статистики
  по метаданным конфигурации (объём, количество объектов).
- `ЦентрМониторингаСлужебный.ЗаписатьОперациюБизнесСтатистикиСлужебная(ПараметрыЗаписи) Экспорт`
  — ⚠️ служебный (`СлужебныеПроцедурыИФункции`); низкоуровневая запись в
  буфер; используйте стабильный `ЦентрМониторинга.ЗаписатьОперациюБизнесСтатистики`.
