# Резервное копирование ИБ

Подсистема **РезервноеКопированиеИБ** — общие модули `РезервноеКопированиеИБКлиент`
(клиентский стабильный API — открытие формы), `РезервноеКопированиеИБСервер`
(серверный, ⚠️ целиком в служебных регионах — чтение/запись настроек, статус,
навигационная ссылка), `РезервноеКопированиеИБВызовСервера` (⚠️ служебный —
серверные вызовы с клиента), `РезервноеКопированиеИБКлиентПереопределяемый`
(хук отключения предупреждений). Покрывает открытие формы копирования,
программную работу с настройками, расчёт даты следующего копирования, сброс
признака после восстановления.

Подсистема поддерживает **только файловый** вариант ИБ; в клиент-серверном
варианте методы автоматического копирования возвращают `Ложь`, а UI-форма
бесполезна — используйте средства СУБД и признак
`РезервноеКопированиеНастроено = Истина` без
`ВыполнятьАвтоматическоеРезервноеКопирование`. Выгрузка в облачные хранилища
(Google Drive, Яндекс.Диск) в этой версии API не предоставляется — используется
локальный каталог файловой системы (`КаталогХраненияРезервныхКопий`).

## Модули

- `РезервноеКопированиеИБКлиент` — **единственный стабильный** клиентский
  метод: `ОткрытьФормуРезервногоКопирования`. Остальные экспортные методы модуля
  — обработчики событий подсистемы (`ПриНачалеРаботыСистемы`,
  `ПередЗавершениемРаботыСистемы`, `ПриПредложенииПользователюСоздатьРезервнуюКопию`)
  в `СлужебныйПрограммныйИнтерфейс` — вызываются БСП, не из прикладного кода.
  Тонкий / Толстый клиент.
- `РезервноеКопированиеИБСервер` — ⚠️ **целиком служебный**: в
  `СлужебныйПрограммныйИнтерфейс` — `УстановитьНастройкиРезервногоКопирования`,
  `ТекущаяНастройкаРезервногоКопирования`,
  `НавигационнаяСсылкаОбработкиРезервногоКопирования`; в
  `СлужебныеПроцедурыИФункции` — `ПараметрыРезервногоКопирования`,
  `СброситьПризнакРезервногоКопирования`, `УстановитьДатуПоследнегоНапоминания`,
  `УстановитьЗначениеНастройки`, `ЗавершитьРезервноеКопирование`. Обратная
  совместимость не гарантируется.
- `РезервноеКопированиеИБВызовСервера` — ⚠️ служебный (серверный вызов с
  клиента): `УстановитьЗначениеНастройки`, `ДатаСледующегоАвтоматическогоКопирования`,
  `УстановитьДатуПоследнегоНапоминания` (в `СлужебныеПроцедурыИФункции`).
- `РезервноеКопированиеИБКлиентПереопределяемый` — **хук**
  `ПриОпределенииНеобходимостиПоказаПредупрежденийОРезервномКопировании`.
- `РезервноеКопированиеОбластейДанных` / `…Клиент` — заглушки модели сервиса
  (SaaS), в локальной ИБ — пустые.

⚠️ **Не существует** модулей `РезервноеКопированиеИБСлужебный`,
`РезервноеКопированиеИБПереопределяемый` (без `Клиент`),
`РезервноеКопированиеИБКлиентСервер`. Служебная логика встроена в основные
модули, а переопределение — только клиентское. Перед вызовом сверяйтесь с
реальным каталогом общих модулей.

## Сценарии

### 1. Открыть форму резервного копирования с проверкой файлового варианта

**Задача:** из прикладной команды открыть форму создания/восстановления
резервной копии, предварительно убедившись, что ИБ файловая.

**Функция:**
`РезервноеКопированиеИБКлиент.ОткрытьФормуРезервногоКопирования(Параметры = Неопределено) Экспорт`
— Процедура, регион: `#Область ПрограммныйИнтерфейс` (stable). Тонкий / Толстый
клиент.

**Параметры:**
- `Параметры` (Структура / Неопределено) — параметры формы. Для запуска при
  завершении работы: `Новый Структура("РежимРаботы", "ВыполнитьПриЗавершенииРаботы")`.

**Пример:**
```bsl
&НаКлиенте
Процедура СоздатьРезервнуюКопию(Команда)
    Если ОбщегоНазначенияКлиент.ИнформационнаяБазаФайловая() Тогда
        РезервноеКопированиеИБКлиент.ОткрытьФормуРезервногоКопирования();
    Иначе
        ПоказатьПредупреждение(,
            НСтр("ru = 'В клиент-серверном варианте резервное копирование выполняется средствами СУБД.'"));
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Открывать форму без проверки `ИнформационнаяБазаФайловая()` — в
  клиент-серверном варианте форма откроется «пустой».
- `ОткрытьФормуРезервногоКопирования` — **единственная** стабильная клиентская
  точка входа. Не выдумывайте `ОткрытьФормуНастроекКопирования` — форма
  настроек открывается UI-командой подсистемы или навигационной ссылкой
  (сценарий 3).

### 2. Прочитать и сохранить настройки копирования

**Задача:** из серверской обработки инициализировать расписание, каталог,
период хранения и флаги автоматического копирования; корректно перезаписать
настройки, не потеряв значения по умолчанию.

**Функции:**
`РезервноеКопированиеИБСервер.ПараметрыРезервногоКопирования() Экспорт` — Функция → Структура (см. `НовыеНастройкиРезервногоКопирования`).
`РезервноеКопированиеИБСервер.УстановитьНастройкиРезервногоКопирования(Знач Настройки, Знач Пользователь = Неопределено) Экспорт` — Процедура.
— ⚠️ **служебный** (`СлужебныеПроцедурыИФункции` / `СлужебныйПрограммныйИнтерфейс`);
обратная совместимость не гарантируется. Сервер, Внешнее соединение.

**Параметры:**
- `Настройки` (Структура, см. `НовыеНастройкиРезервногоКопирования`) — полный
  набор полей: `ВыполнятьАвтоматическоеРезервноеКопирование` (Булево),
  `РезервноеКопированиеНастроено` (Булево),
  `РасписаниеКопирования` (Структура расписания регламентного задания),
  `КаталогХраненияРезервныхКопий` (Строка),
  `КаталогХраненияРезервныхКопийПриРучномЗапуске` (Строка),
  `ВариантВыполнения` (`"ПоРасписанию"` / `"ПриЗавершенииРаботы"`),
  `АдминистраторИБ` (Строка), `ПарольАдминистратораИБ` (Строка),
  `ПараметрыУдаления` (Структура: `ТипОграничения`, `КоличествоКопий`,
  `ЕдиницаИзмеренияПериода`, `ЗначениеВЕдиницахИзмерения`),
  `ДатаПоследнегоРезервногоКопирования`, `МинимальнаяДатаСледующегоАвтоматическогоРезервногоКопирования`,
  `РучнойЗапускПоследнегоРезервногоКопирования` и др.
- `Пользователь` (ПользовательСсылка / Неопределено) — если указан, настройки
  дополнительно сохраняются в константу `ПараметрыРезервногоКопирования` для
  передачи в фоновый сеанс при завершении работы.

**Пример:**
```bsl
// Сервер: всегда читаем текущие настройки, затем изменяем нужные поля
Настройки = РезервноеКопированиеИБСервер.ПараметрыРезервногоКопирования();
Настройки.ВыполнятьАвтоматическоеРезервноеКопирование = Истина;
Настройки.ВариантВыполнения = "ПоРасписанию";
Настройки.РасписаниеКопирования = ОбщегоНазначенияКлиентСервер.РасписаниеВСтруктуру(НовоеРасписание);
Настройки.КаталогХраненияРезервныхКопий = "D:\Backups";
Настройки.РезервноеКопированиеНастроено = Истина;
РезервноеКопированиеИБСервер.УстановитьНастройкиРезервногоКопирования(Настройки);
```

**Нюансы / антипаттерны:**
- ❌ Собирать структуру настроек «с нуля» через `Новый Структура` —
  `УстановитьНастройкиРезервногоКопирования` ожидает **полный** набор полей, и
  будут потеряны значения по умолчанию и служебные поля (`ДатаПоследнегоРезервногоКопирования`,
  `МинимальнаяДатаСледующегоАвтоматическогоРезервногоКопирования` и т. п.). Только
  `ПараметрыРезервногоКопирования()` → правка полей → запись.
- ❌ Программно запускать копирование через `ЗавершитьРезервноеКопирование(Результат, ИмяФайлаРезервнойКопии = "")` — это служебный метод **завершения**
  (обработки результата после выполнения), а не запуска. Копирование
  инициируется через форму (`ОткрытьФормуРезервногоКопирования` с
  `РежимРаботы = "ВыполнитьПриЗавершенииРаботы"`) или регламентным заданием.
- Копирование в облако (Google Drive, Яндекс.Диск) в этой версии API не
  предусмотрено — только локальный/сетевой каталог ФС.

### 3. Показать статус и навигационную ссылку в карточке

**Задача:** получить готовую локализованную строку-формулировку текущего
режима копирования и навигационную ссылку на обработку для подстановки в
форматированную строку / email / оповещение.

**Функции:**
`РезервноеКопированиеИБСервер.ТекущаяНастройкаРезервногоКопирования() Экспорт` — Функция → Строка.
`РезервноеКопированиеИБСервер.НавигационнаяСсылкаОбработкиРезервногоКопирования() Экспорт` — Функция → Строка (`e1cib/app/Обработка.РезервноеКопированиеИБ`).
— ⚠️ **служебный** (`СлужебныйПрограммныйИнтерфейс`). Сервер, Внешнее соединение.

**Параметры:** нет.

**Пример:**
```bsl
// Сервер
ТекстСтатуса = РезервноеКопированиеИБСервер.ТекущаяНастройкаРезервногоКопирования();
НавигационнаяСсылка = РезервноеКопированиеИБСервер.НавигационнаяСсылкаОбработкиРезервногоКопирования();

// В форматированной строке / поле HTML
СтрокаHTML = "<a href='" + НавигационнаяСсылка + "'>" + ТекстСтатуса + "</a>";
```

**Нюансы / антипаттерны:**
- `ТекущаяНастройкаРезервногоКопирования` учитывает вариант ИБ: в
  клиент-серверном вернёт «Резервное копирование не выполняется (организовано
  средствами СУБД)»; при отсутствии настроек — «Для настройки резервного
  копирования обратитесь к администратору.». Не формируйте такой текст сами.
- `НавигационнаяСсылкаОбработкиРезервногоКопирования` возвращает ссылку на
  **основную** обработку `РезервноеКопированиеИБ` (форма `РезервноеКопированиеДанных`),
  не на форму настроек.

### 4. Рассчитать дату следующего копирования и изменить одно поле настроек

**Задача:** в клиентской форме настроек показать дату следующего
автоматического копирования и сохранить изменение одного поля без
перечитывания всей структуры (из обработчика изменения элемента формы).

**Функции:**
`РезервноеКопированиеИБВызовСервера.ДатаСледующегоАвтоматическогоКопирования(ОтложитьРезервноеКопирование = Ложь) Экспорт` — Функция → Дата.
`РезервноеКопированиеИБВызовСервера.УстановитьЗначениеНастройки(ИмяЭлемента, ЗначениеЭлемента) Экспорт` — Процедура.
— ⚠️ **служебный** (`СлужебныеПроцедурыИФункции`). Серверный вызов с клиента
(Тонкий/Толстый клиент → Сервер).

**Параметры:**
- `ОтложитьРезервноеКопирование` (Булево) — `Истина` сдвигает минимальную дату
  на 1 час вперёд (для кнопки «Отложить»).
- `ИмяЭлемента` (Строка) — имя поля настроек, напр. `"КаталогХраненияРезервныхКопий"`.
- `ЗначениеЭлемента` (Произвольный) — новое значение поля.

**Пример:**
```bsl
&НаКлиенте
Процедура РасписаниеКопированияПриИзменении(Элемент)
    ДатаСледующего = РезервноеКопированиеИБВызовСервера.ДатаСледующегоАвтоматическогоКопирования();
    Элементы.ДатаСледующегоКопирования.Заголовок =
        НСтр("ru = 'Следующее копирование:'") + " "
        + Формат(ДатаСледующего, "ДФ='dd.MM.yyyy HH:mm'");
КонецПроцедуры

&НаКлиенте
Процедура КаталогХраненияПриОкончанииВводаТекста(Элемент, Текст, Отказ)
    // Сохранить одно поле без перечитывания всей структуры настроек
    РезервноеКопированиеИБВызовСервера.УстановитьЗначениеНастройки(
        "КаталогХраненияРезервныхКопий", Текст);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ⚠️ `УстановитьЗначениеНастройки` и `УстановитьДатуПоследнегоНапоминания`
  определены в **двух** модулях — `…ВызовСервера` (клиентский серверный вызов)
  и `…Сервер` (серверный контекст). Из клиентского кода — `ВызовСервера`, из
  серверного — `Сервер`. Уточняйте через
  `python scripts/bsp_api.py method УстановитьЗначениеНастройки --src src/cf`.
- `ДатаСледующегоАвтоматическогоКопирования` имеет смысл только при
  `ВариантВыполнения = "ПоРасписанию"` и файловом варианте ИБ.

### 5. Сбросить признак копирования после восстановления

**Задача:** после восстановления ИБ из резервной копии сбросить флаг
`ПроведеноКопирование` (чтобы БСП не считала копирование актуальным) и при
наличии подсистемы мониторинга зафиксировать операцию.

**Функция:**
`РезервноеКопированиеИБСервер.СброситьПризнакРезервногоКопирования() Экспорт`
— Процедура, регион `#Область СлужебныеПроцедурыИФункции` (⚠️ служебный).
Сервер, Внешнее соединение.

**Параметры:** нет.

**Пример:**
```bsl
// Сервер: после восстановления из копии
РезервноеКопированиеИБСервер.СброситьПризнакРезервногоКопирования();
```

**Нюансы / антипаттерны:**
- ❌ Сбрасывать признак «на всякий случай» в обычном сеансе — это служебный
  метод для сценария восстановления. В рабочем режиме признак выставляется
  самим механизмом копирования.
- Также метод записывает операцию в центр мониторинга при наличии подсистемы
  мониторинга — отдельный вызов не нужен.

### 6. Переопределение: отключить предупреждения о настройке копирования

**Задача:** в компании бэкапы делаются сторонними средствами, и БСП-баннеры о
необходимости настроить копирование мешают — отключить их через хук.

**Функция (хук):**
`РезервноеКопированиеИБКлиентПереопределяемый.ПриОпределенииНеобходимостиПоказаПредупрежденийОРезервномКопировании(ПоказыватьПредупреждение) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс`. **Хук**: БСП вызывает,
прикладной код реализует в одноимённом модуле прикладной конфигурации.
Тонкий / Толстый клиент.

**Параметры:**
- `ПоказыватьПредупреждение` (Булево, выходной) — установить `Ложь`, чтобы БСП
  не показывала предупреждения.

**Пример:**
```bsl
// В модуле РезервноеКопированиеИБКлиентПереопределяемый прикладной конфигурации
Процедура ПриОпределениеНеобходимостиПоказаПредупрежденийОРезервномКопировании(ПоказыватьПредупреждение) Экспорт
    // Бэкапы делает внешняя система — не показываем предупреждения БСП
    ПоказыватьПредупреждение = Ложь;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать хук как обычный метод `РезервноеКопированиеИБКлиентПереопределяемый.ПриОпределениеНеобходимостиПоказаПредупреждений…(Ложь)`
  из прикладного кода — это **не вызывается**, а **реализуется**. БСП сама
  вызовет вашу реализацию.
- Не путайте с отключением самой подсистемы — хук глушит только предупреждения,
  регламентное задание и API настроек продолжают работать.

## Дополнительно

Прочие служебные методы (полные сигнатуры — через
`python scripts/bsp_api.py method <Имя> --src src/cf`):

- `РезервноеКопированиеИБСервер.НастройкиРезервногоКопирования(НачалоРаботы = Ложь)` — ⚠️ служебный, вариант чтения настроек с признаком начала работы.
- `РезервноеКопированиеИБСервер.УстановитьДатуПоследнегоНапоминания(ДатаНапоминания)` — ⚠️ служебный; зафиксировать дату напоминания пользователю (дублируется в `…ВызовСервера`).
- `РезервноеКопированиеИБСервер.ЗавершитьРезервноеКопирование(Результат, ИмяФайлаРезервнойКопии = "")` / `ЗавершитьВосстановление(Результат)` — ⚠️ служебные обработчики **результата** выполнения, не запуск.
- `РезервноеКопированиеИБСервер.ИнформацияОПользователе()` — ⚠️ служебный, данные пользователя для копирования.

Обработки подсистемы:

- `Обработка.РезервноеКопированиеИБ` — основная форма `РезервноеКопированиеДанных` (создание/восстановление копии). Открывается через `ОткрытьФормуРезервногоКопирования`.
- `Обработка.НастройкаРезервногоКопированияИБ` — форма настроек; открывается UI-командой подсистемы или навигационной ссылкой (сценарий 3), прямого экспортного метода открытия нет.

Хранилище настроек: ключ `ПараметрыРезервногоКопирования` в хранилище общих
настроек + дублирующая константа `ПараметрыРезервногоКопирования` (тип
`ХранилищеЗначения`, сжатие 9) для передачи параметров в фоновый сеанс при
завершении работы. Метод `ПараметрыРезервногоКопирования` возвращает
`Структуру` (не `ФиксированнаяСтруктура`) с фиксированным набором полей —
не добавляйте свои поля, `УстановитьНастройкиРезервногоКопирования` запишет
только исходный набор.
