# Коммуникации БСП: почта, SMS, шаблоны сообщений, обсуждения, взаимодействия

Пять тесно связанных подсистем: **РаботаСПочтовымиСообщениями** (отправка/загрузка
email через SMTP/IMAP/POP3), **ОтправкаSMS** (через внешнего провайдера),
**ШаблоныСообщений** (формирование писем/SMS из шаблонов с подстановкой реквизитов),
**Обсуждения** (чат через систему взаимодействия 1С:Диалог), **Взаимодействия**
(хранение входящих/исходящих писем и сообщений SMS в базе, привязка к предмету).
Объединены, потому что все решают задачу «отправить сообщение пользователю» —
разными каналами.

## Модули

Суффиксная система (корень = имя подсистемы + контекст):

- `РаботаСПочтовымиСообщениями` — стабильный серверный API (отправка, загрузка,
  проверки, учётные записи).
- `РаботаСПочтовымиСообщениямиКлиент` — стабильный клиентский (открытие форм,
  проверка учётной записи).
- `ОтправкаSMS` — стабильный серверный API (отправка, статусы, настройки).
- `ОтправкаSMSКлиент` — стабильный клиентский (UI отправки с проверкой настроек).
- `ШаблоныСообщений` — стабильный серверный API (формирование сообщений,
  параметры, инициализация структур).
- `ШаблоныСообщенийКлиент` — стабильный клиентский (открытие форм, выбор шаблона).
- `Обсуждения` — стабильный серверный API (отправка сообщений, описание,
  проверки подключения, маппинг пользователей системы взаимодействия).
- `ОбсужденияКлиент` — стабильный клиентский (подключение, отключение,
  проверка доступности).
- `Взаимодействия` — стабильный серверный API (хранение писем/SMS, предмет
  взаимодействия, использование подсистемы).

`*Переопределяемый` модули (`РаботаСПочтовымиСообщениямиПереопределяемый`,
`ОтправкаSMSПереопределяемый`, `ШаблоныСообщенийПереопределяемый`) — **хуки**: БСП
вызывает, прикладной код реализует (копирует модуль-переопределитель и
переопределяет тело). Не вызываются из прикладного кода напрямую.

⚠️ **Не существуют:** `РаботаСПочтовымиСообщениямиСервер`, `ОтправкаSMSСервер`
(базовые модули носят имя подсистемы без суффикса). `ШаблоныСообщенийСервер`
существует (серверный вариант-обёртка `ШаблоныСообщений`, напр.
`ШаблоныСообщенийСервер.ТаблицаПараметров()`) — это не ошибка.
`РаботаСПочтовымиСообщениями.ОтправитьПочтовоеСообщение` — ⚠️ устаревший (регион
`УстаревшиеПроцедурыИФункции`); в новом коде — `ОтправитьПисьмо` /
`ОтправитьПисьма`. `ПодставитьПараметрыВШаблонСообщения` — не существует
(ошибка «по аналогии»); формирование текста по шаблону — через
`ШаблоныСообщений.СформироватьСообщение`.

## Сценарии

### 1. Отправить email через настроенную учётную запись

**Задача:** программно отправить одно или несколько писем через SMTP,
обработав сетевые ошибки и ошибки получателей.

**Функции:**
`РаботаСПочтовымиСообщениями.ОтправитьПисьмо(УчетнаяЗаписьИлиСоединение, Письмо) Экспорт`
— Функция → `Структура` (`ОшибочныеПолучатели` — `Соответствие` адрес→текст
ошибки), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`РаботаСПочтовымиСообщениями.ОтправитьПисьма(УчетнаяЗаписьИлиСоединение, Письма, ТекстОшибки = Неопределено) Экспорт`
— Функция → `Соответствие` (ключ — `ИнтернетПочтовоеСообщение`, значение —
результат отправки), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `УчетнаяЗаписьИлиСоединение` (`СправочникСсылка.УчетныеЗаписиЭлектроннойПочты`
  / `ИнтернетПочта` — установленное соединение). Пароль учётной записи
  автоматически читается из безопасного хранилища.
- `Письмо` (`ИнтернетПочтовоеСообщение`) — одно; `Письма` (`Массив` из
  `ИнтернетПочтовоеСообщение`) — пакет.
- `ТекстОшибки` (Строка) — выходной, сообщение об ошибке, если отправлено не
  все. При частичной отправке исключение **не** выбрасывается — проверяйте
  результат.

**Пример:**
```bsl
Письмо = Новый ИнтернетПочтовоеСообщение;
Письмо.Тема = НСтр("ru = 'Счёт на оплату'");
Письмо.Тексты.Добавить(НСтр("ru = 'Здравствуйте, высылаем счёт.'"));
Письмо.Получатели.Добавить(Новый ИнтернетПочтовыйАдрес("client@example.com"));

Попытка
    Результат = РаботаСПочтовымиСообщениями.ОтправитьПисьмо(УчетнаяЗапись, Письмо);
    Если Результат.ОшибочныеПолучатели.Количество() > 0 Тогда
        ОбщегоНазначения.СообщитьПользователю(
            НСтр("ru = 'Часть получателей отвергнута.'"), , , , Отказ);
    КонецЕсли;
Исключение
    ОбщегоНазначения.СообщитьПользователю(
        НСтр("ru = 'Не удалось отправить письмо.'"), , , , Отказ);
КонецПопытки;
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `ОтправитьПисьмо` без `Попытка / Исключение` — метод
  документированно может вызвать исключение при сетевой ошибке / неверном
  пароле / лимите провайдера.
- ❌ `РаботаСПочтовымиСообщениями.ОтправитьПочтовоеСообщение(...)` — устаревший
  (регион `УстаревшиеПроцедурыИФункции`), не использовать в новом коде.
- ❌ Хранить пароль учётной записи в коде или писать напрямую в реквизит
  (`Учетка.Пароль = "..."`) — обходит безопасное хранилище. Пароль — через
  `ОбщегоНазначения.ЗаписатьДанныеВБезопасноеХранилище(Учетка, Пароль, "Пароль")`.
- Для массовых рассылок оборачивайте `ОтправитьПисьма` в фоновое задание
  (`ДлительныеОперации.ВыполнитьФункцию`).

### 2. Проверить доступность канала и получить учётные записи

**Задача:** перед UI-командой «Отправить письмо» проверить, что есть
настроенная учётная запись; получить список учётных записей с фильтром по
назначению.

**Функции:**
`РаботаСПочтовымиСообщениями.ДоступнаОтправкаПисем() Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`РаботаСПочтовымиСообщениями.ДоступныеУчетныеЗаписи(Знач ДляОтправки = Неопределено, Знач ДляПолучения = Неопределено, Знач ВключатьСистемнуюУчетнуюЗапись = Истина) Экспорт`
— Функция → `ТаблицаЗначений` (`Ссылка / Наименование / Адрес`), регион
`#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ОтправкаSMS.ДоступнаОтправкаSMS() Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ОтправкаSMS.НастройкаОтправкиSMSВыполнена() Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ДляОтправки` (Булево) — `Истина` — только для отправки; `ДляПолучения`
  (Булево) — `Истина` — только для получения; оба `Неопределено` — все.
- `ВключатьСистемнуюУчетнуюЗапись` (Булево) — по умолчанию `Истина`.

**Пример:**
```bsl
// Перед UI-командой отправки письма
Если РаботаСПочтовымиСообщениями.ДоступнаОтправкаПисем() Тогда
    // открыть форму отправки
КонецЕсли;

// Список учётных записей только для отправки
ТЗ = РаботаСПочтовымиСообщениями.ДоступныеУчетныеЗаписи(Истина, Ложь);

// Перед отправкой SMS — проверить готовность провайдера
Если ОтправкаSMS.НастройкаОтправкиSMSВыполнена() Тогда
    // можно отправлять
КонецЕсли;
```

**Нюансы / антипаттерны:**
- `ДоступнаОтправкаПисем` / `ДоступнаОтправкаSMS` проверяют и наличие настройки,
  и права текущего пользователя — используйте перед UI-командой.
- На клиенте `СоздатьНовоеПисьмо` сам откроет мастер настройки при отсутствии
  учётной записи — отдельная проверка не обязательна.

### 3. Открыть форму нового письма на клиенте

**Задача:** из обработчика команды формы открыть форму нового письма с
предзаполненными получателями/темой/вложениями, с оповещением о закрытии.

**Функции:**
`РаботаСПочтовымиСообщениямиКлиент.СоздатьНовоеПисьмо(ПараметрыОтправкиПисьма = Неопределено, ОповещениеОЗакрытииФормы = Неопределено) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.
`РаботаСПочтовымиСообщениямиКлиент.ПараметрыОтправкиПисьма() Экспорт`
— Функция → `Структура` (`Отправитель / Получатель / Тема / Текст /
Вложения / Копии / СкрытыеКопии / Предмет / …`), регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `ПараметрыОтправкиПисьма` (Структура / `Неопределено`) — предзаполнение
  письма.
- `ОповещениеОЗакрытииФормы` (`ОписаниеОповещения` / `Неопределено`) —
  обработчик результата после закрытия формы.

**Пример:**
```bsl
&НаКлиенте
Процедура КомандаОтправитьПисьмо(Команда)
    Параметры = РаботаСПочтовымиСообщениямиКлиент.ПараметрыОтправкиПисьма();
    Параметры.Получатель = Контрагент;
    Параметры.Тема       = НСтр("ru = 'Документы по заказу %1'", Документ.Номер);
    РаботаСПочтовымиСообщениямиКлиент.СоздатьНовоеПисьмо(
        Параметры,
        Новый ОписаниеОповещения("ПослеОтправкиПисьма", ЭтаФорма));
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- При отсутствии настроенной учётной записи `СоздатьНовоеПисьмо` автоматически
  открывает мастер настройки — отдельная проверка не нужна.
- `ОповещениеОЗакрытииФормы` — единственный способ узнать результат (форма
  асинхронна); не блокируйте поток ожиданием.

### 4. Отправить SMS и отследить доставку

**Задача:** отправить SMS через настроенного провайдера (SMS4B / SMS.RU /
SMS-ЦЕНТР / Билайн / МТС) и при необходимости запросить статус доставки.

**Функции:**
`ОтправкаSMS.ОтправитьSMS(НомераПолучателей, Знач Текст, ИмяОтправителя = Неопределено, ПеревестиВТранслит = Ложь) Экспорт`
— Функция → `Структура` (`ОтправленныеСообщения` — `Массив` из `Структура`
(`НомерПолучателя / ИдентификаторСообщения`); `ОписаниеОшибки` — Строка, пустая
= успех), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ОтправкаSMSКлиент.ОтправитьSMS(НомераПолучателей, Текст, ДополнительныеПараметры) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.
`ОтправкаSMS.СтатусДоставки(Знач ИдентификаторСообщения) Экспорт`
— Функция → Строка (`"НеОтправлялось" / "Отправляется" / "Отправлено" /
"Доставлено" / "НеДоставлено" / "Ошибка"`), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `НомераПолучателей` (`Массив из Строка`) — номера в формате `+7XXXXXXXXXX`.
- `Текст` (Строка) — текст SMS (макс. длина зависит от оператора).
- `ИмяОтправителя` (Строка / `Неопределено`) — имя, отображаемое вместо номера.
- `ПеревестиВТранслит` (Булево) — транслитерировать текст перед отправкой.
- `ДополнительныеПараметры` (Структура) — для клиентского варианта: `ИмяОтправителя`,
  `ПеревестиВТранслит`.

**Пример:**
```bsl
МассивНомеров = Новый Массив;
МассивНомеров.Добавить("+79991234567");

Если ОтправкаSMS.НастройкаОтправкиSMSВыполнена() Тогда
    Результат = ОтправкаSMS.ОтправитьSMS(МассивНомеров, "Ваш заказ подтверждён.");
    Если Не ПустаяСтрока(Результат.ОписаниеОшибки) Тогда
        ОбщегоНазначения.СообщитьПользователю(
            СтроковыеФункцииКлиентСервер.ПодставитьПараметрыВСтроку(
                НСтр("ru = 'Ошибка отправки SMS: %1'"), Результат.ОписаниеОшибки));
    Иначе
        Идентификатор = Результат.ОтправленныеСообщения[0].ИдентификаторСообщения;
        // Позже: Статус = ОтправкаSMS.СтатусДоставки(Идентификатор);
    КонецЕсли;
Иначе
    // провайдер не настроен — открыть форму настроек
    ОтправкаSMSКлиент.ОткрытьФормуНастроек(Новый ОписаниеОповещения("ПослеНастройкиSMS", ЭтаФорма));
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Парсить строки номеров вручную (`СтрРазделить("+7 999...", ",")`) — не
  учитывает формат `+7XXX...` и пробелы. Нормализуйте в `+7XXXXXXXXXX` до вызова
  или берите телефон через подсистему контактной информации.
- `ОтправитьSMS` при ненастроенном провайдере **не** выбрасывает исключение, а
  возвращает `ОписаниеОшибки` — легко пропустить при отладке. Сначала проверяйте
  `НастройкаОтправкиSMSВыполнена()`.
- `ОтправкаSMSКлиент.ОтправитьSMS` сам проверяет настройки и открывает мастер
  при отсутствии — используйте на клиенте для UI-команд.

### 5. Сформировать сообщение по шаблону и отправить

**Задача:** сформировать email/SMS по шаблону с подстановкой реквизитов
предмета (документ, контрагент), либо сразу отправить; открыть интерактивную
форму выбора шаблона.

**Функции:**
`ШаблоныСообщений.СформироватьСообщение(Шаблон, Предмет, УникальныйИдентификатор, ДополнительныеПараметры = Неопределено) Экспорт`
— Функция → `Структура` (`Тема / Текст / Получатель / Вложения /
СообщенияПользователю`), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ШаблоныСообщений.СформироватьСообщениеИОтправить(Шаблон, Предмет, УникальныйИдентификатор, ДополнительныеПараметры = Неопределено) Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ШаблоныСообщений.ПараметрыОтправкиПисьмаПоШаблону() Экспорт`
— Функция → `Структура` (`УчетнаяЗапись / ОтправитьСразу /
ПреобразовыватьHTMLДляФорматированногоДокумента`), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ШаблоныСообщенийКлиент.СформироватьСообщение(ПредметСообщения, ВидСообщения, ОписаниеОповещенияОЗакрытии = Неопределено, ВладелецШаблона = Неопределено, ПараметрыСообщения = Неопределено) Экспорт`
— Процедура (открывает интерактивную форму), регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.
`ШаблоныСообщенийКлиент.ВыбратьШаблон(Оповещение, ВидСообщения = "Письмо", ПредметШаблона = Неопределено, ВладелецШаблона = Неопределено) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `Шаблон` (`СправочникСсылка.ШаблоныСообщений`).
- `Предмет` (Произвольный — типы из `ОпределяемыйТип.ПредметШаблонаСообщения`).
- `УникальныйИдентификатор` (`УникальныйИдентификатор`) — для размещения вложений
  во временном хранилище; при серверном вызове без формы — любой идентификатор.
- `ДополнительныеПараметры` (Структура) — из `ПараметрыОтправкиПисьмаПоШаблону()`;
  `УчетнаяЗапись` — учётка для отправки; `ОтправитьСразу` (Булево, по умолчанию
  `Ложь`) — `Истина` отправить немедленно, `Ложь` — в папку Исходящих.
- `ВидСообщения` (Строка) — `"Письмо"` (email) или `"СообщениеSMS"`.

**Пример:**
```bsl
// Сервер: сформировать и сразу отправить
ДопПараметры = ШаблоныСообщений.ПараметрыОтправкиПисьмаПоШаблону();
ДопПараметры.УчетнаяЗапись  = УчетнаяЗапись;
ДопПараметры.ОтправитьСразу = Истина;

Результат = ШаблоныСообщений.СформироватьСообщениеИОтправить(
    ШаблонСсылка,              // ссылка на шаблон
    ЗаказКлиента,              // предмет — реквизиты подставляются из него
    УникальныйИдентификатор,   // ЭтаФорма.УникальныйИдентификатор или Новый УникальныйИдентификатор()
    ДопПараметры);

// Клиент: интерактивная форма формирования
ШаблоныСообщенийКлиент.СформироватьСообщение(ЗаказКлиента, "Письмо",
    Новый ОписаниеОповещения("ПослеФормирования", ЭтаФорма));
```

**Нюансы / антипаттерны:**
- ❌ Своя подстановка `СтрЗаменить(Шаблон.Текст, "[Номер]", Документ.Номер)` —
  шаблоны БСП поддерживают параметры СКД, реквизиты табличных частей, вложенные
  объекты, условное форматирование; ручная подстановка этого не даст.
- ❌ Искать `ШаблоныСообщений.ПодставитьПараметрыВШаблонСообщения` — метод **не
  существует**. Формирование текста — через `СформироватьСообщение`.
- `ПараметрыОтправкиПисьмаПоШаблону()` — фабричный метод с правильными ключами;
  не создавайте структуру параметров вручную.
- Хук `ШаблоныСообщенийПереопределяемый.ПриФормированииСообщения(Сообщение,
  НазначениеШаблона, ПредметСообщения, ПараметрыШаблона)` — реализуется во
  внедрении для нестандартной логики формирования; не вызывается напрямую.

### 6. Отправить сообщение в обсуждение (1С:Диалог)

**Задача:** отправить сообщение пользователю/группе через систему
взаимодействия, с привязкой к объекту (контекстное обсуждение) или без неё.

**Функции:**
`Обсуждения.ОписаниеСообщения(Знач Текст) Экспорт`
— Функция → `Структура` (`Текст` — `ФорматированнаяСтрока` / `Вложения` —
`Массив` / `Данные` / `Действия`), регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`Обсуждения.ОписаниеВложения(Поток, Наименование) Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`Обсуждения.ОтправитьСообщение(Знач Автор, Знач Получатели, Сообщение, ОбсуждениеКонтекст = Неопределено) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`Обсуждения.ОбсужденияДоступны() Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ОбсужденияКлиент.ПоказатьПодключение(ОписаниеЗавершения = Неопределено) Экспорт`
`ОбсужденияКлиент.ПоказатьОтключение() Экспорт`
— Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `Текст` (Строка / `ФорматированнаяСтрока`) — текст сообщения.
- `Автор` (`СправочникСсылка.Пользователи` /
  `ПользовательСистемыВзаимодействия`).
- `Получатели` (`Массив` из `СправочникСсылка.Пользователи` /
  `ПользовательСистемыВзаимодействия`).
- `Сообщение` (Структура — из `ОписаниеСообщения`).
- `ОбсуждениеКонтекст` (`ЛюбаяСсылка` — контекстное обсуждение, привязано к
  объекту; `ИдентификаторОбсужденияСистемыВзаимодействия` — в существующее;
  `Неопределено` — негрупповое 1:1 при одном получателе, групповое при
  нескольких).

**Пример:**
```bsl
Если Не Обсуждения.ОбсужденияДоступны() Тогда
    Возврат;  // система взаимодействия не подключена
КонецЕсли;

Описание = Обсуждения.ОписаниеСообщения(
    НСтр("ru = 'Документ согласован, можно приступать к отгрузке.'"));
// Описание.Вложения.Добавить(Обсуждения.ОписаниеВложения(ПотокФайла, "akt.pdf"));

МассивПолучателей = Новый Массив;
МассивПолучателей.Добавить(Ответственный);

Обсуждения.ОтправитьСообщение(
    Пользователи.ТекущийПользователь(),  // автор
    МассивПолучателей,                    // получатели
    Описание,                             // сообщение (структура, не строка!)
    ЗаказКлиента);                        // контекст — обсуждение привязано к документу
```

**Нюансы / антипаттерны:**
- ❌ `Обсуждения.ОтправитьСообщение(Автор, Получатели, "Привет!")` — `Сообщение`
  имеет тип `Структура` (из `ОписаниеСообщения`), голая строка не пройдёт.
- `ОтправитьСообщение` бросает исключение, если не удалось отправить —
  оборачивайте в `Попытка / Исключение` при необходимости.
- Перед отправкой проверяйте `ОбсужденияДоступны()` (метод проверяет
  `ИспользованиеДоступно` системы взаимодействия и отсутствие блокировки
  администратором).
- Подключение/отключение обсуждений из UI — `ОбсужденияКлиент.ПоказатьПодключение`
  / `ПоказатьОтключение`.

### 7. Взаимодействия: хранение писем и SMS в базе

**Задача:** включить хранение входящих/исходящих писем и SMS в базе как
объектов взаимодействия, привязанных к предмету (документ, контрагент).

**Функции:**
`Взаимодействия.ИспользуетсяПочтовыйКлиент() Экспорт`
`Взаимодействия.ИспользуютсяПрочиеВзаимодействия() Экспорт`
— Функции → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`Взаимодействия.УстановитьИспользованиеПочтовогоКлиента(Знач Значение) Экспорт`
`Взаимодействия.УстановитьИспользованиеПрочегоВзаимодействия(Знач Значение) Экспорт`
— Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`Взаимодействия.ПредметВзаимодействия(Взаимодействие) Экспорт`
— Функция, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Значение` (Булево) — включить/выключить использование подсистемы.
- `Взаимодействие` (`СправочникСсылка` / `ДокументСсылка` взаимодействия) — для
  `ПредметВзаимодействия` возвращает предмет, к которому привязано.

**Пример:**
```bsl
// Проверить, хранятся ли письма в базе
Если Взаимодействия.ИспользуетсяПочтовыйКлиент() Тогда
    // отправленные через РаботаСПочтовымиСообщениями письма
    // сохраняются как объекты взаимодействия и привязываются к предмету
КонецЕсли;

// Предмет, к которому привязано письмо-взаимодействие
Предмет = Взаимодействия.ПредметВзаимодействия(ПисьмоСсылка);
```

**Нюансы / антипаттерны:**
- ⚠️ Большинство низкоуровневых методов `Взаимодействия` (создание письма, SMS,
  контакты, поиск по email) — в регионе `СлужебныйПрограммныйИнтерфейс` /
  `СлужебныеПроцедурыИФункции`, обратная совместимость не гарантируется. Для
  отправки используйте стабильный `РаботаСПочтовымиСообщениями.ОтправитьПисьмо`
  (при включённом почтовом клиенте письмо само сохранится как взаимодействие).
- `Взаимодействия.СоздатьПисьмо` / `СоздатьИОтправитьСообщениеSMS` — служебные
  (`СлужебныйПрограммныйИнтерфейс`); не использовать как основной способ —
  стабильный аналог: `ШаблоныСообщений.СформироватьСообщениеИОтправить` или
  `РаботаСПочтовымиСообщениями.ОтправитьПисьмо`.

## Редкие методы

Прочие стабильные методы (регион `ПрограммныйИнтерфейс`), полные сигнатуры —
через `python scripts/bsp_api.py method <Имя> [--module <М>] --src src/cf`:

- `РаботаСПочтовымиСообщениями.УчетнаяЗаписьНастроена(УчетнаяЗапись,
  ДляОтправки, ДляПолучения)` — настроена ли конкретная учётка.
- `РаботаСПочтовымиСообщениями.ПодготовитьПисьмо(УчетнаяЗапись,
  ПараметрыПисьма)` / `ПодключениеКПочте(УчетнаяЗапись, ДляПолучения = Ложь)` —
  низкоуровневая подготовка письма/соединения.
- `ОтправкаSMS.СтатусыДоставки(ИдентификаторыСообщений)` — ⚠️
  `СлужебныйПрограммныйИнтерфейс`; пакетный запрос статусов (стабильный аналог —
  `СтатусДоставки` по одному).
- `ШаблоныСообщений.ОписаниеПараметровШаблона()` / `ПараметрыШаблона(Шаблон)` /
  `ТаблицаПараметров()` — описание и параметры шаблона.
- `ШаблоныСообщений.ИнициализироватьСтруктуруСообщения()` /
  `ИнициализироватьСтруктуруПолучатели()` — фабрики структур для ручного
  формирования.
- `Обсуждения.ПользовательСистемыВзаимодействия(Пользователь,
  ТолькоИдентификатор = Ложь)` / `ПользовательИнформационнойБазы(...)` /
  `ПользователиСистемыВзаимодействия(...)` — маппинг между пользователями ИБ и
  системы взаимодействия.
- Хуки `*Переопределяемый` (`РаботаСПочтовымиСообщениямиПереопределяемый.ПослеОтправкиПисьма`,
  `ОтправкаSMSПереопределяемый.ОтправитьSMS`,
  `ШаблоныСообщенийПереопределяемый.ПриФормированииСообщения`) — реализуются во
  внедрении, БСП вызывает сама; не вызывать напрямую.
