# Внешние компоненты и интерфейс OData

Подсистема **ВнешниеКомпоненты** — подключение/установка внешних компонент
технологии Native API и COM на клиенте и сервере (сканеры, ККТ, терминалы сбора
данных и т. п.). Подсистема **ИнтерфейсOData** — стандартный REST-интерфейс
платформы 1С; из прикладного кода используется единственный хук переопределения
зависимых таблиц.

Прямой вызов произвольных COM-объектов (`Новый COMОбъект(...)`) — **платформенный**
механизм, БСП его не оборачивает. Скил покрывает только тот COM-сценарий, который
сама подсистема `ВнешниеКомпоненты` использует для обратной совместимости с
компонентами 1С 7.7 (`ПодключитьКомпонентуИзРеестраWindows`).

## Модули

ВнешниеКомпоненты:

- `ВнешниеКомпонентыСервер` — серверный стабильный API: подключение, информация,
  используемые компоненты, обновление. регион `ПрограммныйИнтерфейс` (stable).
- `ВнешниеКомпонентыКлиент` — асинхронное подключение/установка/загрузка из файла
  (с диалогами пользователя). Клиент, stable.
- `ВнешниеКомпонентыКлиентЛокализация` — поиск/обновление компонент с портала.
- `ВнешниеКомпонентыВызовСервера` — ⚠️ `ИнформацияОКомпоненте` здесь в регионе
  `УстаревшиеПроцедурыИФункции` (deprecated). Используйте
  `ВнешниеКомпонентыСервер.ИнформацияОКомпоненте` (через `&НаСервере`).
- `ВнешниеКомпонентыСлужебный` / `…СлужебныйКлиент` / `…СлужебныйВызовСервера` /
  `…ВМоделиСервисаСлужебный` / `…ВМоделиСервисаСлужебныйКлиент` — ⚠️ служебные,
  обратная совместимость не гарантируется.

ИнтерфейсOData:

- `ИнтерфейсODataПереопределяемый` — **единственный стабильный хук** для прикладного
  кода: переопределение зависимых таблиц OData. Метод — хук (БСП вызывает,
  прикладной код реализует, не вызывается напрямую).
- `ИнтерфейсODataСлужебный` / `ИнтерфейсODataСлужебныйПовтИсп` — ⚠️ служебные:
  формирование модели по метаданным, кэш. Прямого стабильного API для OData из
  прикладного кода почти нет — только переопределение и настройка через
  Конфигуратор (роль `РольИнтерфейсаOData`).

> **Клиент/сервер.** Клиентские методы — **асинхронные**, через `ОписаниеОповещения`.
> Серверные — **синхронные**, возвращают `Структура` с `Подключено` (`Булево`) и
> `ПодключаемыйМодуль` (`ОбъектВнешнейКомпоненты`).

## Сценарии

### 1. Подключить компоненту на клиенте (асинхронно)

**Задача:** подключить Native API/COM-компоненту на клиентском компьютере с
пояснением пользователю и обработкой результата в оповещении.

**Функции:**
`ВнешниеКомпонентыКлиент.ПараметрыПодключения() Экспорт`
— Функция → `Структура` (`Кэшировать, ПредложитьУстановить, ПредложитьЗагрузить, ТекстПояснения, ИдентификаторыСозданияОбъектов, Изолированно, ОбновлятьАвтоматически`), регион `ПрограммныйИнтерфейс` (stable). Клиент.
`ВнешниеКомпонентыКлиент.ПодключитьКомпоненту(Оповещение, Идентификатор, Версия = Неопределено, ПараметрыПодключения = Неопределено) Экспорт`
— Процедура (асинхронная), регион `ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `Оповещение` (`ОписаниеОповещения`) — обработчик `Результат` (`Структура`:
  `Подключено` (`Булево`), `ПодключаемыйМодуль` (`ОбъектВнешнейКомпоненты` /
  `ФиксированноеСоответствие` при `ИдентификаторыСозданияОбъектов`), `ОписаниеОшибки`).
- `Идентификатор` (`Строка`) — идентификатор объекта внешней компоненты.
- `Версия` (`Строка` / `Неопределено`) — `Неопределено` = последняя доступная.
- `ПараметрыПодключения` (`Структура` из `ПараметрыПодключения`). `ТекстПояснения` —
  зачем нужна компонента; `ПредложитьУстановить` / `ПредложитьЗагрузить` — БСП сама
  покажет диалоги в тонком/веб-клиенте.

**Пример:**
```bsl
&НаКлиенте
Процедура ПодключитьСканер(Команда)
    Параметры = ВнешниеКомпонентыКлиент.ПараметрыПодключения();
    Параметры.ТекстПояснения =
        НСтр("ru = 'Для работы со сканером требуется внешняя компонента (NativeApi).'");

    ВнешниеКомпонентыКлиент.ПодключитьКомпоненту(
        Новый ОписаниеОповещения("ПодключитьСканерЗавершение", ЭтотОбъект),
        "InputDevice", , Параметры);
КонецПроцедуры

&НаКлиенте
Процедура ПодключитьСканерЗавершение(Результат, ДопПараметры) Экспорт
    Если Результат.Подключено Тогда
        ПодключаемыйМодуль = Результат.ПодключаемыйМодуль;   // ОбъектВнешнейКомпоненты
        // далее — вызовы методов конкретной компоненты
    ИначеЕсли НЕ ПустаяСтрока(Результат.ОписаниеОшибки) Тогда
        ПоказатьПредупреждение(, Результат.ОписаниеОшибки);
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ `Результат = ВнешниеКомпонентыКлиент.ПодключитьКомпоненту(...)` — метод
  возвращает `Неопределено`; результат приходит в `ОписаниеОповещения`. Клиентский
  вариант **всегда асинхронный**.
- ❌ `Новый COMОбъект("InputDevice.BarcodeScanner")` в обход подсистемы — не пройдёт
  проверки безопасности, не обновится автоматически, не будет работать в модели
  сервиса. Только через `ПодключитьКомпоненту`.

### 2. Подключить компоненту на сервере (синхронно)

**Задача:** подключить компоненту в серверном коде (фоновые задания, регламентная
обработка) — синхронно, с immediate результатом.

**Функции:**
`ВнешниеКомпонентыСервер.ПараметрыПодключения() Экспорт`
— Функция → `Структура` (`ИдентификаторыСозданияОбъектов, Изолированно, ПолноеИмяМакета`), регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ВнешниеКомпонентыСервер.ПодключитьКомпоненту(Знач Идентификатор, Версия = Неопределено, ПараметрыПодключения = Неопределено) Экспорт`
— Функция → `Структура` (`Подключено, ПодключаемыйМодуль, ОписаниеОшибки`), stable. Сервер.

**Параметры:**
- `Идентификатор` (`Строка`), `Версия` (`Строка` / `Неопределено`).
- `ПараметрыПодключения` (`Структура` из `ПараметрыПодключения`):
  `ИдентификаторыСозданияОбъектов` (`Массив` из `Строка`) — для компонент с
  несколькими идентификаторами создания объектов; `Изолированно` (`Булево` /
  `Неопределено`) — `Истина` загружает в отдельный процесс ОС; `ПолноеИмяМакета`
  (`Строка`) — путь к общему макету конфигурации (`"ОбщийМакет.КомпонентаСканера"`).

**Пример:**
```bsl
// Сервер (фоновое задание)
Результат = ВнешниеКомпонентыСервер.ПодключитьКомпоненту("InputDevice", , );

Если Результат.Подключено Тогда
    Попытка
        Результат.ПодключаемыйМодуль.Подключить("COM", 0);
    Исключение
        // Записать в журнал регистрации
    КонецПопытки;
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Передавать `ПолноеИмяМакета` через `ВнешниеКомпонентыКлиент.ПараметрыПодключения()` —
  этого поля в клиентском конструкторе **нет**, оно только в серверном
  `ВнешниеКомпонентыСервер.ПараметрыПодключения()`.
- В **модели сервиса** разрешено только подключение **общих** внешних компонент,
  одобренных администратором сервиса.
- `ПодключаемыйМодуль` доступен до конца серверного вызова; явно отключать не нужно.

### 3. Установить компоненту с портала ИТС

**Задача:** если компонента не установлена, предложить пользователю установить её
с портала ИТС или из общего макета.

**Функции:**
`ВнешниеКомпонентыКлиент.ПараметрыУстановки() Экспорт`
— Функция → `Структура` (`ТекстПояснения, ПредложитьЗагрузить, ПредложитьУстановить`), stable. Клиент.
`ВнешниеКомпонентыКлиент.УстановитьКомпоненту(Оповещение, Идентификатор, Версия = Неопределено, ПараметрыУстановки = Неопределено) Экспорт`
— Процедура (асинхронная), stable. Клиент.

**Параметры:**
- `Оповещение` (`ОписаниеОповещения`) — `Результат` (`Структура`:
  `Установлено` (`Булево`), `ОписаниеОшибки` (`Строка`; пустая при отмене пользователем)).
- `Идентификатор` (`Строка`), `Версия` (`Строка` / `Неопределено`).
- `ПараметрыУстановки` (`Структура` из `ПараметрыУстановки`). `ПредложитьЗагрузить` —
  предложить загрузить с сайта ИТС; `ПредложитьУстановить` (по умолчанию `Ложь`).

**Пример:**
```bsl
&НаКлиенте
Процедура УстановитьСканерПриНеобходимости()
    ПараметрыУстановки = ВнешниеКомпонентыКлиент.ПараметрыУстановки();
    ПараметрыУстановки.ТекстПояснения = НСтр("ru = 'Требуется установить компоненту сканера.'");
    ПараметрыУстановки.ПредложитьЗагрузить = Истина;

    ВнешниеКомпонентыКлиент.УстановитьКомпоненту(
        Новый ОписаниеОповещения("УстановкаЗавершение", ЭтотОбъект),
        "InputDevice", , ПараметрыУстановки);
КонецПроцедуры

&НаКлиенте
Процедура УстановкаЗавершение(Результат, ДопПараметры) Экспорт
    Если НЕ Результат.Установлено И НЕ ПустаяСтрока(Результат.ОписаниеОшибки) Тогда
        ПоказатьПредупреждение(, Результат.ОписаниеОшибки);
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Ждать результат `УстановитьКомпоненту` как возврат значения — асинхронный,
  результат в оповещении.
- В тонком/веб-клиенте БСП сама показывает диалоги установки/загрузки —
  управляется флагами `ПредложитьУстановить` / `ПредложитьЗагрузить`.

### 4. Загрузить файл компоненты в справочник

**Задача:** администратор загружает `.zip` компоненты в справочник
`ВнешниеКомпоненты` из локального файла.

**Функции:**
`ВнешниеКомпонентыКлиент.ПараметрыЗагрузки() Экспорт`
— Функция → `Структура` (`Идентификатор, Версия, ПараметрыПоискаДополнительнойИнформации`), stable. Клиент.
`ВнешниеКомпонентыКлиент.ЗагрузитьКомпонентуИзФайла(Оповещение, ПараметрыЗагрузки = Неопределено) Экспорт`
— Процедура (асинхронная), stable. Клиент.

**Параметры:**
- `Оповещение` (`ОписаниеОповещения`) — `Результат` (`Структура`: `Загружена`
  (`Булево`), `Идентификатор`, `Версия`, `Наименование`, `ДополнительнаяИнформация`).
- `ПараметрыЗагрузки` (`Структура` из `ПараметрыЗагрузки`): `Идентификатор` /
  `Версия` — необязательные; `ПараметрыПоискаДополнительнойИнформации` —
  `Соответствие` для запроса доп. информации о компоненте.

**Пример:**
```bsl
&НаКлиенте
Процедура ЗагрузитьКомпонентуИзФайла(Команда)
    Параметры = ВнешниеКомпонентыКлиент.ПараметрыЗагрузки();
    // Идентификатор/Версия необязательны — определятся из файла

    ВнешниеКомпонентыКлиент.ЗагрузитьКомпонентуИзФайла(
        Новый ОписаниеОповещения("ЗагрузкаЗавершение", ЭтотОбъект), Параметры);
КонецПроцедуры

&НаКлиенте
Процедура ЗагрузкаЗавершение(Результат, ДопПараметры) Экспорт
    Если Результат.Загружена Тогда
        // Результат.Идентификатор, Результат.Версия, Результат.Наименование
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- Метод для администраторов (загрузка в справочник `ВнешниеКомпоненты`); для
  прикладного сценария «подключить и использовать» достаточно `ПодключитьКомпоненту`
  с `ПредложитьУстановить = Истина`.

### 5. Получить информацию о компоненте и список используемых

**Задача:** до подключения проверить, существует ли компонента с указанным
идентификатором/версией и доступна ли на редактирование; получить список
компонент конфигурации.

**Функции:**
`ВнешниеКомпонентыСервер.ИнформацияОКомпоненте(Знач Идентификатор, Знач Версия = Неопределено) Экспорт`
— Функция → `Структура` (`Существует, ДоступноРедактирование, Идентификатор, Версия, Наименование, ОписаниеОшибки`), регион `ПрограммныйИнтерфейс` (stable). Сервер.
`ВнешниеКомпонентыСервер.ИспользуемыеКомпоненты(Вариант) Экспорт`
— Функция → `ТаблицаЗначений` (`Идентификатор, Версия, Наименование, ДатаВерсии`), stable. Сервер.

**Параметры:**
- `Идентификатор` (`Строка`), `Версия` (`Строка` / `Неопределено`).
- `Вариант` (`Строка`): `"ДляОбновления"` — с признаком обновления из Интернета;
  `"ДляЗагрузки"` — используемые в конфигурации; `"Поставляемые"` — поставляемые
  компоненты в модели сервиса.

**Пример:**
```bsl
&НаСервере
Функция КомпонентаДоступна(Идентификатор)
    Инфо = ВнешниеКомпонентыСервер.ИнформацияОКомпоненте(Идентификатор);
    Возврат Инфо.Существует;
КонецФункции

// Список компонент, используемых в конфигурации
Таблица = ВнешниеКомпонентыСервер.ИспользуемыеКомпоненты("ДляЗагрузки");
```

**Нюансы / антипаттерны:**
- ❌ `ВнешниеКомпонентыВызовСервера.ИнформацияОКомпоненте(...)` — ⚠️ устаревший
  (регион `УстаревшиеПроцедурыИФункции`), док-комментарий явно указывает:
  «Устарела. Следует использовать `ВнешниеКомпонентыСервер.ИнформацияОКомпоненте`».
  В новом коде — серверный метод через `&НаСервере`.
- ❌ `ВнешниеКомпонентыСлужебный.ДанныеВнешнихКомпонент("ДляОбновления")` —
  служебный модуль. Стабильный аналог — `ВнешниеКомпонентыСервер.ИспользуемыеКомпоненты("ДляОбновления")`.

### 6. Переопределить зависимые таблицы OData

**Задача:** добавить свои объекты в список таблиц, права на которые необходимы для
записи таблиц, включённых в стандартный интерфейс OData (чтобы при выгрузке
объекта автоматически подтягивались связанные).

**Процедура (хук):**
`ИнтерфейсODataПереопределяемый.ПриЗаполненииЗависимыхТаблицДляВыгрузкиЗагрузкиOData(Таблицы) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс`. **Хук переопределения**: реализуется в
одноимённом модуле прикладной конфигурации, БСП вызывает при формировании модели.

**Параметры:**
- `Таблицы` (`Массив` из `Строка`) — полные имена объектов метаданных. Массив
  изменяется в теле процедуры (добавляются свои таблицы).

**Пример:**
```bsl
// В модуле ИнтерфейсODataПереопределяемый прикладной конфигурации
Процедура ПриЗаполненииЗависимыхТаблицДляВыгрузкиЗагрузкиOData(Таблицы) Экспорт
    // Таблицы, не входящие в выгрузку OData, но права на которые нужны
    // для записи таблиц, включённых в интерфейс
    Таблицы.Добавить("РегистрСведений.СостоянияЗаказов");
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `ПриЗаполненииЗависимыхТаблицДляВыгрузкиЗагрузкиOData` из прикладного
  кода напрямую — это хук, БСП сама его вызывает. Только реализовать тело.
- ❌ `ИнтерфейсODataСлужебныйПовтИсп.ОписаниеМоделиДанныхКонфигурации()` — служебный
  модуль; модель пересобирается при обновлении. Стабильная точка расширения —
  только `ИнтерфейсODataПереопределяемый`.
- Права/роли на стандартный интерфейс OData настраиваются в Конфигураторе
  (роль `РольИнтерфейсаOData`), не через программный код.

## Дополнительно

Прочие стабильные методы (регион `ПрограммныйИнтерфейс`), полные сигнатуры — через
`python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`:

- `ВнешниеКомпонентыСервер.ОбновитьВнешниеКомпоненты(ДанныеВнешнихКомпонент, АдресРезультата = Неопределено)` —
  обновление компонент (для регламентных обработчиков обновления).
- `ВнешниеКомпонентыСервер.ОписаниеПоставляемойОбщейКомпоненты()` /
  `ОбновитьОбщуюКомпоненту(ОписаниеКомпоненты)` — общие компоненты в модели сервиса.
- `ВнешниеКомпонентыСервер.АвтоматическиОбновляемыеКомпоненты()` — список
  компонент с признаком автоматического обновления.
- `ВнешниеКомпонентыКлиент.ПодключитьКомпонентуИзРеестраWindows(Оповещение, Идентификатор, ИдентификаторСозданияОбъекта = Неопределено)` —
  подключение COM-компоненты из реестра Windows (обратная совместимость с 1С 7.7).
- `ВнешниеКомпонентыКлиент.ПараметрыПоискаДополнительнойИнформации()` — параметры
  запроса доп. информации о компоненте (для `ПараметрыЗагрузки`).

⚠️ Служебные модули (не использовать в прикладном коде — обратная совместимость не
гарантируется): `ВнешниеКомпонентыСлужебный`, `…СлужебныйКлиент`,
`…СлужебныйВызовСервера`, `…ВМоделиСервисаСлужебный`, `…ВМоделиСервисаСлужебныйКлиент`,
`ИнтерфейсODataСлужебный`, `ИнтерфейсODataСлужебныйПовтИсп`. Если нужен
функционал из них — ищите стабильный аналог в `ВнешниеКомпонентыСервер` /
`ВнешниеКомпонентыКлиент` или через хук `ИнтерфейсODataПереопределяемый`.
