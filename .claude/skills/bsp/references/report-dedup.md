# Поиск дублей, групповое изменение и структура подчинённости БСП

Три смежные подсистемы для **массовой обработки данных**:
**ПоискИУдалениеДублей** (программный и интерактивный поиск одинаковых
элементов, нечёткое сравнение строк, объединение, замена ссылок),
**ГрупповоеИзменениеОбъектов** (команда «Изменить выделенные» в формах списков,
групповая обработка реквизитов) и **СтруктураПодчиненности** (отчёт «Связанные
документы» — дерево родительских и дочерних объектов по ссылке). Подсистемы
связаны через обработку `ГрупповоеИзменениеРеквизитов` и механизм
подключаемых команд.

## Модули

- `ПоискИУдалениеДублей` — серверный API: программный поиск дублей, нечёткое
  сравнение строк, проверка допустимости и выполнение замены ссылок.
- `ПоискИУдалениеДублейКлиент` — клиентский API: формы объединения, замены,
  мест использования.
- `ПоискИУдалениеДублейПереопределяемый` — хуки: какие объекты участвуют в
  поиске, прикладные правила, whitelist методов для «произвольного алгоритма».
- `ГрупповоеИзменениеОбъектовКлиент` — клиентский API: `ИзменитьВыделенные`.
- `ГрупповоеИзменениеОбъектов` — серверный модуль с методами
  `ПриОпределенииВидовПодключаемыхКоманд` / `ПриОпределениеКомандПодключенныхКОбъекту`
  (вызываются БСП автоматически при подключении команд к формам).
- `ГрупповоеИзменениеОбъектовПереопределяемый` — хуки: в каких формах выводить
  команду «Изменить выделенные», состав редактируемых реквизитов.
- `СтруктураПодчиненностиПереопределяемый` — **единственный** модуль подсистемы
  «Структура подчинённости»; все методы — хуки переопределения. Сам отчёт
  `Связанные документы` (`Отчет.СвязанныеДокументы`) формируется БСП по ссылке
  на панели навигации/команде; стабильного программного API для формирования
  отчёта нет — только хуки для влияния на вывод.

⚠️ У подсистем `ПоискИУдалениеДублей` и `ГрупповоеИзменениеОбъектов` **нет**
модулей с суффиксами `Служебный` / `ВызовСервера` / `ПовтИсп` / `Глобальный` —
служебные методы встроены в основные модули под областями
`СлужебныйПрограммныйИнтерфейс` / `СлужебныеПроцедурыИФункции`.

⚠️ Типовые ошибки:
- `СтруктураПодчиненности` (без `Переопределяемый`) — модуль **не существует**.
  Только `СтруктураПодчиненностиПереопределяемый` (хуки).
- Для замены ссылок стабильный API — `ОбщегоНазначения.ЗаменитьСсылки` (см.
  `base-common.md`), а **не** служебный `ПоискИУдалениеДублей.ЗаменитьСсылки`.

## Сценарии

### 1. Программный поиск дублей элемента

**Задача:** найти все элементы справочника/плана счетов/плана видов
характеристик, похожие на эталон, перед объединением или для отчёта.

**Функция:**
`ПоискИУдалениеДублей.НайтиДублиЭлемента(Знач ОбластьПоиска, Знач ЭталонныйОбъект, Знач ДополнительныеПараметры) Экспорт`
— Функция → `ТаблицаЗначений` (колонки `Ссылка`, `Код`, `Наименование`,
  `Родитель`, `ДругиеПоля`), регион `#Область ПрограммныйИнтерфейс` (stable).
  Сервер.

**Параметры:**
- `ОбластьПоиска` (Строка) — полное имя метаданных области поиска, напр.
  `"Справочник.Номенклатура"`. Поддерживаются справочники, планы видов
  характеристик, виды расчётов, планы счетов.
- `ЭталонныйОбъект` (ЛюбаяСсылка / СправочникОбъект) — элемент, для которого
  ищутся дубли.
- `ДополнительныеПараметры` (Произвольный) — параметр для передачи в
  обработчики событий менеджера (`ПриОпределенииПараметровПоискаДублей`,
  `ПриПоискеДублей`).

**Пример:**
```bsl
Эталон = Справочники.Номенклатура.НайтиПоНаименованию("Стол обеденный");
Если Эталон.Пустая() Тогда
    Возврат;
КонецЕсли;

Дубли = ПоискИУдалениеДублей.НайтиДублиЭлемента(
    "Справочник.Номенклатура", Эталон, Неопределено);

ПарыЗамен = Новый Соответствие;
Для Каждого СтрокаДубля Из Дубли Цикл
    Если СтрокаДубля.Ссылка = Эталон Тогда
        Продолжить;  // пропускаем сам эталон
    КонецЕсли;
    ПарыЗамен.Вставить(СтрокаДубля.Ссылка, Эталон);
КонецЦикла;
```

**Нюансы / антипаттерны:**
- Метод сам строит схему компоновки по доступным реквизитам, инициализирует
  `КомпоновщикПредварительногоОтбора` и через обработку `ПоискИУдалениеДублей`
  получает группы дублей. Учитывает прикладные правила
  (`ПоискИУдалениеДублейПереопределяемый.ПриОпределенииПараметровПоискаДублей`)
  и пост-обработку (`ПриПоискеДублей`).
- `Родитель` пустой у элемента-родителя группы дублей; у самих дублей —
  ссылка на родителя группы.
- ❌ Искать дубли прямым запросом с `ГДЕ Наименование = …` — не учитывает
  нечёткие совпадения и прикладные правила поиска. Используйте
  `НайтиДублиЭлемента`.

### 2. Нечёткий поиск похожих строк (Левенштейн-Дамерау)

**Задача:** найти в массиве строк те, что похожи на заданную (опечатки,
разный порядок слов), с настраиваемым порогом совпадения и списком
слов-исключений (`ООО`, `ИП`).

**Функции:**
`ПоискИУдалениеДублей.НайтиПохожиеСтроки(ИсходнаяСтрока, СтрокаПоиска, Разделитель = "~", ПараметрыПоиска = Неопределено) Экспорт`
— Функция → `Строка` (индексы похожих строк через запятую, нумерация с `0`),
  регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.
`ПоискИУдалениеДублей.ПараметрыПоискаПохожихСтрок(ПодключитьКомпоненту = Истина) Экспорт`
— Функция → `Структура` (`ПроцентСовпаденияСтрок` = 90,
  `ПроцентСовпаденияНебольшихСтрок` = 80, `ДлинаНебольшихСтрок` = 30,
  `СловаИсключения` — Массив, `КомпонентаПоиска` — объект внешней компоненты
  `FuzzyStringMatchExtension`). Сервер.

**Параметры:**
- `ИсходнаяСтрока` (Строка) — набор строк через `Разделитель`, в которых
  ищем.
- `СтрокаПоиска` (Строка) — строка, вхождение которой ищем.
- `Разделитель` (Строка) — символ-разделитель в `ИсходнаяСтрока`; `"~"` по
  умолчанию.
- `ПараметрыПоиска` (Структура) — из `ПараметрыПоискаПохожихСтрок`;
  `Неопределено` — метод сам получит параметры (с подключением компоненты).
- `ПодключитьКомпоненту` (Булево) — `Истина` (по умолчанию) — подключить
  внешнюю компоненту поиска и вернуть её в `КомпонентаПоиска`; `Ложь` — если
  компонента уже подключена выше по стеку.

**Пример:**
```bsl
// Оптимизация для массовых вызовов: один раз получаем параметры с компонентой
Параметры = ПоискИУдалениеДублей.ПараметрыПоискаПохожихСтрок();  // ПодключитьКомпоненту = Истина

ИсходнаяСтрока = "Токарный станок~Вентилятор настольный~Стол обеденный~Стул обеденный";
Для Каждого Запрос Из МассивЗапросов Цикл
    Индексы = ПоискИУдалениеДублей.НайтиПохожиеСтроки(ИсходнаяСтрока, Запрос, "~", Параметры);
    // Напр. для "Стол обеденный" -> "2,3" (индексы "Стол обеденный" и "Стул обеденный")
КонецЦикла;
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `НайтиПохожиеСтроки` в цикле **без** `ПараметрыПоиска` — каждый
  вызов заново подключает внешнюю компоненту (дорого). Один раз получите
  параметры через `ПараметрыПоискаПохожихСтрок()` и передавайте их в цикл.
- `ПараметрыПоискаПохожихСтрок(Ложь)` — валидный сценарий, если компонента уже
  подключена где-то выше; возвращает структуру без повторного подключения.
- Алгоритм игнорирует регистр и порядок слов; `СловаИсключения` (`"ООО"`,
  `"ИП"`, `"ОАО"`) исключаются из сравнения.

### 3. Заменить ссылки во всей ИБ

**Задача:** программно заменить один элемент на другой везде, где он
используется (документы, регистры, реквизиты), с пометкой/удалением дубля и
контролем прикладных правил замены.

**Функции:**
`ОбщегоНазначения.ЗаменитьСсылки(Знач ПарыЗамен, Знач ПараметрыЗамены = Неопределено) Экспорт`
— Функция → `ТаблицаЗначений` ошибок (колонки `Ссылка`, `ОбъектОшибки`,
  `ТипОшибки`, `ИнформацияОбОшибке`, `ТекстОшибки`), регион
  `#Область ПрограммныйИнтерфейс` (stable). Сервер. **Это стабильный
  рекомендуемый путь.**
`ОбщегоНазначения.ПараметрыЗаменыСсылок() Экспорт`
— конструктор структуры `ПараметрыЗамены` (`СпособУдаления` = `""` /
  `"Пометка"` / `"Непосредственно"`; `УчитыватьПрикладныеПравила` — Булево;
  `ВключатьБизнесЛогику` — Булево). Сервер.
`ПоискИУдалениеДублей.ПроверитьВозможностьЗаменыЭлементов(ПарыЗамен, ПараметрыЗамены) Экспорт`
— Функция → `Соответствие` ошибок (пустое, если замена разрешена); ⚠️
  `СлужебныйПрограммныйИнтерфейс`. Опрашивает `ВозможностьЗаменыЭлементов`
  модуля менеджера объекта и
  `ПоискИУдалениеДублейПереопределяемый.ПриОпределенииВозможностиЗаменыЭлементов`.
  Сервер.
`ПоискИУдалениеДублей.ДополнитьДублиСвязаннымиПодчиненнымиОбъектами(ПарыЗамен, ПараметрыЗамены) Экспорт`
— Процедура, регион `#Область ПрограммныйИнтерфейс` (stable). Дополняет
  `ПарыЗамен` связанными подчинёнными объектами (напр., при замене
  контрагента — заменить и его банковский счёт). Сервер.
`ПоискИУдалениеДублей.ЗаменитьСсылки(Параметры, Знач АдресРезультата) Экспорт`
— Процедура, ⚠️ `СлужебныеПроцедурыИФункции`. Сервер. Обёртка над
  `ОбщегоНазначения.ЗаменитьСсылки` с принудительным
  `ВключатьБизнесЛогику = Истина`; результат (таблица ошибок) помещается в
  `АдресРезультата` (временное хранилище) — для запуска в фоновом задании.

**Параметры:**
- `ПарыЗамен` (Соответствие) — `Ключ` = что заменяем (дубль, `ЛюбаяСсылка`),
  `Значение` = на что заменяем (оригинал). Ссылки сами на себя и пустые
  игнорируются.
- `ПараметрыЗамены` (Структура) — из `ПараметрыЗаменыСсылок()`:
  - `СпособУдаления` (Строка): `""` — не трогать дубль; `"Пометка"` — пометить
    на удаление; `"Непосредственно"` — удалить.
  - `УчитыватьПрикладныеПравила` (Булево): `Истина` — для каждой пары
    вызывается `ВозможностьЗаменыЭлементов` модуля менеджера (требует подсистему
    «Поиск и удаление дублей»); `Ложь` по умолчанию.
  - `ВключатьБизнесЛогику` (Булево): `Истина` (по умолчанию) — места
    использования записываются в обычном режиме; `Ложь` — в режиме
    `ОбменДанными.Загрузка = Истина` (без перепроведения).

**Пример:**
```bsl
ПарыЗамен = Новый Соответствие;
ПарыЗамен.Вставить(ДублирующийЭлемент, ОригинальныйЭлемент);

ПараметрыЗамены = ОбщегоНазначения.ПараметрыЗаменыСсылок();
ПараметрыЗамены.СпособУдаления = "Пометка";
ПараметрыЗамены.УчитыватьПрикладныеПравила = Истина;  // проверка ВозможностьЗаменыЭлементов

// 1) Проверить допустимость (прикладные правила: запрет менять валюту в проведённых документах и т. п.)
Ошибки = ПоискИУдалениеДублей.ПроверитьВозможностьЗаменыЭлементов(ПарыЗамен, ПараметрыЗамены);
Если Ошибки.Количество() > 0 Тогда
    // ...вывести ошибки через ОбщегоНазначения.СообщитьПользователю
    Возврат;
КонецЕсли;

// 2) Дополнить связанными подчинёнными объектами (банковский счёт, ключи аналитики)
ПоискИУдалениеДублей.ДополнитьДублиСвязаннымиПодчиненнымиОбъектами(ПарыЗамен, ПараметрыЗамены);

// 3) Выполнить замену (стабильный API возвращает таблицу ошибок напрямую)
ОшибкиЗамены = ОбщегоНазначения.ЗаменитьСсылки(ПарыЗамен, ПараметрыЗамены);
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `ПоискИУдалениеДублей.ЗаменитьСсылки` как основной путь — это
  ⚠️ служебный метод (`СлужебныеПроцедурыИФункции`), обратная совместимость не
  гарантируется. Стабильный API — `ОбщегоНазначения.ЗаменитьСсылки`.
  Служебный вариант нужен только для запуска в фоновом задании, когда
  результат надо положить во временное хранилище по `АдресРезультата` (см.
  `longs-and-jobs.md`, длительные операции).
- ❌ Выполнять `ЗаменитьСсылки` без `ПроверитьВозможностьЗаменыЭлементов`
  (когда `УчитыватьПрикладныеПравила = Истина`) — замена обойдёт прикладные
  правила (запрет менять валюту в проведённых документах, запрет менять
  организацию при закрытом периоде). Сначала проверка, потом замена.
- Замена происходит **с транзакциями по изменяемому объекту и его связям**, не
  по анализируемой ссылке. При вызове в неразделённом сеансе ссылки в
  разделённых областях не выявляются.
- Для запуска замены в фоне с прогрессом и отменой — оберните вызов через
  `ДлительныеОперации` (см. `longs-and-jobs.md`).

### 4. Команды объединения, замены и мест использования в форме списка

**Задача:** в форме списка добавить команды «Объединить выделенные»,
«Заменить выделенные», «Места использования» (через обработчики подключаемых
команд).

**Функции:**
`ПоискИУдалениеДублейКлиент.ОбъединитьВыделенные(Знач ОбъединяемыеЭлементы, ДополнительныеПараметры = Неопределено) Экспорт`
— открывает форму мастера объединения элементов. Клиент.
`ПоискИУдалениеДублейКлиент.ЗаменитьВыделенные(Знач ЗаменяемыеЭлементы, ДополнительныеПараметры = Неопределено) Экспорт`
— открывает форму замены и удаления. Клиент.
`ПоискИУдалениеДублейКлиент.ПоказатьМестаИспользования(Знач Элементы, Знач ПараметрыОткрытия = Неопределено) Экспорт`
— открывает отчёт `МестаИспользованияСсылок` с отбором по переданным ссылкам;
  вспомогательные данные (наборы записей с ведущим измерением) **не
  включаются**. Клиент.
— все Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable).

**Параметры:**
- `ОбъединяемыеЭлементы` / `ЗаменяемыеЭлементы` (`ТаблицаФормы` / `Массив из
  ЛюбаяСсылка` / `СписокЗначений` с реквизитом `Ссылка`) — выделенные элементы
  списка.
- `Элементы` (тот же тип) — ссылки для отчёта по местам использования.
- `ПараметрыОткрытия` (Структура) — опц. параметры открытия формы отчёта.

**Пример:**
```bsl
// Модуль формы списка справочника
&НаКлиенте
Процедура Подключаемый_ОбъединитьВыделенные(Команда)
    ПоискИУдалениеДублейКлиент.ОбъединитьВыделенные(Элементы.Список);
КонецПроцедуры

&НаКлиенте
Процедура Подключаемый_ЗаменитьВыделенные(Команда)
    ПоискИУдалениеДублейКлиент.ЗаменитьВыделенные(Элементы.Список);
КонецПроцедуры

&НаКлиенте
Процедура Подключаемый_ПоказатьМестаИспользования(Команда)
    ПоискИУдалениеДублейКлиент.ПоказатьМестаИспользования(Элементы.Список);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Добавлять эти кнопки в форму вручную при подключённой подсистеме
  `ПодключаемыеКоманды` — дублирование. Подключаемые команды уже добавляют их
  автоматически через
  `ПоискИУдалениеДублей.ПриОпределенииКомандПодключенныхКОбъекту` (⚠️
  `СлужебныйПрограммныйИнтерфейс`, вызывается БСП). Если `ПодключаемыеКоманды`
  не используются — добавьте кнопки вручную, как в примере.
- `ОбъединитьВыделенные` открывает мастер сопоставления реквизитов;
  пользователь подтверждает замену. `ЗаменитьВыделенные` — форму выбора новых
  значений и пометки/удаления исходных.

### 5. Команда «Изменить выделенные» и ограничение редактируемых реквизитов

**Задача:** в форме списка добавить команду группового изменения реквизитов и
ограничить состав реквизитов, доступных для группового изменения конкретного
объекта (напр., запретить менять `ИНН`/`КПП` контрагента массово).

**Функция:**
`ГрупповоеИзменениеОбъектовКлиент.ИзменитьВыделенные(СписокЭлемент, Знач СписокРеквизит = Неопределено) Экспорт`
— Процедура, открывает форму `Обработка.ГрупповоеИзменениеРеквизитов` для
  выделенных строк. Регион `#Область ПрограммныйИнтерфейс` (stable). Клиент.

**Параметры:**
- `СписокЭлемент` (`ТаблицаФормы` / `Массив из ЛюбаяСсылка`) — элемент формы со
  списком.
- `СписокРеквизит` (`ДинамическийСписок`) — реквизит формы со списком;
  опционально, для передачи отбора.

**Пример:**
```bsl
&НаКлиенте
Процедура Подключаемый_ИзменитьВыделенные(Команда)
    // Проверить право "Изменение" до открытия формы
    Если Не ПравоДоступа("Изменение", Метаданные.Справочники.Контрагенты) Тогда
        Возврат;
    КонецЕсли;

    // Проверить, что нет помеченных на удаление среди выделенных
    МожноРедактировать = Истина;
    Для Каждого ВыделеннаяСтрока Из Элементы.Список.ВыделенныеСтроки Цикл
        ТекДанные = Элементы.Список.ДанныеСтроки(ВыделеннаяСтрока);
        Если ТекДанные <> Неопределено И Не ТекДанные.Ссылка.Пустая()
            И ТекДанные.Ссылка.ПометкаУдаления Тогда
            МожноРедактировать = Ложь;
            Прервать;
        КонецЕсли;
    КонецЦикла;

    Если МожноРедактировать Тогда
        ГрупповоеИзменениеОбъектовКлиент.ИзменитьВыделенные(Элементы.Список, Список);
    КонецЕсли;
КонецПроцедуры
```

Ограничение редактируемых реквизитов — через модуль менеджера и хук:

```bsl
// Модуль менеджера справочника Контрагенты
Функция РеквизитыНеРедактируемыеВГрупповойОбработке() Экспорт
    Результат = Новый Массив;
    Результат.Добавить("ИНН");
    Результат.Добавить("КПП");
    Результат.Добавить("ОсновнойДоговорКонтрагента");
    Возврат Результат;
КонецФункции

// ГрупповоеИзменениеОбъектовПереопределяемый (модуль прикладной конфигурации)
Процедура ПриОпределенииОбъектовСРедактируемымиРеквизитами(Объекты) Экспорт
    Объекты.Вставить(Метаданные.Справочники.Контрагенты.ПолноеИмя(),
        "РеквизитыНеРедактируемыеВГрупповойОбработке");
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `ИзменитьВыделенные` без проверки права «Изменение» — форма
  откроется, но запись упадёт при первой ошибке доступа. Проверяйте право в
  вызывающем коде (`ПравоДоступа` или `УправлениеДоступом.ИзменениеРазрешено`
  для учёта RLS — см. `users-access.md`).
- `РеквизитыНеРедактируемыеВГрупповойОбработке` имеет **приоритет** над
  `РеквизитыРедактируемыеВГрупповойОбработке` при заполнении обоих списков.
- Групповое изменение работает в двух режимах (управляются в форме
  `ГрупповоеИзменениеРеквизитов`): **контролируемый** (в транзакции — при ошибке
  одного объекта откатываются все, режим по умолчанию) и **неконтролируемый**
  (вне транзакции, каждый объект записывается независимо, флажок
  `ПрерыватьПоОшибке`).
- ❌ Реализовывать хуки `ПриОпределенииОбъектовСРедактируемымиРеквизитами` в
  основном модуле `ГрупповоеИзменениеОбъектовКлиент` — нельзя. Хук должен быть
  в `ГрупповоеИзменениеОбъектовПереопределяемый` **прикладной** конфигурации.

### 6. Структура подчинённости: отчёт «Связанные документы» и хуки

**Задача:** отчёт `Связанные документы` формируется БСП по ссылке объекта
(панель навигации / команда «Отчёты – Связанные документы»). Прикладная
конфигурация влияет на вывод: переопределяет представление объектов, сумму/
валюту документов, скрывает служебные объекты.

Подсистема `СтруктураПодчиненности` не имеет стабильного программного API для
формирования отчёта — только **хуки переопределения** в
`СтруктураПодчиненностиПереопределяемый` (БСП вызывает, прикладной код
реализует; не вызывается напрямую).

**Хуки:**
`СтруктураПодчиненностиПереопределяемый.ПриОпределенииНастроек(Настройки) Экспорт`
— настройка реквизитов суммы/валюты и состава реквизитов для представления
  объектов в отчёте.
`СтруктураПодчиненностиПереопределяемый.ПриПолученииПредставления(ТипДанных, Данные, Представление, СтандартнаяОбработка) Экспорт`
— формирование представления объекта для вывода в отчёт (для объектов,
  перечисленных в `Настройки.РеквизитыДляПредставления`).
`СтруктураПодчиненностиПереопределяемый.ПередВыводомСвязанногоОбъекта(Объект, СвойстваОбъекта, Отказ) Экспорт`
— позволяет скрыть объект из отчёта (`Отказ = Истина`) или отметить как
  служебный (`СвойстваОбъекта.ЭтоСлужебный = Истина`).
— все Процедуры, регион `#Область ПрограммныйИнтерфейс` (хуки переопределения).
  Сервер.

**Параметры:**
- `Настройки` (Структура):
  - `Реквизиты` (Соответствие) — ключ: полное имя метаданных объекта;
    значение: соответствие реквизитов `Валюта` и `СуммаДокумента` с реальными
    реквизитами объекта (если не задано — берутся реквизиты `Валюта` и
    `СуммаДокумента`).
  - `РеквизитыДляПредставления` (Соответствие) — ключ: полное имя метаданных;
    значение: `Массив` имён реквизитов для формирования представления (для этих
    объектов будет вызван `ПриПолучениеПредставления`).
- `Объект` (СправочникСсылка / ДокументСсылка / ЗадачаСсылка /
  БизнесПроцессСсылка / ПланВидовХарактеристикСсылка) — выводимый объект.
- `СвойстваОбъекта` (Структура) — признаки: `ЭтоОсновной`, `ЭтоСлужебный`,
  `ЭтоПодчиненный`, `Выводился` (структура `Итого` / `ВПодчиненных`).
- `Отказ` (Булево) — `Истина` → объект не выводится (но подчинённые ему
  объекты выводятся).

**Пример:**
```bsl
// СтруктураПодчиненностиПереопределяемый (модуль прикладной конфигурации)
Процедура ПриОпределенииНастроек(Настройки) Экспорт
    // Переопределить реквизиты суммы/валюты для счёта на оплату
    Реквизиты = Новый Соответствие;
    Реквизиты.Вставить("СуммаДокумента",
        Метаданные.Документы.СчетНаОплатуПокупателю.Реквизиты.СуммаОплаты.Имя);
    Реквизиты.Вставить("Валюта",
        Метаданные.Документы.СчетНаОплатуПокупателю.Реквизиты.ВалютаДокумента.Имя);
    Настройки.Реквизиты.Вставить(
        Метаданные.Документы.СчетНаОплатуПокупателю.ПолноеИмя(), Реквизиты);

    // Состав реквизитов для представления исходящего письма
    РеквизитыДляПредставления = Новый Массив;
    РеквизитыДляПредставления.Добавить(
        Метаданные.Документы.ЭлектронноеПисьмоИсходящее.Реквизиты.ДатаОтправления.Имя);
    РеквизитыДляПредставления.Добавить(
        Метаданные.Документы.ЭлектронноеПисьмоИсходящее.Реквизиты.Тема.Имя);
    Настройки.РеквизитыДляПредставления.Вставить(
        Метаданные.Документы.ЭлектронноеПисьмоИсходящее.ПолноеИмя(),
        РеквизитыДляПредставления);
КонецПроцедуры

Процедура ПередВыводомСвязанногоОбъекта(Объект, СвойстваОбъекта, Отказ) Экспорт
    // Скрыть черновики писем из структуры подчинённости
    Если ТипЗнч(Объект) = Тип("ДокументСсылка.ЭлектронноеПисьмоИсходящее")
        И Объект.СтатусПисьма = Перечисления.СтатусыИсходящегоПисьма.Черновик Тогда
        Отказ = Истина;  // не выводить, но подчинённые выведутся
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `СтруктураПодчиненностиПереопределяемый.ПриОпределенииНастроек(…)`
  из прикладного кода — это хук, его вызывает БСП при формировании отчёта.
  Прикладной код только **реализует** его в одноимённом модуле.
- ❌ Искать модуль `СтруктураПодчиненности` (без суффикса) — его **не
  существует**. Подсистема полностью состоит из переопределяемого модуля и
  отчёта `Отчет.СвязанныеДокументы`.
- `Отказ = Истина` в `ПередВыводомСвязанногоОбъекта` скрывает сам объект, но
  **не** подчинённые ему — они будут выведены. Чтобы скрыть всё поддерево,
  используйте `СвойстваОбъекта.ЭтоСлужебный = Истина` (объект не обязателен к
  выводу, но может быть показан при необходимости).
- Устаревшие хуки `МассивРеквизитовОбъектаДляФормированияПредставления`,
  `ПредставлениеОбъектаДляВыводаВОтчет`, `ИмяРеквизитаДокумента` (регион
  `УстаревшиеПроцедурыИФункции`) — не использовать в новом коде; их заменяют
  `ПриОпределенииНастроек` и `ПриПолучениеПредставления`.

## Дополнительно

Прочие методы (полные сигнатуры — через
`python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`):

- `ПоискИУдалениеДублей.ПроверитьВозможностьЗаменыЭлементовСтрока(ПарыЗамен, ПараметрыЗамены)`
  — ⚠️ `СлужебныйПрограммныйИнтерфейс`: строковый вариант проверки
  допустимости замены (возвращает текст ошибок, а не `Соответствие`).
- `ПоискИУдалениеДублей.ТипыИсключаемыеИзВозможныхДублей()` — ⚠️
  `СлужебныйПрограммныйИнтерфейс`: типы, исключаемые из поиска дублей самой
  подсистемой. Для расширения состава исключаемых типов реализуйте хук
  `ИнтеграцияПодсистемБСП.ПриДобавленииТиповИсключаемыхИзВозможныхДублей`, а не
  вызывайте этот метод.
- `ГрупповоеИзменениеОбъектовКлиент.ОбработчикКоманды(Знач МассивСсылок, Знач ПараметрыВыполнения)`
  — ⚠️ `СлужебныйПрограммныйИнтерфейс`: обработчик команды группового
  изменения для подключаемых команд (вызывается БСП).
- `ГрупповоеИзменениеОбъектовКлиент.НачатьИзменениеВыделенныхСОповещением(ПараметрыГрупповогоИзменения, ОписаниеОповещения)`
  — ⚠️ `СлужебныйПрограммныйИнтерфейс`: запуск изменения с оповещением о
  завершении (используется внутри обработчика команды).

Хуки переопределения (модули `*Переопределяемый` — БСП вызывает, прикладной
код реализует; не вызывается напрямую):
- `ПоискИУдалениеДублейПереопределяемый`:
  - `ПриОпределенииОбъектовСПоискомДублей(Объекты)` — какие объекты участвуют в
    поиске.
  - `ПриОпределенииПараметровПоискаДублей(...)` — прикладные правила поиска.
  - `ПриПоискеДублей(...)` — пост-обработка результатов.
  - `ПриОпределенииВозможностиЗаменыЭлементов(Знач ИмяОбъектаМетаданных, Знач ПарыЗамен, Знач ПараметрыЗамены, НедопустимыеЗамены)` —
    запрет замены дубликатов по прикладным критериям.
  - `ПриОпределенииОбъектовСКомандамиОбъединенияДублейЗаменыСсылок(Объекты)` —
    в каких формах объектов выводить команды объединения дубликатов и замены
    ссылок на найденный дубль.
- `ГрупповоеИзменениеОбъектовПереопределяемый`:
  - `ПриОпределенииОбъектовСКомандойГрупповогоИзмененияОбъектов(Объекты)` — в
    каких формах списков выводить команду «Изменить выделенные».
  - `ПриОпределенииОбъектовСРедактируемымиРеквизитами(Объекты)` — для каких
    объектов менять состав доступных реквизитов через функции модуля
    менеджера `РеквизитыРедактируемыеВГрупповойОбработке` /
    `РеквизитыНеРедактируемыеВГрупповойОбработке`.
  - `ПриОпределенииРедактируемыхРеквизитовОбъекта(Объект, РедактируемыеРеквизиты, НередактируемыеРеквизиты)`
    — программное ограничение для конкретного объекта метаданных.

Для поиска сигнатуры/региона любого из этих методов —
`python scripts/bsp_api.py method <Имя> --src src/cf`.
