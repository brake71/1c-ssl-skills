# Базовая функциональность БСП: утилиты общего назначения

Подсистема **БазоваяФункциональность** — общие модули `ОбщегоНазначения*`,
`СтроковыеФункции*`, `ФайловаяСистема*`. Покрывает задачи, встречающиеся в
каждом прикладном модуле: сообщения пользователю, сериализация XML/JSON,
чтение реквизитов по ссылке, безопасное хранение секретов, форматирование
строк, парсинг дат, временные каталоги.

## Модули

Суффиксная система имён БСП (один базовый корень + контекст исполнения):

- `ОбщегоНазначения` — серверный код.
- `ОбщегоНазначенияКлиент` — клиентский код.
- `ОбщегоНазначенияКлиентСервер` — общий (и клиент, и сервер).
- `ОбщегоНазначенияВызовСервера` — клиентский код с вызовом сервера без
  контекста формы.
- `ОбщегоНазначенияСлужебныйКлиент` / `ОбщегоНазначенияСлужебныйКлиентСервер` —
  служебный API, ⚠️ обратная совместимость не гарантируется.
- ⚠️ `ОбщегоНазначенияСлужебный` (без суффикса) **не существует** — типовая
  ошибка: вызвать несуществующий модуль.
- `СтроковыеФункцииКлиентСервер` — строковые утилиты, вызывается и с клиента,
  и с сервера (используется в этом файле как универсальный).
- `ФайловаяСистема` (сервер), `ФайловаяСистемаКлиент` (клиент).
  ⚠️ `ФайловаяСистемаКлиентСервер` **не существует** — клиент вызывает сервер
  через контекст формы.

`БезопасноеХранилище` — это **регистр сведений**, а **не общий модуль**. Доступ
к нему — только через обёртки `ОбщегоНазначения.*ДанныеВБезопасноеХранилище*`.

## Сценарии

### 1. Вывести сообщение пользователю с привязкой к реквизиту

**Задача:** показать сообщение об ошибке рядом с полем формы и прервать
операцию (`Отказ = Истина`).

**Функция:**
`ОбщегоНазначения.СообщитьПользователю(Знач ТекстСообщенияПользователю, Знач КлючДанных = Неопределено, Знач Поле = "", Знач ПутьКДанным = "", Отказ = Ложь) Экспорт`
— Процедура, регион: `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый
клиент, Внешнее соединение.

**Параметры:**
- `ТекстСообщенияПользователю` (Строка) — текст сообщения; для локализации
  оборачивать в `НСтр("ru = '…'")`.
- `КлючДанных` (Произвольный) — объект/ссылка/ключ записи ИБ, к которому
  относится сообщение. По умолчанию `Неопределено`.
- `Поле` (Строка) — наименование реквизита формы (привязка сообщения к полю).
- `ПутьКДанным` (Строка) — путь к данным (путь к реквизиту формы, напр.
  `"Объект"`).
- `Отказ` (Булево) — выходной параметр; метод всегда устанавливает `Истина`.

**Пример:**
```bsl
Попытка
    // ...бизнес-логика...
Исключение
    // Сообщение у поля реквизита объекта, прерывает транзакцию формы
    ОбщегоНазначения.СообщитьПользователю(
        НСтр("ru = 'Не удалось провести документ.'"),
        ,                              // КлючДанных
        "Объект.НомерСтроки",           // Поле (путь внутри Объект)
        ,                              // ПутьКДанным
        Отказ);                         // Отказ = Истина
КонецПопытки;
```

**Нюансы / антипаттерны:**
- ❌ `Сообщить("Ошибка!")` — не привязывается к реквизиту формы и не влияет на
  `Отказ`. Используйте `СообщитьПользователю` с `Поле` и `Отказ`.
- ❌ Передавать одновременно `КлючДанных` и `ПутьКДанным` — конфликт привязки.
  Либо ссылка + `Поле`, либо `ПутьКДанным`.
- ❌ `ОбщегоНазначения.ТекущаяДатаСеанса()` — метод **не существует**
  (ошибка компиляции). `ТекущаяДатаСеанса()` — платформенный глобальный метод,
  вызывается без префикса модуля.
- В фоновом задании длительной операции (вне транзакции) сообщение
  записывается в служебный регистр и отправляется на клиент, если подключена
  система взаимодействия — отдельная обработка не нужна.

### 2. Сохранить и прочитать секрет в безопасном хранилище

**Задача:** сохранить пароль внешнего API/токен для учётной записи и потом
прочитать его в момент вызова, минуя прямой доступ к регистру.

**Функции:**
`ОбщегоНазначения.ЗаписатьДанныеВБезопасноеХранилище(Владелец, Данные, Ключ = "Пароль") Экспорт`
`ОбщегоНазначения.ПрочитатьДанныеИзБезопасногоХранилища(Владелец, Ключи = "Пароль", ОбщиеДанные = Неопределено, ОбластьДанных = Неопределено) Экспорт`
`ОбщегоНазначения.УдалитьДанныеИзБезопасногоХранилища(Знач Владелец, Знач Ключи = Неопределено) Экспорт`
— все Процедуры/Функции, регион `#Область ПрограммныйИнтерфейс` (stable).
Сервер, Толстый клиент, Внешнее соединение.

**Параметры:**
- `Владелец` (ПланОбменаСсылка / СправочникСсылка / Строка до 128 символов) —
  объект-владелец секрета. Для не-ссылочных типов — строка-ключ с именем
  подсистемы, напр. `"СтандартныеПодсистемы.УправлениеДоступом"`; для нескольких
  хранилищ на подсистему — `"….<Уточнение>"`.
- `Данные` (Произвольный) — секрет; `Неопределено` удаляет все данные
  владельца. Если `Ключ = Неопределено` — `Данные` это `Структура`
  (ключ структуры = имя ключа данных, значение = секрет).
- `Ключ` (Строка) — ключ сохраняемых данных, по умолчанию `"Пароль"`. Имя
  ключа по правилам идентификаторов (буква/_ первым символом).
- `Ключи` (Строка / Неопределено) — имена через запятую; `Неопределено` —
  вернуть все данные владельца. Возвращает одно значение (один ключ) или
  `Структуру` (несколько), или `Неопределено` (нет данных).
- `ОбщиеДанные` (Булево) — `Истина` для общих данных в модели сервиса.
- `ОбластьДанных` (Число) — идентификатор области данных для чтения из
  неразделённого сеанса; в остальных случаях игнорируется.

**Пример:**
```bsl
// Запись (при настройке интеграции) — в привилегированном режиме
УстановитьПривилегированныйРежим(Истина);
ОбщегоНазначения.ЗаписатьДанныеВБезопасноеХранилище(УчётнаяЗапись, Пароль);          // Ключ по умолчанию "Пароль"
ОбщегоНазначения.ЗаписатьДанныеВБезопасноеХранилище(УчётнаяЗапись, ТокенAPI, "Token");
УстановитьПривилегированныйРежим(Ложь);

// Чтение (в момент HTTP-вызова)
УстановитьПривилегированныйРежим(Истина);
Пароль = ОбщегоНазначения.ПрочитатьДанныеИзБезопасногоХранилища(УчётнаяЗапись);
Токен  = ОбщегоНазначения.ПрочитатьДанныеИзБезопасногоХранилища(УчётнаяЗапись, "Token");
УстановитьПривилегированныйРежим(Ложь);
```

**Нюансы / антипаттерны:**
- ❌ Прямой доступ к регистру `РегистрыСведений.БезопасноеХранилище.СоздатьНаборЗаписей()`
  — обходит шифрование и контроль доступа. Только через `ОбщегоНазначения.*`.
- ⚠️ Вызывающий код **сам** устанавливает привилегированный режим перед
  записью/чтением — обёртка этого не делает. Без него чтение чужих данных
  упадёт в правах.
- Хранилище не отдаёт данные в пользовательский интерфейс напрямую (кроме
  администраторов).

### 3. Сериализовать значение в XML/JSON и обратно

**Задача:** сериализовать структуру/объект в строку для хранения/передачи и
десериализовать обратно; распарсить JSON-ответ внешнего сервиса с контролем
типов.

**Функции:**
`ОбщегоНазначения.ЗначениеВСтрокуXML(Значение) Экспорт` — → XML-строка.
`ОбщегоНазначения.ЗначениеИзСтрокиXML(СтрокаXML) Экспорт` — ← XML-строка.
`ОбщегоНазначения.ЗначениеВJSON(Знач Значение) Экспорт` — → JSON-строка; даты в ISO (`YYYY-MM-DDThh:mm:ssZ`).
`ОбщегоНазначения.JSONВЗначение(Знач Строка, Знач ИменаСвойствСоЗначениямиДата = Неопределено, Знач ПрочитатьВСоответствие = Истина) Экспорт` — ← JSON-строка.
— все Функции, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Значение` (Произвольный) — сериализуемое значение (только типы,
  сериализуемые платформой — см. синтакс-помощник).
- `ИменаСвойствСоЗначениямиДата` (Строка через запятую / Массив из Строка) —
  имена свойств JSON, которые надо десериализовать как `Дата` (формат ISO).
- `ПрочитатьВСоответствие` (Булево) — `Истина` (по умолчанию) → `Соответствие`;
  `Ложь` → `Структура`.

**Пример:**
```bsl
// Сериализация
СтрокаXML = ОбщегоНазначения.ЗначениеВСтрокуXML(СтруктураДанных);
Обратно    = ОбщегоНазначения.ЗначениеИзСтрокиXML(СтрокаXML);

// Разбор JSON-ответа с датами
Данные = ОбщегоНазначения.JSONВЗначение(ТелоОтвета, "ДатаОтправки,ДатаПолучения");
// ПрочитатьВСоответствие = Ложь -> Структура вместо Соответствия
ДанныеСтр = ОбщегоНазначения.JSONВЗначение(ТелоОтвета, , Ложь);
```

**Нюансы / антипаттерны:**
- ❌ `ОбщегоНазначения.JSONСтрокой(Структура)` — метод **не существует**.
  Стабильный API — `ЗначениеВJSON`.
- ❌ Конкатенация строк для форматируемых сообщений ломает локализацию —
  см. сценарий 6 (`ПодставитьПараметрыВСтроку`).
- Объекты JSON по умолчанию → `Соответствие`; для `Структуры` передавайте
  `ПрочитатьВСоответствие = Ложь`.

### 4. Прочитать реквизит объекта по ссылке

**Задача:** быстро прочитать отдельные реквизиты объекта по ссылке, не
обращаясь к ним через точку (что грузит весь объект); с учётом прав доступа.

**Функции:**
`ОбщегоНазначения.ЗначениеРеквизитаОбъекта(Ссылка, ИмяРеквизита, ВыбратьРазрешенные = Ложь, Знач КодЯзыка = Неопределено) Экспорт`
`ОбщегоНазначения.ЗначенияРеквизитовОбъекта(Ссылка, Знач Реквизиты, ВыбратьРазрешенные = Ложь, Знач КодЯзыка = Неопределено) Экспорт`
`ОбщегоНазначения.ЕстьРеквизитОбъекта(ИмяРеквизита, МетаданныеОбъекта) Экспорт` — проверить наличие реквизита.
`ОбщегоНазначения.ЭтоСсылка(ПроверяемыйТип) Экспорт` — ссылочный ли тип.
— все Функции, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `Ссылка` (ЛюбаяСсылка / Строка) — объект или полное имя предопределённого
  элемента.
- `ИмяРеквизита` (Строка) — имя реквизита, можно через точку (`"Контрагент.ИНН"`).
- `Реквизиты` (Строка через запятую / Структура / Массив) — для
  `ЗначенияРеквизитовОбъекта` можно задать псевдонимы через структуру
  (ключ = псевдоним, значение = имя поля).
- `ВыбратьРазрешенные` (Булево) — `Истина` → запрос с учётом RLS: при
  ограничении записи вернёт `Неопределено` для недоступных полей; `Ложь` →
  исключение при отсутствии прав.
- `ПроверяемыйТип` (Тип) — тип для `ЭтоСсылка`; для `Неопределено` возвращает `Ложь`.

**Пример:**
```bsl
// Один реквизит
Контрагент = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(ДокументСсылка, "Контрагент");

// Несколько реквизитов сразу (один запрос к БД)
Реквизиты = ОбщегоНазначения.ЗначенияРеквизитовОбъекта(ДокументСсылка, "Контрагент, Сумма, Ответственный");

// Проверка наличия реквизита перед обращением
Если ОбщегоНазначения.ЕстьРеквизитОбъекта("ИНН", Метаданные.Документы.Заказ) Тогда
    Инн = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(Ссылка, "ИНН");
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ `ОбщегоНазначения.ЭтоСсылка("СправочникСсылка.Контрагенты")` — метод ждёт
  `Тип`, а не строку. Оберните: `ЭтоСсылка(Тип("СправочникСсылка.Контрагенты"))`.
- Для чтения реквизитов **независимо от прав** текущего пользователя —
  предварительный переход в привилегированный режим.
- `ЗначенияРеквизитовОбъекта` эффективнее нескольких вызовов
  `ЗначениеРеквизитаОбъекта` — один запрос к БД.

### 5. Проверить, подключена ли подсистема БСП

**Задача:** опционально вызвать функциональность подсистемы, не требуя её
обязательного подключения в конфигурации.

**Функция:**
`ОбщегоНазначения.ПодсистемаСуществует(ПолноеИмяПодсистемы) Экспорт`
— Функция → Булево, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `ПолноеИмяПодсистемы` (Строка) — полное имя подсистемы без слова
  `"Подсистема."`, с учётом регистра, напр.
  `"СтандартныеПодсистемы.ЭлектроннаяПодпись"`.

**Пример:**
```bsl
Если ОбщегоНазначения.ПодсистемаСуществует("СтандартныеПодсистемы.ЭлектроннаяПодпись") Тогда
    МодульЭП = ОбщегоНазначения.ОбщийМодуль("ЭлектроннаяПодпись");
    МодульЭП.<ИмяМетода>();  // условный вызов API ЭП
КонецЕсли;
```

**Нюансы / антипаттерны:**
- Для вызова из **клиентского** кода — `ОбщегоНазначенияКлиент.ПодсистемаСуществует`
  (серверный вариант недоступен на клиенте).
- Применять совместно с `ОбщегоНазначения.ОбщийМодуль(<Имя>)` для получения
  модуля необязательной подсистемы.

### 6. Отформатировать строку с параметрами / разобрать строку / распарсить дату

**Задача:** собрать локализуемое сообщение из шаблона с подстановкой
параметров; разбить CSV-строку в массив; привести строку к дате.

**Функции:**
`СтроковыеФункцииКлиентСервер.ПодставитьПараметрыВСтроку(Знач ШаблонСтроки, Знач Параметр1, Знач Параметр2 = Неопределено, …, Знач Параметр9 = Неопределено) Экспорт` — `%1…%9`.
`СтроковыеФункцииКлиентСервер.РазложитьСтрокуВМассивПодстрок(Знач Значение, Знач Разделитель = ",", Знач ПропускатьПустыеСтроки = Неопределено, СокращатьНепечатаемыеСимволы = Ложь) Экспорт`
`СтроковыеФункцииКлиентСервер.СтрокаВДату(Знач Значение, ЧастьДаты = Неопределено) Экспорт` — строка → Дата; если дату не удалось распознать — `01.01.01 00:00:00`. Простой вариант без управления частями даты — `ОбщегоНазначенияКлиентСервер.СтрокаВДату(Знач Значение) Экспорт` (1 параметр).
— все Функции, регион `#Область ПрограммныйИнтерфейс` (stable). Клиент + Сервер.

**Параметры:**
- `ШаблонСтроки` (Строка) — шаблон с плейсхолдерами `%1…%9`; до 9 параметров.
- `Параметр1…9` (Произвольный) — значения подстановки; неиспользуемые = `Неопределено`.
- `Разделитель` (Строка) — по умолчанию `","`.
- `ПропускатьПустыеСтроки` (Булево) — `Истина` отбрасывает пустые элементы.
- `СокращатьНепечатаемыеСимволы` (Булево) — `Истина` обрезает пробелы/непечатаемые.
- `Значение` (Строка) — для `СтрокаВДату`: дата в формате `"ДД.ММ.ГГГГ"`, `"ДД/ММ/ГГ"` или `"ДД-ММ-ГГ ЧЧ:ММ:СС"`, напр. `"23.02.1980"`.
- `ЧастьДаты` (ЧастиДаты) — допустимые части даты; по умолчанию `ЧастиДаты.Дата` (можно `ЧастиДаты.Время` / `ЧастиДаты.ДатаВремя`).

**Пример:**
```bsl
// Локализуемое сообщение (НСтр + шаблон)
Текст = СтроковыеФункцииКлиентСервер.ПодставитьПараметрыВСтроку(
    НСтр("ru = 'Ошибка в строке %1, колонке %2.'"), НомерСтроки, ИмяКолонки);

// Разбор CSV
Части = СтроковыеФункцииКлиентСервер.РазложитьСтрокуВМассивПодстрок(CSVСтрока, ",", Истина);

// Строка -> Дата (по умолчанию ЧастиДаты.Дата); нераспознанная дата -> 01.01.01
Дата = СтроковыеФункцииКлиентСервер.СтрокаВДату("31.12.2024");
ДатаВремя = СтроковыеФункцииКлиентСервер.СтрокаВДату("23-02-1980 09:15:45", ЧастиДаты.ДатаВремя);
```

**Нюансы / антипаттерны:**
- ❌ Конкатенация `"Ошибка в строке " + Номер` ломает мультиязычность.
  Используйте `ПодставитьПараметрыВСтроку` + `НСтр`.
- ⚠️ В БСП **два** метода `СтрокаВДату`: `СтроковыеФункцииКлиентСервер` (2
  параметра, контроль частей даты) и `ОбщегоНазначенияКлиентСервер` (1
  параметр, проще). Уточнить нужный — `python scripts/bsp_api.py method
  СтрокаВДату --module <Модуль> --src src/cf`.
- Для > 9 параметров — `ПодставитьПараметрыВСтрокуИзМассива` того же модуля.

### 7. Создать временный каталог / открыть Проводник

**Задача:** получить уникальный временный каталог на сервере для файлов;
открыть Проводник Windows на клиенте с фокусом на пути.

**Функции:**
`ФайловаяСистема.СоздатьВременныйКаталог(Знач Расширение = "") Экспорт` — сервер, возвращает полный путь.
`ФайловаяСистема.УдалитьВременныйКаталог(Знач Путь) Экспорт` — сервер, удаляет каталог, созданный `СоздатьВременныйКаталог`.
`ФайловаяСистемаКлиент.ОткрытьПроводник(ПутьККаталогуИлиФайлу) Экспорт` — клиент.
— Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable).

**Параметры:**
- `Расширение` (Строка) — расширение временного каталога (напр. `"xml"`); по
  умолчанию `""`.
- `Путь` (Строка) — путь к временному каталогу (значение, возвращённое
  `СоздатьВременныйКаталог`).
- `ПутьККаталогуИлиФайлу` (Строка) — путь, на котором сфокусируется Проводник.

**Пример:**
```bsl
// Сервер: временный каталог для выгрузки
Каталог = ФайловаяСистема.СоздатьВременныйКаталог("xml");
Попытка
    // ...записать файлы в Каталог, обработать...
Исключение
    // ...обработка ошибки...
КонецПопытки;
ФайловаяСистема.УдалитьВременныйКаталог(Каталог);  // убрать за собой обязательно

// Клиент: открыть Проводник на скачанном файле
ФайловаяСистемаКлиент.ОткрытьПроводник(ПолныйПутьКФайлу);
```

**Нюансы / антипаттерны:**
- ⚠️ `СоздатьВременныйКаталог` возвращает каталог, который **вы обязаны
  удалить** сами после использования (`ФайловаяСистема.УдалитьВременныйКаталог`),
  иначе временные файлы копятся. Не существует метода `ФайловаяСистема.УдалитьКаталог`.
- `ФайловаяСистема` — серверный модуль; `ФайловаяСистемаКлиент` — клиентский.
  ⚠️ `ФайловаяСистемаКлиентСервер` **не существует**.

## Дополнительно

Прочие стабильные методы `ОбщегоНазначения` (регион `ПрограммныйИнтерфейс`),
полные сигнатуры — через `python scripts/bsp_api.py module ОбщегоНазначения --src src/cf`:

- `ЗначенияРеквизитовОбъектов(Ссылки, Реквизиты, …)` / `ЗначениеРеквизитаОбъектов(МассивСсылок, ИмяРеквизита, …)` — пакетное чтение по массиву ссылок.
- `УстановитьЗначениеРеквизита(Объект, ИмяРеквизита, Значение, КодЯзыка)` / `УстановитьЗначенияРеквизитов(Объект, Значения)` — запись реквизитов объекта.
- `ПредопределенныйЭлемент(ПолноеИмяПредопределенного)` — ссылка на предопределённый элемент по имени.
- `ЕстьСсылкиНаОбъект(СсылкаИлиМассив, ИскатьСредиСлужебныхОбъектов = Ложь)` — проверка использования объекта перед удалением.
- `ОбщийМодуль(Имя)` — получить модуль необязательной подсистемы (для условного вызова с `ПодсистемаСуществует`).
- `ПодставитьПараметрыВСтрокуИзМассива(ШаблонСтроки, Параметры)` — форматирование с числом параметров > 9.

Для поиска сигнатуры/региона любого из этих методов —
`python scripts/bsp_api.py method <Имя> --src src/cf`.
