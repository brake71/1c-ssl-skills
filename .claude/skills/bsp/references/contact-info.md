# Контактная информация и адресный классификатор

Подсистемы **КонтактнаяИнформация** (хранение адресов, телефонов, e-mail у
произвольных владельцев в табличной части `КонтактнаяИнформация` в формате JSON)
и **АдресныйКлассификатор** (проверка адресов по КЛАДР/ФИАС через веб-сервис 1С,
коды регионов, ОКТМО, ИФНС). Тесно связана подсистема **РаботаСАдресами** — разбор
JSON/XML адреса на поля, сборка адреса по ФИАС-идентификатору.

## Модули

Семейство `УправлениеКонтактнойИнформацией*` (суффиксы контекста — те же, что в
`base-common.md`):

- `УправлениеКонтактнойИнформацией` — серверный API: чтение/запись КИ, виды,
  разбор. регион `ПрограммныйИнтерфейс` (stable).
- `УправлениеКонтактнойИнформациейКлиент` — UI: открытие форм, обработчики
  команд, звонок/e-mail/карта. Клиент.
- `УправлениеКонтактнойИнформациейКлиентСервер` — безопасный код: детекторы
  формата, формирование представления. Клиент + Сервер.
- `УправлениеКонтактнойИнформациейЛокализация` / `…КлиентЛокализация` /
  `…КлиентСерверЛокализация` — региональные особенности (РФ, ЕАЭС, иностранные).
- `УправлениеКонтактнойИнформациейПереопределяемый` — хуки: БСП вызывает,
  прикладной код реализует (не вызывается напрямую).
- `УправлениеКонтактнойИнформациейСлужебный` / `…СлужебныйВызовСервера` /
  `…СлужебныйПовтИсп` — ⚠️ служебные, обратная совместимость не гарантируется.
  ⚠️ `УправлениеКонтактнойИнформациейСлужебныйКлиент` / `…СлужебныйКлиентСервер`
  **не существуют** — все служебные варианты только серверные.

Подсистема `АдресныйКлассификатор`:

- `АдресныйКлассификатор` — стабильный серверный API: проверка адресов, коды
  регионов, ОКТМО, сведения о загрузке.
- `АдресныйКлассификаторКлиент` — UI загрузки/очистки классификатора.
- `АдресныйКлассификаторПовтИсп` — кэш. `АдресныйКлассификаторСлужебный` — ⚠️ служебный.

Подсистема `РаботаСАдресами`:

- `РаботаСАдресами` — разбор адреса на поля/регион/город; ФИАС; конвертация
  JSON↔XML; сборка адреса по идентификатору.
- `РаботаСАдресамиКлиент` / `РаботаСАдресамиКлиентСервер` / `РаботаСАдресамиПовтИсп`.
  ⚠️ `РаботаСАдресамиСлужебный` (без суффикса) **не существует** — только
  `РаботаСАдресамиСлужебныйКлиент`.

⚠️ Не путайте подсистемы: записать КИ в объект — `УправлениеКонтактнойИнформацией.*`,
узнать код региона по имени — `АдресныйКлассификатор.*`, разобрать JSON адреса на
поля — `РаботаСАдресами.*`.

## Сценарии

### 1. Прочитать контактную информацию объекта

**Задача:** прочитать адрес/телефон/e-mail объекта — таблицей с JSON или строкой
представления.

**Функции:**
`УправлениеКонтактнойИнформацией.КонтактнаяИнформацияОбъекта(СсылкаИлиОбъект, ВидИлиТипКонтактнойИнформации = Неопределено, Дата = Неопределено, ТолькоПредставление = Истина) Экспорт`
— Функция, регион `ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент, Внешнее соединение.
`УправлениеКонтактнойИнформацией.КонтактнаяИнформацияОбъектов(СсылкиИлиОбъекты, Знач ТипыКонтактнойИнформации = Неопределено, Знач ВидыКонтактнойИнформации = Неопределено, Дата = Неопределено) Экспорт`
— Функция, регион `ПрограммныйИнтерфейс` (stable). Пакетное чтение массива объектов одним запросом.

**Параметры:**
- `СсылкаИлиОбъект` (`ОпределяемыйТип.ВладелецКонтактнойИнформации` / `СправочникОбъект` / `ДокументОбъект`) — владелец КИ.
- `ВидИлиТипКонтактнойИнформации` (`СправочникСсылка.ВидыКонтактнойИнформации` / `ПеречислениеСсылка.ТипыКонтактнойИнформации`) — отбор по виду или типу; `Неопределено` = все.
- `Дата` (`Дата`) — дата действия записи (если владелец хранит историю).
- `ТолькоПредставление` (`Булево`) — `Истина` (по умолчанию) → возвращает **строку**; `Ложь` → **таблицу значений** с колонками `Объект, Вид, Тип, Значение(JSON), Представление, Дата`.
- `СсылкиИлиОбъекты` (Массив) — для пакетного варианта; `ТипыКонтактнойИнформации`/`ВидыКонтактнойИнформации` — отборы.

**Пример:**
```bsl
ВидАдреса = УправлениеКонтактнойИнформацией.ВидКонтактнойИнформацииПоИмени("ЮрАдресКонтрагента");

// Таблица с JSON-значениями (ТолькоПредставление = Ложь — критично)
ТЗ = УправлениеКонтактнойИнформацией.КонтактнаяИнформацияОбъекта(
    Контрагент, ВидАдреса, , Ложь);
Если ТЗ.Количество() > 0 Тогда
    JSONАдреса   = ТЗ[0].Значение;      // строка JSON БСП
    Представление = ТЗ[0].Представление;
КонецЕсли;

// Пакетно по массиву ссылок — один запрос к БД (не N+1)
ТЗВсе = УправлениеКонтактнойИнформацией.КонтактнаяИнформацияОбъектов(
    МассивКонтрагентов, Перечисления.ТипыКонтактнойИнформации.Адрес);
```

**Нюансы / антипаттерны:**
- ❌ `ТЗ[0].Значение` при `ТолькоПредставление = Истина` — метод вернёт **строку**, а
  не таблицу; обращение к индексу упадёт. Явно передавайте `Ложь`, когда нужен JSON.
- ❌ В цикле по списку вызывать `КонтактнаяИнформацияОбъекта` для каждой ссылки —
  `N+1` запросов. Используйте `КонтактнаяИнформацияОбъектов` (один запрос).
- Получение строкой-представлением через `КонтактнаяИнформацияОбъекта(..., Истина)`
  помечено в док-комментарии как устаревшее; для представления используйте
  `ПредставлениеКонтактнойИнформацииОбъекта` (см. сценарий 3).

### 2. Записать контактную информацию в объект

**Задача:** записать адрес/телефон в табличную часть `КонтактнаяИнформация`
объекта из JSON.

**Функции:**
`УправлениеКонтактнойИнформацией.ЗаписатьКонтактнуюИнформацию(Объект, Знач Значение, ВидИнформации, ТипИнформации, ИдентификаторСтроки = 0, Дата = Неопределено) Экспорт`
— Процедура (не Функция), регион `ПрограммныйИнтерфейс` (stable).
`УправлениеКонтактнойИнформацией.УстановитьКонтактнуюИнформациюОбъекта(СсылкаИлиОбъект, Знач КонтактнаяИнформация, Замещать = Истина) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Добавляет/изменяет КИ по таблице (из `НоваяКонтактнаяИнформация`).
`УправлениеКонтактнойИнформацией.УстановитьКонтактнуюИнформациюОбъектов(КонтактнаяИнформация, Замещать = Истина) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Пакетно для нескольких владельцев.

**Параметры:**
- `Объект` (`СправочникОбъект` / `ДокументОбъект`) — **объект**, не ссылка.
- `Значение` (`Строка`) — КИ во внутреннем формате JSON БСП.
- `ВидИнформации` (`СправочникСсылка.ВидыКонтактнойИнформации`), `ТипИнформации` (`ПеречислениеСсылка.ТипыКонтактнойИнформации`).
- `ИдентификаторСтроки` (`Число`) — `0` для реквизита объекта; `> 0` для строки табличной части.
- `Дата` (`Дата`) — дата действия (при хранении истории).
- `КонтактнаяИнформация` (`ТаблицаЗначений`) — для `Установить*`: таблица с колонками из `НоваяКонтактнаяИнформация`.
- `Замещать` (`Булево`) — `Истина` (по умолчанию) заменяет всю КИ по виду; `Ложь` добавляет запись.

**Пример:**
```bsl
ВидАдреса = УправлениеКонтактнойИнформацией.ВидКонтактнойИнформацииПоИмени("ЮрАдресКонтрагента");
ОбъектСпр = Контрагент.ПолучитьОбъект();   // обязательно объект, не ссылка

УправлениеКонтактнойИнформацией.ЗаписатьКонтактнуюИнформацию(
    ОбъектСпр, JSONАдреса, ВидАдреса,
    Перечисления.ТипыКонтактнойИнформации.Адрес);
ОбъектСпр.Записать();
```

**Нюансы / антипаттерны:**
- ❌ Передавать **ссылку** в `ЗаписатьКонтактнуюИнформацию` — запись уйдёт в копию
  в памяти, реальная запись в БД не выполнится. Только `СправочникОбъект` + `.Записать()`.
- `УстановитьКонтактнуюИнформациюОбъекта` сама запишет владельца, если передана
  **ссылка**; если передан **объект** — запись нужно выполнить отдельно.
- ⚠️ Не существует методов `ПолучитьКонтактнуюИнформацию` / `УстановитьКонтактнуюИнформацию`
  (без суффикса `Объекта`) — типовая ошибка по аналогии. Реальные имена —
  `КонтактнаяИнформацияОбъекта`, `УстановитьКонтактнуюИнформациюОбъекта`.

### 3. Получить строковое представление КИ (для UI/печати)

**Задача:** получить готовую строку адреса/телефона для отчёта, печатной формы
или реквизита формы.

**Функция:**
`УправлениеКонтактнойИнформацией.ПредставлениеКонтактнойИнформацииОбъекта(СсылкаИлиОбъект, ВидКонтактнойИнформации, Разделитель = ",", Дата = Неопределено, ДополнительныеПараметры = Неопределено) Экспорт`
— Функция, регион `ПрограммныйИнтерфейс` (stable). Сервер.

**Параметры:**
- `СсылкаИлиОбъект` (Произвольный) — владелец КИ.
- `ВидКонтактнойИнформации` (`СправочникСсылка.ВидыКонтактнойИнформации`).
- `Разделитель` (`Строка`) — между несколькими записями; по умолчанию `", "` (запятая
  с пробелом).
- `Дата` (`Дата`) — обязательна, если владелец хранит историю изменений.
- `ДополнительныеПараметры` (`Структура`): `ТолькоПервая` (`Булево`, по умолчанию
  `Ложь`) — только основная запись; `БезПробелов` (`Булево`, по умолчанию `Ложь`) —
  без пробела после разделителя.

**Пример:**
```bsl
ВидТелефона = УправлениеКонтактнойИнформацией.ВидКонтактнойИнформацииПоИмени("ТелефонКонтрагента");
СтрокаКИ = УправлениеКонтактнойИнформацией.ПредставлениеКонтактнойИнформацииОбъекта(
    Контрагент, ВидТелефона, ", ", ,
    Новый Структура("ТолькоПервая, БезПробелов", Ложь, Ложь));
ОбластьМакета.Параметры.Телефон = СтрокаКИ;
```

**Нюансы / антипаттерны:**
- ❌ Подставлять `ТЗ[0].Значение` (JSON/XML) напрямую в макет печатной формы —
  попадёт нечитаемый JSON. Сначала `ПредставлениеКонтактнойИнформацииОбъекта`,
  затем строку в макет.
- `КонтактнаяИнформацияОбъекта(..., ТолькоПредставление = Истина)` для получения
  представления устарел — используйте `ПредставлениеКонтактнойИнформацииОбъекта`.

### 4. Открыть форму ввода контактной информации

**Задача:** из обработчика команды формы открыть управляемую форму
ввода/редактирования КИ (адрес/телефон/e-mail) с возвратом результата в оповещение.

**Функции:**
`УправлениеКонтактнойИнформациейКлиент.ПараметрыФормыКонтактнойИнформации(ВидКонтактнойИнформации, Значение, Представление = Неопределено, Комментарий = Неопределено, ТипКонтактнойИнформации = Неопределено) Экспорт`
— Функция-конструктор структуры параметров. Клиент.
`УправлениеКонтактнойИнформациейКлиент.ОткрытьФормуКонтактнойИнформации(Параметры, Владелец = Неопределено, Оповещение = Неопределено) Экспорт`
— Функция (возвращает `ФормаКлиентскогоПриложения`), регион `ПрограммныйИнтерфейс` (stable). Клиент.
`УправлениеКонтактнойИнформациейКлиент.ОткрытьФормуВводаАдреса(Форма, Результат) Экспорт`
— Процедура, регион `ПрограммныйИнтерфейс` (stable). Открытие формы **только адреса** из подключаемой команды.

**Параметры:**
- `Параметры` (`Структура`) — из `ПараметрыФормыКонтактнойИнформации`: ключи `Вид`, `Значение` (JSON), `Представление`, `Комментарий`, `Тип`.
- `Владелец` (Произвольный) — для открываемой формы.
- `Оповещение` (`ОписаниеОповещения`) — обработчик закрытия; `Результат` = `Неопределено` (отмена) или структура с `Значение` (новый JSON) и `Представление`.

**Пример:**
```bsl
&НаКлиенте
Процедура КомандаРедактироватьАдрес(Команда)
    ВидАдреса = УправлениеКонтактнойИнформацией.ВидКонтактнойИнформацииПоИмени("ЮрАдресКонтрагента");

    ПараметрыФормы = УправлениеКонтактнойИнформациейКлиент.ПараметрыФормыКонтактнойИнформации(
        ВидАдреса, ЭтаФорма[ВидАдреса.ИмяРеквизита], ЭтаФорма[ВидАдреса.ИмяРеквизита + "Представление"]);

    УправлениеКонтактнойИнформациейКлиент.ОткрытьФормуКонтактнойИнформации(
        ПараметрыФормы, ЭтаФорма,
        Новый ОписаниеОповещения("ПослеРедактированияАдреса", ЭтаФорма));
КонецПроцедуры

&НаКлиенте
Процедура ПослеРедактированияАдреса(Результат, ДопПараметры) Экспорт
    Если Результат <> Неопределено Тогда
        // Результат.Значение — новый JSON, Результат.Представление — строка
    КонецЕсли;
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- `ОткрытьФормуКонтактнойИнформации` — универсальная (адрес/телефон/e-mail);
  `ОткрытьФормуВводаАдреса` — только адрес, вызывается из подключаемой команды.
- Результат приходит в `ОписаниеОповещения`, не возвращается значением — не
  ждите его синхронно.

### 5. Распознать формат хранимой КИ и разобрать адрес на поля

**Задача:** перед разбором строки КИ определить формат (JSON или XML), затем
извлечь регион/город/индекс/коды из адреса.

**Функции:**
`УправлениеКонтактнойИнформациейКлиентСервер.ЭтоКонтактнаяИнформацияВJSON(Знач Текст) Экспорт`
`УправлениеКонтактнойИнформациейКлиентСервер.ЭтоКонтактнаяИнформацияВXML(Знач Текст) Экспорт`
— Функции → `Булево`, регион `ПрограммныйИнтерфейс` (stable). Клиент + Сервер.
`РаботаСАдресами.СведенияОбАдресе(Адрес, ДополнительныеПараметры = Неопределено) Экспорт`
— Функция, регион `ПрограммныйИнтерфейс` (stable). Разбор адреса на поля.
`РаботаСАдресами.РегионАдресаКонтактнойИнформации(Знач Адрес) Экспорт`
— Функция → `Строка` (наименование субъекта РФ; пустая строка, если не определён; **исключение**, если строка не адрес). stable. Сервер.

**Параметры:**
- `Текст` (`Строка`) — проверяемая строка КИ.
- `Адрес` (`Строка` / `Структура`) — JSON или XML, соответствующий XDTO-пакету `Адрес`.
- `ДополнительныеПараметры` (`Структура`) — для `СведенияОбАдресе`: `КодыКЛАДР` (`Булево`) — добавить коды КЛАДР; `ПроверитьАдрес`; `КодыАдреса` (ФИАС); `НаименованиеВключаетСокращение`.

**Пример:**
```bsl
JSONАдреса = ТЗ[0].Значение;   // из КонтактнаяИнформацияОбъекта

Если УправлениеКонтактнойИнформациейКлиентСервер.ЭтоКонтактнаяИнформацияВJSON(JSONАдреса) Тогда
    Регион = РаботаСАдресами.РегионАдресаКонтактнойИнформации(JSONАдреса);
    Данные = РаботаСАдресами.СведенияОбАдресе(JSONАдреса, Новый Структура("КодыКЛАДР", Истина));
    // Данные.Регион, Данные.Город, Данные.ПочтовыйИндекс, Данные.КодКЛАДР, Данные.КодИФНСФЛ ...
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Парсить JSON/XML КИ вручную через `СтрРазделить`/`ЧтениеJSON` — ломается при
  изменении внутренней структуры КИ в новой версии БСП. Только через методы БСП.
- ⚠️ Метод `СведенияОбАдресе` определён в **двух** модулях с разными сигнатурами:
  `РаботаСАдресами.СведенияОбАдресе(Адрес, ДополнительныеПараметры = Неопределено)` и
  `УправлениеКонтактнойИнформацией.СведенияОбАдресе(Адрес = Неопределено, ДополнительныеПараметры = Неопределено)`.
  Уточняйте модуль через `python scripts/bsp_api.py method СведенияОбАдресе --module <Модуль>`.
- Перед разбором **всегда** проверяйте формат: старые записи могут быть в XML.

### 6. Проверить пакет адресов по классификатору (в фоне)

**Задача:** проверить массив адресов на соответствие КЛАДР/ФИАС через веб-сервис 1С
(занимает до 20 секунд — оборачивать в длительную операцию).

**Функции:**
`АдресныйКлассификатор.АдресныйКлассификаторЗагружен() Экспорт`
— Функция → `Булево`, регион `ПрограммныйИнтерфейс` (stable). Сервер. **До 7 секунд** (проверяет доступность веб-сервиса) — вызывать в фоне.
`АдресныйКлассификатор.ПроверитьАдреса(Адреса) Экспорт`
— Функция, регион `ПрограммныйИнтерфейс` (stable). Сервер. Возвращает массив результатов с полями `Ошибки`, `Варианты` (`КодКЛАДР`, `OKATO`, `ОКТМО`, `КодИФНСФЛ`, `КодИФНСЮЛ`, `КодУчасткаИФНСФЛ`, `КодУчасткаИФНСЮЛ`).

**Параметры:**
- `Адреса` (`Массив` из `Структура`) — каждый элемент `{Адрес: <JSON или XML>}`.

**Пример:**
```bsl
// Сервер: обёртка для фонового вызова
&НаСервере
Функция ПроверитьАдресаВФоне(МассивJSON)
    ПараметрыВыполнения = ДлительныеОперации.ПараметрыВыполненияФункции(УникальныйИдентификатор);
    ПараметрыВыполнения.НаименованиеФоновогоЗадания = НСтр("ru = 'Проверка адресов'");
    Возврат ДлительныеОперации.ВыполнитьФункцию(
        ПараметрыВыполнения, "АдресныйКлассификатор.ПроверитьАдреса", МассивJSON);
КонецФункции

// Клиент: ожидание завершения
&НаКлиенте
Процедура КомандаПроверитьАдреса(Команда)
    ДлительнаяОперация = ПроверитьАдресаВФоне(МассивJSONАдресов);
    Оповещение = Новый ОписаниеОповещения("ПослеПроверкиАдресов", ЭтаФорма);
    ДлительныеОперацииКлиент.ОжидатьЗавершение(ДлительнаяОперация, Оповещение,
        ДлительныеОперацииКлиент.ПараметрыОжидания(ЭтаФорма));
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `АдресныйКлассификатор.ПроверитьАдреса` синхронно с клиента — форма
  «зависнет» на 20 секунд. Только через `ДлительныеОперации.ВыполнитьФункцию`.
- `АдресныйКлассификаторЗагружен` тоже до 7 секунд (HTTP-проверка веб-сервиса) —
  не вызывайте в обработчике `ПриОткрытии` формы без фоновой обёртки.
- ⚠️ `РаботаСАдресами.ПроверитьАдреса(Знач Адреса, ДополнительныеПараметры = Неопределено)` —
  отдельный метод (другой модуль, другая сигнатура); не путать с
  `АдресныйКлассификатор.ПроверитьАдреса`.

### 7. Коды региона и ОКТМО / сборка адреса по ФИАС

**Задача:** по наименованию субъекта получить код региона; по коду ОКТМО —
сведения об адресе; по ФИАС-идентификатору собрать полный адрес.

**Функции:**
`АдресныйКлассификатор.КодРегионаПоНаименованию(Название) Экспорт`
— Функция → `Число` (напр. `50`) / `Неопределено`, регион `ПрограммныйИнтерфейс` (stable). Сервер.
`АдресныйКлассификатор.СведенияПоОКМТО(ОКТМО) Экспорт`
— Функция → `Структура` (`КодРегиона, Регион, РегионТипПолный, РегионТипКраткий, ИдентификаторРегиона, МуниципальныйРайон, Поселение, НаселенныйПункт, ПочтовыйИндекс, OKATO, КодКЛАДР, КодИФНСФЛ, КодИФНСЮЛ` …). stable.
`РаботаСАдресами.АдресПоИдентификатору(ИдентификаторАдреса, ДополнительнаяИнформацияАдреса = Неопределено) Экспорт`
— Функция → `Строка` (XML по умолчанию; JSON, если `ДополнительнаяИнформацияАдреса.АдресВJSON = Истина`). stable.

**Параметры:**
- `Название` (`Строка`) — наименование или полное наименование **субъекта** РФ с сокращением, напр. `"Московская"` или `"Московская обл"`.
- `ОКТМО` (`Строка`) — 8 или 11 цифр.
- `ИдентификаторАдреса` (`Строка`) — GUID адресного объекта ФИАС.
- `ДополнительнаяИнформацияАдреса` (`Структура`) — `НомерДома, НомерОфиса, НомерСтроения, ПочтовыйИндекс, ДополнительнаяИнформация` (комментарий), `АдресВJSON` (`Булево`), `Муниципальный` (`Булево`).

**Пример:**
```bsl
Код = АдресныйКлассификатор.КодРегионаПоНаименованию("Московская");   // 50
ДанныеОКТМО = АдресныйКлассификатор.СведенияПоОКМТО("45380000");     // структура

АдресJSON = РаботаСАдресами.АдресПоИдентификатору(
    "d2dc5217-...-...",
    Новый Структура("НомерДома, НомерОфиса, АдресВJSON", "5", "12", Истина));
```

**Нюансы / антипаттерны:**
- ❌ `КодРегионаПоНаименованию("Москва")` вернёт **`77`**, а не `Неопределено`:
  Москва/СПб/Севастополь — города федерального значения, они же **субъекты** РФ
  (коды 77/78/92), функция их находит. `Неопределено` возвращается для
  **населённого пункта, не являющегося субъектом** (напр. `"Подольск"` — город в
  Московской обл.). Для городов/районов используйте `СведенияПоОКМТО` или
  `РаботаСАдресами.СведенияОбАдресе`.
- ❌ Хранить КИ в собственных строковых реквизитах в обход табличной части
  `КонтактнаяИнформация` — обходит журнал регистрации, историю изменений и
  проверки. Только через `УправлениеКонтактнойИнформацией.*`.

## Дополнительно

Прочие стабильные методы (регион `ПрограммныйИнтерфейс`), полные сигнатуры — через
`python scripts/bsp_api.py method <Имя> --module <Модуль> --src src/cf`:

- `УправлениеКонтактнойИнформацией.НоваяКонтактнаяИнформация(КолонкаОбъект = Истина)` —
  пустая таблица КИ с нужными колонками (для `УстановитьКонтактнуюИнформациюОбъекта`).
- `УправлениеКонтактнойИнформацией.ДобавитьКонтактнуюИнформацию(СсылкаИлиОбъект, ЗначениеИлиПредставление, ВидКонтактнойИнформации, Дата = Неопределено, Замещать = Истина, РаспознатьАдрес = Истина)` —
  добавить КИ по представлению (БСП сама распознает адрес).
- `УправлениеКонтактнойИнформацией.КонтактнаяИнформацияВJSON(Знач КонтактнаяИнформация, Знач ОжидаемыйВид = Неопределено)` /
  `КонтактнаяИнформацияВXML(Знач ЗначенияПолей, Знач Представление = "", Знач ОжидаемыйВид = Неопределено)` —
  конвертация между форматами.
- `УправлениеКонтактнойИнформацией.ДанныеКлассификатораСтранМираПоКоду(Знач Код, Знач ТипКода = "КодСтраны")` /
  `ДанныеКлассификатораСтранМираПоНаименованию(Знач Наименование)` —
  справочник стран мира (не адресный классификатор).
- `УправлениеКонтактнойИнформацией.СодержитКонтактнуюИнформацию(ПроверяемыйОбъект)` —
  признак, есть ли у объекта КИ.
- `УправлениеКонтактнойИнформацией.ВидыКонтактнойИнформацииОбъекта(ВладелецКонтактнойИнформации, ТипКонтактнойИнформации = Неопределено)` —
  виды КИ, доступные владельцу.
- `УправлениеКонтактнойИнформациейКлиентСервер.КонтактнаяИнформацияЗаполнена(Значение)` —
  признак заполненности (не `Неопределено`/пустая).
- `АдресныйКлассификатор.СубъектыРФ()` / `КоличествоЗагруженныхРегионов()` /
  `РазрешенДоступВИнтернет()` / `НаименованиеРегионаПоКоду(КодСубъектаРФ)` —
  справочные функции по регионам.
- `АдресныйКлассификатор.КодыАдреса(Адрес, Источник = Неопределено)` —
  коды адреса (КЛАДР/ФИАС) по строке КИ.

Хуки переопределения (модуль `УправлениеКонтактнойИнформациейПереопределяемый`,
регион `ПрограммныйИнтерфейс`): БСП вызывает, прикладной код реализует в
одноимённом модуле — не вызывать напрямую:
- `ПриОпределенииНастроек(Настройки)` — переопределение общих настроек подсистемы КИ.
- `ПриПолученииНаименованийВидовКонтактнойИнформации(Наименования, КодЯзыка)` — локализация наименований видов КИ.
- `ПриНастройкеНачальногоЗаполненияЭлементов(Настройки)` — настройка начального заполнения предопределённых элементов.
- `ПриНачальномЗаполненииЭлементов(КодыЯзыков, Элементы, ТабличныеЧасти)` — начальное заполнение элементов при внедрении.
- `ПриНачальномЗаполненииЭлемента(Объект, Данные, ДополнительныеПараметры)` — начальное заполнение одного элемента.

> ⚠️ Методы `ПриОпределенииТипаКонтактнойИнформации`,
> `ПриОпределениеКомандТипаКонтактнойИнформации` (имя с «и»),
> `ПриКонвертированииКонтактнойИнформацииИзXML`,
> `ПриПроверкеСтраныПослеПоискаСтраныПоКлассификатору` и подобные живут в
> модулях `…Локализация` (БСП сама реализует их для региональной специфики
> РФ/ЕАЭС), а **не** в `…Переопределяемый` — это не прикладные хуки, точки
> расширения БСП для локализации; не копируйте их как хуки переопределения.
