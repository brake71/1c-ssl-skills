# БСП — навигация

**Библиотека стандартных подсистем 1С (БСП) 3.1.11**, корневая подсистема
`СтандартныеПодсистемы` — 70 подсистем верхнего уровня, встраиваемых в
прикладную конфигурацию как подмножество. БСП физически присутствует в дереве
выгрузки конфигурации (`src/cf`): исходники общих модулей в
`CommonModules/<Имя>/Ext/Module.bsl`, подсистемы — в
`Subsystems/СтандартныеПодсистемы/Subsystems/<Имя>.xml`.

Этот файл — **L0-навигатор**: суффиксная система имён модулей, карта подсистем,
логика стабильности регионов, алгоритм поиска метода. Здесь **нет**
workflow-сценариев вызова (`Модуль.Метод(...)` с параметрами) — они в
`references/<подсистема>.md`. Цель — научить агента **найти нужный модуль** и
**отличить стабильный API от служебного** за 3 шага.

## Суффиксы модулей

Общие модули БСП **не имеют единого префикса** (нет `БСП_…`). Корень имени —
по подсистеме напрямую: `ОбщегоНазначения`, `Пользователи`, `УправлениеПечатью`,
`АдресныйКлассификатор`. Разделение по контексту исполнения и режиму — через
**суффиксы**. Суффиксы комбинируются
(`ОбщегоНазначенияСлужебныйКлиентСервер` = служебный + клиент-сервер).

Реальные суффиксы, встречающиеся в `src/cf/CommonModules/` (проверено по 3.1.11):

| Суффикс | Что означает | Пример |
|---|---|---|
| (без суффикса) | Серверный код | `ОбщегоНазначения`, `Пользователи`, `УправлениеПечатью` |
| `Клиент` | Клиентский код (тонкий/веб-клиент). Не вызывает сервер сам — через `ВызовСервера` или контекст формы | `ОбщегоНазначенияКлиент`, `ПользователиКлиент` |
| `КлиентСервер` | Безопасный код — работает и на сервере, и на клиенте, без прямого обращения к БД | `ОбщегоНазначенияКлиентСервер`, `СтроковыеФункцииКлиентСервер` |
| `ВызовСервера` | Клиентский модуль, помеченный «Вызов сервера» — **делает серверный вызов** без контекста формы | `ОбщегоНазначенияВызовСервера`, `БизнесПроцессыИЗадачиВызовСервера` |
| `Глобальный` | Процедуры глобального контекста — вызываются по короткому имени без префикса модуля | `ОбщегоНазначенияГлобальный`, `УправлениеПечатьюГлобальный` |
| `ПовтИсп` | Режим «Повторное использование возвращаемых значений» — кэширование на сеансе | `ОбщегоНазначенияКлиентПовтИсп`, `АдресныйКлассификаторПовтИсп` |
| `КлиентПовтИсп` | Клиентский + кэширование (комбинация) | `ОбщегоНазначенияКлиентПовтИсп`, `ПодключаемыеКомандыКлиентПовтИсп` |
| `Служебный` | Внутренний API подсистем. Экспортные методы есть, но **обратная совместимость не гарантируется**. Только если нет стабильного аналога | `РегламентныеЗаданияСлужебный`, `УправлениеДоступомСлужебный` |
| `СлужебныйКлиент` / `СлужебныйКлиентСервер` / `СлужебныйВызовСервера` / `СлужебныйПовтИсп` | Служебный API + контекст исполнения (комбинации) | `ОбщегоНазначенияСлужебныйКлиент`, `УправлениеДоступомСлужебныйКлиентСервер` |
| `Переопределяемый` | **Хук переопределения**. БСП вызывает эти методы, прикладной код **реализует** их (копирует модуль-переопределитель в конфигурацию и переопределяет тело). НЕ вызываются напрямую из прикладного кода | `ОбщегоНазначенияПереопределяемый`, `ПодключаемыеКомандыПереопределяемый` |
| `КлиентПереопределяемый` | Хук переопределения, клиентский контекст | `ОбщегоНазначенияКлиентПереопределяемый`, `ЭлектроннаяПодписьКлиентПереопределяемый` |
| `ВМоделиСервиса` | Вариант для модели сервиса (мультитенантность). Часто служебный | `УправлениеДоступомСлужебныйВМоделиСервиса`, `ОбменДаннымиВМоделиСервиса` |
| `Локализация`, `РФ` | Региональные/контекстные варианты | `СтроковыеФункцииКлиентСерверЛокализация`, `УправлениеПечатьюРФ` |
| `БТС` | Базовые табличные структуры — **отдельная подсистема**, не «префикс БСП» | `ОбщегоНазначенияБТС`, `ОбщегоНазначенияБТСПовтИсп` |

### ⚠️ Несуществующие модули (типовые ошибки по аналогии)

Перед использованием модуля **всегда** проверяй `CommonModules/<Имя>/Ext/Module.bsl`.
Имена, которые легко «выдумать» по аналогии, но которых в БСП 3.1.11 **нет**
(проверено по `src/cf`):

- **`ОбщегоНазначенияСлужебный`** (без суффикса) — нет. Служебные варианты:
  `ОбщегоНазначенияСлужебныйКлиент`, `ОбщегоНазначенияСлужебныйКлиентСервер`.
- **`ФайловаяСистемаКлиентСервер`** — нет. Есть `ФайловаяСистема` (сервер),
  `ФайловаяСистемаКлиент` (клиент), `ФайловаяСистемаСлужебныйКлиент`,
  `ФайловаяСистемаСлужебныйКлиентСервер`.
- **`ДлительныеОперацииСлужебный`** — нет. Служебные функции «Длительных
  операций» встроены в `ДлительныеОперации`, `ДлительныеОперацииКлиент`,
  `ДлительныеОперацииВызовСервера`.
- **`РегламентныеЗадания`** (как общий модуль без суффикса) — нет. Серверный
  модуль: `РегламентныеЗаданияСервер`. Подсистема `РегламентныеЗадания`
  существует — это объект метаданных, **не** общий модуль.
- **`БизнесПроцессыИЗадачи`** (без суффикса) — нет. Серверный модуль:
  `БизнесПроцессыИЗадачиСервер`.
- **`УправлениеДоступомКлиент`** — нет. Клиентский вариант только в служебном
  слое: `УправлениеДоступомСлужебныйКлиент`. Серверный: `УправлениеДоступом`
  (без суффикса).
- **`МашиночитаемыеДоверенности`** (без суффикса `ФНС`) — нет. Все модули МЧД
  имеют суффикс `ФНС`: `МашиночитаемыеДоверенностиФНС`,
  `МашиночитаемыеДоверенностиФНСКлиент`, `...Служебный` и т. д.
- **`БезопасноеХранилище`** как общий модуль — нет. Это **регистр сведений**
  `БезопасноеХранилищеДанных`. Работа с ним — только через методы
  `ОбщегоНазначения.ЗаписатьДанныеВБезопасноеХранилище` /
  `ПрочитатьДанныеИзБезопасногоХранилища` /
  `УдалитьДанныеИзБезопасногоХранилища`.

## Регионы и стабильность

Внутри общего модуля БСП код разделён на **области** директивой препроцессора
`#Область` / `#КонецОбласти`. Это валидные конструкции языка 1С, видимые
Конфигуратору как структурные блоки (не комментарии `//`).

Стандартные регионы (имена из реальных модулей `src/cf`, БСП 3.1.11):

| Регион | Стабильность | Можно ли вызывать из прикладного кода |
|---|---|---|
| `ПрограммныйИнтерфейс` | **stable** — обратная совместимость поддерживается между минорными версиями БСП | **Да.** Это основной API подсистемы |
| `СлужебныйПрограммныйИнтерфейс` | ⚠️ служебный — обратная совместимость **не гарантируется** | Только если нет стабильного аналога, с пометкой ⚠️ в коде |
| `СлужебныеПроцедурыИФункции` | ⚠️ внутренние, обычно не экспортные | **Нет** |
| `УстаревшиеПроцедурыИФункции` | ⚠️ **deprecated** — устаревший, не использовать в новом коде | **Нет**, ищи альтернативу в `ПрограммныйИнтерфейс` |

Прочие встречающиеся регионы (`ОбработчикиСобытийПодсистемКонфигурации`,
`ДляВызоваИзДругихПодсистем`, `ВспомогательныеПроцедурыИФункции`,
`ОбновлениеИнформационнойБазы`, `ОбработчикиРегламентныхЗаданий` и т. п.) — это
внутренние/служебные блоки БСП, не предназначенные для прикладного вызова.

### Хуки переопределения

Основной механизм переопределения поведения БСП — **модули с суффиксом
`Переопределяемый`** (и `КлиентПереопределяемый`). Методы в них живут в регионе
`ПрограммныйИнтерфейс`, но семантически это **хуки**: БСП сам вызывает эти методы
в точках расширения, а прикладной код **реализует** их, скопировав
модуль-переопределитель в конфигурацию и переопределив тело. **Из прикладного
кода их не вызывают** как `Модуль.Метод(...)`.

> ⚠️ В БСП 3.1.11 регион `#Область Переопределение` **не существует** — это
> частая ошибка в старых описаниях. Реальные override-регионы в `src/cf`
> единичны: `ПереопределениеВызовов` и `ПереопределениеТекстаЗапросаНабораДанных`
> (встречаются в 1-2 модулях). Массовый механизм хуков — именно модули
> `*Переопределяемый`, а не отдельный регион `Переопределение`.

Пример (модуль-переопределитель):

```bsl
// ОбщегоНазначенияПереопределяемый (модуль-хук, регион ПрограммныйИнтерфейс)
// БСП вызывает этот метод; прикладной код реализует тело под свои нужды.
Процедура ПриЗаполненииПараметровРаботыПользователя(Параметры) Экспорт
    // своя логика: добавить параметры сеанса
КонецПроцедуры
```

## Как найти метод

Алгоритм из 4 шагов — от задачи к сигнатуре:

1. **Определить подсистему** по таблице «Карта подсистем» выше (печать →
   `Печать` → `print-reports.md`; пользователи → `Пользователи` →
   `users-access.md`).
2. **Греп-шаблон** из `SKILL.md` по `src/cf/CommonModules/` — найти экспортные
   методы модуля или метод по имени во всех модулях:
   ```bash
   # все экспортные методы модуля
   grep -Pn "^(Функция|Процедура)\s+\w+.*Экспорт" src/cf/CommonModules/<Модуль>/Ext/Module.bsl
   # метод по имени во всех модулях
   grep -rPl "^(Функция|Процедура)\s+<Метод>\b" src/cf/CommonModules/
   ```
3. **Скрипт `bsp_api.py`** (даёт сигнатуру + имя региона + doc-комментарий +
   путь): `python scripts/bsp_api.py method <Имя> [--module <Модуль>] --src src/cf`
   — для уточнения модуля, если метод в нескольких; `python scripts/bsp_api.py
   module <Модуль> --src src/cf` — все экспортные методы модуля с регионами.
4. **Для редких/служебных методов** — прямой греп по `src/cf/CommonModules/`
   (регион `СлужебныйПрограммныйИнтерфейс`, `УстаревшиеПроцедурыИФункции`).
   Классифицируй стабильность по имени региона (см. таблицу выше).

Перед использованием модуля **всегда** проверяй его существование
(`CommonModules/<Имя>/Ext/Module.bsl`) — особенно если имя похоже на
«выдуманное» (см. раздел «Несуществующие модули»). Аналогии в БСП часто
обманывают: корень имени может не иметь варианта без суффикса
(`БизнесПроцессыИЗадачи` → только `...Сервер`), а служебный слой может быть
только с дополнительным суффиксом (`ОбщегоНазначенияСлужебныйКлиент`, но не
`...Служебный`).
