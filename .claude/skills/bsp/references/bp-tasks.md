# Бизнес-процессы и задачи

Подсистема **БизнесПроцессыИЗадачи** — общие модули `БизнесПроцессыИЗадачиСервер`
(серверный стабильный API), `БизнесПроцессыИЗадачиКлиент` (клиентские обёртки UI),
`БизнесПроцессыИЗадачиВызовСервера` (серверный вызов с клиента, ⚠️ служебный), и
переопределяемые модули-хуки `БизнесПроцессыИЗадачиПереопределяемый` /
`БизнесПроцессыИЗадачиКлиентПереопределяемый`. Покрывает исполнение задач,
перенаправление, принятие к исполнению, остановку/активацию бизнес-процессов,
получение формы выполнения задачи и регламентные обработчики контроля сроков.

## Модули

Суффиксная система имён (один корень + контекст исполнения):

- `БизнесПроцессыИЗадачиСервер` — **стабильный серверный API** (выполнение,
  перенаправление, принятие, остановка, контроль сроков). Сервер, Толстый
  клиент, Внешнее соединение.
- `БизнесПроцессыИЗадачиКлиент` — **стабильный клиентский API** (открытие форм
  перенаправления, доп. информации, предмета задачи, принятие к исполнению из
  формы). Тонкий / Толстый клиент.
- `БизнесПроцессыИЗадачиВызовСервера` — ⚠️ **служебный**: серверные вызовы с
  клиента. Методы-дублёры серверных в `СлужебныеПроцедурыИФункции`
  (`ПеренаправитьЗадачи`, `ПринятьЗадачиКИсполнению`, `ОстановитьБизнесПроцесс`,
  `СделатьАктивнымБизнесПроцесс` и др.) и в `УстаревшиеПроцедурыИФункции`
  (`ВыполнитьЗадачу`, `ФормаВыполненияЗадачи`, `ЭтоВедущаяЗадача`,
  `ЭтоЗадачаИсполнителю`, `СформироватьДанныеВыбораИсполнителя`). Обратная
  совместимость не гарантируется — из прикладного кода не вызывать, использовать
  клиентские обёртки.
- `БизнесПроцессыИЗадачиКлиентСервер` — безопасный код без серверного вызова и
  без БД.
- `БизнесПроцессыИЗадачиПереопределяемый` (сервер) — **хуки**: БСП вызывает,
  прикладной код реализует (не вызывает напрямую).
- `БизнесПроцессыИЗадачиКлиентПереопределяемый` (клиент) — **хук**
  `ПриВыбореИсполнителя`.
- `БизнесПроцессыИЗадачиСобытия` — обработчики событий приложения/сеанса
  (внутреннее).

⚠️ **Общего модуля `БизнесПроцессыИЗадачи` (без суффикса) НЕ существует** — это
имя подсистемы и определяемого типа `ОпределяемыйТип.БизнесПроцесс`, но не общий
модуль. Все серверные методы — в `…Сервер`, все клиентские — в `…Клиент`.

⚠️ **«Фантомы»** — методов `БизнесПроцессыИЗадачиСервер.ФормированиеЗадач`,
`…Сервер.СоздатьЗадачу`, `…Клиент.ОткрытьФормуЗадачи` **не существует**. Создание
задачи — дело кода бизнес-процесса (`БизнесПроцессы.<Имя>.СоздатьЗадачу(...)`);
для открытия формы задачи — `ОткрытьЗначение(...)` либо
`ОткрытьФорму("Задача.ЗадачаИсполнителя.ФормаВыполненияЗадачи", ...)` после
`ФормаВыполненияЗадачи`.

## Сценарии

### 1. Выполнить задачу с авто-обработчиком по умолчанию

**Задача:** отметить задачу исполнителя выполненной и при необходимости вызвать
обработчик бизнес-процесса.

**Функция:**
`БизнесПроцессыИЗадачиСервер.ВыполнитьЗадачу(ЗадачаСсылка, ДействиеПоУмолчанию = Ложь) Экспорт`
— Процедура, регион: `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый
клиент, Внешнее соединение.

**Параметры:**
- `ЗадачаСсылка` (ЗадачаСсылка.ЗадачаИсполнителя) — задача для выполнения.
- `ДействиеПоУмолчанию` (Булево) — `Истина` → после стандартной отметки вызвать
  процедуру `ОбработкаВыполненияПоУмолчанию` модуля менеджера бизнес-процесса
  задачи (если она определена). `Ложь` — только стандартная отметка выполнения.

**Пример:**
```bsl
&НаКлиенте
Процедура ВыполнитьЗадачу(Команда)
    ТекущиеДанные = Элементы.СписокЗадач.ТекущиеДанные;
    Если ТекущиеДанные = Неопределено Тогда
        Возврат;
    КонецЕсли;
    ВыполнитьЗадачуНаСервере(ТекущиеДанные.Ссылка);
    ОповеститьОбИзменении(Тип("ЗадачаСсылка.ЗадачаИсполнителя"));
КонецПроцедуры

&НаСервере
Процедура ВыполнитьЗадачуНаСервере(ЗадачаСсылка)
    // ДействиеПоУмолчанию = Истина — вызовет ОбработкаВыполненияПоУмолчанию
    БизнесПроцессыИЗадачиСервер.ВыполнитьЗадачу(ЗадачаСсылка, Истина);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ `БизнесПроцессыИЗадачи.ВыполнитьЗадачу(...)` — модуль без суффикса не
  существует, ошибка компиляции. Только `БизнесПроцессыИЗадачиСервер`.
- ❌ Вызывать серверный метод напрямую из `&НаКлиенте` тонкого приложения —
  модуль `…Сервер` недоступен на клиенте. Оборачивайте в `&НаСервере` или
  используйте `БизнесПроцессыИЗадачиВызовСервера` (⚠️ служебный).
- Без `ДействиеПоУмолчанию = Истина` выполнится только отметка, прикладная
  логика бизнес-процесса не вызовется — типичная ошибка при копировании
  «выполнить задачу» без флага.

### 2. Перенаправить задачи новому исполнителю с предварительной проверкой

**Задача:** перенаправить массив задач на другого исполнителя/в другую роль,
сначала проверив возможность без побочных эффектов.

**Функция:**
`БизнесПроцессыИЗадачиСервер.ПеренаправитьЗадачи(Знач ПеренаправляемыеЗадачи, Знач ИнфоОПеренаправлении, Знач ТолькоПроверка = Ложь, ПеренаправленныеЗадачи = Неопределено) Экспорт`
— Функция → Булево (Истина = успех), регион: `#Область ПрограммныйИнтерфейс`
(stable). Сервер, Толстый клиент, Внешнее соединение.

**Параметры:**
- `ПеренаправляемыеЗадачи` (Массив из ЗадачаСсылка.ЗадачаИсполнителя) — задачи.
- `ИнфоОПеренаправлении` (Структура) — новые значения реквизитов адресации
  задачи: ключи `Исполнитель`, `РольИсполнителя`, `ОсновнойОбъектАдресации`,
  `ДополнительныйОбъектАдресации`.
- `ТолькоПроверка` (Булево) — `Истина` → только проверить возможность, без
  физического изменения; `Ложь` — выполнить перенаправление.
- `ПеренаправленныеЗадачи` (Массив, выходной) — успешно перенаправленные задачи;
  может быть короче исходного массива, если перенаправлены не все.

**Пример:**
```bsl
ИнфоОПеренаправлении = Новый Структура;
ИнфоОПеренаправлении.Вставить("Исполнитель", НовыйИсполнитель);
ИнфоОПеренаправлении.Вставить("РольИсполнителя", Неопределено);

// Шаг 1 — только проверка (быстро, без изменения состояния)
МожноПеренаправить = БизнесПроцессыИЗадачиСервер.ПеренаправитьЗадачи(
    МассивЗадач, ИнфоОПеренаправлении, Истина);
Если Не МожноПеренаправить Тогда
    Возврат; // Сообщение пользователю уже сформировано внутри
КонецЕсли;

// Шаг 2 — собственно перенаправление
ПеренаправленныеЗадачи = Новый Массив;
Успех = БизнесПроцессыИЗадачиСервер.ПеренаправитьЗадачи(
    МассивЗадач, ИнфоОПеренаправлении, Ложь, ПеренаправленныеЗадачи);
```

**Нюансы / антипаттерны:**
- ❌ Сразу вызывать с `ТолькоПроверка = Ложь` без предварительной проверки — при
  частично неподходящем массиве часть задач изменится до ошибки. Сначала
  `ТолькоПроверка = Истина`, потом `Ложь`.
- Если `Исполнитель` — пользователь/внешний пользователь, БСП дополнительно
  проверяет, что он не помечен на удаление и действителен; иначе —
  `ОбщегоНазначения.СообщитьПользователю`.
- ❌ Вызывать серверный `ПеренаправитьЗадачи` прямо из `&НаКлиенте`. С клиента —
  `БизнесПроцессыИЗадачиКлиент.ПеренаправитьЗадачи` (открывает форму, см.
  сценарий 3). Программный серверный вызов с клиента через
  `БизнесПроцессыИЗадачиВызовСервера.ПеренаправитьЗадачи` — ⚠️ служебный,
  обратная совместимость не гарантируется.

### 3. Открыть форму перенаправления из списка задач

**Задача:** из клиентской формы списка задач открыть управляемую форму
перенаправления с предварительной серверной проверкой возможности.

**Функция:**
`БизнесПроцессыИЗадачиКлиент.ПеренаправитьЗадачи(ПеренаправляемыеЗадачи, ФормаВладелец) Экспорт`
— Процедура, регион: `#Область ПрограммныйИнтерфейс` (stable). Тонкий / Толстый
клиент.

**Параметры:**
- `ПеренаправляемыеЗадачи` (Массив из ЗадачаСсылка.ЗадачаИсполнителя) —
  выделенные в списке задачи.
- `ФормаВладелец` (ФормаКлиентскогоПриложения) — форма-владелец для открываемой
  формы перенаправления.

**Пример:**
```bsl
&НаКлиенте
Процедура ПеренаправитьЗадачи(Команда)
    ВыделенныеЗадачи = Элементы.СписокЗадач.ВыделенныеСтроки;
    Если ВыделенныеЗадачи.Количество() = 0 Тогда
        ПоказатьПредупреждение(, НСтр("ru = 'Не выбраны задачи.'"));
        Возврат;
    КонецЕсли;
    // Открывает форму; серверный вызов проверки (ТолькоПроверка = Истина)
    // делается внутри самой обёртки
    БизнесПроцессыИЗадачиКлиент.ПеренаправитьЗадачи(ВыделенныеЗадачи, ЭтаФорма);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- Эта клиентская обёртка **не дублирует** серверный метод — она открывает форму,
  где пользователь вводит нового исполнителя/роль; серверный вызов проверки
  делается внутри. Не пытайтесь «программно перенаправить» через неё — для
  программного перенаправления используйте серверный метод (сценарий 2) в
  серверном контексте.
- ❌ `БизнесПроцессыИЗадачиКлиент.ОткрытьФормуЗадачи(...)` — метод не существует.
  Для открытия самой задачи — `ОткрытьЗначение(ЗадачаСсылка)` либо
  параметризованное открытие через `ФормаВыполненияЗадачи` (сценарий 6).

### 4. Принять задачи к исполнению и отменить принятие

**Задача:** отметить массив задач как принятые текущим пользователем, либо
снять принятие (смена исполнителя, отказ от роли).

**Функции:**
`БизнесПроцессыИЗадачиСервер.ПринятьЗадачиКИсполнению(Задачи) Экспорт`
`БизнесПроцессыИЗадачиСервер.ОтменитьПринятиеЗадачКИсполнению(Задачи) Экспорт`
— Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый
клиент, Внешнее соединение. Клиентские обёртки —
`БизнесПроцессыИЗадачиКлиент.ПринятьЗадачиКИсполнению(Знач МассивЗадач) Экспорт`
и `БизнесПроцессыИЗадачиКлиент.ОтменитьПринятиеЗадачКИсполнению(Знач МассивЗадач) Экспорт`.

**Параметры:**
- `Задачи` / `МассивЗадач` (Массив из ЗадачаСсылка.ЗадачаИсполнителя) — задачи
  для принятия/отмены.

**Пример:**
```bsl
&НаКлиенте
Процедура ПринятьКИсполнению(Команда)
    ВыделенныеЗадачи = Элементы.СписокЗадач.ВыделенныеСтроки;
    БизнесПроцессыИЗадачиКлиент.ПринятьЗадачиКИсполнению(ВыделенныеЗадачи);
КонецПроцедуры

// На сервере (например, из обработки обновления)
&НаСервере
Процедура ПринятьНаСервере(МассивЗадач)
    БизнесПроцессыИЗадачиСервер.ПринятьЗадачиКИсполнению(МассивЗадач);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- Принятие фиксирует, что исполнитель взял задачу в работу; отмена — снимает
  признак. Серверный вариант не показывает вопросов пользователю; клиентская
  обёртка `ПринятьЗадачуКИсполнению(Форма, ТекущийПользователь)` (с формой)
  дополнительно задаёт вопрос подтверждения.
- ❌ Передавать одну ссылку вместо массива — методы ждут `Массив`. Оберните:
  `Новый Массив; Массив.Добавить(Ссылка)`.

### 5. Остановить и возобновить бизнес-процесс

**Задача:** приостановить активный бизнес-процесс (например, перед
администрированием) и затем сделать его активным снова.

**Функции:**
`БизнесПроцессыИЗадачиСервер.ОстановитьБизнесПроцесс(БизнесПроцесс) Экспорт`
`БизнесПроцессыИЗадачиСервер.СделатьАктивнымБизнесПроцесс(БизнесПроцесс) Экспорт`
— Процедуры, регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый
клиент, Внешнее соединение.

**Параметры:**
- `БизнесПроцесс` (ОпределяемыйТип.БизнесПроцесс) — ссылка на бизнес-процесс
  любого вида, удовлетворяющего определяемому типу.

**Пример:**
```bsl
// Остановить
БизнесПроцессыИЗадачиСервер.ОстановитьБизнесПроцесс(БизнесПроцессСсылка);

// Возобновить (отменить остановку)
БизнесПроцессыИЗадачиСервер.СделатьАктивнымБизнесПроцесс(БизнесПроцессСсылка);
```

**Нюансы / антипаттерны:**
- Не нужно приводить к конкретному виду (`БизнесПроцессСсылка.Согласование` и
  т. п.) — определяемый тип сам разрешает нужный.
- Для пакетной обработки есть `ОстановитьБизнесПроцессы(БизнесПроцессы)` и
  `СделатьАктивнымБизнесПроцессы(БизнесПроцессы)` (массив) того же модуля.
- Перед остановкой из формы объекта используйте клиентскую обёртку
  `БизнесПроцессыИЗадачиКлиент.ОстановитьБизнесПроцессИзФормыОбъекта(Форма)` /
  `ПродолжитьБизнесПроцессИзФормыОбъекта(Форма)` — они корректно проверяют
  права через `БизнесПроцессыИЗадачиПереопределяемый.ПриПроверкеПравНаОстановкуБизнесПроцесса`.

### 6. Получить параметры формы выполнения задачи и открыть форму

**Задача:** для задачи получить структуру параметров формы выполнения (имя
формы, ключ, параметры), зависящую от бизнес-процесса, и открыть форму; плюс
проверить, ведущая ли задача.

**Функции:**
`БизнесПроцессыИЗадачиСервер.ФормаВыполненияЗадачи(Знач ЗадачаСсылка) Экспорт`
— Функция → Структура параметров формы (состав ключей определяется хуком
`БизнесПроцессыИЗадачиПереопределяемый.ПриПолученииФормыВыполненияЗадачи`).
`БизнесПроцессыИЗадачиСервер.ЭтоВедущаяЗадача(ЗадачаСсылка) Экспорт`
— Функция → Булево, является ли задача ведущей (родительской) для своего
бизнес-процесса.
— регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент,
Внешнее соединение.

**Параметры:**
- `ЗадачаСсылка` (ЗадачаСсылка.ЗадачаИсполнителя) — задача.

**Пример:**
```bsl
&НаКлиенте
Процедура ОткрытьФормуВыполнения(ЗадачаСсылка)
    ПараметрыФормы = БизнесПроцессыИЗадачиСервер.ФормаВыполненияЗадачи(ЗадачаСсылка);
    Если ПараметрыФормы = Неопределено ИЛИ ПараметрыФормы.Количество() = 0 Тогда
        // У задачи нет бизнес-процесса — открываем обычную форму задачи
        ОткрытьЗначение(ЗадачаСсылка);
        Возврат;
    КонецЕсли;
    ОткрытьФорму("Задача.ЗадачаИсполнителя.ФормаВыполненияЗадачи",
        ПараметрыФормы, ЭтаФорма);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Прямое `ОткрытьФорму("Задача.ЗадачаИсполнителя.ФормаОбъекта", ...)` в обход
  `ФормаВыполненияЗадачи` — антипаттерн: не пройдёт проверку прав и не подменит
  форму выполнения, если она задана хуком.
- `ФормаВыполненияЗадачи` — универсальный контейнер; имя формы, ключ и
  параметры зависят от прикладного бизнес-процесса и переопределяются в
  `БизнесПроцессыИЗадачиПереопределяемый.ПриПолученииФормыВыполненияЗадачи`.

### 7. Регламентные задания: контроль сроков и уведомления

**Задача:** запустить обработчики регламентных заданий `МониторингЗадач`
(просроченные задачи) и `УведомлениеИсполнителейОНовыхЗадачах` (рассылка о
новых задачах), обычно — самим регламентным заданием, реже — принудительно.

**Функции:**
`БизнесПроцессыИЗадачиСервер.ПроконтролироватьЗадачи() Экспорт`
`БизнесПроцессыИЗадачиСервер.УведомитьИсполнителейОНовыхЗадачах() Экспорт`
— Процедуры без параметров, регион `#Область ПрограммныйИнтерфейс` (stable).
Сервер, Толстый клиент, Внешнее соединение.

**Параметры:** нет.

**Пример:**
```bsl
// Обработчик регламентного задания МониторингЗадач —
// рассылка уведомлений о просроченных задачах от системной учётной записи
БизнесПроцессыИЗадачиСервер.ПроконтролироватьЗадачи();

// Обработчик регламентного задания УведомлениеИсполнителейОНовыхЗадачах —
// рассылка о новых задачах за период с момента предыдущей рассылки
БизнесПроцессыИЗадачиСервер.УведомитьИсполнителейОНовыхЗадачах();
```

**Нюансы / антипаттерны:**
- Запускать вручную из прикладного кода обычно **не нужно** — регламентное
  задание вызовет их само. Принудительный запуск допустим (например, в
  обработке обновления ИБ), но учитывайте длительность рассылки.
- `ПроконтролироватьЗадачи`: если задача направлена «в никуда» (роль с пустым
  списком исполнителей) — создаётся новая задача ответственному за настройку
  ролей.
- Рассылка идёт по почте **от системной учётной записи** и может занять
  заметное время; не вызывайте в интерактивном обработчике.
- ❌ Создавать задачу вручную `Задачи.ЗадачаИсполнителя.СоздатьЗадачу()` без
  заполнения `БизнесПроцесс` и `ТочкаМаршрута` — связь
  «задача ↔ бизнес-процесс ↔ точка маршрута» обязательна для работы
  регламентных обработчиков и `ЭтоВедущаяЗадача`.

## Дополнительно

Прочие стабильные методы `БизнесПроцессыИЗадачиСервер` (регион
`ПрограммныйИнтерфейс`), полные сигнатуры — через
`python scripts/bsp_api.py method <Имя> --module БизнесПроцессыИЗадачиСервер --src src/cf`:

- `ИсполнительСтрокой(Знач Исполнитель, Знач РольИсполнителя, Знач ОсновнойОбъектАдресации = Неопределено, Знач ДополнительныйОбъектАдресации = Неопределено)` — строковое представление исполнителя для отчётов/UI.
- `РольСтрокой(Знач РольИсполнителя, Знач ОсновнойОбъектАдресации = Неопределено, Знач ДополнительныйОбъектАдресации = Неопределено)` — строковое представление роли.
- `БизнесПроцессыВедущейЗадачи(ЗадачаСсылка, ДляИзменения = Ложь)` / `БизнесПроцессыГлавнойЗадачи(ЗадачаСсылка, ДляИзменения = Ложь)` — бизнес-процессы ведущей/главной задачи.
- `ДатаЗавершенияБизнесПроцесса(БизнесПроцессСсылка)` — дата завершения.
- `УстановитьПометкуУдаленияЗадач(БизнесПроцессСсылка, ПометкаУдаления)` — массовая пометка задач на удаление.
- `ЗаблокироватьБизнесПроцессы(БизнесПроцессы)` / `ЗаблокироватьЗадачи(Задачи)` — блокировка по ссылкам (для изменения в транзакции).
- `ГруппаИсполнителейЗадач(РольИсполнителя, ОсновнойОбъектАдресации, ДополнительныйОбъектАдресации)` — группа исполнителей роли.
- `ЗаполнитьГлавнуюЗадачу(БизнесПроцессОбъект, ДанныеЗаполнения)` — заполнение главной задачи.
- Отложенный старт: `ДобавитьПроцессДляОтложенногоСтарта(Процесс, ДатаСтарта)`, `ОтключитьОтложенныйСтартПроцесса(Процесс)`, `СтартоватьОтложенныйПроцесс(БизнесПроцесс)`, `ПараметрыОтложенногоПроцесса(Процесс)`, `ДатаОтложенногоСтартаПроцесса(БизнесПроцесс)`.

Хуки переопределения (модуль `БизнесПроцессыИЗадачиПереопределяемый`, регион
`ПрограммныйИнтерфейс` — **БСП вызывает, прикладной код реализует**):

- `ПриПолученииФормыВыполненияЗадачи(ИмяБизнесПроцесса, ЗадачаСсылка, ТочкаМаршрутаБизнесПроцесса, ПараметрыФормы)` — заполнить параметры формы выполнения задачи (используется `ФормаВыполненияЗадачи`).
- `ПриОпределенииБизнесПроцессов(ПодключенныеБизнесПроцессы)` — список подключённых бизнес-процессов.
- `ПриЗаполненииГлавнойЗадачиБизнесПроцесса(БизнесПроцессОбъект, ДанныеЗаполнения, СтандартнаяОбработка)` — заполнение главной задачи.
- `ПриПроверкеПравНаОстановкуБизнесПроцесса(БизнесПроцесс, ЕстьПрава, СтандартнаяОбработка)` — права на остановку.

Клиентский хук `БизнесПроцессыИЗадачиКлиентПереопределяемый.ПриВыбореИсполнителя(ЭлементИсполнитель, РеквизитИсполнитель, ТолькоПростыеРоли, БезВнешнихРолей, СтандартнаяОбработка)` —
реализуется в прикладном модуле, вызывается БСП при выборе исполнителя.
