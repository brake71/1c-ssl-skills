# Административные инструменты: сеансы, удаление помеченных, профили безопасности

Три подсистемы администрирования ИБ: **ЗавершениеРаботыПользователей**
(общие модули `СоединенияИБ*` — блокировка ИБ и сеансов областей данных,
информация о соединениях, режим завершения работы), **УдалениеПомеченныхОбъектов**
(`УдалениеПомеченныхОбъектов*` — программное/интерактивное удаление с контролем
ссылочной целостности, регламентное удаление по расписанию, видимость помеченных
в формах списков), **ПрофилиБезопасности** (`РаботаВБезопасномРежиме*` —
разрешения на внешние ресурсы: каталоги ФС, COM-классы, интернет-ресурсы,
внешние модули/компоненты, привилегированный режим).

## Модули

**ЗавершениеРаботыПользователей:**

- `СоединенияИБ` — **стабильный серверный API**: блокировка ИБ, блокировка
  сеансов областей данных, параметры блокировки, информация о соединениях.
  Сервер, Толстый клиент, Внешнее соединение.
- `СоединенияИБКлиент` — **стабильный клиентский API**: режим завершения
  работы, признак завершения сеансов, форма параметров администрирования.
  Тонкий / Толстый клиент.
- `СоединенияИБКлиентСервер` / `СоединенияИБВызовСервера` — ⚠️ служебные.
- `СоединенияИБПереопределяемый` — **хук** `ПриОпределенииПараметровБлокировкиСеансов`.

**УдалениеПомеченныхОбъектов:**

- `УдалениеПомеченныхОбъектов` — **стабильный серверный API**: получение
  помеченных, ссылки на удаляемые, программное удаление, настройки отображения,
  режим расписания. Сервер, Толстый клиент, Внешнее соединение.
- `УдалениеПомеченныхОбъектовКлиент` — **стабильный клиентский API**:
  интерактивное удаление, видимость помеченных, расписание.
- `УдалениеПомеченныхОбъектовПереопределяемый` — **хуки** перед/после удаления
  группы, определение объектов с командой «Показать помеченные».
- `УдалениеПомеченныхОбъектовПовтИсп` / `…Служебный` /
  `…СлужебныйВызовСервера` / `…СлужебныйКлиентСервер` — ⚠️ служебные.

**ПрофилиБезопасности:**

- `РаботаВБезопасномРежиме` — **стабильный серверный API**: конструкторы
  разрешений `РазрешениеНа*`, `ЗапросНаИспользованиеВнешнихРесурсов`,
  `ЗапросНаОтменуРазрешений…`, `УстановленБезопасныйРежим`. Сервер, Толстый
  клиент, Внешнее соединение.
- `РаботаВБезопасномРежимеКлиент` — **стабильный клиентский API**:
  `ПрименитьЗапросыНаИспользованиеВнешнихРесурсов`,
  `ОткрытьДиалогНастройкиИспользованияПрофилейБезопасности`.
- `РаботаВБезопасномРежимеПереопределяемый` — **хуки**: проверка возможности
  использования/настройки профилей, заполнение разрешений, запросы
  создания/удаления профиля.

⚠️ Подсистема «ЗавершениеРаботыПользователей» реализована в модулях с корнем
`СоединенияИБ` (а не «ЗавершениеРаботы…»), «ПрофилиБезопасности» — в модулях с
корнем `РаботаВБезопасномРежиме`. Модулей с буквальными именами подсистем не
существует — типовая ошибка искать `ЗавершениеРаботыПользователей.Установить…`.

## Сценарии

### 1. Заблокировать ИБ перед администрированием и снять после

**Задача:** перед обновлением/миграцией установить блокировку соединений ИБ с
отсрочкой и длительностью, гарантированно снять её даже при ошибке.

**Функции:**
`СоединенияИБ.УстановитьБлокировкуСоединений(Знач ТекстСообщения = "", Знач КодРазрешения = "КодРазрешения", Знач ОжиданиеНачалаБлокировки = 0, Знач ДлительностьБлокировки = 0) Экспорт`
— Функция → Булево (успех).
`СоединенияИБ.РазрешитьРаботуПользователей() Экспорт` — Функция → Булево.
`СоединенияИБ.УстановленаБлокировкаСоединений() Экспорт` — Функция → Булево.
— регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент,
Внешнее соединение.

**Параметры:**
- `ТекстСообщения` (Строка) — текст, который увидят подключающиеся при отказе.
- `КодРазрешения` (Строка) — код для входа администратора в заблокированную ИБ;
  по умолчанию `"КодРазрешения"`.
- `ОжиданиеНачалаБлокировки` (Число, минуты) — отсрочка до вступления
  блокировки в силу (даёт пользователям завершить работу).
- `ДлительностьБлокировки` (Число, минуты) — сколько держать блокировку; `0` —
  до явного снятия.

**Пример:**
```bsl
УстановленаБлокировка = Ложь;
Попытка
    УстановленаБлокировка = СоединенияИБ.УстановитьБлокировкуСоединений(
        "Технические работы. Вход — с кодом разрешения.",
        "СекретныйКод", 5, 60);  // отсрочка 5 мин, длительность 60 мин
    Если Не УстановленаБлокировка Тогда
        ВызватьИсключение "Не удалось установить блокировку ИБ";
    КонецЕсли;
    // ...администрирование, обновление, миграция...
Исключение
    Если УстановленаБлокировка Тогда
        СоединенияИБ.РазрешитьРаботуПользователей();
    КонецЕсли;
    ВызватьИсключение;
КонецПопытки;

Если УстановленаБлокировка Тогда
    СоединенияИБ.РазрешитьРаботуПользователей();
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Устанавливать блокировку без `Попытка…Исключение` — при падении кода между
  установкой и снятием ИБ останется заблокированной. Всегда гарантированно
  снимайте.
- Нюанс: если вызывается из сеанса с установленными значениями разделителей,
  `УстановитьБлокировкуСоединений` устанавливает **блокировку сеансов области
  данных**, а не всей ИБ. Для явной блокировки области данных используйте
  `УстановитьБлокировкуСеансовОбластиДанных` (сценарий 2).
- Перед записью/длительной операцией проверяйте
  `УстановленаБлокировкаСоединений()` и предупредите пользователя.

### 2. Блокировка сеансов области данных (модель сервиса)

**Задача:** в модели сервиса заблокировать сеансы одной области данных на
окно времени, потом прочитать текущую блокировку.

**Функции:**
`СоединенияИБ.НовыеПараметрыБлокировкиСоединений() Экспорт` — Функция → Структура
параметров.
`СоединенияИБ.УстановитьБлокировкуСеансовОбластиДанных(Знач Параметры, Знач ПоМестномуВремени = Истина, Знач ОбластьДанных = -1) Экспорт` — Процедура.
`СоединенияИБ.ПолучитьБлокировкуСеансовОбластиДанных(Знач ПоМестномуВремени = Истина) Экспорт` — Функция → Структура.
`СоединенияИБ.ПараметрыБлокировкиСеансов(Знач ПолучитьКоличествоСеансов = Ложь) Экспорт` — Функция → Структура параметров текущей блокировки ИБ.
— регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент,
Внешнее соединение.

**Параметры:**
- `Параметры` (Структура, см. `НовыеПараметрыБлокировкиСоединений`) — содержит
  `Начало`, `Конец` (Дата), `Сообщение` (Строка), `Установлена` (Булево),
  `КодРазрешения` (Строка).
- `ПоМестномуВремени` (Булево) — `Истина` → начало/конец в местном времени
  сеанса; `Ложь` → в универсальном.
- `ОбластьДанных` (Число) — номер области; `-1` — текущая (из разделителей
  сеанса). Из сеанса с разделителями можно передать только совпадающее или
  опустить; из сеанса без разделителей — обязательный.

**Пример:**
```bsl
Параметры = СоединенияИБ.НовыеПараметрыБлокировкиСоединений();
Параметры.Начало     = '20260101230000';  // 23:00 1 января
Параметры.Конец      = '20260102060000';  // 06:00 2 января
Параметры.Сообщение  = "Область данных заблокирована для обновления";
Параметры.Установлена = Истина;

// Установить блокировку для области данных №3
СоединенияИБ.УстановитьБлокировкуСеансовОбластиДанных(Параметры, Истина, 3);

// Снять — та же процедура с Установлена = Ложь
Параметры.Установлена = Ложь;
СоединенияИБ.УстановитьБлокировкуСеансовОбластиДанных(Параметры, Истина, 3);

// Прочитать текущую блокировку
Текущая = СоединенияИБ.ПолучитьБлокировкуСеансовОбластиДанных(Истина);
```

**Нюансы / антипаттерны:**
- ❌ Путать `УстановитьБлокировкуСоединений` (вся ИБ) и
  `УстановитьБлокировкуСеансовОбластиДанных` (одна область в SaaS). Это разные
  уровни блокировки.
- `РазрешитьРаботуПользователей` снимает блокировку ИБ; для областей данных
  снятие — повторный вызов `УстановитьБлокировкуСеансовОбластиДанных` с
  `Параметры.Установлена = Ложь`.
- `ПараметрыБлокировкиСеансов(Истина)` возвращает структуру с полем
  `КоличествоСеансов` — пригодится для контроля перед блокировкой.

### 3. Режим завершения работы и параметры администрирования (клиент)

**Задача:** из клиентского кода включить режим завершения работы
пользователей, пометить чужие сеансы к завершению, открыть форму ввода
параметров администрирования ИБ/кластера; получить информацию о соединениях.

**Функции:**
`СоединенияИБКлиент.УстановитьРежимЗавершенияРаботыПользователей(Знач ЗавершитьРаботу) Экспорт` — Процедура.
`СоединенияИБКлиент.УстановитьПризнакЗавершитьВсеСеансыКромеТекущего(Значение) Экспорт` — Процедура.
`СоединенияИБКлиент.ПоказатьПараметрыАдминистрирования(ОписаниеОповещенияОЗакрытии, ЗапрашиватьПараметрыАдминистрированияИБ, ЗапрашиватьПараметрыАдминистрированияКластера, ПараметрыАдминистрирования = Неопределено, Заголовок = "", ПоясняющаяНадпись = "") Экспорт` — Процедура.
`СоединенияИБ.ИнформацияОСоединениях(ПолучатьСтрокуСоединения = Ложь, СообщенияДляЖурналаРегистрации = Неопределено, ПортКластера = 0) Экспорт` — Функция → Структура.
— клиентские — регион `ПрограммныйИнтерфейс` (stable), Тонкий/Толстый клиент;
`ИнформацияОСоединениях` — серверный, Сервер/Толстый клиент/Внешнее соединение.

**Параметры:**
- `ЗавершитьРаботу` (Булево) — `Истина` включает режим завершения.
- `Значение` (Булево) — признак «завершить все сеансы, кроме текущего».
- `ОписаниеОповещенияОЗакрытии` (ОписаниеОповещения) — обработчик после закрытия
  формы параметров.
- `ЗапрашиватьПараметрыАдминистрированияИБ` / `…Кластера` (Булево) — какие
  группы параметров запросить.
- `ПараметрыАдминистрирования` (Структура) — начальные значения.
- `ПолучатьСтрокуСоединения` (Булево) — добавить строку соединения в результат
  `ИнформацияОСоединениях`.

**Пример:**
```bsl
&НаКлиенте
Процедура НачатьАдминистрирование(Команда)
    Если СоединенияИБ.УстановленаБлокировкаСоединений() Тогда
        ПоказатьПредупреждение(, "ИБ заблокирована. Обратитесь к администратору.");
        Возврат;
    КонецЕсли;
    // Отправить предупреждение работающим пользователям
    СоединенияИБКлиент.УстановитьРежимЗавершенияРаботыПользователей(Истина);
    // Не завершать текущий сеанс
    СоединенияИБКлиент.УстановитьПризнакЗавершитьВсеСеансыКромеТекущего(Истина);

    // Открыть форму параметров администрирования ИБ
    Оповещение = Новый ОписаниеОповещения("ПослеПараметровАдминистрирования", ЭтотОбъект);
    СоединенияИБКлиент.ПоказатьПараметрыАдминистрирования(Оповещение, Истина, Ложь);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `ПоказатьПараметрыАдминистрирования` без проверки подсистемы — если
  `ЗавершениеРаботыПользователей` не внедрена, будет ошибка. Предварительно
  `ОбщегоНазначенияКлиент.ПодсистемаСуществует("СтандартныеПодсистемы.ЗавершениеРаботыПользователей")`.
- `ИнформацияОСоединениях` — серверный метод; из клиентского кода вызовите его
  в `&НаСервере` и передайте результат на клиента.

### 4. Программно удалить помеченные объекты с контролем результата

**Задача:** получить помеченные на удаление (с фильтром по метаданным),
удалить их с контролем ссылочной целостности и корректно обработать
препятствующие ссылки.

**Функции:**
`УдалениеПомеченныхОбъектов.ПомеченныеНаУдаление(Знач ОтборМетаданных = Неопределено, ИскатьТехнологическиеОбъекты = Ложь) Экспорт` — Функция → Массив.
`УдалениеПомеченныхОбъектов.УдалитьПомеченныеОбъекты(УдаляемыеОбъекты, РежимУдаления = "Стандартный") Экспорт` — Функция → Структура.
`УдалениеПомеченныхОбъектов.СсылкиНаУдаляемыеОбъекты(Источник) Экспорт` — Функция → Соответствие.
— регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент,
Внешнее соединение.

**Параметры:**
- `ОтборМетаданных` (СписокЗначений из Строка / Неопределено) — полные имена
  метаданных, напр. `"Справочник.Номенклатура"`; `Неопределено` — без отбора.
- `ИскатьТехнологическиеОбъекты` (Булево) — включать технологические объекты.
- `УдаляемыеОбъекты` (Массив из *Ссылка) — объекты для удаления.
- `РежимУдаления` (Строка) — `"Стандартный"` (контроль + многопользовательская
  работа), `"Монопольный"` (с установкой монопольного режима; при неудаче —
  исключение), `"Упрощенный"` (контроль только в непомеченных; в помеченных
  ссылки на удаляемые **очищаются**).
- `Источник` (СправочникОбъект / ДокументОбъект / РегистрСведенийНаборЗаписей) —
  объект, в котором искать ссылки на удаляемые.

**Пример:**
```bsl
// Все помеченные
Помеченные = УдалениеПомеченныхОбъектов.ПомеченныеНаУдаление();
Если Помеченные.Количество() = 0 Тогда
    Возврат;
КонецЕсли;

// Фильтр по типу (опционально)
ОтборМетаданных = Новый СписокЗначений;
ОтборМетаданных.Добавить("Справочник.Контрагенты");
ПомеченныеКонтрагенты = УдалениеПомеченныхОбъектов.ПомеченныеНаУдаление(ОтборМетаданных);

Результат = УдалениеПомеченныхОбъектов.УдалитьПомеченныеОбъекты(ПомеченныеКонтрагенты, "Стандартный");
Если Не Результат.Успешно Тогда
    Для Каждого СтрокаПрепятствия Из Результат.ПрепятствующиеУдалению Цикл
        ОбщегоНазначения.СообщитьПользователю(
            "Нельзя удалить " + Строка(СтрокаПрепятствия.УдаляемыйСсылка)
            + ": используется в " + Строка(СтрокаПрепятствия.МестоИспользования));
    КонецЦикла;
КонецЕсли;
```

**Нюансы / антипаттерны:**
- ❌ Вызывать `УдалитьПомеченныеОбъекты` внутри явной транзакции — метод сам
  управляет транзакциями и порционностью. Внешняя транзакция приведёт к
  конфликтам.
- ❌ Игнорировать результат — объекты могут быть не удалены из-за ссылочной
  целостности; всегда проверяйте `Результат.Успешно` и обходите
  `ПрепятствующиеУдалению` (поля `УдаляемыйСсылка`, `МестоИспользования`,
  `ОписаниеОшибки`, `ПодробноеОписаниеОшибки`).
- ❌ `УдалениеПомеченныхОбъектовСлужебный.<Метод>` — служебный модуль, обратная
  совместимость не гарантируется. Только стабильный API.
- `СсылкиНаУдаляемыеОбъекты` для наборов записей, подчинённых регистратору,
  возвращает пустой список — это сделано намеренно для производительности и
  бесперебойной работы механизмов формирования движений.

### 5. Интегрировать видимость помеченных в форму списка

**Задача:** в форме с динамическим списком настроить видимость помеченных на
удаление и пометку кнопки «Показать помеченные».

**Функции:**
`УдалениеПомеченныхОбъектов.ПриСозданииНаСервере(Форма, Знач НастройкиОтображенияПомеченныхОбъектов) Экспорт` — Процедура.
`УдалениеПомеченныхОбъектов.НастройкиОтображенияПомеченныхОбъектов() Экспорт` — Функция → ТаблицаЗначений (колонки `ИмяЭлементаФормы`, `ТипыМетаданных`, `ИмяСписка`).
`УдалениеПомеченныхОбъектов.УстановитьПометкуКомандыПоказатьПомеченные(Форма, ТаблицаФормы, КнопкаФормы) Экспорт` — Процедура.
— регион `#Область ПрограммныйИнтерфейс` (stable). Сервер, Толстый клиент,
Внешнее соединение.

**Параметры:**
- `Форма` (ФормаКлиентскогоПриложения) — форма с динамическим списком.
- `НастройкиОтображенияПомеченныхОбъектов` (см. `НастройкиОтображенияПомеченныхОбъектов` / ТаблицаФормы) — либо таблица настроек, либо элемент формы
  динамического списка (для одного списка).
- `ТаблицаФормы` / `КнопкаФормы` — элементы формы для `УстановитьПометкуКомандыПоказатьПомеченные`.

**Пример:**
```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    // Вариант 1: один динамический список — передаём элемент формы
    УдалениеПомеченныхОбъектов.ПриСозданииНаСервере(ЭтотОбъект, Элементы.Список);
КонецПроцедуры

&НаСервере
Процедура ПриСозданииНаСервере_НесколькоСписков(Отказ, СтандартнаяОбработка)
    // Вариант 2: несколько списков — таблица настроек
    Настройки = УдалениеПомеченныхОбъектов.НастройкиОтображенияПомеченныхОбъектов();
    Настройка = Настройки.Добавить();
    Настройка.ИмяЭлементаФормы = "Список1";
    ОсновныеТаблицы = Новый СписокЗначений;
    ОсновныеТаблицы.Добавить("Справочник.Номенклатура");
    Настройка.ТипыМетаданных = ОсновныеТаблицы;
    Настройка = Настройки.Добавить();
    Настройка.ИмяЭлементаФормы = "Список2";
    УдалениеПомеченныхОбъектов.ПриСозданииНаСервере(ЭтотОбъект, Настройки);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- Второй параметр `ПриСозданииНаСервере` принимает **либо** элемент формы
  (ТаблицаФормы, для одного списка), **либо** таблицу настроек из
  `НастройкиОтображенияПомеченныхОбъектов` (для нескольких). Не передавайте
  произвольную структуру — только эти два варианта.
- `ТипыМетаданных` (СписокЗначений из Строка) — для перехода к списку помеченных
  с предустановленным отбором по типу.

### 6. Регламентное удаление по расписанию

**Задача:** прочитать настройки регламентного удаления помеченных по
расписанию и изменить флажок использования.

**Функции:**
`УдалениеПомеченныхОбъектов.РежимУдалятьПоРасписанию() Экспорт` — Функция →
Структура (`Расписание`, `Использование`).
`УдалениеПомеченныхОбъектов.ЗначениеФлажкаУдалятьПоРасписанию() Экспорт` — ⚠️
устаревший (регион `УстаревшиеПроцедурыИФункции`); альтернатива —
`РежимУдалятьПоРасписанию`.
— серверные. Клиентская обёртка переключения флажка —
`УдалениеПомеченныхОбъектовКлиент.ПриИзмененииФлажкаУдалятьПоРасписанию(АвтоматическиУдалятьПомеченныеОбъекты, ОповещениеОбИзменении = Неопределено) Экспорт`.

**Параметры:**
- `АвтоматическиУдалятьПомеченныеОбъекты` (Булево) — новое значение флажка.

**Пример:**
```bsl
// Сервер: прочитать режим
Режим = УдалениеПомеченныхОбъектов.РежимУдалятьПоРасписанию();
Если Режим.Использование Тогда
    // Расписание = Режим.Расписание (см. РегламентныеЗаданияСервер.РасписаниеРегламентногоЗадания)
КонецЕсли;

// Клиент: обработчик переключения флажка в форме настроек
&НаКлиенте
Процедура АвтоматическиУдалятьПомеченныеОбъектыПриИзменении(Элемент)
    УдалениеПомеченныхОбъектовКлиент.ПриИзмененииФлажкаУдалятьПоРасписанию(
        Элементы.АвтоматическиУдалятьПомеченныеОбъекты.Проверять);
КонецПроцедуры
```

**Нюансы / антипаттерны:**
- ❌ Использовать `ЗначениеФлажкаУдалятьПоРасписанию` в новом коде — метод
  устаревший (регион `УстаревшиеПроцедурыИФункции`). Замена —
  `РежимУдалятьПоРасписанию`.
- Расписание хранится в настройках регламентного задания
  `УдалениеПомеченныхОбъектов`; для интерактивного изменения расписания
  используйте `УдалениеПомеченныхОбъектовКлиент.НачатьИзменениеРасписанияРегламентногоЗадания(ОповещениеОбИзменении)`.
- Сама процедура удаления запускается регламентным заданием; программно
  вызывать `УдалитьПомеченныеОбъекты` (сценарий 4) и регламентное расписание —
  разные сценарии: первое — разовое удаление, второе — фоновое по расписанию.

### 7. Запросить разрешения на внешние ресурсы (профили безопасности)

**Задача:** перед выполнением внешнего вызова (HTTP, COM-класс, каталог ФС,
внешняя компонента) запросить разрешения профиля безопасности, и применить их
на клиенте через диалог.

**Функции:**
`РаботаВБезопасномРежиме.РазрешениеНаИспользованиеИнтернетРесурса(Знач Протокол, Знач Адрес, Знач Порт = Неопределено, Знач Описание = "") Экспорт` — Функция → ОбъектXDTO.
`РаботаВБезопасномРежиме.РазрешениеНаИспользованиеКаталогаФайловойСистемы(Знач Адрес, Знач ЧтениеДанных = Ложь, Знач ЗаписьДанных = Ложь, Знач Описание = "") Экспорт` — Функция → ОбъектXDTO.
`РаботаВБезопасномРежиме.РазрешениеНаСозданиеCOMКласса(Знач ProgID, Знач CLSID, Знач ИмяКомпьютера = "", Знач Описание = "") Экспорт` — Функция → ОбъектXDTO.
`РаботаВБезопасномРежиме.РазрешениеНаИспользованиеВнешнейКомпоненты(Знач ИмяМакета, Знач Описание = "") Экспорт` / `РазрешениеНаИспользованиеВнешнегоМодуля(Знач Имя, Знач КонтрольнаяСумма, Знач Описание = "") Экспорт` / `РазрешениеНаИспользованиеКаталогаВременныхФайлов(…) Экспорт` / `РазрешениеНаИспользованиеКаталогаПрограммы(…) Экспорт` / `РазрешениеНаИспользованиеПриложенияОперационнойСистемы(…) Экспорт` / `РазрешениеНаИспользованиеПривилегированногоРежима(Знач Описание = "") Экспорт`.
`РаботаВБезопасномРежиме.ЗапросНаИспользованиеВнешнихРесурсов(Знач НовыеРазрешения, Знач Владелец = Неопределено, Знач РежимЗамещения = Истина) Экспорт` — Функция.
`РаботаВБезопасномРежиме.УстановленБезопасныйРежим() Экспорт` — Функция → Булево.
`РаботаВБезопасномРежимеКлиент.ПрименитьЗапросыНаИспользованиеВнешнихРесурсов(Знач Идентификаторы, ФормаВладелец, ОповещениеОЗакрытии) Экспорт` — Процедура.
— регион `#Область ПрограммныйИнтерфейс` (stable). Серверные — Сервер/Толстый
клиент/Внешнее соединение; клиентская обёртка — Тонкий/Толстый клиент.

**Параметры:**
- `Протокол` (Строка) — `IMAP`, `POP3`, `SMTP`, `HTTP`, `HTTPS`, `FTP`, `FTPS`,
  `WS`, `WSS`.
- `Адрес` (Строка) — адрес ресурса без протокола.
- `Порт` (Число) — номер порта.
- `ProgID` / `CLSID` (Строка) — идентификаторы COM-класса (напр. `"Excel.Application"`).
- `ИмяМакета` (Строка) — имя макета внешней компоненты в конфигурации.
- `НовыеРазрешения` (Массив) — объекты-разрешения из `РазрешениеНа*`.
- `Владелец` (ЛюбаяСсылка) — объект ИБ, с которым логически связаны разрешения
  (напр. элемент справочника «ТомаХраненияФайлов» для каталогов томов).
- `РежимЗамещения` (Булево) — `Истина` замещает прежние разрешения владельца.
- `Описание` (Строка) — причина запроса (видна администратору).

**Пример:**
```bsl
// Сервер: подготовить запрос на HTTP-доступ к API и каталогу выгрузки
Разрешения = Новый Массив;
Разрешения.Добавить(РаботаВБезопасномРежиме.РазрешениеНаИспользованиеИнтернетРесурса(
    "HTTPS", "api.example.com", 443, "Запрос курсов валют"));
Разрешения.Добавить(РаботаВБезопасномРежиме.РазрешениеНаИспользованиеКаталогаФайловойСистемы(
    "D:\Uploads", , Истина, "Каталог выгрузки"));

// Создать запрос (применяется при следующем сеансе/обновлении профиля)
Идентификаторы = РаботаВБезопасномРежиме.ЗапросНаИспользованиеВнешнихРесурсов(
    Разрешения, СправочникСсылка.ИнтеграцияСAPI);

// Клиент: применить запросы через диалог профилей безопасности
РаботаВБезопасномРежимеКлиент.ПрименитьЗапросыНаИспользованиеВнешнихРесурсов(
    Идентификаторы, ЭтаФорма, ОписаниеОповещения);
```

**Нюансы / антипаттерны:**
- ❌ Выполнять внешний вызов без запроса разрешения в профиле безопасности —
  при включённых профилях вызов упадёт с ошибкой доступа. Сначала
  `ЗапросНаИспользованиеВнешнихРесурсов`, потом — применение через
  `ПрименитьЗапросыНаИспользованиеВнешнихРесурсов` (клиентский диалог).
- `УстановленБезопасныйРежим()` проверяет безопасный режим, **игнорируя** профиль
  безопасности с уровнем привилегий конфигурации — пригодится для условного
  кода.
- Есть `ЗапросНаОтменуРазрешенийИспользованияВнешнихРесурсов(Владелец, ОтменяемыеРазрешения)` и
  `ЗапросНаОчисткуРазрешенийИспользованияВнешнихРесурсов(Владелец)` — для снятия
  более не нужных разрешений (напр. при удалении интеграции).
- `Владелец` связывает разрешения с объектом ИБ — при удалении владельца БСП
  может очистить его разрешения.

## Редкие методы

Прочие стабильные методы (полные сигнатуры — через
`python scripts/bsp_api.py method <Имя> --src src/cf`):

- `СоединенияИБ.ИнформацияОСоединениях(ПолучатьСтрокуСоединения = Ложь, СообщенияДляЖурналаРегистрации = Неопределено, ПортКластера = 0)` — информация о текущих соединениях; `СообщенияДляЖурналаРегистрации` (СписокЗначений) — записывает события в журнал.
- `УдалениеПомеченныхОбъектовКлиент.НачатьУдалениеПомеченных(УдаляемыеОбъекты, ПараметрыУдаления = Неопределено, Владелец = Неопределено, ОписаниеОповещенияОЗакрытии = Неопределено)` — открыть форму интерактивного удаления; `ПараметрыУдаления` — из `УдалениеПомеченныхОбъектовКлиент.ПараметрыИнтерактивногоУдаления()`.
- `УдалениеПомеченныхОбъектовКлиент.ПоказатьПомеченныеНаУдаление(Форма, ТаблицаФормы, КнопкаФормы)` / `ПерейтиКПомеченнымНаУдаление(Форма, ТаблицаФормы = Неопределено)` — переключение видимости помеченных и переход к рабочему месту удаления.
- `РаботаВБезопасномРежиме.ЗапросыОбновленияРазрешенийКонфигурации(Знач ВключаяЗапросСозданияПрофиляИБ = Истина)` — pending-запросы обновления разрешений; `КонтрольныеСуммыФайловКомплектаВнешнейКомпоненты(Знач ИмяМакета)` — контрольные суммы комплекта компоненты.

Хуки переопределения (модули `*Переопределяемый`, регион `ПрограммныйИнтерфейс`
— **БСП вызывает, прикладной код реализует**):

- `УдалениеПомеченныхОбъектовПереопределяемый.ПередУдалениемГруппыОбъектов(Контекст, УдаляемыеОбъекты)` — **вне транзакции** перед удалением группы; можно проинициализировать `Контекст` для передачи в `ПослеУдаления`.
- `УдалениеПомеченныхОбъектовПереопределяемый.ПослеУдаленияГруппыОбъектов(Контекст, Успешно)` — **вне транзакции** после; для логирования, очистки внешних данных.
- `УдалениеПомеченныхОбъектовПереопределяемый.ПриОпределенииОбъектовСКомандойПоказатьПомеченные(Объекты)` — добавить метаданные объектов, в формах списков которых будут команды «Показать помеченные»/«Перейти к помеченным».
- `СоединенияИБПереопределяемый.ПриОпределенииПараметровБлокировкиСеансов(ПараметрыБлокировкиСеансов)` — модифицировать параметры при установке блокировки ИБ.
- `РаботаВБезопасномРежимеПереопределяемый.ПриЗаполненииРазрешенийНаДоступКВнешнимРесурсам(ЗапросыРазрешений)` — заполнить разрешения по умолчанию. `ПриВключенииИспользованияПрофилейБезопасности()`, `ПриЗапросеСозданияПрофиляБезопасности(...)`, `ПриЗапросеУдаленияПрофиляБезопасности(...)`, `ПриПроверкеВозможностиИспользованияПрофилейБезопасности(Отказ)` — хуки жизненного цикла профилей.
