
Скил покрывает две смежные подсистемы БСП 3.1.11, отвечающие за **защиту реквизитов** и **пользовательские свойства** объектов на управляемых формах:

- **ЗапретРедактированияРеквизитовОбъектов** — «ключевые» реквизиты, которые нельзя менять после записи, плюс команда «Разрешить редактирование реквизитов» в форме.
- **Свойства** — дополнительные реквизиты и сведения (ПВХ `ДополнительныеРеквизитыИСведения`), подключаемые к форме объекта через стандартные обработчики БСП.

Третий слой — **проверка заполнения** — в БСП 3.1.11 не выделен в отдельный модуль `ПроверкаЗаполненияСлужебный` (модуль с таким именем отсутствует; это не ошибка плана, а реальность выгрузки). Валидация дополнительных реквизитов делается через форму-обработчик `УправлениеСвойствами.ОбработкаПроверкиЗаполнения`, а общий механизм проверки заполнения — **платформенный**: обработчики модуля формы `ОбработкаПроверкиЗаполненияНаСервере(Отказ, ПроверяемыеРеквизиты)` и модуля объекта `ОбработкаПроверкиЗаполнения(Отказ, ПроверяемыеРеквизиты)`. Сообщения пользователю привязываются через `ОбщегоНазначения.СообщитьПользователю`.

## When to use

- Заблокировать редактирование «ключевых» реквизитов объекта на форме после записи (например, валюта кассы, организация договора) — `ЗапретРедактированияРеквизитовОбъектов.ЗаблокироватьРеквизиты`.
- Объявить в модуле менеджера объекта список блокируемых реквизитов через функцию `ПолучитьБлокируемыеРеквизитыОбъекта` и подключить объект к подсистеме через переопределяемый модуль.
- Подключить к форме объекта дополнительные реквизиты/сведения и единым обработчиком сохранять их перед записью — `УправлениеСвойствами.ПриСозданииНаСервере` / `ПередЗаписьюНаСервере`.
- Реализовать проверку заполнения обязательных дополнительных реквизитов в форме объекта с привязкой сообщений к реквизитам — `УправлениеСвойствами.ОбработкаПроверкиЗаполнения` + `ОбщегоНазначения.СообщитьПользователю`.
- Открыть список наборов дополнительных реквизитов/сведений из команды формы (`УправлениеСвойствамиКлиент.ОткрытьСписокСвойств`) или обработать оповещение об изменении набора в форме со свойствами (`ОбрабатыватьОповещения`).
- Обновить видимость/доступность/обязательность зависимых дополнительных реквизитов после изменения значения управляющего реквизита формы — `УправлениеСвойствамиКлиент.ОбновитьЗависимостиДополнительныхРеквизитов`.

## Не использовать, если

- Нужно просто вывести сообщение пользователю с привязкой к реквизиту формы без запрета редактирования и без свойств — переключайтесь на `bsp-base-common` (`ОбщегоНазначения.СообщитьПользователю`).
- Нужна печатная форма, отчёт или дополнительная обработка с командой — переключайтесь на `bsp-print-reports` / `bsp-commands-external`.
- Нужна авторизация, проверка прав или отбор по текущему пользователю — переключайтесь на `bsp-users-access`.
- Нужно изменить состав дат запрета изменения данных (`ДатыЗапретаИзменения`) — это **отдельная** подсистема БСП, в этом скиле не покрыта.
- Задача не связана с управляемыми формами объектов (например, работа с регистрами сведений напрямую) — подсистема «Свойства» оперирует **только** формами объектов-владельцев.

## Core concepts

### Подсистема «ЗапретРедактированияРеквизитовОбъектов»

**Цель:** защитить «ключевые» реквизиты объекта (например, `Валюта` справочника `Кассы`) от случайного изменения после начала учёта. Поведение по умолчанию: после записи объекта платформа вызывает блокировку реквизитов, указанных в модуле менеджера объекта. В форме появляется команда «Разрешить редактирование реквизитов» (доступна пользователю с ролью `ПолныеПрава` или `РедактированиеРеквизитовОбъектов`).

**Концепция «квитанции блокировки»:** модуль менеджера объекта экспортирует функцию `ПолучитьБлокируемыеРеквизитыОбъекта()`, которая возвращает массив строк вида `"Валюта"` или `"Партнер;Партнер"`, где после `;` перечисляются имена элементов формы, которые нужно блокировать вместе с реквизитом. Этот массив — единственный источник истины о том, что блокировать. Прикладной код в форме не должен повторять эту логику; он вызывает `ЗапретРедактированияРеквизитовОбъектов.ЗаблокироватьРеквизиты(Форма, ...)` и получает заполненный реквизит формы `ПараметрыЗапретаРедактированияРеквизитов` (служебная `ТаблицаЗначений` с колонками `ИмяРеквизита`, `Представление`, `РедактированиеРазрешено`, `БлокируемыеЭлементы`, `ПравоРедактирования`, `ЭтоРеквизитФормы`).

**Общие модули семейства (реальные имена в 3.1.11):**

| Модуль | Контекст | Назначение |
|---|---|---|
| `ЗапретРедактированияРеквизитовОбъектов` | Сервер | Инициализация блокировки + конструкторы описаний (`ЗаблокироватьРеквизиты`, `НовыйБлокируемыйРеквизит`, `БлокируемыеРеквизитыОбъекта`) |
| `ЗапретРедактированияРеквизитовОбъектовКлиент` | Клиент | UI-разблокировка и доступность элементов формы (`РазрешитьРедактированиеРеквизитовОбъекта`, `УстановитьДоступностьЭлементовФормы`, `Реквизиты`, `ПоказатьПредупреждениеВсеВидимыеРеквизитыРазблокированы`) |
| `ЗапретРедактированияРеквизитовОбъектовСлужебный` | Сервер (служебный) | Создание служебных реквизитов формы, кеширование |
| `ЗапретРедактированияРеквизитовОбъектовПереопределяемый` | Сервер (переопределение) | Регистрация объектов, у которых есть блокируемые реквизиты (`ПриОпределенииОбъектовСЗаблокированнымиРеквизитами`) и точечная донастройка (`ПриОпределенииЗаблокированныхРеквизитов`) |
| `ЗапретРедактированияРеквизитовОбъектовСлужебныйКлиент` | Клиент (служебный) | Обработчики оповещений формы разблокировки |

> ⚠️ **Не существует** модуля `ЗапретРедактированияРеквизитовОбъектовКлиентСервер` и метода `СнятьЗапрет` — частая ошибка по аналогии с другими подсистемами. Реальное API клиент-серверной части — `ЗапретРедактированияРеквизитовОбъектов` (сервер, `ЗаблокироватьРеквизиты`) + `ЗапретРедактированияРеквизитовОбъектовКлиент` (клиент, `РазрешитьРедактированиеРеквизитовОбъекта`).

### Подсистема «Свойства»

**Цель:** дать пользователю/администратору возможность **без изменения конфигурации** добавлять к объектам произвольные дополнительные реквизиты (отображаются на форме, сохраняются в табличной части `ДополнительныеРеквизиты`) и дополнительные сведения (отображаются в отчётах/печатных формах, хранятся в РС `ДополнительныеСведения`).

**Концепция «квитанции свойств»:** модуль менеджера объекта **не** описывает свойства — они задаются в справочнике `НаборыДополнительныхРеквизитовИСведений` администратором. Прикладной код в форме объекта вызывает `УправлениеСвойствами.ПриСозданииНаСервере(Форма)` и получает набор служебных реквизитов формы (`Свойства_ИспользоватьСвойства`, `Свойства_ОписаниеДополнительныхРеквизитов`, `ПараметрыСвойств` и т. д.), которые БСП использует для отрисовки полей. Сохранение значений — через `УправлениеСвойствами.ПередЗаписьюНаСервере(Форма, ТекущийОбъект)`.

**Общие модули семейства (реальные имена в 3.1.11):**

| Модуль | Контекст | Назначение |
|---|---|---|
| `УправлениеСвойствами` | Сервер | Инициализация формы, сохранение значений, программное чтение/запись свойств (`ПриСозданииНаСервере`, `ПередЗаписьюНаСервере`, `ПриЧтенииНаСервере`, `ОбработкаПроверкиЗаполнения`, `ОбновитьЭлементыДополнительныхРеквизитов`, `ЗначениеСвойства`, `ЗаписатьЗначенияСвойств`) |
| `УправлениеСвойствамиКлиент` | Клиент | Обработка оповещений, обновление зависимостей, команды формы (`ОбрабатыватьОповещения`, `ОбновитьЗависимостиДополнительныхРеквизитов`, `ПослеЗагрузкиДополнительныхРеквизитов`, `ВыполнитьКоманду`, `ОткрытьСписокСвойств`, `РедактироватьМетки`) |
| `УправлениеСвойствамиПереопределяемый` | Сервер (переопределение) | Настройка состава наборов свойств для типов владельцев (`ЗаполнитьНаборыСвойствОбъекта`, `ПриПолученииПредопределенныхНаборовСвойств`, `ПриПолученииНаименованийНаборовСвойств`) |
| `УправлениеСвойствамиПовтИсп` | Сервер (кеш) | Предопределённые наборы и виды свойств (`ПредопределенныеНаборыСвойств`, `ВидыСвойствНабора`) |
| `УправлениеСвойствамиСлужебный` | Сервер (служебный) | Создание реквизитов формы, обновление элементов |

> ⚠️ **Не существует** модуля `УправлениеСвойствамиКлиентСервер` — клиентский и серверный код разделены на `УправлениеСвойствами` и `УправлениеСвойствамиКлиент` (как в большинстве подсистем БСП).

### Проверка заполнения в формах с дополнительными реквизитами

Платформенный контракт:

1. Перед записью объекта платформа вызывает обработчик модуля формы `ОбработкаПроверкиЗаполненияНаСервере(Отказ, ПроверяемыеРеквизиты)`.
2. Если объект записывается, платформа вызывает обработчик модуля объекта `ОбработкаПроверкиЗаполнения(Отказ, ПроверяемыеРеквизиты)`.

В обоих случаях платформа уже выполнила проверку «по умолчанию» (по свойству `АвтоОтметкаНезаполненного` и `ПроверкаЗаполнения` реквизитов). Прикладной код **дополняет** её собственными проверками и **выводит сообщения** через `ОбщегоНазначения.СообщитьПользователю`, чтобы они привязались к нужному реквизиту формы.

Для форм с подсистемой «Свойства» обработчик `ОбработкаПроверкиЗаполненияНаСервере` формы должен вызвать `УправлениеСвойствами.ОбработкаПроверкиЗаполнения(Форма, Отказ, ПроверяемыеРеквизиты)` — это **расширит** проверку заполнения обязательными дополнительными реквизитами, описанными в ПВХ `ДополнительныеРеквизитыИСведения`. Без этого вызова обязательность дополнительных реквизитов игнорируется при записи.

| `УправлениеСвойствамиКлиент.ОбрабатыватьОповещения` | `ОбрабатыватьОповещения(Форма, ИмяСобытия, Параметр)` | Клиент | Возвращает `Истина`, если оповещение относится к набору свойств / дополнительному реквизиту / сведению формы. Вызывается из `ОбработкаОповещения` модуля формы | `Если УправлениеСвойствамиКлиент.ОбрабатыватьОповещения(ЭтаФорма, ИмяСобытия, Параметр) Тогда ОбновитьНаФорме(); КонецЕсли;` | стабильный |
| `УправлениеСвойствамиКлиент.ОткрытьСписокСвойств` | `ОткрытьСписокСвойств(ИмяКоманды)` | Клиент | Открывает форму списка `Справочник.НаборыДополнительныхРеквизитовИСведений` с отбором по виду свойств (`ДополнительныеРеквизиты` / `ДополнительныеСведения` / `Метки`). Вызывается из обработчика команды формы. В исходном коде — `#Область СлужебныйПрограммныйИнтерфейс`; БСП не гарантирует обратную совместимость | `УправлениеСвойствамиКлиент.ОткрытьСписокСвойств("ДополнительныеРеквизиты");` | ⚠️ служебный |
| `УправлениеСвойствамиПереопределяемый.ЗаполнитьНаборыСвойствОбъекта` | `ЗаполнитьНаборыСвойствОбъекта(Знач Объект, ТипСсылки, НаборыСвойств, СтандартнаяОбработка, КлючНазначения)` | Сервер (переопределение) | **Реализуется** в прикладной конфигурации для привязки типов владельцев к наборам свойств. Подменяет стандартную логику БСП. Не вызывается из прикладного кода напрямую | (см. Pattern 4) | стабильный (переопределение) |

## Patterns

### 1. Подключение запрета редактирования реквизитов к форме объекта

```bsl
// Модуль формы документа / справочника

&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    Если Объект.Ссылка.Пустая() Тогда
        Возврат;
    КонецЕсли;
    ЗапретРедактированияРеквизитовОбъектов.ЗаблокироватьРеквизиты(ЭтаФорма);
КонецПроцедуры

&НаКлиенте
Процедура Подключаемый_РазрешитьРедактированиеРеквизитовОбъекта(Команда)
    ЗапретРедактированияРеквизитовОбъектовКлиент.РазрешитьРедактированиеРеквизитовОбъекта(
        ЭтаФорма, Новый ОписаниеОповещения("ПослеРазблокировки", ЭтотОбъект));
КонецПроцедуры

&НаКлиенте
Процедура ПослеРазблокировки(Результат, ДополнительныеПараметры) Экспорт
    Если Результат = Истина Тогда
        // Реквизиты разблокированы
    КонецЕсли;
КонецПроцедуры
```

Кнопка `РазрешитьРедактированиеРеквизитовОбъекта` и её обработчик `Подключаемый_РазрешитьРедактированиеРеквизитовОбъекта` **добавляются в форму автоматически** при вызове `ЗаблокироватьРеквизиты` (если у пользователя есть роль `ПолныеПрава` или `РедактированиеРеквизитовОбъектов` и есть право на редактирование объекта).

### 2. Объявление блокируемых реквизитов в модуле менеджера объекта

```bsl
// Модуль менеджера справочника "Кассы"

Функция ПолучитьБлокируемыеРеквизитыОбъекта() Экспорт
    БлокируемыеРеквизиты = Новый Массив;
    БлокируемыеРеквизиты.Добавить("Валюта");
    БлокируемыеРеквизиты.Добавить("Организация");
    
    // Расширенное описание с предупреждением
    Реквизит = ЗапретРедактированияРеквизитовОбъектов.НовыйБлокируемыйРеквизит();
    Реквизит.Имя = "Ответственный";
    Реквизит.Предупреждение = НСтр("ru = 'Смена ответственного влияет на все ранее введённые документы.'");
    БлокируемыеРеквизиты.Добавить(Реквизит);
    
    Возврат БлокируемыеРеквизиты;
КонецФункции
```

Регистрация самого объекта как участника подсистемы — в переопределяемом модуле `ЗапретРедактированияРеквизитовОбъектовПереопределяемый.ПриОпределенииОбъектовСЗаблокированнымиРеквизитами(Объекты)` (прикладной код, см. `bsp-fundamentals`).

### 3. Подключение подсистемы «Свойства» к форме объекта

```bsl
// Модуль формы справочника / документа, к которому подключены свойства

&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    УправлениеСвойствами.ПриСозданииНаСервере(ЭтаФорма);
КонецПроцедуры

&НаСервере
Процедура ПриЧтенииНаСервере(ТекущийОбъект)
    УправлениеСвойствами.ПриЧтенииНаСервере(ЭтаФорма, ТекущийОбъект);
КонецПроцедуры

&НаСервере
Процедура ПередЗаписьюНаСервере(Отказ, ТекущийОбъект, ПараметрыЗаписи)
    УправлениеСвойствами.ПередЗаписьюНаСервере(ЭтаФорма, ТекущийОбъект);
КонецПроцедуры

&НаСервере
Процедура ОбработкаПроверкиЗаполненияНаСервере(Отказ, ПроверяемыеРеквизиты)
    УправлениеСвойствами.ОбработкаПроверкиЗаполнения(ЭтаФорма, Отказ, ПроверяемыеРеквизиты);
КонецПроцедуры
```

Без вызова `ОбработкаПроверкиЗаполнения` обязательные дополнительные реквизиты **не** проверяются при записи. `ПриЧтенииНаСервере` нужен, если форма открывается из списка (без `ПриСозданииНаСервере`).

### 4. Привязка типа объекта к набору свойств (прикладной переопределяемый модуль)

```bsl
// Модуль УправлениеСвойствамиПереопределяемый прикладной конфигурации

Процедура ЗаполнитьНаборыСвойствОбъекта(Знач Объект, ТипСсылки, НаборыСвойств,
        СтандартнаяОбработка, КлючНазначения) Экспорт
    Если ТипСсылки = Тип("СправочникСсылка.Контрагенты") Тогда
        Набор = НаборыСвойств.Строки.Добавить();
        Набор.Имя = "Справочник_Контрагенты";
        Набор.Заголовок = НСтр("ru = 'Дополнительные реквизиты контрагента'");
    КонецЕсли;
КонецПроцедуры

Процедура ПриПолученииПредопределенныхНаборовСвойств(Наборы) Экспорт
    Набор = Наборы.Строки.Добавить();
    Набор.Имя = "Справочник_Контрагенты_Основное";
    Набор.Идентификатор = Новый УникальныйИдентификатор("...");
    Набор.ЭтоГруппа = Ложь;
КонецПроцедуры
```

Это **переопределение**, не вызов. Модуль `УправлениеСвойствамиПереопределяемый` создаётся в прикладной конфигурации автоматически при внедрении БСП; БСП сама вызывает его процедуры.

### 5. Привязка сообщения об ошибке заполнения к реквизиту формы

```bsl
// Модуль формы, произвольная прикладная проверка

&НаСервере
Процедура ОбработкаПроверкиЗаполненияНаСервере(Отказ, ПроверяемыеРеквизиты)
    УправлениеСвойствами.ОбработкаПроверкиЗаполнения(ЭтаФорма, Отказ, ПроверяемыеРеквизиты);
    
    Если Объект.Сумма <= 0 Тогда
        ОбщегоНазначения.СообщитьПользователю(
            НСтр("ru = 'Сумма должна быть больше нуля.'"),
            ,
            "Объект.Сумма",
            ,
            Отказ);
    КонецЕсли;
КонецПроцедуры
```

`Поле = "Объект.Сумма"` привязывает сообщение к реквизиту `Сумма` в форме. `Отказ = Истина` прерывает запись. Подробности сигнатуры — в `bsp-base-common` (`## Key methods`).

## Anti-patterns

### ❌ Вызывать несуществующий `ЗапретРедактированияРеквизитовОбъектовКлиентСервер.СвойстваРеквизитов`

```bsl
// ❌ ОШИБКА КОМПИЛЯЦИИ: модуль и метод не существуют
Массив = ЗапретРедактированияРеквизитовОбъектовКлиентСервер.СвойстваРеквизитов(Форма);
```

```bsl
// ✅ Клиентский модуль называется без "Сервер", и метод называется Реквизиты
Массив = ЗапретРедактированияРеквизитовОбъектовКлиент.Реквизиты(Форма, Истина, Истина);
```

`ЗапретРедактированияРеквизитовОбъектовКлиентСервер` **не существует** в 3.1.11. Семейство: `ЗапретРедактированияРеквизитовОбъектов` (сервер), `…Клиент` (клиент), `…Служебный`, `…СлужебныйКлиент`, `…Переопределяемый`. Для получения имён блокируемых реквизитов на сервере — `ЗапретРедактированияРеквизитовОбъектов.БлокируемыеРеквизитыОбъекта(ИмяОбъекта)`.

### ❌ Вызывать несуществующий `ЗапретРедактированияРеквизитовОбъектов.СнятьЗапрет`

```bsl
// ❌ ОШИБКА КОМПИЛЯЦИИ: метода нет
ЗапретРедактированияРеквизитовОбъектов.СнятьЗапрет(Форма);
```

```bsl
// ✅ Снятие блокировки инициирует пользователь через команду формы;
//   метод "ЗаблокироватьРеквизиты" вызывается однократно при открытии формы
ЗапретРедактированияРеквизитовОбъектов.ЗаблокироватьРеквизиты(Форма);
```

«Снять запрет» в BSP делается через пользовательский интерфейс: команда `РазрешитьРедактированиеРеквизитовОбъекта` → форма `РазблокированиеРеквизитов` → возврат `Истина` в оповещении. Серверного метода `СнятьЗапрет`/`Разблокировать` нет — это намеренно, чтобы избежать случайного снятия блокировки кодом.

### ❌ Вызывать несуществующий `УправлениеСвойствамиКлиентСервер`

```bsl
// ❌ ОШИБКА КОМПИЛЯЦИИ: модуль не существует
УправлениеСвойствамиКлиентСервер.ОбновитьЗависимостиДополнительныхРеквизитов(ЭтаФорма);
```

```bsl
// ✅ Клиентский код — в УправлениеСвойствамиКлиент (без "Сервер")
УправлениеСвойствамиКлиент.ОбновитьЗависимостиДополнительныхРеквизитов(ЭтаФорма);
```

В 3.1.11 модуля `УправлениеСвойствамиКлиентСервер` **нет**. Клиентский код целиком в `УправлениеСвойствамиКлиент`, серверный — в `УправлениеСвойствами`. По аналогии с `ОбщегоНазначения*` суффикс `КлиентСервер` появляется только там, где код **безопасен** для обоих контекстов (нет обращений к БД / форме).

### ❌ Вызывать несуществующий `ПроверкаЗаполненияСлужебный.ПроверитьЗаполнениеРеквизитовОбъекта`

```bsl
// ❌ ОШИБКА КОМПИЛЯЦИИ: модуль не существует
ПроверкаЗаполненияСлужебный.ПроверитьЗаполнениеРеквизитовОбъекта(Объект, Отказ);
```

```bsl
// ✅ Проверка заполнения дополнительных реквизитов — в форме через БСП-мост
//   Модуль формы:
&НаСервере
Процедура ОбработкаПроверкиЗаполненияНаСервере(Отказ, ПроверяемыеРеквизиты)
    УправлениеСвойствами.ОбработкаПроверкиЗаполнения(ЭтаФорма, Отказ, ПроверяемыеРеквизиты);
    // ... прикладные проверки с ОбщегоНазначения.СообщитьПользователю
КонецПроцедуры
```

Модуля `ПроверкаЗаполненияСлужебный` **нет** в 3.1.11 (как и отдельной подсистемы «ПроверкаЗаполнения»). Валидация дополнительных реквизитов делается вызовом `УправлениеСвойствами.ОбработкаПроверкиЗаполнения` из обработчика формы; произвольные прикладные проверки — через `ОбщегоНазначения.СообщитьПользователю` в том же обработчике.

### ❌ Дублировать логику блокировки собственной разметкой формы

```bsl
// ❌ Вручную перебирать реквизиты и расставлять ТолькоПросмотр
Для Каждого Реквизит Из Метаданные.Справочники.Кассы.Реквизиты Цикл
    Если Реквизит.Имя = "Валюта" Тогда
        Элементы.Валюта.ТолькоПросмотр = Истина;
    КонецЕсли;
КонецЦикла;
```

```bsl
// ✅ Один вызов — БСП сам добавляет служебные реквизиты, делает элементы только для чтения,
//   добавляет команду «Разрешить редактирование» (если есть право)
ЗапретРедактированияРеквизитовОбъектов.ЗаблокироватьРеквизиты(ЭтаФорма);
```

Ручная разметка теряет: (a) проверку прав, (b) добавление кнопки разблокировки, (c) условную блокировку (для новых объектов), (d) интеграцию с групповым изменением реквизитов.

### ❌ Забыть вызвать `ОбработкаПроверкиЗаполнения` в форме с подсистемой «Свойства»

```bsl
// ❌ Обязательные дополнительные реквизиты НЕ проверяются при записи
&НаСервере
Процедура ОбработкаПроверкиЗаполненияНаСервере(Отказ, ПроверяемыеРеквизиты)
    // только прикладные проверки
КонецПроцедуры
```

```bsl
// ✅ Первая строка обработчика — вызов БСП-моста
&НаСервере
Процедура ОбработкаПроверкиЗаполненияНаСервере(Отказ, ПроверяемыеРеквизиты)
    УправлениеСвойствами.ОбработкаПроверкиЗаполнения(ЭтаФорма, Отказ, ПроверяемыеРеквизиты);
    // ... прикладные проверки с ОбщегоНазначения.СообщитьПользователю
КонецПроцедуры
```

Без вызова `УправлениеСвойствами.ОбработкаПроверкиЗаполнения` флаг `ЗаполнятьОбязательно` в ПВХ `ДополнительныеРеквизитыИСведения` **игнорируется**, и пользователь запишет объект с пустым обязательным дополнительным реквизитом.

Для поиска методов/модулей в выгрузке конфигурации используйте `python scripts/bsp_ui_search.py method <имя>`.
