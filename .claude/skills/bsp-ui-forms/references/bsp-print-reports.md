
Скил по двум смежным подсистемам БСП 3.1.11, образующим инфраструктуру **печатных форм** и **отчётов** в управляемом приложении:

1. **`Печать`** — формирование и вывод печатных форм (табличных документов и офисных документов) из формы объекта, программная печать пакета объектов, настройка подменю «Печать» в формах.
2. **`ВариантыОтчетов`** — хранение вариантов отчётов в справочнике `ВариантыОтчетов`, подключение вариантов к подсистемам, программное открытие формы отчёта и фоновое формирование через СКД.

> 🔗 **Связь с `bsp-commands-external`:** команды печати — это **подмножество** подключаемых команд. `ПодключаемыеКоманды.ПриСозданииНаСервере` в форме объекта автоматически создаёт подменю «Печать» из коллекции, собранной в `ДобавитьКомандыПечати` модуля менеджера объекта. Инфраструктура размещения команд — в `bsp-commands-external`, конкретный вид команд (печать) — здесь. API `УправлениеПечатью*` намеренно **не дублируется** в `bsp-commands-external`.

> 🔗 **Связь с `bsp-longs-and-jobs`:** формирование тяжёлой печатной формы или отчёта — типичный сценарий для `ДлительныеОперации.ВыполнитьФункцию`. Прямой синхронный вызов `УправлениеПечатью.СформироватьПечатныеФормы` блокирует управляемую форму. Тонкая обёртка `УправлениеПечатью.СформироватьПечатныеФормыВФоне` использует фоновые задания изнутри, но оборачивает «свою» модель — для нового кода предпочтительно `ДлительныеОперации.ВыполнитьФункцию` + `ДлительныеОперацииКлиент.ОжидатьЗавершение`.

## When to use

- Сформировать печатную форму (табличный документ или офисный документ) из формы документа/справочника по команде «Печать» — `УправлениеПечатьюКлиент.ВыполнитьКомандуПечати` в обработчике команды формы.
- Зарегистрировать новую печатную форму в подменю «Печать» объекта — добавить процедуру `ДобавитьКомандыПечати(КомандыПечати, Параметры)` в модуль менеджера объекта, заполнить через `УправлениеПечатью.СоздатьКоллекциюКомандПечати` + `КомандаПечати = КомандыПечати.Добавить()`.
- Переопределить общие настройки подсистемы «Печать» (отключить подписи/печати, задать список объектов с командами печати) — `УправлениеПечатьюПереопределяемый.ПриОпределенииНастроекПечати`.
- Программно напечатать пакет документов (с предварительным просмотром или сразу на принтер) — `УправлениеПечатью.СформироватьПечатныеФормы` + возврат в `ТабличныйДокумент`.
- Найти ссылку варианта отчёта по отчёту и ключу варианта — `ВариантыОтчетов.ВариантОтчета(Отчет, КлючВарианта)`.
- Открыть форму отчёта (внутреннего или дополнительного) по ссылке на вариант — `ВариантыОтчетовКлиент.ОткрытьФормуОтчета(ФормаВладелец, Вариант)`.
- Программно сформировать отчёт (с проверкой заполнения, с получением табличного документа) — `ВариантыОтчетов.СформироватьОтчет(Параметры, ПроверятьЗаполнение, ПолучатьФлажокПустой)`.
- Зарегистрировать новый вариант отчёта в подсистеме (в обработчике `ВариантыОтчетовПереопределяемый.НастроитьВариантыОтчетов`) — `ВариантыОтчетов.НастроитьОтчетВМодулеМенеджера(Настройки, МетаданныеОтчета)`.
- Запустить формирование тяжёлой печатной формы или отчёта в фоне (без блокировки формы) — `ДлительныеОперации.ВыполнитьФункцию` (предпочтительно) или устаревший `ДлительныеОперации.ВыполнитьВФоне`.

## Не использовать, если

- Нужно добавить **произвольную** подключаемую команду (отчёт, заполнение, «Создать на основании», шаблон сообщения) — переключайтесь на `bsp-commands-external`. Там — общая инфраструктура размещения команд в форме и регистрации внешних обработок; `УправлениеПечатью` — это только **вид** команды «Печать».
- Нужно просто открыть форму внешнего отчёта/обработки (без вариантов) — `ОткрытьЗначение(Ссылка)` (платформенный) или `ДополнительныеОтчетыИОбработкиКлиент.ОткрытьВариантДополнительногоОтчета` (см. `bsp-commands-external`).
- Нужны базовые утилиты (сообщения пользователю, безопасное хранилище, форматирование строк) — `bsp-base-common`.
- Нужна фоновая печать/отчёт в виде `ДлительныеОперации` (а не печатной формы как таковой) — `bsp-longs-and-jobs` (там API `ДлительныеОперации.*` и шаблоны `ПараметрыВыполненияФункции` / `ОжидатьЗавершение`).
- Нужно сформировать печатную форму через прямое обращение к макету `MXL` (`ПолучитьОбщийМакет(...)`) — это обходит БСП, ломает подменю «Печать», отключает замеры производительности и контроль прав. Только `УправлениеПечатью.СформироватьПечатныеФормы`.

## Core concepts

### Архитектура подсистемы «Печать»

Подсистема `Печать` состоит из **набора общих модулей** с разделением по контексту. **Не путать** подсистему `Печать` (в дереве `Subsystems/СтандартныеПодсистемы/Subsystems/Печать.xml`) с общим модулем `УправлениеПечатью` (в `CommonModules/УправлениеПечатью/Ext/Module.bsl`): первое — это объект метаданных, второе — это серверный модуль с экспортными функциями.

Жизненный цикл печатной формы:

1. **Объявление** — в модуле менеджера объекта (документа, справочника) определена процедура `ДобавитьКомандыПечати(КомандыПечати, Параметры)`, в которой через `УправлениеПечатью.СоздатьКоллекциюКомандПечати` создаётся таблица команд и заполняется (идентификатор, представление, картинка, `МенеджерПечати`, `ТипыОбъектовПечати`, условия видимости, формат сохранения).
2. **Формирование** — клик по команде вызывает `УправлениеПечатьюКлиент.ВыполнитьКомандуПечати`, который через серверный вызов открывает форму `ПечатьДокументов`. В форме пользователь видит табличный документ, может сохранить (PDF, XLSX, MXL, DOCX, HTML), отправить по e-mail или сразу отправить на принтер.
3. **Печать через внешнюю обработку** — для печати в форме внешней обработки с видом `ПечатнаяФорма` экспортный метод `Печать(МассивОбъектов, ПараметрыПечати, КоллекцияПечатныхФорм, ОбъектыПечати, ПараметрыВывода)` вызывается БСП-инфраструктурой автоматически.
4. **Постобработка** — `УправлениеПечатьюПереопределяемый.ПриПечати` (переопределяемый, не вызывается) выполняется после формирования и позволяет модифицировать `КоллекцияПечатныхФорм` (добавить дату, подпись, штрихкод).
5. **Регистрация в форме** — на форме объекта в `ПриСозданииНаСервере` вызывается `ПодключаемыеКоманды.ПриСозданииНаСервере(Форма, ПараметрыРазмещения)` (см. `bsp-commands-external`), БСП автоматически создаёт подменю «Печать» и помещает туда кнопки команд.

### Стабильный vs служебный API

- **Стабильный API** — `#Область ПрограммныйИнтерфейс` в `УправлениеПечатью`, `УправлениеПечатьюКлиент`, `ВариантыОтчетов`, `ВариантыОтчетовКлиент` и `*КлиентСервер`. Методы в `## Key methods` — из стабильных областей. Обратная совместимость поддерживается между минорными версиями БСП.
- **Переопределяемый API** — `#Область ПрограммныйИнтерфейс` в модулях `*Переопределяемый`. Экспортные, но **не вызываются** из прикладного кода — в них реализуют свою логику поверх БСП (настройки, постобработка печати, состав вариантов отчётов).
- **Служебный API** — `#Область СлужебныйПрограммныйИнтерфейс` (модули `*Служебный*`). ⚠️ Обратная совместимость не гарантируется. Используйте только если стабильного аналога нет, с явной пометкой в коде.

### `ВыполнитьВФоне` — deprecated, предпочтительно `ВыполнитьФункцию`

В модуле `ДлительныеОперации` есть две группы методов:

- **`ВыполнитьФункцию(ПараметрыВыполнения, ИмяФункции, Параметр1, …, Параметр7)`** и **`ВыполнитьПроцедуру(ПараметрыВыполнения, ИмяПроцедуры, Параметр1, …, Параметр7)`** — современный API с произвольным числом параметров (до 7). **Предпочтительны** для нового кода.
- **`ВыполнитьВФоне(ИмяПроцедуры, ПараметрыПроцедуры, ПараметрыВыполнения)`** — устаревший, требует упаковки параметров в `Структуру` и явного `АдресХранилища` в обработчике. Оставлен для обратной совместимости, в `bsp-longs-and-jobs` помечен как ⚠️.

В `bsp-print-reports` оба варианта легитимны для сценариев «тяжёлая печать в фоне», но **новый код** — через `ВыполнитьФункцию`/`ВыполнитьПроцедуру`. Подробное описание фоновых заданий, прогресса и `ОжидатьЗавершение` — в `bsp-longs-and-jobs`.

## Patterns

### 1. Добавление новой печатной формы в подменю «Печать» объекта

```bsl
// В модуле менеджера документа.
Процедура ДобавитьКомандыПечати(КомандыПечати, Параметры) Экспорт
    КомандаПечати = КомандыПечати.Добавить();
    КомандаПечати.Идентификатор = "СписаниеТоваров";
    КомандаПечати.Представление = НСтр("ru = 'Списание товаров'");
    КомандаПечати.МенеджерПечати = "Документ._ДемоСписаниеТоваров";
    КомандаПечати.Картинка = БиблиотекаКартинок.ПечатьMXL;
    КомандаПечати.Порядок = 10;
    КомандаПечати.ПроверкаПроведенияПередПечатью = Истина;
    КомандаПечати.ДополнительныеПараметры = Новый Структура("ПоказыватьЦены", Истина);
КонецПроцедуры
```

В `ДобавитьКомандыПечати` инициализируется `КомандыПечати` параметром, заполнять через `УправлениеПечатью.СоздатьКоллекциюКомандПечати` не нужно — параметр приходит уже заполненным.

### 2. Реализация менеджера печати объекта (формирование табличного документа)

```bsl
// В модуле менеджера документа — экспортная процедура Печать,
// вызывается БСП-инфраструктурой при выборе команды печати.
Процедура Печать(МассивОбъектов, ПараметрыПечати, КоллекцияПечатныхФорм, ОбъектыПечати, ПараметрыВывода) Экспорт
    Если УправлениеПечатью.НужноПечататьМакет(КоллекцияПечатныхФорм, "СписаниеТоваров") Тогда
        ТабДок = Новый ТабличныйДокумент;
        Для Каждого Ссылка Из МассивОбъектов Цикл
            НачалоОбласти = ТабДок.ВысотаТаблицы + 1;
            // ... формирование строк ТабДок по данным Ссылка ...
            УправлениеПечатью.ЗадатьОбластьПечатиДокумента(ТабДок, НачалоОбласти, ОбъектыПечати, Ссылка);
        КонецЦикла;
        УправлениеПечатью.ВывестиТабличныйДокументВКоллекцию(
            КоллекцияПечатныхФорм, "СписаниеТоваров", НСтр("ru = 'Списание товаров'"), ТабДок);
    КонецЕсли;
КонецПроцедуры
```

`МассивОбъектов`, `КоллекцияПечатныхФорм`, `ОбъектыПечати`, `ПараметрыВывода` — стандартные параметры менеджера печати; `ПараметрыПечати` — структура, переданная из `УправлениеПечатьюКлиент.ВыполнитьКомандуПечати`.

### 3. Программная печать пакета документов из серверного кода

```bsl
&НаСервере
Функция ПодготовитьПечатныеФормыНаСервере(МассивСсылок)
    ПараметрыПечати = Новый Структура;
    ПараметрыПечати.Вставить("ЗаголовокФормы", НСтр("ru = 'Печатные формы документов'"));
    Возврат УправлениеПечатью.СформироватьПечатныеФормы(
        "Документ._ДемоСписаниеТоваров",
        "СписаниеТоваров,АктСписания",
        МассивСсылок,
        ПараметрыПечати);
КонецФункции

&НаКлиенте
Процедура КомандаПечатиПакета(Команда)
    Результат = ПодготовитьПечатныеФормыНаСервере(МассивСсылок);
    Для Каждого Строка Из Результат.КоллекцияПечатныхФорм Цикл
        // Строка.ТабличныйДокумент — готовый табличный документ
    КонецЦикла;
КонецПроцедуры
```

Результат содержит `КоллекцияПечатныхФорм` (таблица с колонками `ИмяМакета`, `СинонимМакета`, `ТабличныйДокумент`), `ОбъектыПечати`, `ПараметрыВывода`.

### 4. Открытие формы отчёта по варианту

```bsl
&НаКлиенте
Процедура ОткрытьОтчет(Команда)
    // Ссылка на вариант отчёта (из ВариантыОтчетов.ВариантОтчета или из макета).
    Вариант = ПолучитьСсылкуВариантаНаСервере();
    ВариантыОтчетовКлиент.ОткрытьФормуОтчета(ЭтотОбъект, Вариант);
КонецПроцедуры

&НаСервереБезКонтекста
Функция ПолучитьСсылкуВариантаНаСервере()
    Возврат ВариантыОтчетов.ВариантОтчета("Отчет.ЗависимостиПодсистем", "Основной");
КонецФункции
```

Для внешнего отчёта (тип `СправочникСсылка.ДополнительныеОтчетыИОбработки`) тот же вызов открывает форму дополнительного отчёта.

### 5. Программное формирование отчёта с проверкой заполнения

```bsl
&НаСервере
Функция СформироватьОтчетСервер(ПараметрыОтчета)
    ПараметрыФормирования = ВариантыОтчетов.ПараметрыФормированияОтчета();
    ПараметрыФормирования.Отчет = Метаданные.Отчеты.ЗависимостиПодсистем;
    ПараметрыФормирования.КлючВарианта = "Основной";
    ЗаполнитьЗначенияСвойств(ПараметрыФормирования, ПараметрыОтчета);
    Возврат ВариантыОтчетов.СформироватьОтчет(ПараметрыФормирования, Истина, Истина);
КонецФункции
```

Возвращаемая структура содержит `ТабличныйДокумент`, `Успех`, `ТекстОшибки`, `Расшифровка`, `НастройкиКД` и др.

### 6. Тяжёлая печать/отчёт в фоне (предпочтительный путь)

```bsl
// Серверная функция, которая будет выполнена в фоне.
Функция СформироватьТяжелыйОтчет(ПараметрыОтчета) Экспорт
    ДлительныеОперации.СообщитьПрогресс(10, "Загрузка данных");
    // ...тяжёлая работа...
    ДлительныеОперации.СообщитьПрогресс(90, "Формирование табличного документа");
    Возврат Результат;
КонецФункции

&НаСервере
Функция ЗапуститьФормирование(ПараметрыОтчета)
    ПараметрыВыполнения = ДлительныеОперации.ПараметрыВыполненияФункции(УникальныйИдентификатор);
    Возврат ДлительныеОперации.ВыполнитьФункцию(
        ПараметрыВыполнения,
        "Обработка.МойОтчет.СформироватьТяжелыйОтчет",
        ПараметрыОтчета);
КонецФункции

&НаКлиенте
Процедура Сформировать(Команда)
    ДлительнаяОперация = ЗапуститьФормирование(ПараметрыОтчета);
    Оповещение = Новый ОписаниеОповещения("ПослеФормирования", ЭтотОбъект);
    ДлительныеОперацииКлиент.ОжидатьЗавершение(
        ДлительнаяОперация, Оповещение,
        ДлительныеОперацииКлиент.ПараметрыОжидания(ЭтотОбъект));
КонецПроцедуры
```

Для миграции с устаревшего `ДлительныеОперации.ВыполнитьВФоне` — см. `bsp-longs-and-jobs` (раздел Anti-patterns).

### 7. Регистрация вариантов отчётов в переопределяемом модуле

```bsl
// В своём ВариантыОтчетовПереопределяемый.
Процедура НастроитьВариантыОтчетов(Настройки) Экспорт
    ВариантыОтчетов.НастроитьОтчетВМодулеМенеджера(Настройки, Метаданные.Отчеты.ЗависимостиПодсистем);
    ВариантыОтчетов.НастроитьОтчетВМодулеМенеджера(Настройки, Метаданные.Отчеты.СтатистикаВыполненияОбработчиковОбновления);
КонецПроцедуры
```

В модуле менеджера отчёта при этом определена экспортная процедура `НастроитьВариантыОтчета(Настройки, НастройкиОтчета)`, которая и наполняет варианты.

## Anti-patterns

### ❌ Вызывать `УправлениеПечатью.СформироватьПечатныеФормы` синхронно из обработчика команды формы

```bsl
// ❌ Форма «висит» на десятки секунд, пользователь не понимает, что происходит
&НаКлиенте
Процедура СформироватьПечатнуюФорму(Команда)
    Результат = СформироватьНаСервере(); // 30 секунд синхронно
КонецПроцедуры
```

```bsl
// ✅ Через ДлительныеОперации.ВыполнитьФункцию (см. Patterns §6) или через подменю «Печать»,
//   которое само открывает форму ПечатьДокументов в фоне.
```

То же касается программного формирования тяжёлого отчёта: `ВариантыОтчетов.СформироватьОтчет` блокирует управляемую форму.

### ❌ Использовать устаревший `ДлительныеОперации.ВыполнитьВФоне` в новом коде

```bsl
// ❌ Legacy API: лишний уровень (Структура `ПараметрыПроцедуры`, обязательный `АдресХранилища`)
ПараметрыПроцедуры = Новый Структура("ПараметрыОтчета", ПараметрыОтчета);
ДлительнаяОперация = ДлительныеОперации.ВыполнитьВФоне(
    "Обработка.МойОтчет.Сформировать", ПараметрыПроцедуры,
    ДлительныеОперации.ПараметрыВыполненияВФоне(УникальныйИдентификатор));
// в `Сформировать` обязательно 2 параметра: (ПараметрыПроцедуры, АдресРезультата)
```

```bsl
// ✅ Произвольное число параметров (до 7), без обёртки `Структура`/`АдресРезультата`
ДлительнаяОперация = ДлительныеОперации.ВыполнитьФункцию(
    ДлительныеОперации.ПараметрыВыполненияФункции(УникальныйИдентификатор),
    "Обработка.МойОтчет.СформироватьТяжелыйОтчет", ПараметрыОтчета);
```

### ❌ Создавать свой общий модуль «ПечатныеФормы» / «ПечатьМоейПодсистемы»

```bsl
// ❌ Дублирует инфраструктуру УправлениеПечатью, ломает подменю «Печать»,
//    отключает замеры производительности, контроль прав, переключение языка
//    печатных форм, рассылку отчётов и т. п.
Функция СформироватьМоюПечатнуюФорму(Ссылка) Экспорт
    // ...свой код...
КонецФункции
```

```bsl
// ✅ Использовать УправлениеПечатью.СформироватьПечатныеФормы / ВыполнитьКомандуПечати
//   + ДобавитьКомандыПечати в модуле менеджера объекта.
```

### ❌ Открывать форму отчёта через `ОткрытьЗначение` / `ПолучитьФорму`

```bsl
// ❌ Обходит подсистему вариантов, ломает кэширование настроек, панель отчётов
ОткрытьЗначение(СсылкаВарианта);
```

```bsl
// ✅ Через ВариантыОтчетовКлиент.ОткрытьФормуОтчета
ВариантыОтчетовКлиент.ОткрытьФормуОтчета(ЭтотОбъект, СсылкаВарианта);
```

### ❌ Прямой вызов методов из модуля `УправлениеПечатьюСлужебный*`

```bsl
// ⚠️ Служебный API — обратная совместимость не гарантируется.
```
