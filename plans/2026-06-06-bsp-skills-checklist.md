# Чек-лист скилов по БСП 3.1.11 — трекинг прогресса

> **Назначение:** отмечать готовность каждого скила. Ведётся параллельно с большим планом `2026-06-06-bsp-skills-roadmap.md`.
>
> **Конвенция:**
> - `- [ ]` — не начат / в работе / не прошёл DoD
> - `- [x]` — скил опубликован и прошёл §9 DoD (RED/GREEN/REFACTOR)
> - `- [-]` — отменён / не нужен (с обоснованием в `## Baseline log` §9.0)
> - `[YYYY-MM-DD]` после галочки — дата публикации (опционально)
>
> **Правило обновления** (см. §14 большого плана): после того как скил прошёл §9 DoD (все три фазы RED → GREEN → REFACTOR), агент **обязан** отметить соответствующий чек-бокс в этом файле. Без этого скил считается «не завершённым», даже если `SKILL.md` написан.
