- **ПараметрыПодключения в Key methods** — две строки (Клиент и Сервер) с разным составом полей. Обе попали в Key methods, поскольку у них разные контракты и применимость. Это компромисс с минимальным «дублированием» и не выходит за лимит 10 строк.
- **COM** — упомянут в Core concepts как платформенный механизм, отдельной строки в Key methods не получил. Метод `ПодключитьКомпонентуИзРеестраWindows` намеренно не включён (узкий, deprecated, не рекомендуется к использованию).
- **OData** — в таблице Key methods 2 строки (переопределение + один служебный метод с ⚠️). Это намеренный сигнал: стабильное API OData для прикладного кода ограничено переопределением; остальное — служебное.
