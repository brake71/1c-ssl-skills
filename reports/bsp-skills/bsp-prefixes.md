**Задача 1 из `## When to use`:** «Извлечь чистый номер документа для отображения — убрать префикс ИБ, префикс организации, пользовательский префикс».

**Ожидаемый RED-результат (без скила, по аналогии с §0.1 п.6 плана):**
- Агент, скорее всего, напишет «свою» функцию разбора через `Сред(Номер, 6)`, `СтрНайти(Номер, "-")` и т. п. — типичная ошибка: не различает 3-символьный (только ИБ) и 5-символьный (ИБ+организация) префиксы, не снимает незначащие нули, не сохраняет пользовательский префикс.
- Агент может попытаться вызвать `ПрефиксацияОбъектов.НомерНаПечать(...)` — **ошибка компиляции**, модуль `ПрефиксацияОбъектов` без суффикса не существует (см. Core concepts).
- Агент может вызвать `ПрефиксацияОбъектовСлужебный.НомерНаПечать` — **ошибка компиляции**, в `ПрефиксацияОбъектовСлужебный` нет такого экспорта; стабильный API в `ПрефиксацияОбъектовКлиентСервер`.

**Ожидаемое решение:** Case 1 из §9.0 — LLM без скила **не знает** идиоматический способ разбора номера в БСП и попытается реализовать разбор «с нуля», скорее всего с ошибками (особенно на нестандартных форматах вроде `АБВГ0012345` без дефиса). Скил оправдан.

**Задача 2:** «Программно изменить префикс ИБ при миграции данных».

**Ожидаемая ловушка:** агент установит значение константы `ПрефиксУзлаРаспределеннойИнформационнойБазы` напрямую через `Константы.ПрефиксУзлаРаспределеннойИнформационнойБазы.Установить("НБ")` — нарушит транзакционную целостность, не выполнит перенумерацию (`ПродолжитьНумерацию`), и при активной подписке `УстановитьПрефиксИнформационнойБазы` на вновь создаваемых документах будет расхождение.

**Задача 3:** «Реализовать прикладной переопределяемый модуль для документа с нестандартным форматом номера `АБВГ0012345`».

**Ожидаемая ловушка:** агент не найдёт переопределяемый модуль `ПрефиксацияОбъектовПереопределяемый` и попытается либо реализовать `ПриИзмененииНомера` в самом документе (не вызывается БСП), либо пропустит переопределение и оставит стандартный алгоритм (который вернёт ошибочный базовый номер).

**Дополнительные gotchas, зафиксированные в `## Core concepts`:**

- ⚠️ `ПрефиксацияОбъектов` (без суффикса) **не существует** как общий модуль — стабильный API в `ПрефиксацияОбъектовКлиентСервер` (аналог `РегламентныеЗаданияСервер` по «выдуманному» суффиксу). В отличие от `ОбновлениеИнформационнойБазы` (без суффикса), в семействе `ПрефиксацияОбъектов` базовое имя не используется.
- ⚠️ `ПрефиксацияОбъектовСобытия.ПриОпределенииПрефиксаОрганизации` возвращает `ПрефиксОрганизации = Ложь` (булево!) для пустой ссылки — не пустую строку. Это обрабатывается в `ПрефиксацияОбъектовСобытия.УстановитьПрефикс` через `?(ПрефиксОрганизации = Ложь, "", ПрефиксОрганизации)`. Прикладной код, читающий префикс через этот метод, должен учитывать эту особенность.
- ⚠️ В `ПрефиксацияОбъектовСобытия.ПриОпределенииПрефиксаОрганизации` используется **демо-функциональная опция** `_ДемоПрефиксыОрганизаций` с параметром `_ДемоОрганизация` (в демо-конфигурации). В реальной конфигурации заменяется на `ПрефиксыОрганизаций` / `Организация`. Если в определяемом типе `Организация` разрешён `Строка`, префикс всегда будет пустым (отдельная ветка в коде).
- ⚠️ `ПрефиксацияОбъектовПовтИсп.ПрефиксообразующиеРеквизиты()` возвращает `ФиксированноеСоответствие` (кэш на сеанс); первое обращение идёт через `ПрефиксацияОбъектовПереопределяемый.ПолучитьПрефиксообразующиеРеквизиты`.
